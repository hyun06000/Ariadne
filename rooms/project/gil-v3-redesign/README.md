# 프로젝트: gil-v3-redesign

**v3를 상현님이 디자인한 대로 재설계한다.** 지금의 v3는 스텝 트리·open/step/close를 갖췄지만
두 지점에서 디자인과 어긋난다.

## 정복할 간극

### 1. 폴더 난립 → 커밋 그래프
- **지금**: `gil v3 open <dir>`이 디렉토리를 사이클 정체성으로 쓴다. 사이클마다 `C00x/` 폴더가
  쌓이고, 그 안에 `steps.yaml`·`steps/*.md`·`cycle.yaml`이 생긴다.
- **디자인**: 사이클·스텝은 폴더가 아니라 **커밋 그래프의 노드**다. append-only, 앞으로만
  (v3-view/C001: 깃=append-only 전진 기록, gil=백트래킹·분기 지능).
- **목표**: 프로젝트 디렉토리 하나 + 커밋 그래프. 사이클을 여는 것 = 커밋 노드를 새기는 것.

### 2. 본문 md 파일 → 커밋 로그
- **지금**: 스텝 본문이 `steps/sN.md` 디스크 파일. 계보만 커밋 trailer로 옮겼다(C041).
- **디자인**: 짧은 커밋 제목(노드 위계·이름·id) + 긴 커밋 본문(스텝 디테일). 뷰어는 `git log`
  에서 본문을 읽는다. md 파일 폐지.
- **git 사실 확인**: 커밋은 제목(subject, 첫 줄) + 본문(body, 빈 줄 뒤 긴 설명) + trailer(본문
  끝 `Key: value`)로 구성. `git log --format=%s`(제목)·`%b`(본문)로 각각 읽힌다. 상현님 구상이
  정확히 git 네이티브 구조다.

## 새 구조 (rooms 확장, 상현님 확정)

- `rooms/`: existence(정체성) · deployment(배포) · **project**(신설, 이 방).
- `rooms/project/<프로젝트>/` 아래 사용자가 자유롭게: `source`·`figure`·`md`·`field` 등.
- 버저닝·로깅은 git. gil은 위계 있는 노드 생성(템플릿화된 로깅)만.

## 작업 방식

- 이 프로젝트 자체는 **부트스트랩** — 현재 gil v3(폴더+md)로 열되, 여기서 만든 결과물이
  폴더리스·커밋로그 구조를 실현한다. 자기를 재설계하는 도그푸딩.
- 사이클은 이 프로젝트 안에서 돈다. 첫 사이클: 새 데이터 모델·명령 인터페이스 설계.
