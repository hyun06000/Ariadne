# 프로젝트의 방 (Room of Projects)

이 방에는 Ariadne에서 진행되는 **프로젝트**가 산다. 프로젝트는 사용자가 무언가를 만드는
작업 공간이다 — 도구, 뷰어, 문서, 실험, 무엇이든.

## rooms의 세 방 (2026-07-23 확장, 상현님 확정)

Ariadne의 rooms는 세 방으로 구성된다:

| 방 | 담는 것 |
|---|---|
| [existence](../existence/README.md) | **정체성** — 이 레포에서 활동하는 LLM 존재들 (identity·will·memory·relations) |
| [deployment](../deployment/README.md) | **배포** — 검증을 통과해 봉인·버저닝된 산출물 |
| **project** (이 방) | **프로젝트** — 사용자가 자유롭게 만드는 작업 공간 |

## 프로젝트의 구조 — 자유롭되 위계 있는 로깅

각 프로젝트는 자기 디렉토리 하나를 가진다. 그 아래는 **사용자가 자유롭게** 만든다:

```
rooms/project/<프로젝트명>/
├── source/   # 소스 코드
├── figure/   # 그림·캡쳐·다이어그램
├── md/       # 문서
├── field/    # 필드 데이터·실측
└── ...       # 무엇이든 필요한 것
```

이 하위 구조는 **강제가 아니다** — 프로젝트가 필요로 하는 것을 만든다.

## 핵심 원칙 — 버저닝·로깅은 git, 위계는 gil

- **모든 버저닝과 로깅은 git으로** 한다. 프로젝트의 모든 변경은 커밋으로 쌓인다 (append-only,
  앞으로만 — v3 정신, v3-view/C001).
- **gil은 템플릿화된 로깅**을 담당한다 — 커밋 그래프 위에 **위계 있는 노드**(사이클·스텝)를
  생성한다. gil이 사이클을 "여는" 것은 폴더를 만드는 게 아니라 **커밋 그래프에 노드를 새기는**
  것이다.
- **스텝 본문은 커밋 로그에** 남긴다 — 짧은 커밋 제목(노드의 위계·이름·id) + 긴 커밋 본문(스텝
  디테일). 별도 md 파일을 스텝마다 만들지 않는다. 뷰어는 `git log`에서 본문을 읽는다.

## 이름 규약 (영구) — "gilv3"는 없다

**`gilv3`라는 이름은 폐기됐고 다시 부활하지 않는다.** 이건 별도 물건이 아니라 **gil의 v3**다.
명령·문서·코드 어디에도 `gilv3`, `gil v3` 같은 별도 이름을 쓰지 않는다 — 그냥 **gil**이다.
v3는 gil의 현재 세대이며, gil에 통합된다. (2026-07-23, 상현님 확정.)

## 왜 이 방이 생겼나 (2026-07-23)

v3(스텝 트리·gilv3 open/step/close)를 지어왔으나, 상현님이 짚었다 — "v3가 완성됐다지만
내가 디자인한 바와 크게 다르다." 두 간극:

1. **폴더 난립**: 사이클마다 `C00x/` 폴더가 생긴다. append-only 커밋 그래프 정신과 어긋난다.
2. **본문이 md 파일**: 계보는 커밋 trailer로 옮겼으나(C041) 본문은 아직 `steps/sN.md` 파일.
   "깃 로깅을 쓰자"의 절반만 실현.

이 방은 그 간극을 메우는 **v3 재설계**의 집이다. 첫 프로젝트가 그 재설계다.
