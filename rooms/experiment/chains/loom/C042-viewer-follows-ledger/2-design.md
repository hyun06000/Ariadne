# 2. 실험 설계

**정답을 도구보다 먼저 고정한다** (C001의 절차).

## 설계 원칙

### 원칙 1 — 창은 원장을 따른다 (커밋 여부와 무관하게)

원장(`cycle.yaml`)이 바뀌면 창(뷰어)도 바뀐다. 커밋은 그 다음 문제다 — **깃 없는 사용자도 창은 갱신받아야 한다** (C028: *로컬-only 사용자를 일급으로*).

### 원칙 2 — 뷰어는 이름이 아니라 **자기보고**로 식별된다

`gil web -o <아무 이름>`이므로 파일명 목록은 반드시 낡는다(C039). 대신 **도구가 만든 것을 도구가 알아본다**:

> **뷰어 = 탐색 루트의 HTML 중 `<script … id="gil-data">` 훅을 가진 것.**

이 훅은 **이미 계약면이다**(§5·§7). 목록이 없으니 낡을 수도 없다.

**탐색 루트** = 깃 저장소면 저장소 루트, 아니면 현재 작업 디렉토리. **비재귀** (예측 가능·저비용).

### 원칙 3 — 없으면 아무것도 만들지 않는다

뷰어가 없는 저장소에서 gil은 **뷰어를 창조하지 않는다.** 사용자가 `gil web`을 한 번 돌린 사실 자체가 *"나는 뷰어를 쓴다"* 는 **선언**이다. 도구는 그 선언을 존중할 뿐 **강요하지 않는다.**

### 원칙 4 — 산출물은 자기를 어떻게 다시 굽는지 스스로 말한다

재굽기가 사용자의 `--title`이나 `--chain` 필터를 **잃어버리면** 그것은 갱신이 아니라 **훼손**이다.

**추론은 안 된다**: *"체인이 하나만 들어 있으니 필터된 뷰어겠지"* 는 **거짓일 수 있다** — 뷰어를 구운 뒤 새 체인이 생겼을 뿐일 수도 있고, 그 경우 **새 체인이 영원히 안 보이게 된다.**

그래서 **추론하지 않고 기록한다** — `gil-data` JSON에 굽기 조건을 넣는다:

```json
{"version": "0.4", "bake": {"title": "…", "chain": null}, "chains": {…}}
```

> C040의 문장 그대로다 — **도구는 모르는 것을 지어내지 않는다.** 추측 대신 **기록**한다.

JSON 훅 버전 `0.3 → 0.4` (최상위 키 추가). 기존 소비자는 `chains`만 읽으므로 **하위 호환**이다.

### 원칙 5 — 꼬리가 개를 흔들지 않는다

- **뷰어는 사이클이 아니다.** 사이클 커밋은 **사이클 디렉토리만** 담는다(§4, `GIT-CLOSE`·`OPEN-GIT`가 집행). 뷰어를 그 커밋에 섞으면 **태그가 사이클 밖의 것을 봉인한다.** → 뷰어는 **자기 커밋**을 갖는다.
- **뷰어 갱신 실패는 명령 실패가 아니다.** 원장의 각인은 이미 끝났다. 실패는 **stderr 경고**로 알리고 **종료 코드는 건드리지 않는다.**

## 계약 (SPEC §5.2 신설 예정)

| 항목 | 내용 |
|---|---|
| 적용 명령 | 원장을 바꾸는 것 전부 — `open` · `step` · `close` · `supersede` · `correct` |
| 시점 | 원장 커밋 **이후** |
| 발견 | 탐색 루트의 **비재귀** `*.html` 중 `id="gil-data"` 훅 보유 |
| 재굽기 | 각 뷰어의 `bake.title` · `bake.chain`을 **그대로** 써서 다시 굽는다 |
| 커밋 | 내용이 바뀐 뷰어만, **별도 커밋** `gil: web 갱신 — <chain>/<id>` |
| opt-out | `--no-web` |
| 실패 | stderr 경고, **종료 코드 불변** (원장이 우선이다) |
| 뷰어 없음 | **아무것도 하지 않는다** |

## 판정기 확장 (50 → 54)

**항목마다 자기 샌드박스** (C040 교훈 6).

| # | 항목 | 판정 |
|---|---|---|
| 51 | `WEB-AUTO-REFRESH` | 뷰어가 있는 저장소에서 `step` → 그 HTML이 **새 상태를 반영**(`"step": "2"`) |
| 52 | `WEB-AUTO-PURE-COMMIT` | 같은 상황에서 **사이클 커밋에 뷰어가 섞이지 않는다** + 뷰어는 **별도 커밋**에 있다 |
| 53 | `WEB-AUTO-NONE` | 뷰어 없는 저장소에서 `step`·`close` → **HTML이 생기지 않는다** (강요 금지) |
| 54 | `WEB-BAKE-META` | 사용자 `--title`로 구운 뷰어가 자동 재굽기 후에도 **그 제목 유지**(`bake` 자기보고) + `--no-web`이면 **손대지 않는다** |

## 변이 시험 (구현이 아니라 **행동**을 판정하는가)

| 변이 | 내용 | 기대 |
|---|---|---|
| **M1** | 뷰어를 사이클 커밋에 함께 담는다 | **52 실패** |
| **M2** | 뷰어가 없어도 기본 이름으로 만들어낸다 | **53 실패** |
| **M3** | 재굽기가 `bake.title`을 무시(기본 제목으로 덮어씀) | **54 실패** |
| **M4** | 갱신을 하지 않는다 (v2.1의 행동 그대로) | **51 실패** |

## 실데이터

우리 저장소에는 뷰어 파일이 **없다**(Pages로 굽는다). **그래서 이 결함은 우리 눈에 안 보였다** — 세 사이클 연속 밀린 이유이기도 하다. maru의 경로를 픽스처로 재현하고, **뷰어를 둔 클론에서 이 사이클의 남은 스텝을 실제로 밟아** 창이 따라오는지 관측한다.
