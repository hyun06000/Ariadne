# 5. 결과 보고

## 요약

"워크플로의 빌드 스텝을 신선한 클론에서 그대로 실행하면 배포 가능한 정적 사이트가 재현된다"를 검증했다 — 판정 4/4 통과, **가설 채택**. 배선 커밋 `af12256`.
이제 이 저장소가 GitHub에 push되는 순간, main의 모든 커밋이 자동으로 체인 그래프 웹 뷰어가 된다. "모두의 공유"의 마지막 배관이 놓였다.

## 교훈

1. **"정의가 곧 테스트" 패턴의 3연승**: 퀵스타트(C006) → 워크플로(C007). 실행 가능한 정의에서 검증이 명령을 추출하는 한, 문서·정의·검증은 어긋날 수 없다. Ariadne의 표준 검증 패턴으로 승격할 가치가 있다.
2. **배포는 릴리스를 따른다.** 빌드가 실험의 방이 아닌 배포된 스냅샷(v0.1.0)만 참조하므로, 파이프라인 의존성이 릴리스 단위로 고정된다. 대가: 도구 진화가 뷰어에 반영되려면 릴리스 승격이 필요 — C006이 예고한 버전 승격 규칙이 두 번째로 요구됨.
3. **한계의 명시가 가설을 지켰다**: Actions 러너·Pages 끝단은 원격 없이는 반증 불가능하므로 가설에서 제외했다. 원격 연결 후 사용자 확인 사항: ① GitHub에 push, ② Settings → Pages → Source = "GitHub Actions", ③ 첫 워크플로 실행과 URL 확인.
