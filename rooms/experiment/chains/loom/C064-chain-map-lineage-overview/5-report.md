# 5. 결과 보고

## 요약

위계(드릴다운) 뷰어 상단에 **체인 지도(L0)** 를 얹었다 — 원=체인(크기 ∝ 사이클 수, 색=상태), 초록 점선 화살표=체인 간 lineage(굵기·숫자 ∝ 건수), 원을 누르면 아래 그 체인이 펼쳐진다(프래그먼트→조상 details 자동 오픈, JS 0). 좌표를 삼각함수 없이 **sqrt·사칙연산만**으로 계산해 양 구현(gil.py·go/main.go)이 무옵션·`--flat` 모두 **바이트 동일**, 참조 90/90·Go 83/83 회귀 0 — 가설 채택.

## 교훈

1. **양 구현 바이트 동일 계약은 렌더 좌표의 수치 연산까지 규율한다.** 초월함수(sin/cos/atan2)는 IEEE correctly-rounded 미보장 → Python libm과 Go math가 ULP에서 갈릴 수 있어 parity를 깬다. **sqrt·+·−·×·÷만 쓰면**(sqrt는 IEEE 보장) Python↔Go 비트 동일. 기하가 삼각함수로 표현되면 벡터 정규화로 바꿔 같은 그림을 얻는다. C063의 "무옵션 바이트 동일"을 렌더 내부 수치까지 확장한 판.
2. **렌더는 계약이 아니지만 parity는 계약이다.** 체인맵은 `gil-data` JSON을 안 바꿔 conformance가 무수정 통과 — 잠글 것은 렌더 구조가 아니라 두 구현의 바이트 일치. 그래서 판정기를 늘리지 않고 parity 대조를 검증 산출물로 남겼다(SPEC §102 "렌더는 계약이 아니다"의 재확인, C021·C063 계열).
3. **개관과 드릴다운은 층으로 공존한다.** L0(체인 지도, lineage 한눈에) → L1(체인 목록) → L2(체인 그래프) → L3(사이클 5스텝). 각 층은 다음 층으로의 입구이고, JS 없이 `<details>`+프래그먼트로만 잇는다. loomlight/C002가 깐 자동-오픈 메커니즘을 재사용해 새 상호작용을 공짜로 얻었다.
