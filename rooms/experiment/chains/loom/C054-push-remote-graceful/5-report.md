# 5. 결과 보고

**부모**: loom/C053-windows-entry (그 보고서 추천 (A) — 상현님 발의)

## 요약

C053이 발견한 거친 모서리 — `--push`인데 원격이 없으면 gil이 **날것 git fatal(rc1)** 또는
**침묵(check=False)** 으로 답하던 두 얼굴 — 을, C052가 git 부재를 다룬 방식의 원격판으로 정복했다.
모든 push 경로를 단일 관문(`_has_push_remote`/`_warn_no_remote_once`/`_push`·`gitPush`, 참조·Go 대칭)에
모아 원격 부재를 앞에서 감지 → **원인 한 줄 + 커밋 보존 + rc0**로 우아하게 강등한다. 판정기
`NO-REMOTE-GRACEFUL` 신설(양 구현 75/75), 변이로 계약 집행 확인, 회귀 0. README·README.ko에
네트워크 자세(자체 호출 0·텔레메트리 0) 명문화. **판정: supported · 이탈 0 · v2.13.0.**

## 교훈

1. **한 결함은 여러 얼굴로 흩어진다 — 관문을 세우면 하나로 접힌다.** "원격 없이 push"라는 단일
   조건이 push 호출부 6곳에서 세 가지 다른 실패(날것 fatal ×2 계열, 침묵 ×1 계열)로 나타났다.
   각 호출부를 개별로 고쳤다면 다음 push 경로가 생길 때 또 새는 결함이었다. **단일 관문(`_push`)이
   "원격 부재 = 우아한 강등"을 구조적으로 보증한다** — C037("문지기의 증거는 없는 파일이다")의
   push판, C044(디렉토리 경계로 누락을 불가능하게)의 함수 경계판.
2. **C052의 교훈이 한 겹 더 일반화됐다 — "침묵보다 나쁜 것은 잘못된 신호다"의 양방향.** git *부재*는
   크래시(오도)를, 원격 *부재*는 날것 fatal(오도) *또는* 침묵을 낳았다. 두 실패 모드가 같은 처방
   (앞에서 감지 → 원인 지목 → rc0 강등)으로 풀린다. **가용성을 앞에서 감지하는 것이 견고함의 공통
   골격**이다 — git·원격 다음은 무엇이든 이 패턴.
3. **크래시 없음이 견고함이 아니었다(C052 재확인) — 그리고 rc0도 아니었다.** `check=False` 경로는
   rc0였지만 침묵으로 사용자를 속였다. 판정기가 `NO-REMOTE-GRACEFUL`로 "rc0 **그리고** 커밋 보존
   **그리고** 무크래시"를 함께 요구하기 전까지, 침묵하는 rc0도 날것 fatal도 각자 "정상"처럼 보였다.
   C051("관측되지 않으면 없는 계약")의 배포판.
4. **환경 함정이 계약 검증을 가리려 했다.** 판정기 첫 실행에서 참조만 FAIL한 진짜 범인은 push가
   아니라 macOS 심볼릭 링크(C028·C029)였다. **실패 로그를 끝까지 읽어**(C036) `git add -- ../../../`
   붕괴 경로를 보고서야 push 무관임을 알았다 — 앞부분만 봤다면 멀쩡한 push 로직을 뜯었을 것.
5. **두 구현의 계약은 같되 이름은 언어를 따른다.** Go `push` 함수가 `reserveCommitPush`의 `push bool`
   파라미터에 가려져 `gitPush`로 개명. 계약(NO-REMOTE-GRACEFUL·강등 문면)은 동일, 표현만 다르다.

## 다음 사이클을 위한 제안

- **(A) 경로 해석 비대칭 — Go `relToRepo`는 심볼릭 링크를 해석하고 참조 `os.path.relpath`는 안 한다.**
  이 사이클이 realpath로 우회한 선재 결함. 절대 `--root`가 심볼릭 링크된 경로면 참조가 깨진다 —
  두 구현이 갈리는 지점이라 계약면 후보. C028·C029 환경 함정의 데이터가 또 쌓였다.
- **(B) close의 deviations 카운트 조정** (C053 슬립이 드러낸 공백) — `gil close`가 deviations.yaml ↔
  `cycle.yaml` 카운트 불일치를 감지·경고하지 않는다. "규율이 문서의 훈계로만 남으면 새어나간다"(C038).
- **(C) Windows end-to-end 수신자 검증** — v2.13.0부터 `.exe` 게시(C053 B), 진짜 Windows에서 상현님/친구.
- **(D) verify·release의 git 부재/원격 부재 경로 점검** — 이 아크(git-optional·원격-optional)의 마지막
  미점검 명령들. verify는 태그를 읽고 release는 태그를 push한다.
