# 1. 가설 수립 — 뷰어 헤더에 gil 버전 표시 (발의: 박상현)

## 이전 사이클의 교훈

부모 **loom/C061**: 배포 계보(`releases.current = _GIL_VERSION`)를 gil-data에 담아 배포 패널에 "현재 배포 버전 v{cur}"를 표시했다. 그러나 그건 **CHANGELOG가 있을 때(배포 패널이 뜰 때)만** 보인다.

## 문제 분할

상현님: "길 뷰어에 길 버전 나타나게 했으면 좋겠음." 뷰어를 보는 사람이 **이 뷰어를 어느 gil 버전이 구웠는지** 항상 알 수 있어야 한다(드리프트·재현·버그리포트에 필수). 현재는 배포 패널이 있는 저장소에서만, 그것도 하단에 있다. → 헤더 통계 줄에 항상 표시.

## 가설

> **가설**: 뷰어 헤더(hierarchy·flat 두 몸)의 통계 `<p>`에 `_GIL_VERSION`을 `· gil vX.Y.Z`로 추가하면, CHANGELOG 유무와 무관하게 이 뷰어를 구운 도구 버전이 항상 헤더에 보인다. `_GIL_VERSION`은 지어냄 불가(도구 자기보고)이므로 정직하다.

## 기각 조건

- 버전이 flat/hierarchy 중 한 몸에만 나오면 기각(대칭 실패).
- 버전 문자열이 하드코딩 리터럴이면 기각(_GIL_VERSION 단일 소스여야).
- 기존 WEB-* 회귀가 나면 기각.
