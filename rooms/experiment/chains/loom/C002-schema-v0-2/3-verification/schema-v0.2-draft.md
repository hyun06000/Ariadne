# cycle.yaml 스키마 v0.2 (초안)

> ⚠️ 이 문서는 **도구(fsck) 작성 전**에 고정되었다 (2026-07-14). 검증 통과 후 배포의 방 스펙 릴리스(로드맵 C005)의 재료가 된다.

## 필드

| 필드 | 타입 | 필수 | 의미 |
|---|---|---|---|
| `id` | 문자열 | ✅ | 체인 내 사이클 식별자. 형식은 R1 |
| `chain` | 문자열 | ✅ | 소속 체인명. 디렉토리와 일치해야 함 (R4) |
| `parent` | 로컬 id \| 리스트 \| null | ✅ | **같은 체인 내** 계보 간선. 분기=같은 부모 공유, 병합=리스트 |
| `lineage` | 전역 id 리스트 | — | **다른 체인**의 사이클(보고서)로부터 얻은 교훈의 연원. v0.2 신설 |
| `author` | 문자열 | ✅ | 수행한 존재 (존재의 방 이름) |
| `status` | `open` \| `closed` | ✅ | R8과 정합해야 함 |
| `opened` / `closed` | 일자 \| null | ✅ | 개시일 / 종료일 |
| `title` | 문자열 | ✅ | 정복하려는 가장 작은 문제 한 줄 |

## 전역 id 표기

`<chain>/<id>` — 예: `genesis/C001-existence-in-repo`. 체인 경계를 넘는 모든 참조는 전역 표기를 쓴다.

## 규칙

| 규칙 | 내용 | 해소하는 모호성 (C001) |
|---|---|---|
| **R1** | id는 `C<3자리 이상 번호>-<소문자 케밥 슬러그>` 형식이며, 번호는 체인 내 유일하다. 표시 순서의 동순위는 번호 오름차순으로 결정한다 — **번호가 순서의 담지자다** | ① id 단조가 관례일 뿐 |
| **R2** | lineage의 각 참조는 실재하는 사이클로 해소되어야 한다 | ② 체인 간 계보 표현 불가 |
| **R3** | 참조 표기: parent는 로컬 id만(`/` 금지), lineage는 전역 id만(`/` 정확히 1개), lineage로 같은 체인을 가리키는 것 금지(그건 parent의 일) | ②③ 참조 표기의 혼동 |
| **R4** | `chain` 필드 = 소속 체인 디렉토리명 | ④ 이중 진실 |
| **R5** | `id` = 사이클 디렉토리명 | ⑤ 이중 진실 |
| **R6** | parent는 실재하는 사이클로 해소되어야 한다 (끊어진 참조 금지) | (C001의 오류 탐지 승격) |
| **R7** | 체인은 비순환이어야 한다 | (C001의 오류 탐지 승격) |
| **R8** | `status: closed` ⇔ `closed` 일자 존재 | (상태 정합 — 신규) |

## 이주 규정 (닫힌 사이클 불변 원칙과의 조화)

닫힌 사이클의 **5스텝 문서와 검증 산출물은 불변**이다. 단, `cycle.yaml`(메타데이터)에 한해
**의미 무손실 스키마 이주**를 허용한다: 정보의 추가·형식 전환만 가능하고 삭제·의미 변경은 금지하며,
이주 커밋은 메시지에 `[migrate]`를 명시한다.

*근거*: 우리의 참조는 깃의 내용 주소(해시)와 달리 이름 주소이므로 이주가 기술적으로 가능하고,
스키마 진화를 영원한 다중 버전 지원보다 싸게 만든다. 대신 무손실·기록 의무로 불변의 정신을 지킨다.
