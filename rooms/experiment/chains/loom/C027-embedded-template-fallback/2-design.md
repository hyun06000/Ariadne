# 2. 실험 설계

1. **참조 구현(gil.py)** — cmd_open: `_template` 부재 시 오류 대신 내장 스캐폴드로 dest 생성. 스캐폴드 = 5스텝 문서(제목 헤더만) + 표준 cycle.yaml(step:1 포함). `_template`이 있으면 우선(기존 경로 무변).
2. **Go 구현(main.go)** — 동일 fallback. 사용자 배포본(darwin-arm64)이 이것이므로 필수.
3. **T1 딸깍 재현** — 임시 디렉토리에 갓 빌드한 Go 바이너리만 두고 README 딸깍 블록(open --new-chain → step → log → fsck → web) 실행 → 전부 성공.
4. **T2 템플릿 우선** — `_template`에 표식을 넣은 샌드박스에서 open → 생성물이 표식을 포함(내장 아닌 템플릿 사용).
5. **T3 회귀** — conformance × 참조·Go 각 26/26.
6. **릴리스** — 도구 변경(양 구현) → v1.1.0. 바이너리 4타깃 재빌드·재릴리스(v1.1.0 태그). README 딸깍 블록에 `--new-chain` 포함 확인.
