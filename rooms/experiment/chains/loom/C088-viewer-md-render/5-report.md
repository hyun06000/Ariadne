# 5. 결과 보고 — 스텝 문서 마크다운 렌더 토글 + 이미지 (발의: 박상현)

## 요약
뷰어 스텝/존재 문서에 (a) 외부 CDN 없는 인라인 마크다운 파서 `renderMd`, (b) 로컬 이미지 장당 상한 내 base64 임베드(초과·외부·부재·경로이탈 스킵), (c) 기본 원문 <pre> + 토글 버튼을 더했다. C075의 클릭-시-마운트 덕에 렌더는 완전 lazy. 헤드리스 Chrome end-to-end + 이미지 단위 + conformance WEB-MD-RENDER 통과, 회귀 0. **채택(supported).**

## 교훈
1. **렌더는 렌더로 검증해야 한다.** blockquote 버그(esc가 `>`→`&gt;`로 바꿔 정규식이 못 잡음)를 헤드리스 브라우저 렌더 덤프가 잡았다 — 정적 grep·코드 읽기로는 안 보였다. verify 정신("실제 동작 관찰")의 뷰어판.
2. **"필요할 때만"은 이미 구조였다.** 상현님 직관대로, C075 클릭-시-마운트가 렌더를 lazy로 만든다. 새 무게는 이미지 base64뿐이고 장당 상한+폴백으로 통제. 무게 계약(C075) 무손상.
3. **esc 기반 파서가 곧 XSS 방어.** 원문을 항상 이스케이프한 뒤 화이트리스트 태그만 승격 — 안전이 파서 구조에 내장. url은 스킴 화이트리스트(javascript: 차단).
4. **정직한 원문과 렌더의 공존.** 기본을 원문으로 둬 C049의 "정직한 원문" 계약을 지키면서, 토글로 가독을 얻었다 — 둘 중 하나를 버리지 않았다.

## 다음 사이클을 위한 제안
- (A) **Go parity** — renderMd/이미지 임베드의 Go 이식(현재 참조 전용). 이번 세션 뷰어 3연작(C087·C088) 모두 Go 이월.
- (B) 파서 강건화 — 중첩 리스트·`<`/`&`가 든 인라인·펜스 언어 힌트 등(필요 시).
- (C) 토글 상태 기억 — 지금은 세션 내 JS 변수(자기완결). localStorage 없이도 유지할지.
