# 1. 가설 수립 — 스텝 문서 마크다운 렌더링 토글 (발의: 박상현)

## 이전 사이클의 교훈

부모 **loom/C075**(web-weight-lazy-docs): 문서를 초기 DOM에 굽지 않고 gil-data에 내장, **노드 클릭 시 JS가 그 하나의 사이클만** 구축한다(fetch 없이 자기완결). 무게는 초기 DOM 노드 수 — 그래서 지연 마운트가 계보 깊이에 무관하게 뷰어를 가볍게 한다.

## 문제 분할

상현님: "스텝문서 열기 하면 마크다운 날것 텍스트가 그대로 보인다 — 렌더링할 수 있으면. 그림도 같이 보일 수 있을까?" 현재 스텝은 `<pre>esc(content)</pre>`로 원문 그대로(C075/C049의 "JS 파싱 없는 정직한 원문" 선택). 두 요구:
1. **마크다운 렌더**(헤딩·볼드·이탤릭·코드·인라인코드·링크·리스트·이미지) — 보기 좋게.
2. **이미지 표시** — 자기완결 뷰어라 외부 파일 참조 금지 → data URI 임베드 필요.

상현님 직관("필요할 때만 렌더링하면 안 무겁다"): C075 구조가 이미 클릭-시-렌더라, **렌더는 완전 lazy**(클릭한 사이클 5문서만). 이미지 base64만 굽는 시점 임베드라 lazy 불가 → **장당 크기 상한 + 초과 시 링크 폴백**으로 비대화 방지.

## 가설

> **가설**: 뷰어에 (a) 외부 CDN 없는 **인라인 경량 마크다운 파서**를 JS에 넣어 스텝/존재 문서를 클릭 시 렌더하고, (b) 스텝 문서가 참조하는 로컬 이미지를 굽는 시점에 **장당 상한 내에서만 base64 data URI로 임베드**(초과·부재는 링크/alt 폴백)하며, (c) **기본은 원문 `<pre>`, 토글 버튼으로 렌더 전환**하면, "정직한 원문"과 "보기 좋은 렌더"를 둘 다 살리면서 자기완결·무게 계약을 지킨다.

## 기각 조건

- 토글 없이 렌더가 강제되면 기각(원문을 못 봄 — C049의 "정직한 원문" 상실).
- 이미지 임베드가 상한 없이 뷰어를 비대화시키면 기각(C075 무게 정신 위반).
- 렌더가 외부 CDN/네트워크를 요구하면 기각(자기완결 상실).
- 마크다운 렌더가 `esc` 없이 원문을 innerHTML해 XSS 표면을 열면 기각(파서가 이스케이프를 보장해야).
- 기존 WEB-* 회귀가 나면 기각.
