# 5. 결과 보고

## 요약

`close`가 사이클 디렉토리 전체를 불변 태그에 봉인해 오배치 untracked 파일까지 넣던 이슈 #19를 고쳤다(나중에 지우면 verify 변조, 중복 봉인 영구화 — 이 세션 C029 태그 drift의 계열). 봉인 전 **"3-verification/ 밖 + 표준 산출물 밖의 신규(untracked) 파일"**을 게이트해, 있으면 요약 출력 + `--allow-extra` 없이 거부(저장소·태그 무변화). 3-verification/ 안 자유 산출물은 존중(오탐 0). 참조·Go 동시, 참조 **107/107**·Go **93/93**(CLOSE-SEAL 3항목 신설). **채택.**

## 교훈

1. **오배치는 형태로 판단할 수 없다 — 실데이터가 가르쳤다.** 정당한 3-verification 산출물이 하위 디렉토리·임의 파일명을 자유롭게 쓰므로(C005·C020·C076), 형태 기반 차단은 정당 산출물을 오탐한다. 경계를 위치(3-verification/ 경계)+신규성(untracked)으로 옮겨 오탐 0을 달성.
2. **같은 증상, 다른 처방.** step(C080)도 close도 `cycle_dir` 통째 add였으나, step은 "더 좁혀" 해결, close는 그 안 자유도 때문에 "좁히는 대신 게이트(확인)"로. 증상이 같아도 내용의 자유도가 처방을 가른다.
3. **자율의 겸손**: 상현님이 A를 골랐어도 실데이터가 A의 세부를 반증하면 밀지 않고 재확인받았다 — 잘못된 경계는 회귀다.
4. **이 세션 커밋-규율 5연속**: C077→C078→C079→C080→C081. "커밋이 무엇을 담는가"를 open·step·close 전면에서 정련했다.
