# 5. 결과 보고

## 요약

외부 채택자 **결(Gyeol)의 이슈 #8** — *"gil은 사이클의 무결성은 기계로 집행하면서, 정작 자기 자신이 세상에 들어오는 입구(다운로드한 바이너리를 실행하는 순간)에서는 검증을 사람의 선의에 맡긴다"* — 를 정복했다. 설치 스니펫 4곳과 **`gil pages`가 생성하는 워크플로**에 체크섬 대조를 필수 단계로 박고, 대조 통과를 `chmod +x`의 전제로 만들어 **불일치 시 실행 파일 자체가 생기지 않게** 했다. 18/18 통과, conformance 26/26 ×2, 두 구현 산출물 바이트 동일 → **채택**.

## 교훈

1. **문지기의 증거는 "없는 파일"이다.** `exit 1`로 막으면 대화형 셸을 죽여 사용자가 우회하고, `exit`을 빼면 스크립트가 흘러 실행에 도달한다. 대신 **`chmod +x`를 대조 통과에 종속**시키면(`대조 && mv && chmod`), 불일치 시 `gil`이라는 실행 파일이 **태어나지 못한다**. 검증되지 않은 바이너리는 "실행하면 안 되는 파일"이 아니라 **존재하지 않는 파일**이 된다 — 규율이 훈계가 아니라 파일 시스템의 사실이 됐다. 대화형 셸(세션 유지)과 CI(rc≠0) 양쪽에서 같은 안전.

2. **훈계와 코드는 다른 층이다 — 도구가 생성하는 코드가 더 위험하다.** 이슈는 문서 스니펫을 지적했지만, 전수 grep(C010 절차)이 다섯 번째 경로를 찾았다: `gil pages`의 산출 워크플로. 문서는 에이전트가 안 따를 수도 있는 훈계지만 **생성된 워크플로는 사용자 CI에서 매 push마다 무조건 실행되는 코드**다. 우리는 "규율은 기계가 집행한다"고 말하면서 **우리 도구가 만든 기계에게 검증 없는 실행을 시키고 있었다.** 문서만 고쳤다면 가장 확실히 실행되는 경로가 뚫린 채 남았다.

3. **다섯·여섯 번째 "테스트를 먼저 의심하라" — 그리고 그 범인의 정체가 드러났다.** 첫 실행의 실패 4건이 **전부 하네스 결함, 구현 결함 0건**이었다. C016·C021·C029·C035에 이어 이제 패턴이 뚜렷하다: **실패한 단언의 범인은 대개 하네스의 *환경 가정*이다.** 오늘의 범인 셋 — 빌드 호출 문법(`go build .` vs `main.go`), 호스트 아키텍처(macOS에서 리눅스 바이너리 실행), 러너의 작업 디렉토리(빈 폴더 vs checkout 직후). C029의 git 기본 브랜치, C035의 작업 트리 기준선과 같은 계열이다.

4. **실패 메시지를 끝까지 읽어라.** E4-b의 첫 실패 로그에는 `gil-linux-amd64: OK`가 이미 찍혀 있었다 — 대조는 처음부터 통과하고 있었고 실패는 그 다음 줄이었다. 로그 앞부분만 봤다면 "체크섬 로직이 깨졌다"고 오진하고 멀쩡한 구현을 뜯어고쳤을 것이다.

5. **기억보다 저장소가 옳다.** 나는 "C023에서 Go 소스를 패키지로 승격했다"고 기억했지만 `go.mod`은 존재하지 않았다. C007의 교훈("전제는 가정하지 말고 명령 한 줄로 확인하라")이 이번엔 *내 기억*을 상대로 유효했다.

## 알려진 한계

- **전송 불일치 탐지이지 공급망 공격 방어가 아니다.** SHA256SUMS도 같은 CDN에서 온다. 바이너리와 SUMS가 **함께** stale하면 서로 일치해 통과할 수 있다. (결의 실측에서는 바이너리만 stale해 불일치가 드러났다.)
- **`gil pages` 산출물 규율이 아직 계약 밖이다.** 지금은 체크섬 없는 pages를 만드는 구현도 conformance 26/26을 받는다.

## 다음 사이클을 위한 제안

- **(A) pages 산출물의 계약화** — `conformance.py`에 PAGES 판정 항목을 신설해 "산출 워크플로는 체크섬을 대조한다"를 §7 계약으로 승격한다. 이번 사이클에서 **Weft가 `loom/C036`에서 같은 파일(`main.go`·`conformance.py`)을 만지고 있어** 판정 항목 수 경합을 피하려 이월했다. **그의 병합 직후가 적기다.**
- **(B) 번호 예약을 데이터로** — 이 사이클을 여는 순간 `gil open`이 **내가 Weft에게 예약해 둔 C036을 발급**했다(→ C037로 양보). 원장 규율(C016)은 *실행된 것*의 push 경합만 풀고 *예정된 것*은 못 푼다. **세 번째 재발**이며(C015·C018·오늘), 수기 규율은 소환자가 스스로를 못 막는다. 이슈 초안: [3-verification/issue-draft-number-reservation.md](3-verification/issue-draft-number-reservation.md) — 하네스가 자율 이슈 생성을 차단해 **상현님 승인 대기**.
- **(C) 무결성의 뿌리** — 서명 또는 릴리스 노트에 해시를 고정해 "CDN 동반 stale" 한계를 없앤다.
