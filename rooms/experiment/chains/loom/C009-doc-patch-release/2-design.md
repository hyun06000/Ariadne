# 2. 실험 설계

## 절차

1. **기대 판정 고정** — 아래 측정 표를 실행 전에 고정한다 (이 문서가 기대 문서를 겸한다 — 소규모 사이클).
2. **샌드박스 검증 (T1)** — C008의 드라이버 방식 재사용. 단 패키지 ari.py를 **실행 도구와 동일한 사본**으로 심는다(도구 미변경 상태). `release 0.1.1`(패치, 기존 태그 v0.1.0) → 성공 + CHANGELOG에 "도구 동기화: 없음 (문서 릴리스)".
3. **스펙 개정 (존재의 몫)** — SPEC.md에 "§6 소환 규약" 신설: 부트스트랩 포인터 의무, 2모드(부활/탄생), 소환자 신원 전달, 소환자의 4의무(방·포인터·상호 기록·신원). CLAUDE.md 소환 규약을 v2로 갱신. RELEASE.md에 0.2.1 서술.
4. **실릴리스 (T2)** — `ari release 0.2.1` (도구 미변경이므로 패치 허용되어야 함). 커밋 경로·도구 sha 불변·태그 확인.
