# 4. 결과 분석

## 통계적 결과

기대 행동 24건 전부 통과(실패 0). conformance는 두 구현 모두 26/26 — 기존 계약 무회귀.
두 구현의 web HTML은 바이트 단위로 동일했고, `cycle.yaml`·`[migrate]` 커밋 메시지·R11 판정·
supersede 거부 경로도 전부 일치했다.

## 데이터 직접 관찰

수치보다 중요한 것은 `runs/ref/07-cycle-after.yaml`을 직접 들여다본 결과다:

```yaml
id: C001-dirty
...
status: closed
closed: 2026-07-14
verdict: supported
deviations: 0
superseded_by: bench/C002-clean   # ← 이 한 줄이 전부다
```

닫힌 사이클에서 변한 것은 **이 한 줄뿐**이다. 5스텝 문서와 검증 산출물의 해시는 supersede 전후로
동일했고(T2e), 태그는 `[migrate]` 이주 커밋으로 이동해 사유를 남겼으며(T2d), `verify`는 여전히
"변조 없음"이라 답했다(T2a·b). **불변성을 깨지 않고 무효화를 기록한다**는 가설의 핵심 주장이
데이터로 확인됐다 — 결이 제안서에서 예측한 대로, §4 이주 규정이 이미 이 문을 열어두고 있었다.

감사자의 시야에서 본 결과(T3, `runs/ref/13-log.txt`):

```
●  C001-dirty [closed · supported] 오염된 벤치마크: 원샷이라 믿었으나 멀티턴이었다  ↣ superseded: bench/C002-clean
●  C002-clean [closed · supported] 오염 제거 후 재실험: 진짜 원샷
```

`verdict: supported`는 그대로 남아 있다 — 그 사이클은 자기 시점에서 정직하게 성공했다. 그러나
그 결론이 더는 유효하지 않다는 사실이 **같은 줄에** 붙어 있다. 이슈 #6이 지목한 "사슬을 앞으로
정독해야만 보이던 상태"는 한 줄의 조회로 대체됐다.

## 예상과 달랐던 것

1. **`verdict`를 건드리고 싶은 유혹이 있었지만, 건드리지 않은 것이 옳았다.** 설계 중 "무효화된
   사이클의 verdict를 `rejected`로 바꿔야 하나"를 잠깐 생각했다. 그것은 역사 왜곡이다 — 그 사이클의
   결말은 실제로 supported였고, 오염은 나중에 발견됐다. 두 사실은 서로 다른 축이며, 그래서 두
   필드다. **결말(자기 시점의 진실)과 무효화(후대의 판단)는 직교한다.** 이 분리가 없었다면
   "닫힌 사이클은 수정하지 않는다"는 원칙이 안에서부터 무너졌을 것이다.

2. **하네스가 구현보다 먼저 틀렸다 (네 번째).** "거부 시 작업 트리가 비어 있다"는 단언이 실패했고,
   범인은 supersede가 아니라 `open --new-chain`이 만든 미커밋 `chain.md`였다(close는 사이클
   디렉토리만 커밋한다). 무변화의 기준을 "비어 있음"에서 **"기준선과 같음"** 으로 고치자 통과했다.
   loom/C016·C021·C029에 이은 반복 — **테스트가 실패하면 먼저 테스트를 의심하라.**

3. **web의 이중 인코딩이 자연스럽게 요구됐다.** 무효화를 "흐리게"로만 표현하면 투명도를 못 보는
   독자에게는 없는 정보가 된다. 노드 흐림(opacity .5) + 점선 간선(고유 색) + 텍스트(`↣ superseded: X`)
   + 표의 전용 열 + 내장 JSON — 다섯 겹으로 같은 사실을 말한다. loom/C005의 접근성 규율이
   여기서도 문서의 완결성을 만들었다.

4. **표기의 자유는 검증기의 분기로 되갚는다.** `superseded_by`는 로컬 id와 전역 `<chain>/<id>`를
   모두 허용한다(parent는 로컬만, lineage는 전역만). 그래서 R11은 세 갈래 판정(자기참조 / 전역 해소 /
   로컬 해소)이 됐고, web은 `_supersede_ref`로 해소 규칙을 한 곳에 모아야 했다. 다만 `gil supersede`
   CLI는 전역 표기만 받는다 — **데이터는 관대하게, 도구는 엄격하게.**

## 이 사이클이 남긴 공백

- **연쇄 무효화의 의미가 미정의다.** A ↣ B이고 B ↣ C이면 A는 C에 의해 무효인가? 도구는 지금
  한 단계만 말한다. 감사자가 사슬을 따라가야 한다면 이슈 #6의 문제가 축소된 형태로 남는 셈이다.
- **`gil close`가 무효화를 제안하지 않는다.** 재실험 사이클(부모가 오염된 사이클)을 닫을 때
  "부모를 supersede하겠는가"를 물을 수 있다. 지금은 존재의 규율에 맡겨져 있다.
- **뷰어의 무효화 간선은 같은 열에서 활처럼 우회한다.** 사이클이 많은 체인에서 이 곡선이 parent
  간선과 시각적으로 겹칠 여지가 있다(정적 레인 폭 한계 — C005부터의 이월).
