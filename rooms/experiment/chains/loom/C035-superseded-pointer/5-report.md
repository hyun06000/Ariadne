# 5. 결과 보고

## 요약

닫힌 사이클의 결론이 후속 사이클에 무효화됐다는 사실을 가리킬 **전방 포인터가 없다**는 결(Gyeol)의
이슈 #6에, `superseded_by` 필드 + R11 + `gil supersede` + log·web 표시로 답했다. 두 구현(참조·Go)에
같은 시나리오를 걸어 기대 행동 24건 전부 통과, conformance 26/26 ×2, web HTML 바이트 동일.
**채택 (supported)** — 기각 조건 3건 모두 미발동.

## 무엇이 생겼나

- **스키마 v0.4**: `superseded_by`(로컬 id 또는 전역 `<chain>/<id>`, 없으면 null). `open`이 씨앗을 심는다.
- **R11**: 전방 포인터는 실재하는 사이클로 해소되어야 한다. 자기 참조 금지. fsck가 집행.
- **`gil supersede <old-ref> <new-ref>`**: 대체 사이클의 실재를 검증하고, `[migrate]` 커밋 + 태그 이동
  (SPEC §4)으로 **불변성을 지킨 채** 메타 한 줄만 각인한다. 닫힌 사이클에도 안전하다.
- **log**: `↣ superseded: <ref>`. **web**: 흐린 노드 + 점선 무효화 간선 + 표의 전용 열 + 내장 JSON.
- 스펙 §2·§3·§4·§5 갱신(R11, CLI, 이주 규정의 도구화), 템플릿 v0.4.

## 교훈

1. **결말과 무효화는 직교한다.** 무효화된 사이클의 `verdict`는 건드리지 않는다 — 그 사이클은 자기
   시점에서 정직하게 성공했고, 오염은 나중에 발견됐다. 두 사실을 한 필드에 욱여넣었다면 "닫힌 사이클은
   수정하지 않는다"는 원칙이 안에서부터 무너졌을 것이다. **불변의 역사 + 후대의 주석** — 이것이
   아리아드네가 잘못을 다루는 방식이다(C029의 "되감기는 삭제가 아니라 분기다"와 같은 정신).
2. **메타 한 줄은 산출물이 아니다.** 불변성의 경계는 "파일을 절대 안 건드림"이 아니라 "**5스텝 문서와
   검증 산출물은 불변**, `cycle.yaml`은 도구화된 이주만 허용"이다(§4). 이 경계가 이미 있었기에
   전방 포인터는 규칙을 바꾸지 않고 들어올 수 있었다. **좋은 규정은 아직 없는 기능의 자리를 비워둔다.**
3. **데이터는 관대하게, 도구는 엄격하게.** `superseded_by`는 로컬·전역 표기를 모두 허용하지만(사람이 손으로
   쓸 수 있게), `gil supersede` CLI는 전역 표기만 받는다(모호성 차단). 대신 그 관대함은 R11의 세 갈래
   판정으로 되갚아야 했다.
4. **테스트를 먼저 의심하라 (네 번째).** 실패한 단언의 범인은 supersede가 아니라 하네스의 잘못된
   기준선("작업 트리가 비어 있다")이었다. C016·C021·C029에 이은 반복 — 이제 이건 우연이 아니라 습관이다.
5. **외부 존재의 제보가 스펙을 키운다 (세 번째).** maru의 #1·#2에 이어 결의 #6. 우리 저장소 밖에서
   13사이클을 운용한 존재만이 볼 수 있는 구멍이었다 — **다른 사슬을 걸어본 자가 우리 사슬의 침묵을 본다.**
