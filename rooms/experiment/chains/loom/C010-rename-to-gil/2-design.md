# 2. 실험 설계

## 절차 (측정 표를 겸해 선고정)

1. **gil.py 작성** — C008 v6을 개명·수정: 독스트링/prog=`gil`, release의 패키지 도구 참조를 `os.path.basename(__file__)`로 비의존화. 로직 변경은 이 두 가지뿐(동작 불변이 가설이므로).
2. **T1 스모크 (실데이터)** — `gil.py`로 fsck(위반 0)·verify(변조 0)·log(체인 3 렌더)·web(자기완결 생성) 실행.
3. **T2 release 비의존화 (샌드박스)** — 패키지에 낡은 `ari.py`만 있는 샌드박스에서 `gil.py release` → 패키지에 `gil.py`가 동기화(sha 일치)됨을 확인. (낡은 파일 제거는 실릴리스 절차에서 `git rm`으로 — 도구는 자기 파일명만 안다.)
4. **표면 갱신 (존재의 몫)** — SPEC: CLI 표의 명령을 gil로, **§7 구현 독립 계약** 신설(명령·종료 코드·파일 포맷이 계약, 파이썬은 참조 구현, 바이너리 교체 전제 — 상현님 방향). QUICKSTART·README·워크플로를 gil.py로. RELEASE.md에 0.3.0 서술.
5. **실릴리스** — `git rm` 패키지 ari.py → `gil.py release 0.3.0` (도구 변경 → 마이너 강제 규칙의 정상 경로).
6. **T3 문서=테스트 재실행** — C006 방식: 갱신된 QUICKSTART의 bash 블록 추출을 신선 환경에서 실행. C007 방식: 갱신된 워크플로의 run 블록 추출을 신선 클론에서 실행 (릴리스·표면 커밋 후).
