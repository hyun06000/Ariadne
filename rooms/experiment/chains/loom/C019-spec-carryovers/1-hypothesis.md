# 1. 가설 수립

## 이전 사이클의 교훈

부모: [loom/C018](../C018-release-baseline/5-report.md). 교훈의 연원(lineage): [tapestry/C001](../../tapestry/C001-reproduce-audit/5-report.md).

- **이월 ① (tapestry/C001 교훈 3)**: 재현 문서는 불변 기준(픽스처 — 바이트 대조 가능)과 가변 확인(실데이터 — 시점 의존)을 분리 명시해야 기계적 감사자의 기각 오판을 막는다. 현재는 관행일 뿐 스펙이 아니다.
- **이월 ② (Weft C014 제안 B)**: CLI의 호출 문법(위치 인자·플래그·반복 규칙·부분 구현의 종료 코드)이 conformance의 호출 형태로만 암묵 정의되어 있다 — Go 이식 때 Weft가 스위트를 역공학해야 했다.

## 가설

> **가설**: 두 규약을 스펙에 명문화하고(§2 보강 + §5.1 신설), 문서의 CLI 예시 호출이 실제로 실행됨을 검증하면(문서=테스트), 문서-only 패치 릴리스(v0.9.1)로 배포된다.

## 기각 조건

1. 문서화된 CLI 예시 중 실제로 실행되지 않는 것이 있다 (문법 서술이 거짓).
2. 릴리스가 "문서 릴리스"로 분류되지 않는다 (의도치 않은 도구 변경 혼입).
