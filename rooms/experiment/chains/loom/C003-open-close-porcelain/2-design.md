# 2. 실험 설계

## 절차

1. **기대 행동 고정** — 테스트 매트릭스(정상 4건 + 거부 7건)를 `3-verification/fixtures/expected-behavior.md`에 구현보다 먼저 기록한다.
2. **샌드박스 픽스처** — 실제 레포 구조(`rooms/experiment/{_template, chains/}`)를 본뜬 미니 저장소를 `fixtures/sandbox/`에 만든다. lineage 테스트용으로 닫힌 사이클 하나(`seed/C001-origin`)를 심는다.
3. **구현** — C002의 ari.py를 계승·확장(v3): `open`(사전 검증 → 템플릿 복사 → cycle.yaml 생성 → 저장소 전체 fsck 사후 확인, 실패 시 생성물 제거)과 `close`(보고서 존재·비템플릿 검증 → 상태 전이 → fsck, 실패 시 원상 복구). 표준 라이브러리 전용 유지.
4. **테스트 실행** — 드라이버(`tests.py`)가 매 테스트마다 샌드박스를 임시 디렉토리로 복사해 독립 실행하고, 기대 행동과 대조해 PASS/FAIL을 출력한다. 거부 케이스는 exit code ≠ 0 **그리고** 저장소 무변화(부분 생성물 없음)까지 확인한다.
5. **실데이터 도그푸딩** — 검증이 통과하면, **이 사이클 자신(C003)을 `ari close`로 닫는다.** 도구가 만든 첫 실전 상태 전이가 자기 자신의 죽음이 되는 것. 트랜스크립트를 저장한다.

## 준비물

- C002의 산출물: ari v2, 스키마 v0.2 규칙, fsck
- Python 3 표준 라이브러리만 (계승)

## 측정 방법

| # | 항목 | 통과 기준 |
|---|---|---|
| 1 | 정상 계열 | T1~T4 전부 성공: 생성물이 fsck 통과, 번호 자동 증가, 5스텝 문서 복사, close 후 상태 정합 |
