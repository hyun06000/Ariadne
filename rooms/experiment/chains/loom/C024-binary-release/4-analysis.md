## 데이터 직접 관찰

- "딸깍"의 검증을 배포자 관점이 아니라 **수신자 관점**으로 설계했다: 내가 올린 파일이 아니라 **공개 URL에서 내려받은 파일**을 체크섬 대조하고 실행했다 — 사용자의 다른 맥이 겪을 경로 그대로.
- 워크플로의 업로드 스텝에 `GITHUB_ACTIONS` 가드를 넣어, "정의가 곧 테스트" 추출 실행이 실릴리스를 오염시키지 않게 했다 — 실행 가능성과 부작용 격리의 양립.

## 예상과 달랐던 것

- zsh의 단어 분리 함정(`for T in "darwin arm64"` + `set --`)으로 1차 빌드 실패 — 셸 이식성도 환경 계약의 일부라는 잔교훈. 명시적 `GOOS=${T%/*}` 페어로 해소.
