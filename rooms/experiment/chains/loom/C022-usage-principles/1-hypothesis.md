# 1. 가설 수립

## 이전 사이클의 교훈

부모: [loom/C021](../C021-parsing-contract/5-report.md). 발의: 사용자 (박상현, 2026-07-15):
"Weft는 스텝마다 커밋하지 않고 사이클마다 하는 것 같아. 스텝마다 커밋하게 명시해줘. 다른 LLM 쓸 때도 마찬가지일 거야. gil 사용 원칙에 쓸 수 있겠니?"

- 스텝 커밋은 C013의 규율·C015의 소환 규약 5항으로 존재하지만 **흩어져 있고 소환 문맥에 묶여 있다** — 도구를 쓰는 모든 LLM(소환 여부 무관)의 원칙으로 승격 필요.
- 관측된 실패 양상: 긴 스텝(구현 구간)의 침묵이 관전자에게 멈춤과 구별되지 않는다 — 스텝 전이 커밋만으로 부족하고, 스텝 내 진행 커밋 규율이 필요하다 (Weft C017의 "끊길 것 같으면 먼저 커밋" 관례의 일반화).

## 가설

> **가설**: "커밋의 단위는 스텝"을 축으로 한 사용 원칙 4조를 스펙(§2.1)과 부트스트랩(CLAUDE.md)에 명문화하고 퀵스타트에 반영하면, 문서-only 릴리스로 배포되고 퀵스타트는 여전히 실행 가능하다.

## 기각 조건

1. 원칙이 스펙·CLAUDE.md·퀵스타트 세 곳 중 누락되는 곳이 있다.
2. 갱신된 퀵스타트의 실행(문서=테스트)이 깨진다.
3. 문서 릴리스로 분류되지 않는다.
