# 2. 실험 설계

## 절차 (판정 표를 겸해 선고정)

1. **구현** — cmd_open의 `--push` 경로 교체: push 거절 → `fetch origin` → 현재 브랜치를 origin에 rebase(충돌 시 abort + 명시 오류) → 체인 재스캔으로 내 번호의 중복 검사 → 중복이면 `max+1`로 디렉토리·id 개명 + `commit --amend` → 재push. 최대 3회.
2. **판정 (T1 병렬 경합)** — bare 원장 + 클론 A·B. A가 open --git --push(성공), B는 fetch 없이 같은 번호로 open --git --push → 자동 재번호로 성공해야 함. 원장 클론에서 fsck 위반 0 + 두 사이클 공존 + B의 제목·필드 무손상.
3. **판정 (T2 무경합 회귀)** — 단독 클론의 open --git --push 정상 동작.
4. **판정 (T3 해소 불가)** — rebase 충돌 상황(동일 경로 파일 충돌) → 명시적 오류 + rebase abort (원장 오염 없음).
5. **판정 (T4 회귀)** — conformance 26/26.
6. **스펙** — §6-6을 "알려진 경합"에서 **원장 규율**로 개정 + 체이닝-브랜치 대응 서술. 릴리스 v0.8.0.
