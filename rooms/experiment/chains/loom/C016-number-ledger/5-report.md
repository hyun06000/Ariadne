# 5. 결과 보고

## 요약

번호의 진실을 원격 main 원장으로 정하고, `gil open --git --push`가 경합 시 fetch·rebase·자동 재번호로 재시도하게 했다 — 판정 3/3 + 회귀 26/26, **가설 채택**, v0.8.0.
C015에서 실물이 된 병렬 번호 경합이 하루 만에 규율이 됐고, 같은 시간에 Weft의 C014가 무충돌 병합되어 병렬 존재 협업의 전 과정(격리 노동 → 관전 → 병합)이 완결됐다.

## 교훈

1. **깃의 답을 훔쳐라**: push의 낙관적 동시성(거절 → 동기화 → 재시도)이 번호 발급에 그대로 이식됐다. 원장(원격 main)이 있으면 발급은 직렬화된다.
2. **사용자의 직관은 진단이었다**: "브랜치별 사이클 관리"의 실체는 그래프 개조가 아니라 발급 규율 — parent 모델은 이미 병렬을 담는다.
3. **테스트가 실패하면 먼저 테스트를 의심하라**: T3의 1차 실패는 픽스처의 무확인 push였다. 최소 재현으로 격리 진단.
