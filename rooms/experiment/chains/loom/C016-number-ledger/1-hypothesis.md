# 1. 가설 수립

## 이전 사이클의 교훈

부모: [loom/C015](../C015-being-work-visibility/5-report.md) (채택). 발의: 사용자 (박상현, 2026-07-14):
"브랜치명(번호) 충돌이 있었네. 브랜치별로 사이클을 관리하는 방식이 체이닝에도 도입될 필요가 있겠다."

- C015에서 병렬 번호 경합이 실물이 됐다(둘 다 C014 발급). 임시 규율은 소환자 양보 — 수동이고 우연 발견에 기댄다.
- 체인 그래프는 이미 분기·병합(parent)을 모델링한다. 결핍은 그래프가 아니라 **발급**이다: 번호 발급이 브랜치-로컬 관찰에 기대므로, 서로 못 보는 두 브랜치는 같은 번호를 얻는다.
- 깃의 선례: push의 낙관적 동시성 — 거절당하면 fetch·rebase 후 재시도. 번호도 같은 방식이면 된다.

## 문제 분할

(a) 원장 규칙 — 번호의 진실은 원격 main, (b) 자동 경합 해소 — `open --push` 거절 시 fetch·rebase → 중복이면 재번호(디렉토리·id 개명 + 커밋 정정) → 재push (한도 내 반복), (c) 규약 서술 — 격리 브랜치의 존재와 체이닝-브랜치 대응(브랜치 = 잠정 체인 구간, 병합 = 그래프의 분기·병합)을 스펙에.

## 가설

> **가설**: `gil open --git --push`가 push 거절 시 fetch·rebase 후 번호 중복을 감지해 자동 재번호·재시도하면, 서로를 모르는 두 클론이 같은 번호로 병렬 open해도 둘 다 성공하고(하나는 재번호), 원장(main)은 R1 위반 없이 두 사이클을 담는다.

## 기각 조건

1. 병렬 open 시나리오에서 한쪽이 실패로 끝나거나, 원장에 번호 중복(R1)이 남는다.
2. 재번호 과정이 사이클 내용(제목·계보)을 훼손하거나, 경합 없는 정상 open --push의 동작을 바꾼다.
3. rebase 충돌 등 해소 불가 상황에서 조용히 넘어간다 (명시적 오류여야 함).
