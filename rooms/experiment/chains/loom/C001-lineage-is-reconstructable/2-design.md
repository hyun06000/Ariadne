# 2. 실험 설계

## 절차

1. **기대 구조 정의** — 분기 1회·병합 1회를 포함한 가상 체인 픽스처(사이클 6개)의 계보를 먼저 손으로 그려 `3-verification/fixtures/expected-graph.md`에 기록한다. *(도구를 만들기 전에 정답을 먼저 고정한다 — 도구에 맞춰 정답을 바꾸는 것을 방지.)*
2. **픽스처 생성** — 위 구조대로 `cycle.yaml` 6개를 `3-verification/fixtures/chains/test-maze/`에 만든다. 구조: `C001 → C002 → (분기) C003, C004 → (병합: C003+C004) C005 → C006`.
3. **최소 구현** — `ari log` 프로토타입을 작성한다 (`3-verification/ari/`).
   - 언어: Python 3, **표준 라이브러리만** 사용 (cycle.yaml은 평탄한 key-value라 정규식 파싱으로 충분; PyYAML 의존 배제 — 범용 하네스는 설치 장벽이 낮아야 한다).
   - 기능: 체인 디렉토리 스캔 → DAG 구성 → 토폴로지 정렬 → ASCII 그래프 렌더 (id, title, status 표시).
   - 오류 처리: 존재하지 않는 parent ID 참조 시 명시적 오류 보고.
4. **픽스처 검증** — `ari log`를 test-maze에 실행, 출력을 1번의 기대 구조와 대조한다.
5. **실데이터 검증** — 같은 도구를 실제 체인(genesis, loom)에 실행한다. 픽스처에서 안 보이던 문제가 실데이터에서 나오는지 관찰한다.
6. **파괴 테스트** — 픽스처에서 (a) parent를 존재하지 않는 ID로 바꾼 경우, (b) 순환 참조를 만든 경우에 도구가 침묵하지 않고 오류를 보고하는지 확인한다.
7. **모호성 기록** — 진행 중 "parent 필드만으로는 표현/결정 불가"였던 모든 지점을 기각 조건 1에 비추어 기록한다. *(체인 간 참조 — loom이 genesis의 교훈에서 태어난 것 — 이 표현 가능한지가 예상 쟁점.)*

## 준비물

- Python 3 (macOS 기본 탑재 버전이면 충분)
- 이 레포의 기존 체인 2개 (실데이터)
- 외부 의존성: 없음 (의도적)

## 측정 방법

| # | 항목 | 통과 기준 |
|---|---|---|
| 1 | 픽스처 일치 | `ari log` 출력의 그래프 구조가 `expected-graph.md`와 정확히 일치 |
| 2 | 실데이터 | genesis·loom 체인이 오류 없이 올바르게 렌더됨 |
| 3 | 파괴 테스트 | 끊어진 참조·순환 참조를 모두 명시적 오류로 보고 |
| 4 | 모호성 | "표현 불가" 지점의 목록이 작성됨 (개수 자체는 기각 사유가 아니며, **유일 재구성을 깨는 모호성**만 기각 사유) |

1~3 전부 통과 + 유일 재구성을 깨는 모호성 없음 → 채택.
하나라도 실패 → 기각, 실패 양상이 다음 사이클(스키마 개정)의 가설 재료가 된다.
