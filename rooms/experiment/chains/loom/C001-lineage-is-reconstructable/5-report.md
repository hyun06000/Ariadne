# 5. 결과 보고

## 요약

"`cycle.yaml`의 스키마(id, chain, parent)만으로 분기·병합 포함 계보를 기계가 유일하고 정확하게 재구성할 수 있다"는 가설을, 정답 선고정 픽스처(6사이클)·실데이터 2체인·파괴 테스트 2종으로 검증했다.
측정 4항목 전부 통과, 기각 조건 3건 중 발생 0건 — **가설 채택**.
부산물로 Python 3 표준 라이브러리 전용 `ari log` 프로토타입([3-verification/ari/ari.py](3-verification/ari/ari.py))과, 스키마의 침묵 5건의 목록을 얻었다.

## 교훈

1. **계보는 기계의 것이 되었다.** parent 참조는 이제 사람의 약속이 아니라 파싱·검증·렌더 가능한 데이터다. 이 파서가 CLI(`ari`)와 웹 뷰어가 공유할 심장이며, 외부 의존성 0으로 유지됐다.
2. **도구는 스펙의 침묵을 드러내는 탐지기다.** 발견된 모호성 5건([4-analysis.md](4-analysis.md))은 전부 도구 결함이 아니라 스펙 부재였다: ① id 단조 증가가 관례일 뿐, ② 체인 간 계보 표현 불가(주석에만 존재), ③ id가 체인 내에서만 유일, ④ chain 필드↔디렉토리 위치 이중 진실, ⑤ id↔디렉토리명 이중 진실 + 이름 주소 참조의 취약성.
3. **정답 선고정 절차는 싸고 강력하다.** 도구 작성 전에 기대 그래프를 손으로 고정한 것이 검증의 신뢰를 만들었다. 이후 사이클의 표준 관행으로 삼을 가치가 있다.
4. **실데이터는 아직 그래프 검증력이 없다.** 실제 체인들이 단일 노드뿐이라 분기·병합은 픽스처로만 검증됐다. 실제 분기가 처음 발생하는 사이클에서 재검증이 필요하다.
