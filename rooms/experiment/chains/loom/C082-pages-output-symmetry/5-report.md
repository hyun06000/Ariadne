# 5. 결과 보고

## 요약

`gil pages`가 `web`과 달리 `-o`를 안 받고 고정 경로에만 쓰는 비대칭(이슈 #21)을, `pages`에 web과 동형인 `-o/--output`(및 `-o -` stdout)을 이식해 해소한다는 가설을 세웠다. 참조·Go 두 구현에 동시 이식하고 판정기에 T1(PAGES-OUTPUT-PATH)·T2(PAGES-OUTPUT-STDOUT)를 신설했다. 참조 109/109·Go 95/95(수정 전 각 107·93 — 신설 항목 FAIL), parity 0바이트, 이슈가 원한 diff 미리보기 흐름이 한 줄로 작동해 **채택**했다.

## 교훈

1. **대칭은 계약이다 — 명령 간 비대칭이 예측을 깬다.** 이 세션 커밋-규율 5연속(C077~C081)이 "커밋이 무엇을 담나"였다면, C082는 새 결 "명령이 무엇을 받나". 사용자는 `web -o`를 알면 `pages -o`도 되리라 기대한다(이슈의 "예측이 깨진다"). 같은 종류 명령은 같은 인자 형태를 공유해야 한다 — 대칭을 깨는 특례는 마찰의 씨앗.
2. **stdout은 파이프의 계약면 — 순수해야 한다.** `-o -`가 유용하려면 안내 문구 0(stderr로도 0바이트 실측). "무엇을 stdout에 쓰나"가 아니라 "stdout에 오직 그것만 쓰나"가 diff 파이프 안전의 조건. §7.2 부작용 없는 경로(force·존재검사·makedirs 전부 건너뜀)와 짝을 이룬다.
3. **default=None 센티넬로 회귀 0.** 새 인자를 더하되 미지정 시 기존 동작으로 폴백하는 센티넬(참조 None, Go "")이 회귀 가드의 핵심. web은 기본 파일명을 쓰지만 pages는 "기본 경로 폴백"이 기존 동작이라 None이 맞다 — 대칭이 인자 형태이지 기본값까지는 아님(맥락이 다르면 기본값은 달라도 된다).
4. **정적 이식은 parity를 공짜로 준다(C077 재확인).** pagesWorkflow 상수가 두 구현에서 이미 동일하므로, 같은 분기 로직을 얹으면 `pages -o -` 출력이 자동으로 바이트 동일. 참조·Go 동시 수정으로 gil-gate도 안 깨짐.
