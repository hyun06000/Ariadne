# 2. 실험 설계

## 절차

**범위 (상현님 결정): 참조 구현(gil.py) 앱화까지.** Go 이식은 정직히 이월(C043 리듬).

1. **문서를 DOM에서 gil-data JSON으로 이동**: `_render_hierarchy_body`에서 각 사이클의 `_render_cycle_detail`(5스텝 전문 인라인)을 초기 HTML에 굽지 않는다. 대신 사이클별 문서 데이터(head 메타 + 5스텝 텍스트)를 `gil-data` JSON의 각 cycle 노드에 담는다. 초기 HTML에는 그래프(SVG)+메타+빈 마운트 지점만.

2. **JS 앱 추가 (인라인 `<script>`)**: 노드(`<a>` 클릭 → `#cycdoc-*` 해시 변경)를 감지해, gil-data의 해당 문서를 읽어 그 자리에 **DOM을 구축**한다. `hashchange`/`DOMContentLoaded`에서 현재 타겟 하나만 렌더. 이미 그린 것은 캐시. **외부 리소스 0 유지**(fetch 없음, 문서는 내장 JSON) — 자기완결은 지킨다.

3. **SPEC §7 개정**: "JS 0줄"을 **"JS 허용, 단 외부 리소스 0·오프라인/file:// 자기완결 유지"**로. web 계약의 본질을 "JS 없음"에서 **"자기완결(어디서든 열림)"**로 재정의. §5 web 행·§5.2도 갱신.

4. **판정기 재정의**: `WEB-SELFCONTAINED`(외부 URL 0)는 유지. 문서 렌더 계약은 "노드 클릭 시 문서가 나타남"인데 판정기는 브라우저 없이 판정하므로, **문서 데이터가 gil-data JSON에 내장됨**을 검사(WEB-DOCS-EMBEDDED 신설). "초기 HTML에 5스텝 전문이 `<pre>`로 인라인되지 않음"(무게 감소)도 관측 — 초기 DOM 경량화가 계약이 된다. JS 존재를 이유로 실패하던(만약 있었다면) 검사가 없도록 정리.

5. **측정**: 초기 DOM 노드 수·초기 `<pre>`/`<td>` 수(수정 전후), 헤드리스 브라우저로 로드 후 노드 클릭→문서 출현 확인.

## 측정 방법

| 측정 | 성공 기준 |
|---|---|
| 초기 HTML의 `<pre>` 5스텝 전문 수 | 대폭 감소(문서가 초기 DOM에서 빠짐) |
| 초기 DOM 노드 수 | 대폭 감소(≈그래프+메타만) |
| 노드 클릭 → 문서 출현 | 헤드리스에서 클릭 시 5스텝 문서 DOM에 나타남 |
| 외부 리소스 | 0 유지(자기완결) |
| gil-data JSON 문서 내장 | 판정기 WEB-DOCS-EMBEDDED PASS |
| 기존 그래프·계보·메타 | 무손실(노드·엣지·lineage 보존) |

**기각선**: 문서를 빼도 초기 DOM이 유의미하게 안 줄거나, 외부 리소스가 생기거나(자기완결 깨짐), 노드 클릭으로 문서에 닿을 수 없거나, 그래프/계보가 손상되면 기각(1-hypothesis 기각 조건).
