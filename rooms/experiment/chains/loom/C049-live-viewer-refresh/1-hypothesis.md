# 1. 가설 수립 — 새로고침 없는 실시간 관찰

**부모**: loom/C048-sibling-label-spacing
**발의**: 박상현 — "새로고침을 안 눌러도 실시간으로 워크트리의 상태 변화를 관찰할 수 있으면 좋겠다."

## 이전 사이클의 교훈 / 이미 있는 조각

실시간 관찰은 두 조각이 맞물려야 한다:
1. **파일 갱신** — 원장이 바뀌면 뷰어 HTML이 다시 구워져야. **이미 있다**: C042(§5.2) 자동 재굽기 — `gil step`·`close` 등이 커밋 뒤 뷰어를 재생성한다.
2. **브라우저 반영** — 파일이 갱신돼도 브라우저는 수동 새로고침해야 새 내용을 본다. **이것이 빠진 조각.**

## 문제 분할

빠진 것은 **브라우저 자동 리로드**다. gil web은 JS 0줄·외부 리소스 0이 계약(자기완결). JS 폴링은 계약 위반. 그러나 **`<meta http-equiv="refresh" content="N">`** 는 HTML 표준 태그(JS 아님, 외부 리소스 아님)로 N초마다 같은 URL을 리로드한다 — 계약을 지키며 자동 리로드를 준다.

핵심 통합: `--refresh N`을 **bake에 기록**하면(C042 정신: "산출물이 자기 생성 조건을 말한다"), 자동 재굽기가 refresh를 보존한다. 안 그러면 재굽힌 HTML에서 meta가 사라져 리로드가 멈춘다.

## 가설

> **가설**: `gil web --refresh N`이 ① `<head>`에 meta refresh N을 삽입하고 ② `bake`에 refresh를 기록하면, C042 자동 재굽기(gil step 등)와 결합해 **새로고침 없이 실시간 관찰**이 이뤄진다 — gil step이 원장을 바꿀 때마다 파일이 갱신되고 브라우저가 스스로 리로드한다. `--refresh` 없으면 기존과 **바이트 동일**(하위호환).

- **H1 meta 삽입**: `--refresh N` → HTML `<head>`에 `<meta http-equiv="refresh" content="N">`. 자기완결 계약(외부 리소스 0) 유지.
- **H2 bake 보존**: `--refresh N`으로 구운 뷰어를 자동 재굽기(gil step)가 재생성해도 meta refresh가 **유지**된다(bake.refresh에서 읽음).
- **H3 하위호환**: `--refresh` 없으면 meta 없음, bake에 refresh 키 없음 → 기존 산출물과 **바이트 동일**, 두 구현 동일.
- **H4 편의 --watch**(선택): `gil web --watch`는 --refresh를 함축하고 원장 변경을 감시해 재생성 — gil step을 안 거치는 외부 변경(병합·pull)도 반영.

## 기각 조건

1. meta refresh가 자기완결 계약(외부 리소스 0)을 깬다.
2. 자동 재굽기가 refresh를 잃는다(H2 위반 — 리로드가 한 번 후 멈춤).
3. `--refresh` 없는 산출물이 기존과 달라진다(H3 위반, 회귀).
4. 두 구현 web이 달라진다.
