# 5. 결과 보고 — 실시간 자동 리로드를 뷰어 기본값으로 (발의: 박상현)

## 요약
`gil web`의 `--refresh` 기본값을 5초로 두어 옵션 없이도 `<meta http-equiv="refresh">`가 들어가게 했다(실시간이 기본). `--refresh 0`으로 옵트아웃하며, 그 0을 bake에 명시 기록해 C042 재굽기가 옵트아웃을 되돌리지 않는다. bake에 refresh 키가 없는 구버전 뷰어는 재굽으면 기본 5초를 획득한다. 직접 검증 T1~T4 + 신설 conformance WEB-REFRESH-DEFAULT 통과, 회귀 0. **채택(supported).**

## 교훈
1. **"안 되는 것처럼 보임"의 정체는 옵트인이었다.** 실시간의 두 절반(파일 갱신 C042 + 브라우저 반영 C049) 중 후자가 기본에서 꺼져 있었다. 기능이 있어도 기본이 아니면 없는 것과 같다 — 상현님이 로컬에서 겪은 마찰이 그 증거.
2. **기본화는 "명시적 끔"의 기록을 요구한다.** bake 조건을 `if refresh`(truthy)에서 `is not None`으로 바꾼 게 핵심. 기본이 켜짐이 되는 순간, "0(끔)"과 "말 안 함(도구 기본)"을 구별하지 못하면 재굽기가 사용자의 옵트아웃을 지운다. C063의 위계 기본화가 `--flat`이라는 명시 옵트아웃을 요구한 것과 같은 구조.
3. **필드가 이미 다음 문을 열어뒀다.** meta refresh 기본화는 정적 파일 방식의 한계(통째 리로드·고정 주기·상태 리셋)를 남긴다. 이슈 #24가 로컬 HTTP 서버+SSE `--live`를 프로토타입까지 만들어 제안 — 진짜 "앱으로 띄우기"는 그쪽이다.
