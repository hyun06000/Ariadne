# 3. 가설 검증

> **죽은 가지 — 검증 없음.** 이 사이클은 deploy 축 가설을 검증하지 못한 채 절차 위반(열린 부모의 자식 생성)으로 폐기된다. deploy 축 자체의 검증은 C091에서 재출발하는 새 가지에서 수행된다.

## 검증 대신 남기는 것: 위반의 증거

- 위반 사실: `C096.parent = C095-deploy-axis`, C095.status = open (1/5). `gil open`이 부모 닫힘을 요구하지 않아 발생.
- 재현: `python3 gil.py open <chain> <slug> --parent <열린-사이클>` — 부모가 열려 있어도 거부되지 않음(R6는 존재만 검증).
- 이 관찰이 다음 가지의 재료다: "open이 부모 status=closed를 요구해야 하는가"가 새 가지의 후보 가설.
