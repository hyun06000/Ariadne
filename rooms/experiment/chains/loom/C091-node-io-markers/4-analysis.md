# 4. 결과 분석

## 통계적 결과
참조 123/123·Go 105/105(WEB-NODE-IO 신설, 양쪽 PASS). 헤드리스 렌더로 확정 목업(수직 A)과 일치 확인.

## 데이터 직접 관찰
- **방향이 의미를 담았다.** lineage=위로 들어오는 화살표, 배포=아래로 나가는 화살표 — 색(초록/파랑) + 방향(위/아래)의 이중 인코딩이라 색맹 접근성도 유지. 확대 스크린샷에서 화살표 머리·꼬리가 노드에 붙어 하나로 이어짐을 확인(상현님 "붙여줘" 반영).
- **기존 데이터를 이었다.** 배포 화살표는 C007의 released_in, 근거 링크는 C086의 CHANGELOG 근거 사이클 — 새 데이터 0, 이미 있던 두 진실을 노드·패널에 시각화. "뷰어는 새 진실을 만들지 않는다"(loomlight)의 재현.
- **양방향 배포↔사이클.** 노드에선 "이 사이클이 vX 배포됨"(아래 화살표), 배포 패널에선 "이 배포는 이 사이클에서 나옴"(rcyc 링크). 상현님 "어떤 끝단 사이클로부터 배포됐는지"가 뷰어에 양방향으로 드러남.

## 예상과 달랐던 것
- 목업 4차 반복(아이콘→방향화살표→수직→짧게→붙임)이 실제 SVG 좌표(_H_R=9 기준)로 거의 그대로 옮겨졌다 — 목업의 상대 배치가 좌표계에 잘 매핑됐다. 디자인을 먼저 목업으로 확정한 게 구현을 기계적으로 만들었다.

## 판정
**채택(supported).** 노드 수직 화살표(핵심 시각 요구) 참조·Go 완성, 배포 근거 링크 참조 완성. Go 근거 링크·이미지 임베드는 정직한 후속.
