# 5. 결과 보고 — 노드 입출력 마커 + 배포↔근거사이클 (발의: 박상현)

## 요약
상현님 뷰어 요구 둘 — ① 노드의 lineage/배포 표시가 초록 긴 글자로 겹침 ② 배포가 어느 끝단 사이클에서 나왔는지 — 을 해결했다. 확정 목업(수직 A)대로 lineage=노드 위 들어오는 초록 화살표·배포=노드 아래 나가는 파랑 화살표(짧고 노드에 붙음, 이름은 아래로, 호버 툴팁+클릭 상세)로 대체하고, 배포 패널에 근거 사이클 링크(C086 데이터)를 더했다. WEB-NODE-IO 신설, 참조 123/123·Go 105/105. **채택(supported).**

## 교훈
1. **디자인은 목업으로 먼저 확정하면 구현이 기계적이다.** 4차 반복(아이콘→방향→수직→짧게→붙임)으로 상현님과 합의한 뒤 코딩하니, 목업의 상대 배치가 SVG 좌표(_H_R 기준)로 거의 그대로 옮겨졌다. 헤드리스 렌더로 매 반복을 눈으로 확인.
2. **방향이 의미를 담으면 색 없이도 읽힌다.** lineage=위(들어옴)/배포=아래(나감)의 공간 방향 + 색의 이중 인코딩. 아이콘(⤳/⚑)보다 자기설명적이고 색맹 접근성도 낫다.
3. **뷰어는 있는 진실을 잇는다.** 배포 화살표=C007 released_in, 근거 링크=C086 근거 사이클. 새 데이터 없이 두 기존 진실을 노드·패널에 양방향으로 시각화.
