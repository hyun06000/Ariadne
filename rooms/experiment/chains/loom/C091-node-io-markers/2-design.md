# 2. 실험 설계 — 노드 수직 화살표(확정 목업) + 배포↔근거사이클

## 확정 디자인 (상현님과 목업으로 합의)
- **lineage = 노드 위, 들어오는 수직 화살표**(점 → 아래로 → 화살촉이 노드 상단에 붙음), 초록.
- **배포(released_in) = 노드 아래, 나가는 수직 화살표**(노드 하단 → 아래로 → 화살촉 → 점), 파랑.
- 짧고 아담. 머리·꼬리 붙음(선이 화살촉 관통). 노드 이름은 화살표 아래로 내림.
- 이름 없음 — 호버 <title> 툴팁(전체 lineage 이름·배포 버전) + 노드 클릭 상세보기.

## 좌표 (상수: _H_R=9, _H_ROWH=60, 노드 중심 (x,y))
- lineage 점: (x, y-_H_R-13). 선 관통, 화살촉 팁 노드 상단 y-_H_R.
- 배포 선: 노드 하단 y+_H_R에서 시작, 화살촉 아래, 점 (x, y+_H_R+13).
- 노드 번호: y+_H_R+13→ 더 아래로 (배포 화살표와 안 겹치게, 배포 있으면 아래로 더).

## 절차
1. `_render_cycle_graph_h`에서 기존 `lin_txt`(초록 긴 글자) 제거.
2. lineage 있으면 위쪽 수직 화살표 <g>(호버 title=전체 lineage). 배포(released_in) 있으면 아래쪽 화살표(title=vX.Y 배포).
3. 노드 번호 y를 화살표와 안 겹치게 조정(배포 화살표가 아래를 쓰므로 번호를 그 아래로, 또는 노드 위 반대편).
4. **배포 패널 근거 사이클 링크**: `_build_releases_data` entries에 `cycles`(C086 근거) 추가, `_render_releases_panel`이 그 사이클로 링크(#cycdoc-).
5. **CLI**: (releases는 이미 근거 출력) 확인만.
6. Go parity: 같은 세션 내 이식(C094 리듬 — 이월 금지).

## 측정 방법
- 헤드리스 Chrome으로 실제 그래프 렌더: lineage 노드에 위 화살표, 배포 노드에 아래 화살표, 겹침 없음, 이름 안 늘어남.
- 호버 title에 전체 lineage 이름·배포 버전.
- 배포 패널에서 근거 사이클 링크 클릭 → 해당 사이클 상세.
- flat/hierarchy 대칭, 무배포/무lineage 저장소 바이트 영향 최소, WEB 회귀 0.
- 신설 conformance WEB-NODE-IO(마커 존재) + WEB-RELEASE-CYCLE-LINK(근거 링크).
