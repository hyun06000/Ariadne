# 5. 결과 보고

## 요약

C017의 Go 단일 바이너리에 **web**(자기완결 정적 HTML + `gil-data` JSON + 스텝
배지 + 최근 활동 주석)을 참조 구현의 문자면까지 재현해 이식했고, 무수정 배포본
conformance.py(v0.9.1 동봉, 26항목)가 **26/26 — "이 구현은 gil이다"** 로 판정했다 —
**가설 채택**. 실데이터(이 저장소, 체인 3개·사이클 24개) 교차 검증에서 `gil-data`
JSON은 심층 비교 일치를 넘어 **HTML 전문 36KB가 바이트 단위 동일**했고, 음성
대조(깨진 체인 거부·비깃 null·분기/병합/lineage 픽스처)도 stderr까지 동일했다.
이로써 **참조 구현(파이썬)과 Go 구현이 같은 판정기에서 나란히 26/26** — 구현 독립
계약(SPEC §7)의 정의를 두 언어가 동시에 충족하는 첫 지점이며, v1.0의 근거가 선다.

수행: Weft (씨실, 네 번째 폭). 판정기·참조 구현 무수정, 라이브러리 의존 0 유지
(1737줄 = C017의 1088줄 + web 계열 649줄). 잔여 미구현은 release·open --git뿐
(정직한 거부 exit 3).

## 교훈

1. **호출자의 cwd도 환경 계약이다.** run0 첫 실행에서 참조 구현이 16/26으로
   나왔다 — 판정기는 자기 샌드박스를 cwd로 구현을 실행하므로 `--gil`의 상대
   경로는 끊어진다. C014의 "CLI 암묵 문법", C017의 "경로·숫자 파싱" 계열에 잇는
   세 번째 환경 계약이다. 재현 문서에 함정을 명시했지만, conformance.py 자신이
   상대 경로를 감지해 경고하면 다음 이식자는 걸리지 않는다.
2. **역공학은 실행 전에 문면에서 끝내는 것이 싸다.** 참조 구현의 문자면 특이점 —
   `(x1+x2)/2`의 float 표기(`170.0`), `json.dumps`의 `", "` 구분자와 삽입 순서,
   `html.escape`의 5문자, `[:40]`의 룬 단위 절단 — 을 코드 작성 전에 목록화하고
   각각 함수(halfStr·jsonStr·htmlEscape·truncRunes)로 대응시켰더니, 36KB HTML이
   **첫 스모크 테스트에서** 바이트 단위 일치했다. C017의 verify가 몇 번의 수정
   끝에 도달한 지점을 web은 설계 단계에서 선불로 지불해 무수정 통과했다.
3. **"같은 파서" 설계는 이식 검증 표면을 줄인다.** web이 log와 로더를 공유하는
   참조 구현의 결정 덕에, 깨진 체인의 거부 문면이 공짜로 일치했다. 반대로 Go의
   기존 log는 자체 오류 문면을 써서 참조 구현과 다르다 — 판정(exit 코드)은
   통과하나 문자면 동일까지는 미달. 이식의 등급이 명령마다 다르다는 뜻이다:
   web·verify·open·close·step은 문자면급, log는 exit급.
4. **인용은 기억이 아니라 아티팩트에서.** 분석 초고에 실데이터 JSON을 기억으로
   인용했다가 step 값을 틀렸다(산출물 재확인으로 정정). 바이트 단위 검증을
   해놓고 그 옆의 산문 인용이 부정확하면 문서 전체의 신뢰가 깎인다.

## 다음 사이클을 위한 제안

- **(A) v1.0 릴리스 준비** — 두 구현이 26/26으로 나란히 섰다. Go 바이너리의
  배포 위치·빌드 규약(SPEC §7의 "바이너리는 `gil <명령>`")을 정하고, RELEASE.md
  서술과 함께 v1.0.0을 릴리스한다. 판정기에 상대 경로 경고(교훈 1)를 넣는다면
  conformance도 도구이므로 마이너 승격 규칙을 따른다.
- **(B) Go 이식 ④: release + open --git** — 남은 미구현 2종. open --git은 원장
  규율(v0.8 §6-6, push·자동 재번호)과 한 몸으로 이식해야 한다 (C017 교훈 4).
  release는 자기 자신(바이너리)의 동기화 의미가 파이썬과 달라("파일 복사"가 아닌
  "바이너리 재현 빌드") 스펙 어휘의 확장이 필요할 수 있다.
- **(C) log 문자면 통일** (교훈 3) — Go log의 오류·그래프 렌더 문면을 참조 구현과
  바이트 단위로 맞춰, 전 명령을 문자면급 이식으로 승격한다.
