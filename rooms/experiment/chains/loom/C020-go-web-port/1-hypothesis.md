# 1. 가설 수립

## 이전 사이클의 교훈

부모 [C017-go-git-binding](../C017-go-git-binding/5-report.md)에서:

1. **판정기도 걷는다.** 계약 준수는 판정기 버전에 상대적이다 — C017 시점의 배포본
   판정기(v0.8 동봉, 26항목)에서 Go 바이너리는 **24/26**이었고, 남은 실질 FAIL은
   범위 밖으로 미뤄 둔 **WEB 2종**(WEB-SELFCONTAINED · WEB-JSON)뿐이었다.
   현재 배포본은 v0.9.1이지만 conformance.py의 판정 항목은 26개 그대로다.
2. **판정기가 보는 면보다 한 겹 넓게 옮겨라.** exit 코드는 스위트의 계약이고
   출력 포맷은 사람의 계약이다 — 구현 교체(파이썬→Go)가 운영자에게 들키지 않아야
   진짜 교체다. C017의 verify는 실데이터에서 바이트 단위 동일까지 끌어올렸다.
3. C017의 제안 (A): **Go 이식 ③ web** — 문자열 조립로 충분하며, `gil-data` JSON
   훅과 자기완결성(외부 리소스 0)이 판정 대상이다. 이것이 닫히면 Go 바이너리는
   release를 제외한 전 계약을 지킨다.

## 문제 분할

Go 이식 로드맵의 잔여는 ③ web, ④ release, ⑤ open의 깃 경로(원장 규율)다.
이번 폭은 **③ web 하나만** 정복한다:

- **web이 가장 작다**: 읽기 전용 명령이고(쓰기 규율·원상 복구 불필요), C017까지
  이식된 파서·토폴로지·fsck를 그대로 재사용한다. 새로 필요한 것은 레이아웃 계산
  (`_layout_columns`), SVG/HTML 문자열 조립, JSON 직렬화, 그리고 깃 가용 시의
  최근 활동 주석(`_last_activity`·`_ago`)뿐이다.
- **이것으로 26/26이 완성된다**: 판정 26항목 중 release는 항목이 없으므로,
  WEB 2종을 닫으면 참조 구현과 Go 구현이 같은 판정기에서 나란히 26/26이 된다 —
  구현 독립 계약(SPEC §7)의 "전 항목 통과 = 이 구현은 gil이다"를 두 언어가
  동시에 충족하는 첫 지점이다.
- ④ release와 ⑤ open --git(원장 규율과 한 몸)은 규모·위험이 다르므로 다음
  사이클로 미룬다 (C017 교훈 4: 반쪽 이식은 무이식보다 위험하다).

계약의 원문 (SPEC §5·§7):

- `gil web -o out.html` — **자기완결적** 정적 HTML 뷰어 (같은 파서, SVG 렌더,
  **외부 리소스 0**). 그래프 구조는 `<script type="application/json" id="gil-data">`로
  내장(**step 포함**) — 기계 검증의 훅. 열린 사이클은 스텝 인디케이터(●●●○○ n/5) 표시.
- 최근 활동 주석(v0.7): 열린 사이클의 마지막 커밋 시각·제목 — 깃이 없으면 조용히
  생략 (web은 깃 무의존).

## 가설

> **가설**: C017의 Go 단일 바이너리에 참조 구현(gil.py v0.9.1)의 web을 —
> `_layout_columns`·SVG/HTML 렌더·`gil-data` JSON·스텝 배지·최근 활동 주석까지 —
> 같은 조립 규칙으로 이식하면, 무수정 배포본 conformance.py가 **26/26**으로
> 판정하고, 실데이터(이 저장소)에서 Go의 `gil-data` JSON이 참조 구현과
> **구조 동일**(파싱 후 심층 비교 일치)하게 나온다.

## 기각 조건

다음 중 하나라도 나오면 가설을 기각한다:

1. 무수정 배포본 `conformance.py --gil "<Go 바이너리>"`가 26/26 미만
   (특히 WEB-SELFCONTAINED 또는 WEB-JSON이 FAIL로 남는 경우).
2. 기존 24항목 중 하나라도 PASS→FAIL로 퇴행.
3. 실데이터 교차 검증에서 Go와 참조 구현의 `gil-data` JSON이 파싱 후 심층 비교로
   불일치 (단, 생성 시각 경계에서만 갈리는 `last_activity.ago`의 분·시간 표기
   차이는 동시 실행으로 통제하고, 통제 후에도 남으면 불일치로 판정).
4. 참조 구현 자신이 같은 판정기에서 26/26이 아니다 (판정기 건전성 상실 —
   이 경우 가설 검증 불능으로 중단).
