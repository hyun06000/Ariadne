## 데이터 직접 관찰 — 프로브가 잡은 진짜 결함

- **`**`는 세그먼트를 넘지 않는다**: `refs/tags/cycle/**`는 `cycle/probe-sacrifice`(1레벨)를 잡았으나 `cycle/loom/C001`(2레벨)을 놓쳤다. `refs/tags/**`로 넓혀도 `v1.0.0`만 잡고 2레벨 cycle 태그는 여전히 통과 — GitHub 룰셋의 `**`가 fnmatch `*`처럼 슬래시를 넘지 않음이 실측으로 확정됐다. `~ALL`이 유일하게 확실한 해법이었다.
- **프로브가 없었다면 방어선은 거짓 안심이었다**: 룰셋이 "active"로 떠 있고 1레벨 태그(v*)는 실제로 막혔으므로, 프로브 없이 콘솔만 봤다면 "보호 완료"로 오판했을 것이다. 우리 사이클의 태그는 전부 2레벨(cycle/체인/사이클) — 정확히 무방비였던 것들이다. C011의 교훈("스위트가 진짜 뭔가를 잡는지 변이로 확인")이 인프라에서 재현됐다.

## 예상과 달랐던 것

- 파괴 프로브가 실데이터(진짜 C001 태그)를 3회 삭제했다 — 선고정한 복구 계획(로컬 태그 재push) 덕에 매번 무손실 복구. 로컬이 원본의 사본이라는 사실이 안전망이었다. 파괴적 검증에는 복구 계획의 선고정이 필수다.
