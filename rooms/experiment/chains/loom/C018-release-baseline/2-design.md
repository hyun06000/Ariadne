# 2. 실험 설계

## 절차 (판정 표를 겸해 선고정)

1. **구현** — cmd_release: `_changed_vs_last_tag(repo, tag, 경로)` — 태그 blob의 내용과 작업 트리 파일을 비교(태그에 없던 파일이 생겼으면 변경). tool_changed = gil.py 변경 ∨ conformance.py 변경. 태그가 없으면(첫 릴리스) 승격 규칙 미적용(현행 유지). CHANGELOG 항목에 무엇이 변했는지 명시.
2. **T1 (기준 교정)** — 샌드박스: 구버전 gil.py를 커밋·태그(v0.1.0) 후 실행 도구와 같은 내용으로 갱신 커밋(= 패키지 도구 직접 실행과 동등한 상태). `release 0.1.1` → 거부돼야 함(태그 기준 변경 감지). `release 0.2.0` → 성공.
3. **T2 (판정기 = 도구)** — gil.py는 태그와 동일, conformance.py만 변경 → `0.1.1` 거부, `0.2.0` 성공.
4. **T3 (문서-only 회귀)** — 태그 대비 문서만 변경 → `0.1.1` 성공 + "문서 릴리스" 분류.
5. **T4** — conformance 26/26 회귀. 릴리스 v0.9.0 (이 수정 자체가 도구 변경 — 새 규칙의 첫 준수자).
