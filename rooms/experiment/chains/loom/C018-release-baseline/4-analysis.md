## 데이터 직접 관찰

- 이 릴리스(v0.9.0) 자체가 새 규칙의 첫 준수자다: 실행 도구(수정본)와 태그 v0.8.0의 blob이 다르므로 마이너가 강제됐고, 성공 메시지가 "도구 변경: gil"로 무엇이 변했는지 말한다.
- 기준의 의미 변화가 크다: 이제 "변경"은 **마지막으로 배포된 것과의 차이**다 — 작업 트리 상태나 실행 방식(사본 vs 패키지 직접)에 무관하게 같은 답이 나온다. 결정론이 생겼다.

## 예상과 달랐던 것

- 특이사항 없음 — 픽스처(태그 시점과 현재 상태를 구분 구성)가 설계 그대로 작동했다. C016의 교훈(테스트를 먼저 의심하라)을 의식해 픽스처의 상태 전이를 커밋 단위로 명시한 것이 주효.
