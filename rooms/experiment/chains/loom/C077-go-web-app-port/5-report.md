# 5. 결과 보고

## 요약

C075의 완전한 앱화(문서 gil-data JSON 내장 + 클릭 시 JS DOM 구축 + `_json_for_script` 봉인)를 Go `main.go`에 6조각으로 동형 이식했다. 참조↔Go web **바이트 동일**(hierarchy·flat·per-chain), Go 판정기 **86/86**(WEB-DOCS-EMBEDDED PASS, 회귀 0), `<\/` 치환 개수 일치. "두 몸, 한 계약"이 web 앱화에서도 회복돼 gil-gate가 초록으로 돌아온다. **채택.**

## 교훈

1. **정적 JS 문자열은 parity를 공짜로 만든다.** `webAppJS`를 참조와 바이트 동일하게 옮기면 언어 무관하게 브라우저 동작이 같다 — 좌표 없는 마크업(C064·C067)과 같은 원리의 앱판. parity 검증이 `diff` 한 줄로 끝났다.
2. **guard는 author를 보지 예약을 안 본다.** 소환자의 "예약된 open은 통과"라는 예측이 코드와 어긋났다(`_guard_primary_owner`에 예약 예외 없음). 씨실은 우회하지 않고 정직히 멈춰 보고했고, 소환자가 guard를 임시 해제해 weft author로 원장에 새겼다 — **작업자와 기록이 일치**한다. 이 갭(예약된 open을 예약 대상이 하면 허용)은 개선 후보.
3. **범위 이월이 다음 사이클을 정확히 만든다.** C075가 "참조 앱화까지"로 자르고 Go를 이월한 덕에, 이 사이클의 임무·검증 기준·기각선이 처음부터 선명했다(C043 리듬 재확인).
