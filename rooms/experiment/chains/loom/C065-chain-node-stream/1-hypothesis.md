# 1. 가설 수립

## 이전 사이클의 교훈

부모 loom/C064가 위계 상단에 L0 체인 지도를 얹었다. 그러나 지도의 원을 눌러 열리는 **펼쳐진 체인 뷰**는 (a) 단일 체인 SVG 그래프라 교차-체인 lineage가 안 보였고, (b) 그래프 노드가 클릭 불가라 5스텝 문서로 못 갔고, (c) 카드(그래프+표)로 박혀 있어 "저 아래 따로 펼쳐지는" 느낌이었다. 상현님 네 차례 지적으로 방향이 잡혔다: **"카드 잡지 말고, 체인을 누르면 연결된 사이클 노드가 주루룩 나오고, 노드를 누르면 5단계 문서를, lineage도 보이게, 지도 바로 아래로 펼쳐지게."**

## 문제 분할

1. **카드·SVG그래프·표 제거** → 체인 body를 좌측 레일에 매달린 **노드 스트림**(`<ol class="cycstream">`)으로. 각 사이클 = 한 노드 행(● cid · 상태).
2. **lineage 가시화** → 각 노드에 원천을 초록 칩(`⇠ chain/cid`)으로, 잘리지 않고 줄바꿈. 칩은 그 원천 사이클로 점프하는 링크.
3. **노드 클릭 → 5스텝 인라인** → 노드가 곧 `<details>`, 열면 그 자리 아래로 메타+5스텝이 흐른다. (그래프 노드를 따로 만들 필요 없이 스트림 행이 노드이자 문서 입구.)
4. **지도 바로 아래 아코디언** → 체인 `<details name="hchain">` 네이티브 배타 아코디언 — 하나 열면 나머지 접힘. JS 0 유지.

넷은 한 재설계로 함께 풀린다(노드=행이 핵심). 위험: 양 구현 바이트 동일 유지.

## 가설

> **가설**: 펼쳐진 체인 뷰를 카드+SVG 대신 `<ol>` 노드 스트림으로 바꾸고(lineage 칩·노드=`<details>`·`name` 아코디언), 이를 hierarchy 전용 렌더 경로에만 적용하면, 네 요청(카드 제거·lineage 가시·노드 클릭 5스텝·인라인 아코디언)을 JS 0으로 충족하면서 flat 뷰어는 불변, 양 구현(gil.py·go/main.go) 기본·`--flat` 출력이 여전히 바이트 동일하다.

## 기각 조건

- 두 구현의 기본 또는 `--flat` 출력이 1바이트라도 다르면 기각.
- 기존 WEB-* 판정기(참조 90·Go 83)가 하나라도 깨지면 기각.
- 노드 클릭 시 5스텝이 안 열리거나, 지도 원 클릭 시 아코디언이 안 펼쳐지면 기각.
