## 데이터 직접 관찰

- `node-stream.png`: loomlight C001 노드에 lineage 원천 9개가 초록 칩으로 **전부** 보인다(⇠ loom/C005 … loom/C049) — 이전엔 SVG 텍스트로 잘려 "리니지가 하나도 안 보이던" 자리. 각 칩은 그 원천 사이클로 점프하는 링크.
- `cycle-open.png`: `#cyc-loomlight-C003-…`로 이동하니 loomlight 아코디언과 C003 노드가 **함께** 열려, 좌측 레일 아래로 메타표(status·parent·lineage…)와 5스텝 토글(+1 가설 … +5 보고)이 인라인으로 흘렀다. 카드 테두리 없이 레일 하나로 연결된 스트림.
- `parity.txt`: 구 per-chain SVG그래프(aria-label "사이클 체인 그래프") 0개 — 카드+그래프가 실제로 제거됐고, L0 체인 지도(aria-label "체인 지도")는 별개로 유지.

## 예상과 달랐던 것

- **네 요청이 하나의 재설계로 수렴했다.** "카드 제거·주루룩 노드·lineage 가시·노드 클릭 5스텝·아코디언"은 따로가 아니라 **"스트림 행이 곧 노드이자 문서 입구"** 라는 한 결정으로 전부 풀렸다. 별도의 SVG 노드-링크(초기 C064식 접근)나 ghost 노드가 불필요해졌다.
- **좌표 없는 재설계라 parity가 자명했다.** C064는 삼각함수 ULP로 parity가 위태로웠는데, 이번엔 순수 HTML/CSS 문자열(정수 좌표조차 없음)이라 Go 이식이 곧 바이트 동일. 렌더 복잡도를 기하에서 마크업으로 옮긴 것이 parity를 공짜로 만들었다.
- **네이티브 `<details name>` 아코디언**이 JS 0으로 "하나 열면 나머지 접힘 + 지도 바로 아래 인라인"을 정확히 줬다 — 별도 `:target`/`:has()` 트릭 불필요.

## 판정

**가설 채택.** 기각 조건 셋 모두 미발동: 바이트 불일치 0(기본·`--flat`), 판정기 회귀 0, 노드 클릭 5스텝·아코디언 펼침 동작. 네 요청을 JS 0으로 한 재설계에 담았다.
