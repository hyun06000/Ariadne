# 5. 결과 보고

## 요약

위계의 "펼쳐진 체인" 뷰를 카드+SVG그래프+표에서 **좌측 레일 노드 스트림**으로 재설계했다(loom/C065): 체인을 누르면(`<details name="hchain">` 배타 아코디언, 지도 바로 아래) 사이클 노드가 주루룩 흐르고, 각 노드에 lineage 원천이 초록 칩(점프 링크)으로 다 보이며, 노드를 누르면 그 자리 아래로 5스텝 문서가 인라인으로 펼쳐진다. 좌표 계산이 전혀 없는 순수 HTML/CSS라 flat 불변·양 구현 **바이트 동일**(참조 90/90·Go 83/83) — 가설 채택.

## 교훈

1. **여러 UX 요청이 하나의 구조 결정으로 수렴할 수 있다.** "카드 제거·주루룩·lineage 가시·클릭 5스텝·아코디언" 네 가지는 **"스트림 행 = 노드 = 문서 입구"** 라는 단일 결정으로 전부 풀렸다. 요청을 각각 기능으로 쌓지 말고, 그것들을 동시에 만족시키는 하나의 표현을 찾는 것이 적은 코드·적은 parity 위험을 낳는다.
2. **렌더 복잡도를 기하에서 마크업으로 옮기면 parity가 공짜가 된다.** C064는 SVG 좌표(삼각함수)로 Python↔Go ULP parity가 위태로웠다. 이번엔 정수 좌표조차 없는 순수 마크업이라 이식이 곧 바이트 동일. **가능하면 계산을 렌더에서 빼라** — 계약(양 구현 바이트 동일)이 쉬워진다.
3. **네이티브 HTML이 JS 없이 상호작용을 준다.** `<details name>` 배타 아코디언 + 프래그먼트→조상 details 자동 오픈만으로 "지도 클릭→인라인 아코디언 펼침→노드 클릭→5스텝"의 전체 드릴다운을 JS 0으로 구현. `:target`/`:has()` 같은 트릭도 불필요했다. 플랫폼의 표준 기능을 먼저 확인하라(C063·C064 "JS 0" 계승).
4. **정직한 방향 전환.** 처음엔 C064식 SVG-ghost(그래프에 외부 lineage 노드 추가)로 갔으나, 상현님 "따로 카드 잡지 말고"에 그 방향을 버리고 노드 스트림으로 선회 — 미완성 실험 변경을 되돌리고 새로 짰다. 사용자의 상(想)이 명확해지면 앞선 코드에 매몰되지 않는다.
