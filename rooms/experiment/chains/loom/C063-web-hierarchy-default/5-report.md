# 5. 결과 보고

## 요약

`gil web`의 기본을 평면 SVG에서 **위계(드릴다운)로 뒤집고**(`hierarchy = not --flat`), 평면은 `--flat` 옵트아웃으로 보존, `--hierarchy`는 하위호환 no-op 별칭으로 남겼다. 양 구현(gil.py·go/main.go)에 문자 동형 이식 → 무옵션·`--flat` 모두 **바이트 동일**, 참조 90/90·Go 83/83 회귀 0. 새 판정기 `WEB-HIERARCHY-DEFAULT`가 기본=위계를 잠그고 변경 전 바이너리(기본=평면)를 82/83으로 격추 — 가설 채택.

## 교훈

1. **기본값 뒤집기의 안전한 형태는 "제거 없는 반전 + 옵트아웃"이다.** `--hierarchy`를 지우지 않고 no-op 별칭으로 두니 기존 스크립트·근육기억이 깨지지 않고, `--flat`이 평면 계약을 온전히 보존한다. 파괴 없는 계약 변경.
2. **렌더는 계약이 아니지만, "어떤 렌더가 기본인가"는 잠글 가치가 있다.** conformance는 무수정으로도 통과했다(계약면=`gil-data` JSON). 그럼에도 판정기를 추가한 이유는, 뒤집힌 기본값이 다음 리팩터에서 조용히 되돌아가는 회귀를 격추하기 위함 — 계약을 넓히는 게 아니라 **의도를 증언**하는 판정기.
3. **변이 대조군은 "변경 전 바이너리"로 공짜로 얻는다.** 별도 mutant를 만들지 않고 v2.18.0 빌드를 그대로 conformance에 걸어 판정기의 비공허성을 증명했다(82/83). C038 쌍 검증의 값싼 형태.
4. **참조 구현은 절대경로로 호출해야 한다** — conformance 샌드박스가 cwd를 바꾸므로 `python3 gil.py`(상대)는 rc=2로 무더기 위양성 FAIL을 낸다. 도구가 아니라 호출자의 함정.
