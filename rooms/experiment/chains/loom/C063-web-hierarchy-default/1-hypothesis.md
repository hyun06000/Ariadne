# 1. 가설 수립

## 이전 사이클의 교훈

lineage로 잇는 loomlight/C002(위계 뷰어)·C003(Go 이식)이 `--hierarchy` 위계(드릴다운) 뷰어를 **opt-in**으로 낳았다: 체인 목록 → 체인 그래프 → 사이클 5스텝을 JS 없이 `<details>`로 연다. 그러나 opt-in인 탓에 기본 `gil web`은 여전히 평면 SVG 그래프를 굽고, 위계는 플래그를 아는 사람만 본다. 부모 loom/C062까지의 규율: **무옵션 바이트는 양 구현(gil.py·go/main.go)이 동일**해야 하고, 렌더 형식은 계약이 아니되 `gil-data` JSON(§7)이 계약면이다.

## 문제 분할

박상현 발의: **"`gil web --hierarchy`를 디폴트로."** 분할하면 —
1. 기본값 뒤집기 (양 구현) — 무옵션 = 위계.
2. 옵트아웃 경로 — 기존 평면 뷰어를 잃지 않도록 `--flat` 신설. `--hierarchy`는 하위호환 no-op 별칭.
3. 계약 잠금 — conformance 판정기가 "기본은 위계"를 증언하고, 회귀(기본이 평면으로 되돌아감)를 격추.

셋은 분리 불가다 — 한쪽만 바꾸면 parity가 깨진다. 지금 정복할 첫 문제는 **1·2를 양 구현에 대칭 이식하고 3으로 잠그는 것**.

## 가설

> **가설**: `--hierarchy`를 제거하지 않고 기본값만 뒤집어(`hierarchy = not --flat`) 양 구현에 대칭 이식하면, 무옵션 출력이 위계로 바뀌면서도 (a) 두 구현이 여전히 바이트 동일하고 (b) `--flat`으로 기존 평면 계약이 보존되며 (c) 새 판정기 WEB-HIERARCHY-DEFAULT가 기본=위계를 증언하고 회귀를 격추한다.

## 기각 조건

- 두 구현의 무옵션(또는 `--flat`) 출력이 바이트 불일치하면 기각.
- 기존 WEB-* 판정기(SELFCONTAINED·JSON·REFRESH·AUTO-*)가 하나라도 깨지면 기각.
- 새 판정기가 변경 전 바이너리(기본=평면)를 격추하지 못하면(공허한 판정) 기각.
