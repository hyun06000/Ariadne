# 4. 결과 분석

| # | 항목 | 결과 |
|---|---|---|
| 1 | 조회 | ✅ 메타·태그·분기 안내, 두 구현 바이트 동일 |
| 2 | 체크아웃 | ✅ 시점 역행 왕복, 미커밋 시 유실 방지 거부 |
| 3 | 엣지 | ✅ 열린 사이클(태그 없음 고지)·없는 사이클(오류) |
| 4 | 분기 재시작 | ✅ 안내대로 open --parent → 실제 분기 |
| 5 | 회귀 | ✅ conformance 26/26 (양 구현) |

## 데이터 직접 관찰

- goto의 세 줄 출력이 사용자의 세 니즈에 정확히 대응한다: "어디였나"(메타), "그때로 가자"(태그+checkout), "다른 길로"(분기 안내). 타임머신의 세 동작이 한 명령에 모였다.
- `--checkout`의 미커밋 거부가 핵심 안전장치다 — 되감기가 현재 작업을 삼키지 않는다. 유실 방지를 위해 거부하고 커밋/스태시를 안내한다.
- 실증된 아리아드네의 역행: C001에서 wrong-path(막다른 길)와 better-path가 **둘 다 그래프에 남았다**. 되돌아감은 wrong-path를 지우는 게 아니라 C001에서 새 실을 뽑는 것 — 죽은 갈래도 교훈으로 보존된다.

## 예상과 달랐던 것

- 첫 검증 실패가 또 픽스처였다(git 기본 브랜치 master, detached 오염). C016·C021에 이어 세 번째 "테스트를 먼저 의심하라". 도구 검증에서 환경 기본값(브랜치명)은 반복되는 함정 — 픽스처는 `-b main`을 명시해야 한다.

## 판정

**채택.** 기각 조건 3건 중 발동 0건 (픽스처 결함은 교정되어 goto 무결함 확인).
