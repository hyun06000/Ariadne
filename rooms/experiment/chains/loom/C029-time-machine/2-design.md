# 2. 실험 설계

1. **구현(참조·Go)** — `gil goto <chain>/<id> [--checkout] [--root]`:
   - ref를 `<chain>/<id>`로 파싱, 사이클 디렉토리 존재 확인.
   - cycle.yaml 읽기 → 제목·상태·부모·계보 출력.
   - 깃 저장소면 태그 `cycle/<chain>/<id>` 존재·커밋 조회 → "코드 상태로 역행: git checkout <tag>" 안내(닫힌 사이클).
   - 항상: "이 지점에서 새 갈래: gil open <chain> <slug> --parent <id>" 분기 안내.
   - `--checkout`: 깃 저장소 + 태그 존재 + 미커밋 변경 없음이면 `git checkout <tag>` 실행, "돌아오려면: git checkout <원래브랜치>" 안내. 미커밋 있으면 거부(유실 방지).
2. **T1 조회** — 닫힌 사이클 goto → 메타·태그·분기 안내 전부. 두 구현 출력 대조(경로 무관 부분 동일).
3. **T2 체크아웃 왕복** — 샌드박스에서 --checkout → 그 시점 트리 확인 → 복귀. 미커밋 변경 있으면 거부 확인.
4. **T3 엣지** — 없는 사이클/열린 사이클(태그 없음) goto → 명시적·적절한 안내(오류 아님, 태그 없음 고지).
5. **T4 분기 왕복** — goto 안내대로 open --parent 하면 실제 분기 생성.
6. **T5 회귀** — conformance × 두 구현 26/26.
7. **릴리스** — v1.3.0(도구 변경) + 바이너리 재배포, SPEC §5·README.ai.md에 goto 추가.
