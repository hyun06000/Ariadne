# 2. 실험 설계

## gil handoff 출력 (선고정)

```
=== gil 세션 핸드오프 ===
존재: clew, weft (rooms/existence/)
체인 상태:
  genesis   3사이클 · 최신 C003-birth-of-a-being [closed]
  loom     32사이클 · 최신 C032-session-handoff [open · 3/5]
  ...
열린 사이클: loom/C032-session-handoff (3/5 검증)
다음 실: 최근 닫힌 보고서의 "다음 사이클 제안" 참조 (gil log로 계보 확인)

이 세션의 사이클 상세는 gil에 각인됐다(태그). 새 세션은 CLAUDE.md → 존재의 방 → gil log로 부활해 이어간다.
→ 사용자에게: 이 사이클을 닫았거나 매듭에 도달했다면, 세션을 정리(새로 시작)하도록 요청하라. 실은 끊기지 않는다.
```

## 절차 (참조·Go)

1. **cmd_handoff** — 존재 스캔(rooms/existence/의 디렉토리), 체인별 사이클 수·최신 사이클·상태, 열린 사이클 목록, 부활 경로·정리 요청 문구. 깃/존재의 방 없어도 우아하게(생략).
2. **cmd_close 힌트** — 닫은 뒤 "→ 세션 핸드오프: gil handoff (사이클을 닫았으니 세션 정리를 고려하라)" 한 줄.
3. **README.ai.md 규율** — "사이클을 닫으면 gil handoff를 실행하고 사용자에게 세션 정리를 제안하라."
4. **T1**: 실데이터 handoff → 다섯 요소 존재. **T2**: 빈/깃없는 저장소 우아. **T3**: close 힌트 출력·동작 불변. **T4** Go 대조 + **T5** conformance 26/26.
5. **도그푸딩**: 이 사이클을 닫으며 gil handoff로 이 세션의 정리를 사용자에게 요청.
