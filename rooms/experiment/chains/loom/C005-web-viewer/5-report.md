# 5. 결과 보고

## 요약

"같은 파서로 렌더러만 바꾸면 체인 그래프는 자기완결적 정적 HTML이 된다"를 선고정 기대 구조 6판정과 실데이터로 검증했다.
6/6 통과, 실데이터 구조 일치, 외부 리소스 0 — **가설 채택**.
상현님의 두 번째 질문 "사이클 체인 그래프를 웹 UI로"에 대한 답이 생겼다: `ari web` 한 명령, 서버 없음, 어디서든 열린다.

## 교훈

1. **"같은 파서, 다른 렌더러" 원칙이 검증을 공짜로 만든다.** 내장 JSON과 기대 집합의 비교라는 기계 판정 덕에, 터미널 log와 웹 뷰어가 다른 이야기를 할 수 없는 구조가 됐다. 뷰어를 늘릴 때마다(향후: Actions 자동 배포, Hub) 이 원칙을 유지할 것.
2. **접근성 요구가 문서를 완성했다.** 색+모양 이중 인코딩과 테이블 폴백은 규정 준수로 넣었지만, 결과적으로 테이블은 "ari log의 HTML판"이라는 독립 가치를 얻었다.
3. **알려진 한계 2건**: 레인 폭이 정적(긴 id에서 겹침 가능), lineage 간선의 시각 밀도(체인 다수 시 교차) — 규모가 실제로 커질 때의 사이클 재료다.
4. 웹 뷰어는 깃조차 필요 없다 — 표준 라이브러리만으로 동작한다. 범용 하네스의 설치 장벽 원칙이 끝까지 유지됐다.
