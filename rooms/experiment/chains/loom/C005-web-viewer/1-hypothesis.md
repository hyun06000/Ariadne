# 1. 가설 수립

## 이전 사이클의 교훈

부모: [loom/C004](../C004-git-binding/5-report.md) (채택·닫힘).

- CLI 스택이 완성됐다: log(읽기) · fsck(검증) · open/close(쓰기) · verify(무결성). 그러나 이 모든 것은 터미널의 것이다 — 상현님의 두 번째 질문 "**사이클 체인 그래프를 웹 UI로 볼 수 있으려면?**"이 남아 있다.
- 아키텍처 결정(2026-07-13): 웹 뷰어는 **같은 파서, 다른 렌더러**다. 서버 없이 정적 HTML을 생성해 GitHub Pages 등으로 서빙한다. 자기완결(외부 리소스 0)이어야 CSP·오프라인·호스팅 어디서든 동작한다.
- C002의 산출물 lineage 필드 덕에 체인 간 간선을 그릴 수 있게 됐다 (C004 보고서 제안 A).

## 문제 분할

"웹 뷰어"는 (a) 그래프 데이터의 HTML 내장 — 파서의 재구성 결과를 기계 검증 가능한 형태로 싣기,
(b) 시각 렌더 — 분기·병합·체인 간 lineage가 보이는 SVG, (c) 배포 자동화 — GitHub Actions + Pages,
(d) 상호작용(줌·필터 등)으로 나뉜다. (c)는 (a)(b)가 검증된 뒤의 얇은 배선이고 (d)는 사치다.
이번에 정복할 가장 작은 문제는 **(a)+(b): 구조가 옳고 자기완결적인 정적 HTML 한 장**.

## 가설

> **가설**: `ari web`이 log와 **같은 파서**의 재구성 결과를 (a) 기계 검증 가능한 JSON으로 내장하고 (b) SVG로 렌더한 **자기완결적 정적 HTML**을 생성하면, ① 내장 JSON의 구조(노드·간선·분기·병합·lineage)는 파서의 재구성과 동일하고, ② 체인 간 lineage 간선이 시각 요소로 존재하며, ③ 문서는 외부 리소스 참조가 0이고, ④ 깨진 체인에서는 부분 산출물 없이 거부한다.

## 기각 조건

1. **구조 불일치**: 내장 JSON이 기대 구조(선고정)와 다르다 — 노드/간선의 누락·추가·왜곡.
2. **표현 누락**: 분기·병합·lineage 중 시각 요소로 렌더되지 않는 것이 있다.
3. **자기완결 실패**: 생성물에 외부 URL 참조(스크립트·스타일·폰트·이미지)가 존재한다.
4. **문지기 실패**: 깨진 체인에서 exit 0이거나 부분 파일을 남긴다.
