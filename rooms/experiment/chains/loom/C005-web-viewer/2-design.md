# 2. 실험 설계

## 절차

1. **기대 구조 고정** — 픽스처별 내장 JSON의 기대 구조(노드·간선·분기·병합·lineage 집합)와 자기완결 판정 규칙을 `3-verification/fixtures/expected-structure.md`에 구현보다 먼저 기록한다.
2. **픽스처 계승** — 이전 사이클의 픽스처를 재사용한다: C001의 test-maze(분기·병합), C002의 good(alpha/beta — 체인 간 lineage), C001의 broken(끊어진 parent).
3. **구현** — ari v5: `web` 서브커맨드. log와 같은 파서·그래프 재구성을 사용하고, 렌더만 다르다: 레인형 SVG(체인당 세로 레인, 분기는 열 오프셋, lineage는 레인을 건너는 점선) + 접근성 테이블 + `<script type="application/json" id="ari-data">` 내장. 디자인은 dataviz 검증 팔레트의 토큰을 따르고(상태는 색+모양 이중 인코딩: 닫힘=채운 원, 열림=빈 원), 라이트/다크 양 테마를 토큰 레벨로 지원한다. 외부 리소스 0.
4. **테스트 실행** — T1~T6: JSON 추출·집합 비교(기계 판정), 시각 요소 존재(SVG 노드 수·점선 lineage path), 자기완결(외부 URL 정규식 0건), 깨진 체인 거부(부분 파일 없음), HTML 파싱 가능성(표준 파서 무오류).
5. **실데이터** — 실제 레포에 `ari web` 실행. 노드 수 = 실제 사이클 수, lineage 간선(loom→genesis) 존재 확인. 생성물을 상현님이 브라우저로 볼 수 있게 Artifact로 발행한다.

## 준비물

- C004의 ari v4, C001·C002의 픽스처
- Python 3 표준 라이브러리 (렌더는 순수 파이썬 — JS 0줄, 정적 SVG)

## 측정 방법

| # | 항목 | 통과 기준 |
|---|---|---|
| 1 | 구조 동일성 | T1(maze)·T2(lineage): 내장 JSON의 노드·간선·lineage 집합이 기대 구조와 정확히 일치 |
| 2 | 시각 표현 | T3: SVG에 사이클 수만큼의 노드 요소, 분기·병합 간선, 점선 lineage path가 존재 |
| 3 | 자기완결 | T4: `http(s)://` 참조 0건 (src·href·url() 전수 검사) |
| 4 | 문지기 | T5: 깨진 체인 → exit ≠ 0 + 출력 파일 미생성 |
| 5 | 실데이터 | T6 + 실행: 실제 레포 렌더 성공, 노드 6개(genesis 1 + loom 5), loom→genesis lineage 간선 |

전부 통과 → 채택.
