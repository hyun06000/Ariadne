# 2. 실험 설계

## 절차

1. **기대 행동 고정** — T1~T8을 `3-verification/fixtures/expected-behavior.md`에 구현보다 먼저 기록한다.
2. **구현** — C003의 ari.py를 계승·확장(v4): `close --git`(사전: 깃 저장소·태그 미선점 확인 → 닫기 → 사이클 경로만 add·commit → 주석 태그, 실패 시 원상 복구)과 `verify`(닫힌 사이클마다 태그↔작업 트리 대조: 추적 파일 diff + 미추적 신규 파일, 태그 없는 닫힌 사이클은 경고). 외부 의존성은 깃 CLI뿐 — 이 사이클의 주제 자체가 깃이므로 정당하다.
3. **테스트 실행** — 드라이버가 매 테스트마다 샌드박스를 임시 복사하고 깃 저장소로 초기화해 T1~T8을 독립 실행한다.
4. **과거의 각인 (실데이터 백필)** — 이미 닫힌 loom/C001~C003에 태그를 백필한다. 각 사이클을 **마지막으로 정당하게 수정한 커밋**에 태그를 단다: C001·C002 → `a111d05`(C001은 [migrate] 이주가 마지막 정당 수정), C003 → `6d8e98e`. 백필 직후 `ari verify`로 현 작업 트리와 일치함을 확인한다.
5. **도그푸딩** — 이 사이클 자신을 `ari close --git`으로 닫는다. **증거는 파일이 아니라 깃 역사 자체다**: 도구가 만든 커밋과 태그 `cycle/loom/C004-git-binding`이 남는다. (닫힘 후에는 사이클 디렉토리에 어떤 파일도 추가하지 않는다 — verify가 그것을 변조로 볼 것이므로. 이는 규칙이 아니라 이제 기계적 사실이다.)

## 준비물

- C003의 산출물: ari v3 (open/close), 샌드박스 픽스처 (재사용)
