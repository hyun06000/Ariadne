# Chain: loom

## 이 체인이 정복하려는 문제

**사이클을 깃만큼 유용하게 다룰 수 있는 도구를 만든다.**

깃이 커밋이라는 단위로 코드의 역사를 다루듯, LLM은 "사이클"이라는
**행동 체인 기록**을 만들 수 있어야 하고, 인간과 LLM 모두 그 기록을
조회하고(log), 분기하고(branch), 병합하고(merge), 검증할(fsck) 수 있어야 한다.

이 도구는 우리만의 것이 아니다. 어떤 사용자든 자신의 LLM에게 아리아드네
방식을 입힐 수 있는 하네스의 핵심 부품이다 — 새로운 깃 같은 무언가.

## 성공의 정의

Ariadne 레포 구조를 가진 어떤 저장소에서든, LLM 또는 인간이 단일 도구로
사이클 체인을 생성·조회·검증할 수 있고, 그 출력만으로 "어떤 교훈에서
어떤 가설이 태어났는지"의 전체 계보를 추적할 수 있다.

## 깃과의 대응 (설계 나침반)

| 깃 | loom | 의미 |
|---|---|---|
| commit | cycle | 기록의 최소 단위 |
| commit graph (DAG) | chain | parent 참조로 이어진 계보 |
| `git log --graph` | 계보 조회 | 역사를 읽는 능력 — **첫 번째로 정복** |
| branch / merge | 분기 / 병합 | 같은 부모에서 갈라지고, 여러 부모로 합쳐짐 |
| `git fsck` | 체인 검증 | 고아 사이클, 끊어진 parent 참조 탐지 |
