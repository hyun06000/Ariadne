# 5. 결과 보고

## 요약

계약 준수 스위트 `conformance.py`(22항목, 구현은 명령 문자열로만 주입)를 만들어 v0.4.0에 동봉했다 — **가설 채택**.
참조 구현 22/22, 행동 파괴 변이 3종 격추. 백미는 **동등 변이의 생존**: 검증 시점만 바꾼 변이를 스위트가 gil로 인정했다 — 스위트가 구현이 아니라 행동을 판정한다는, §7 구현 독립 계약의 실증.
이로써 바이너리 이식의 전제 조건이 완성됐다: 미래의 Go/Rust `gil`은 `conformance.py --gil "./gil"` 한 줄로 자격을 얻는다.

## 교훈

1. **"이 구현은 gil이다"가 기계 판정이 됐다.** 스펙 §7의 선언이 실행 가능형을 얻었고, 스위트는 패키지와 함께 여행한다.
2. **동등 변이는 버그가 아니라 증명이다.** 행동을 보존한 변이의 생존은 계약 스위트의 정의 그 자체 — 구현 세부를 판정했다면 그게 결함이었다.
3. **기각 조건은 문면이 아니라 의도로 써야 한다.** "변이 전부 통과 → 기각"에는 '변이는 행동 파괴적'이라는 가정이 숨어 있었다. 이후 사이클의 기각 조건에는 실패의 목적어를 명시할 것.
4. 잔재 정리 절차(전수 grep 선행)가 이번엔 작동했다 — `ari`는 이제 역사 속에만 있다 (커밋 접두어까지 `gil:`).
