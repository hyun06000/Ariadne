# 4. 결과 분석

| # | 항목 | 결과 |
|---|---|---|
| 1 | 기본 커밋 | ✅ --git 없이 step/close 자동 커밋(+태그) |
| 2 | opt-out | ✅ --no-commit |
| 3 | 깃 아닌 곳 | ✅ 조용히 스킵 (상태 전이만) |
| 4 | 두 구현·회귀 | ✅ 배치 동일, conformance 26/26 |

## 데이터 직접 관찰

- 문제의 뿌리는 "옵트인의 기본값"이었다. `--git`이 옵트인이라 안 붙이면 커밋 0 — 기본을 on으로 뒤집자 "안 붙이는 사람"이 자동으로 각인된다. 정책을 규율(문서)에서 기본값(도구)으로 옮긴 것.
- conformance 무회귀의 열쇠: 자동 커밋은 **깃 저장소일 때만**. conformance의 비깃 테스트(CLOSE-OK 등)는 스킵되어 무변, 깃 테스트(GIT-CLOSE)는 여전히 커밋·태그. 계약면을 안 건드렸다.
- --git을 하위호환 no-op으로 둔 것: 기존 스크립트·문서가 안 깨진다. 부드러운 강제의 조건.

## 예상과 달랐던 것

- 없음 — 기본값 뒤집기는 국소적이었다. 다만 close의 사전 검증(태그 선점)이 이제 --git 무관하게 돌아, 깃 저장소면 항상 태그 충돌을 먼저 잡는다(개선).

## 판정

**채택 (supported).** 커밋을 안 붙이던 사용자도 스텝마다 각인된다.
