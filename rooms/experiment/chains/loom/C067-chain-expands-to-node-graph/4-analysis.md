## 데이터 직접 관찰

- `graph-expanded.png`: loomlight를 펼치니 세로 목록이 아니라 **가로 노드-엣지 그래프** `C001—C002—C003`가 지도 카드 안에서 펼쳐졌다. C001 아래 초록 `⇠ C005-web-viewer +9`로 교차-체인 lineage 9건이 압축 표시.
- `node-clicked-doc.png`: `#cycdoc-loomlight-C001-…`로 이동하니 그래프는 그대로 위에 있고 **그 아래에** C001 문서가 `:target`으로 나타났다 — head(cid·closed·supported·title)·lineage 칩 전체(점프 링크)·메타표·5스텝 토글. 다른 노드를 누르면 그 문서로 바뀐다(:target은 하나만).
- gnode 75 = cycdoc 75 = 전체 사이클 수. 모든 사이클이 그래프 노드이자 클릭 시 열리는 문서.

## 예상과 달랐던 것

- **목업이 방향 오해를 즉시 드러냈다.** C065~C066 두 사이클을 "세로 목록"으로 소비한 게 상현님 의도(그래픽 노드-엣지)와 어긋났는데, 코드가 아니라 **그림**으로 전/후를 나란히 놓자 한 번에 합의됐다. "진행 전에 그림으로 보여달라"는 상현님 절차가 값비싼 재작업을 막았다(C037 "수신자 경로를 실제로 밟아라"의 디자인판).
- **`:target`이 그래프-문서 분리에 깔끔했다.** 그래프(항상 보임)와 문서(평소 숨김·클릭 시 하나만)를 `<details>` 없이 `:target` CSS만으로 분리 — 노드가 SVG `<a>`라 `<details>` 요약으로 감쌀 수 없는 제약을 `:target`이 우회했다. JS 0 유지.
- **정수 좌표라 parity가 또 자명**(C065·C066 재확인). `_layout_columns`를 전치만 해 가로로 만들었고 삼각함수가 없어 이식이 곧 바이트 동일.

## 판정

**가설 채택.** 기각 조건 넷 미발동: 목업 컨펌 일치, 바이트 불일치 0, 판정기 회귀 0, 노드 클릭 시 가로 그래프 아래 문서 :target. 상현님 스케치 `0—o—o—o`를 JS 0으로 구현했다.
