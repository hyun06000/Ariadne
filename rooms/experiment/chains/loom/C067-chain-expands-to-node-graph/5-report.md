# 5. 결과 보고

## 요약

체인을 누르면 그 자리에서 **가로 사이클 노드-엣지 그래프**(`0—o—o—o`)가 펼쳐지고, 노드를 누르면 그 사이클의 5스텝 문서가 그래프 아래에 `:target`으로 드러나게 했다(loom/C067). `_layout_columns`를 전치해 x=깊이·y=레인의 정수 좌표 SVG로 그렸고, 노드는 `<a href="#cycdoc-*">`. 상현님께 전/후 목업을 먼저 그려 컨펌받고 구현 — flat 불변·양 구현 **바이트 동일**(참조 90/90·Go 83/83), JS 0. 가설 채택.

## 교훈

1. **진행 전에 그림으로 합의하라.** C065·C066 두 사이클을 "세로 목록"으로 만든 게 상현님 의도(그래픽 노드-엣지 그래프)와 어긋났다. 코드가 아니라 **전/후 목업 그림**을 나란히 놓자 한 번에 정렬됐다. 상현님의 "진행 전 그림 컨펌" 절차가 재작업을 막았다 — 말로 합의된 줄 알아도 시각 산출물이 진짜 합의를 만든다(C037 디자인판). **다음부터 UI 변경은 목업 먼저.**
2. **`:target`은 SVG 노드→문서 드러내기의 열쇠.** 노드가 SVG `<a>`라 `<details>` 요약으로 못 감싼다. 그래프는 항상 보이고 문서는 `.cycdoc:target`으로만 하나씩 드러내면, JS 없이 "노드 클릭 → 그 문서" 가 된다. 상호작용마다 알맞은 네이티브 기제가 있다(C063~C066 "JS 0" 계승).
3. **레이아웃은 전치로 재활용한다.** 세로 그래프(`_render_svg`)의 (깊이,레인)을 x↔y만 바꿔 가로로 만들었다. 새 레이아웃 알고리즘 없이 검증된 것을 회전. 정수 좌표라 parity도 공짜(C064의 삼각함수 교훈 재확인).
