# 1. 가설 수립

## 이전 사이클의 교훈

부모 C099(다중부모·죽은가지 문서화, supported). C099가 "같은 체인 다중부모는 `--parent` 반복→`[A,B]`"를 문서화했다. 그런데 **과거의 사이클들 중 다중부모여야 했으나 단일부모로 봉인된 것들**이 있다(C096 발견③). 그때는 다중부모 사용법을 몰라(문서 침묵) 둘째 부모를 놓쳤다. 이제 그것을 복원한다.

**상현님 복원 기준(C096에서 확립)**: **"문서가 부모/근원/만난다로 명시한 것만"** parent로 복원한다 — 재발이력은 제외.

## 문제 분할 — 복원 대상 선별

C096이 지목한 4건을 상현님 기준으로 재검토:

| 사이클 | 현재 부모 | 문서 명시 | 복원할 부모 | 판정 |
|---|---|---|---|---|
| **C018** | C016 | 1-hypothesis:5 "부모: C016 … 근원: C012" | `[C016, C012]` | **복원** |
| **C041** | C040 | 2-design:34 "C029와 C035가 같은 자리에서 만난다" | `[C040, C029, C035]` | **복원** |
| **C057** | C056 | 1-hypothesis "부모: C056 … 기원: C053" | `[C056, C053]` | **복원** |
| **C043** | C042 | 1-hypothesis "부모: C042" — C015·C018·C037은 **재발이력** | 변경 없음 | **제외** (재발이력≠부모) |

→ 복원 대상 **3건(C018·C041·C057)**. C043은 현재 부모 C042가 이미 옳다(재발이력은 부모가 아님, 상현님 기준).

## 가설

> **가설**: `gil correct --field parent --to <A> --field parent --to <B> …`로, 각 사이클의 **봉인된 문서가 증언하는** 잃은 부모를 병합 부모 리스트로 복원할 수 있다 — C018→`[C016, C012]`, C041→`[C040, C029, C035]`, C057→`[C056, C053]`. correct의 C5 증거 검사(새 값이 봉인본 evidence에 문자 그대로 존재)를 통과하고, fsck R6(부모 존재)·R3(로컬 id)를 만족하며, 각 사이클은 무결(변조 0) 상태에서 정정돼 그래프에 다중부모 노드로 나타날 것이다.

## 예상 쟁점 (검증에서 확인)

1. **값 매칭**: parent 값은 로컬 id 풀슬러그(`C016-number-ledger`)여야 하는데(R6), 문서엔 `C016`(링크텍스트)·`../C016-number-ledger/`(경로)로 나타난다. correct C5는 "값이 haystack에 문자 그대로"를 요구하므로, **경로 안 풀슬러그가 부분문자열로 매칭되는지**가 관건(memory "번호접두 vs 풀슬러그 미검"). 안 되면 evidence 선택을 조정하거나, correct 확장(별도 전제 사이클, gil.py 수정 — 단 지금 Sheen이 gil.py 병렬수정 중이라 충돌, 그 경우 이월).
2. **evidence 단일 제약**: correct는 evidence 하나(한 줄 or 파일 전체)에서 **모든 새 값이** 나타나야 한다. C018은 한 줄(1-hyp:5)에 C016·C012 둘 다 ✓. C041·C057은 여러 줄에 흩어져 → **줄번호 생략(파일 전체 evidence)** 필요.

## 기각 조건

1. **증거 없는 복원이면 기각**: correct C5(증거 검사)를 우회하거나, 문서가 증언 안 하는 부모를 넣으면 기각. 오직 봉인 문서가 명시한 것만.
2. **fsck 위반이면 기각**: 복원 후 R6(부모 존재)·R3(로컬 id)·기타 위반이 나면 기각.
3. **무결하지 않은 사이클을 고치면 기각**: correct C6(변조된 사이클 거부)를 어기면 기각 — 봉인 무결한 것만.
4. **C043을 건드리면 기각**: 재발이력을 부모로 넣는 것은 상현님 기준 위반. C043은 불변.
5. **Sheen 파일 충돌이면 기각**: 이 사이클은 원장(cycle.yaml)+corrections.yaml만 고치고 **gil.py 무변경** — 코드 건드리면(예: correct 확장) 기각(그건 이월). Sheen land와 무충돌 유지.

## 범위 밖

- **correct 자체 확장**(다중 evidence·번호접두 매칭): 값 매칭이 실패하면 필요할 수 있으나 gil.py 수정이라 Sheen land 전까지 이월.
- **(D) deploy 축**: gil.py 대폭 수정이라 Sheen land 후.
