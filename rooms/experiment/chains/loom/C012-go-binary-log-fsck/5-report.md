# 5. 결과 보고

## 요약

Go 단일 바이너리(표준 라이브러리만, Mach-O arm64)가 계약 부분집합 11항목(fsck R1~R8·log)을 무수정 스위트로 통과했고, 실제 레포에서 파이썬 gil과 문장 단위로 동일하게 답했다 — **가설 채택**. "구현 교체 가능"이 선언에서 실물이 됐다.
도그푸딩의 이중 수확: 스위트 v1의 부분 구현 취약성(→ v2 항목 독립, v0.5.0), 참조 구현의 SameFileError 결함(→ 수정, 마이너 승격 규칙 정상 발동).

## 교훈

1. **두 몸, 한 계약.** 스펙 §4·§5와 참조 구현만으로 Go 이식이 가능했다(461줄) — 실험의 방 발굴 불필요. 스펙이 정말로 계약 역할을 한다.
2. **판정기는 판정 대상과 함께 자란다.** 부분 구현이라는 새 피판정자가 스위트의 암묵 가정(전체 구현)을 드러냈다 — 판정 항목 간 독립은 부분 이식 로드맵(명령 하나씩 추가)의 전제 조건이었다.
3. **거부형 검사는 수락형과 짝일 때만 의미가 있다** — 미구현의 exit≠0가 거부형을 공허 통과시킨다.
4. 승격 규칙의 공백 2건 기록: conformance.py의 도구 분류, 변경 감지 기준(마지막 릴리스 태그여야 함).
