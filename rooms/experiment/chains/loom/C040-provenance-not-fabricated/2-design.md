# 2. 실험 설계

가설: **출처(`author`·`parent`)의 조용한 기본값을 제거하면 거짓 원장은 태어나지 못한다.**

## 0. 설계 중 발견 — 우리도 걸려 있었다

설계에 앞서 우리 저장소의 체인을 훑었다 (가정하지 말고 명령으로 — C007).

```
$ grep -l '^parent: null' loom/*/cycle.yaml
loom/C001-lineage-is-reconstructable    ← 정당한 루트
loom/C029-time-machine                  ← ???
```

`loom/C029-time-machine/cycle.yaml`은 `parent: null`이다. 그런데 **그 사이클 자신의 불변 문서**는 이렇게 말한다:

> `1-hypothesis.md:5` — *"**부모**: [loom/C028](../C028-pages-command/5-report.md). 발의: 사용자 (박상현)…"*

**문서는 C028이라 말하고 원장은 루트라 말한다.** 나는 C029를 열 때 `--parent`를 잊었고, 도구는 조용히 `null`을 박았다 — **maru와 정확히 같은 실수**다. 그리고:

- `gil log`는 그 뒤로 11사이클 동안 내 화면에 **`root: C001-…, C029-time-machine`** 을 찍고 있었다.
- `gil fsck`는 같은 저장소를 보고 **위반 0건**이라 말해 왔다.
- 하필 그것이 **계보를 되감는 타임머신 사이클**이다.

이 발견이 설계에 두 가지를 고정한다.

1. **탐지는 예방만큼 필수다.** 예방만 넣으면 C029(와 maru의 C012)는 영원히 거짓으로 남는다.
2. **정정은 이 사이클의 범위가 아니다.** 닫힌 사이클(§4 불변)의 출처를 어떻게 정당하게 고치는가는 **별개의 가설**이다 — maru도 *"규칙의 회색지대"* 라 불렀다. C040은 **못 만들게(예방) + 보이게(탐지)** 까지만 하고, **정정 규정은 C041로 사전등록**한다. 탐지 없이 정정을 논할 수 없고, 이 사이클이 그 증거(C029)를 만들었다.

## 절차

### 1. 계약 — SPEC 개정

**§3.2 출처 필드 (provenance) 신설.** 도구가 채우는 필드를 두 종류로 가른다.

| 종류 | 필드 | 근거 |
|---|---|---|
| **파생 가능한 사실** | `id`(번호), `opened`, `status`, `step` | 도구가 계산으로 안다 |
| **출처 (provenance)** | `author`, `parent` | **오직 저자만 안다** |

- **P1. 도구는 출처를 지어내지 않는다.** 출처 필드에 추측값·고유명사 기본값을 넣는 것을 금한다.
- **P2. 모르면 거부한다.** 출처가 확정되지 않으면 `open`은 사이클을 **생성하지 않는다** (저장소 무변화).
- **P3. 파생 가능한 사실은 예외다.** 체인에 사이클이 하나도 없으면 그 첫 사이클이 루트라는 것은 추측이 아니라 계산이다 — 부모 플래그를 요구하지 않는다.

> 깃의 답을 훔친다 (C016): `git commit`은 `user.name`이 없으면 **추측하지 않고 거부한다.**

**`open`의 의무** (§5 CLI 계약):

| # | 조건 | 행동 |
|---|---|---|
| **O1** | `--author` 없음 | **거부** (exit ≠ 0, 무변화). 기본값 없음 |
| **O2** | 체인에 사이클이 있는데 `--parent`도 `--new-root`도 없음 | **거부** — tip을 보여주고 두 길을 안내 |
| **O3** | `--parent`와 `--new-root` 동시 | **거부** (모순) |
| **O4** | 체인이 비어 있음 (`--new-chain` 포함) | `parent: null`로 **생성** — 부모 플래그 불필요 (P3) |
| **O5** | `--new-root` 단독 | `parent: null`로 **생성** — 의도된 두 번째 루트 |

**R12 신설** (§3 규칙 표):

| 규칙 | 내용 |
|---|---|
| **R12** | 한 체인의 루트(부모 없는 사이클)가 2개 이상이면 **경고** (`[다중루트]`). 거의 항상 `--parent` 누락의 흔적 |

**경고이지 위반이 아닌 이유** (과잉 작동 방어): `--new-root`(O5)가 정당한 탈출구인데 그 결과물이 `fsck`를 **실패**시키면 도구가 자기 탈출구를 불법화하는 것이다. R10(결말없음)의 선례를 따른다 — 경고는 `stderr`, exit 0.

### 2. 구현 (두 구현 동시)

두 구현 모두 `open`의 **사전 검증 블록**에 넣는다 — *"저장소를 건드리기 전에 전부 확인한다 (부분 생성물 방지)"* 는 주석이 이미 그 자리를 비워 두고 있었다 (C035: *좋은 규정은 아직 없는 기능의 자리를 비워둔다*).

| 파일 | 변경 |
|---|---|
| `gil.py` | `--author` 기본값 `"clew"` **삭제**, `--new-root` 플래그 신설, 사전 검증 O1~O3, `fsck_collect`에 R12 |
| `go/main.go` | 동일 (`flagVal(flags,"author","clew")` → 기본값 없음), `--new-root`, 동일 검증, 동일 R12 |

거부 메시지는 **길을 알려주는 형태**로 (깃의 *"Please tell me who you are"* 처럼):

```
오류: 저자를 알 수 없다 — 도구는 출처를 지어내지 않는다 (§3.2 P1).
      존재의 이름을 명시하라:  gil open <chain> <slug> --author <이름>
오류: 체인 'loom'에 이미 사이클이 있다 (tip: C039-command-surface-contract).
      부모를 명시하라:  --parent C039-command-surface-contract   (분기면 여러 번)
      정말 새 루트라면:  --new-root
```

> 메시지 **문면**은 계약이 아니다 (§3.1: *렌더는 계약이 아니다*). 계약은 **종료 코드 ≠ 0 + 저장소 무변화**다. 판정기는 그것만 본다.

### 3. 판정기 확장 (`conformance.py`)

> *"판정기가 안 보는 계약은 없는 계약이다."* — Weft (C036). **새 표면을 계약에 적으면 같은 커밋에서 판정기에도 적는다** (C039).

| 항목 | 판정 |
|---|---|
| `OPEN-AUTHOR-REQUIRED` | `--author` 없이 open → exit ≠ 0 **+ 저장소 무변화** |
| `OPEN-PARENT-REQUIRED` | 비어있지 않은 체인, 부모 플래그 없이 open → exit ≠ 0 **+ 무변화** |
| `OPEN-ROOT-CONFLICT` | `--parent` + `--new-root` 동시 → exit ≠ 0 **+ 무변화** |
| `OPEN-NEW-ROOT` | `--new-root` → exit 0, `parent: null`로 생성 |
| `OPEN-ROOT-EMPTY-CHAIN` | **빈 체인의 첫 사이클은 부모 플래그 없이 성공** (과잉 작동 방어) |
| `FSCK-MULTI-ROOT` | 루트 2개 체인 → **경고 출력 + exit 0** (위반 아님 — 과잉 작동 방어) |

**기존 항목 1건 교정** (C011의 *동등 변이* 교훈: *엉뚱한 이유로 통과하는 테스트는 눈먼 테스트다*):
`OPEN-REJECT-SLUG`는 지금 `--author` 없이 open을 부른다. 새 규칙 아래서는 **슬러그가 옳아도 저자 때문에** 거부되므로 그 항목은 더 이상 슬러그 규칙을 판정하지 못한다. `--author fx`를 넣어 **판정 대상을 되돌린다.**

### 4. 변이 주입 (기각 조건 1·2·5의 실증)

판정기가 **정말 보는지** 확인한다 — 통과는 증거가 아니다 (C026: *"프로브만이 증거다"*).

| 변이 | 주입 | 격추해야 할 항목 |
|---|---|---|
| **M1** | `--author` 기본값 `"clew"` 부활 | `OPEN-AUTHOR-REQUIRED` |
| **M2** | O2 부모 검사 삭제 | `OPEN-PARENT-REQUIRED` |
| **M3** | R12 삭제 | `FSCK-MULTI-ROOT` |
| **M4** (과잉) | **빈 체인에서도** 부모 요구 | `OPEN-ROOT-EMPTY-CHAIN` (+ `OPEN-CREATE`) |

M4가 핵심이다 — *"이게 과하게 작동하면 어떤 참이 거짓이 되는가"* (C038). 정당한 루트를 불법화하는 구현은 **반드시 실패해야** 한다.

### 5. 전수 grep (C010의 절차)

`gil open`을 부르는 **모든** 문서와 **도구가 생성하는 코드**를 훑어 `--author`를 넣는다.
C037의 교훈 — ***도구가 생성하는 코드가 더 위험하다*** — 를 잊지 않는다: 사람이 읽는 README보다, 에이전트가 **그대로 실행하는** 온보딩 스크립트(`README.ai.md`)가 먼저다.

대상: `README.md` / `README.ko.md` / `README.ai.md` / `QUICKSTART.md` / `SPEC.md` / `go/README.md` / 워크플로.

### 6. 릴리스 — **v2.0.0**

`--author` 없이 돌던 `open`이 이제 거부된다. **하위 호환이 깨진다.** SemVer는 메이저를 요구하고, 이 저장소의 규율은 *"규칙을 내 편의에 맞춰 고치지 않는다"*(C002)이다.

좋은 파괴다: **조용히 거짓을 쓰던 경로가 시끄럽게 멈춘다.**
