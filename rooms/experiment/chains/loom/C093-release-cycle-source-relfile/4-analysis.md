## 데이터 직접 관찰
- **헬퍼 공유의 함정.** `_mk_release_repo`는 두 목적(drift 게이트 / 근거 사이클)에 재사용됐는데, 두 목적의 요구가 달랐다 — drift는 봉인 전이라 RELEASE.md 불요, 근거 사이클은 봉인 성공이라 필요. 공유 헬퍼가 더 엄격한 소비자의 요구를 못 채웠다.
- **재현의 함정을 직접 밟았다.** C092에서 격리 재현이 통과해 "간섭"으로 오진할 뻔했다 — 내가 재현 시 RELEASE.md를 수동으로 넣었기 때문. 실제 테스트 경로(수동 보강 없이)를 그대로 재현해야 진짜 원인이 보인다.

## 예상과 달랐던 것
- C092에서 "상태 간섭"으로 의심했으나, 실제는 훨씬 단순한 "테스트가 필요 파일을 안 만듦"이었다. 크래시(TypeError)가 걷힌 뒤에야 이 단순한 결함이 또렷이 보였다 — C092 교훈("한 크래시가 다른 결함을 가린다")의 직접 후속.
