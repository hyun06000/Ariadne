# 5. 결과 보고

## 요약

C054가 선재 결함·1순위 후보로 남긴 **경로 해석 비대칭** — 심볼릭 링크를 통과하는 절대
`--root`에서 참조는 `git add … outside repository`로 붕괴(사이클 파일은 생겼으나 커밋
실패)하고 Go는 `EvalSymlinks`로 견디던 "두 몸, 한 계약"의 위반 — 을 정복했다. 참조에
Go의 `relToRepo`와 동치인 단일 헬퍼 `_rel_to_repo`(양쪽 realpath 정규화 후 relpath)를
도입해 git 지향 relpath 22곳을 대체하고, 판정기 `PATH-SYMLINK-GIT`으로 이 계약을 봉인했다.
**수정 전 참조 FAIL(75/76)/Go PASS로 비대칭을 기계가 재현**하고, **수정 후 양 구현
76/76, 회귀 0, 변이 격추(75/76)**. 세 가설 모두 지지, 이탈 0.

## 교훈

1. **한 사이클이 우회한 결함을 다음 사이클이 계약으로 가둔다.** C054는 NO-REMOTE
   테스트에서 `os.path.realpath(make_sandbox(...))`로 경로를 **미리 정규화**해 이 결함을
   범위 밖으로 우회했다. `PATH-SYMLINK-GIT`은 정확히 그 반대 — realpath한 실디렉토리를
   가리키는 **심링크를 새로 만들어 `--root`를 심링크로 통과**시킨다. C054가 지운 비대칭을
   이 항목이 되살려 봉인했다. C009("미완을 정직히 남기면 다음 가설의 재료")의 판정기판:
   *우회의 흔적(테스트의 realpath 한 줄)이 다음 계약면의 설계도였다.*

2. **가용성/환경을 앞에서 흡수하는 것이 견고함의 공통 골격 — 이번엔 경로의 심링크 공간.**
   C052(git 부재)·C054(원격 부재)에 이어 세 번째 환경 함정이었고, 근본 처방은 같았다:
   차이를 **경계에서 정규화**한다. git 부재는 `shutil.which`로, 원격 부재는 `_has_push_remote`로,
   심링크 불일치는 `realpath`로 — 전부 "실행 직전에 환경을 확인·정규화"다. 다른 것은 축뿐이다.

3. **반쪽 상태에는 두 종류가 있다 — 의도된 우아화 vs 의도되지 않은 붕괴.** 둘 다
   `파일=있음, 커밋=없음`이지만, C052의 git 부재는 **rc0 + 안내**(의도된 강등)이고 이번
   심링크 붕괴는 **rc1 + 날것 fatal**(의도되지 않은 실패)이었다. 판정 detail의
   `yaml=True committed=False`를 눈으로 본 것이 둘을 갈랐다 — 통계가 아니라 데이터가 성격을 말한다.

4. **두 구현의 구조적 평행이 리팩터링의 경계를 그어 준다.** "두 번째 인자가 git-toplevel
   `repo`인가"라는 단일 구문 기준이 Go의 `relToRepo(repo, …)` 사용처와 정확히 일대일
   대응해, 대체 대상 22곳과 제외 4곳(자기일관 `_hash_tree`, pages의 repo_root, display용
   getcwd)이 자동으로 갈렸다. 계약이 두 몸을 평행하게 유지하면, 한 몸의 수정 범위를 다른
   몸이 알려 준다.

## 다음 사이클을 위한 제안

- **(A) `close`의 deviations 카운트 조정** (C053 슬립이 드러낸 도구 공백). `gil close`가
  `deviations.yaml` ↔ `cycle.yaml`의 카운트 불일치를 감지·경고하지 않아, 봉인 시 내부
  불일치가 새어들 수 있다. C053에서 실제로 발생한(봉인된 C053의 5-report는 "이탈 1"인데
  cycle.yaml은 `deviations:0`) 재발 방지. C038 "규율이 훈계로만 남으면 새어나간다"의 도구화.
- **(B) verify·release의 경로·환경 계약면 전수 감사.** 이번 수정으로 verify·release도
  `_rel_to_repo`를 쓰게 됐으나, 그들의 git 부재·원격 부재·심링크 경로가 판정기에 **직접**
  걸리는 항목은 아직 없다(open 계열만 PATH-SYMLINK-GIT·NO-GIT·NO-REMOTE로 덮임). 명령별
  환경 계약면의 공백 목록화 — C051·C052·C054·C055가 쌓은 "환경 우아화" 계열의 완결.
- **(C) Windows end-to-end 수신자 검증** (v2.13.0부터 `.exe` 게시). 상현님/친구가
  수신자-테스터 — 비개발자·크로스플랫폼 아크의 실환경 확인.
- **(D) `_hash_tree`(verify 스냅샷)의 심링크 경계 재점검.** 이번엔 `p`와 `root`가 같은
  `os.walk` 기반이라 자기일관으로 제외했으나, 향후 `--root`가 심링크이고 스냅샷 대상이
  저장소 밖을 참조하는 조합이 생기면 재검토 필요 — 지금은 사각이 없음을 확인만.
