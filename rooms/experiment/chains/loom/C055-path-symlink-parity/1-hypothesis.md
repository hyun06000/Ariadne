# 1. 가설 수립

## 이전 사이클의 교훈 (loom/C054)

C054(`--push` 원격 부재 우아화)의 판정기 첫 실행에서 **참조만 FAIL**했고, 진짜 범인은
push가 아니라 경로 해석이었다. macOS tmpdir의 `/var`→`/private/var` 심볼릭 링크 때문에
`git rev-parse --show-toplevel`(realpath)과 절대 `--root`(심링크 그대로)가 어긋나,
참조의 `os.path.relpath`가 `git add -- ../../../…`로 붕괴했다. Go는 `relToRepo`가
`EvalSymlinks`로 견뎠다.

C054는 그 테스트를 `os.path.realpath`로 정규화해 **의도(push 우아화)만 검증**하고,
경로 해석 비대칭 자체는 **선재 결함으로 범위 밖에 남겼다**(C044·C051의 절제). 그리고
보고서 다음 후보 (A)로 명시했다: *"경로 해석 비대칭(Go는 심볼릭 링크 해석, 참조는 안 함
— 두 구현 갈림, 계약면 후보, C028·C029 데이터 축적)."* 이 사이클이 그 매듭이다.

## 문제 분할

정복할 첫 문제: **참조 구현이 심볼릭 링크를 통과하는 절대 `--root`에서 git 각인에
실패**하는 것 — Go는 성공한다. 이는 "두 몸, 한 계약"의 실제 위반이며 두 조각으로 나뉜다.

1. **참조를 Go에 맞춘다** (동작 패리티). Go가 옳으므로 참조를 고친다.
2. **판정기가 이 계약을 보게 한다** (계약면). C051의 교훈 — "판정기가 안 보는 계약은
   없는 계약이다". C054가 realpath로 우회해 버린 바로 그 경로를 판정기가 정면으로 밟게.

### 관찰 (가정하지 않고 직접 재현 — C007)

심볼릭 링크 `symrepo → realdir`를 만들고 그 안에 git 저장소를 초기화한 뒤,
**심링크 경로를 통과하는 절대 `--root`**로 `open --git`을 두 구현에 각각 실행했다.

| 구현 | 결과 | 커밋 | rc |
|---|---|---|---|
| 참조(Python) | `오류: git add -A -- ../symrepo/…/C001-sym-test … 'is outside repository'` | **안 됨** (사이클 디렉토리만 디스크에 남음) | 날것 fatal |
| Go | `열림: demo/C001-sym-test` | **됨** | 0 |

Go가 옳고(의도대로 커밋), 참조가 깨졌다(반쪽 상태 — 파일은 생겼는데 각인이 없다).
C052(git 부재)·C054(원격 부재)와 같은 계열의 "환경을 앞에서 흡수하지 못한" 결함이며,
이번엔 **경로의 심볼릭 링크 공간 불일치**다.

### 근본 원인

참조의 `_repo_root`는 `git rev-parse --show-toplevel`로 **realpath화된 저장소 루트**를
얻지만, git에 넘길 경로들은 사용자가 준 `--root`(심링크 그대로)에서 `os.path.join`으로
조립된 뒤 `os.path.relpath(path, repo)`로 상대화된다. 두 경로가 **서로 다른 심볼릭 링크
공간**에 있으면 relpath가 저장소를 탈출하는 `../…`를 만들고 git이 거부한다.

Go의 `relToRepo(repo, path)`는 `filepath.EvalSymlinks`로 **양쪽을 정규화한 뒤** relpath해
이 불일치를 흡수한다(main.go 주석이 이미 명시: *"macOS의 /tmp·/var 심링크 차이를
EvalSymlinks로 흡수"*). 참조에는 이 정규화가 없다.

## 가설

> **가설 (H1)**: 참조에 Go의 `relToRepo`와 동치인 단일 헬퍼 `_rel_to_repo(path, repo)`
> (양쪽을 `os.path.realpath`로 정규화 후 `os.path.relpath`)를 도입하고 git에 넘어가는
> 모든 `os.path.relpath(…, repo)` 호출을 이 헬퍼로 대체하면, **심볼릭 링크 `--root`에서
> 참조가 Go와 동일하게 정상 커밋**(rc0 ∧ 사이클 파일 ∧ 로컬 커밋 ∧ 무크래시)하고,
> **비심링크 경로에서 회귀 0**(realpath는 심링크 없는 경로에 멱등)이다.

> **가설 (H2 — 계약면)**: 이 계약을 판정기 `PATH-SYMLINK-GIT`으로 신설한다(C054의
> realpath 정규화와 정반대로 **정규화하지 않은 심링크 루트**를 준다). 신설 직후
> **참조 FAIL / Go PASS**로 비대칭을 기계가 재현하고, 참조 수정 후 **양 구현 PASS**로 접힌다.

## 기각 조건

- 참조 수정 후에도 심링크 `--root`에서 참조가 커밋하지 못하면 (H1 실패).
- 비심링크 경로에서 기존 판정 항목(75/75)에 회귀가 생기면 (H1-2 실패).
- `PATH-SYMLINK-GIT`이 수정 전 참조를 통과시키거나(거짓 음성) 수정 후 Go/참조를
  떨어뜨리면 (H2 실패 — 계약면이 실제 비대칭을 포착하지 못함).

## 이 사이클의 성격

참조를 Go에 맞추는 일 — 온전히 **날실(Clew)의 몫**이다. Go는 이미 옳으므로 Weft 소환은
없다. 나는 Go가 새 판정 항목을 통과함을 **교차 검증**하는 관찰자로만 Go를 다룬다.
