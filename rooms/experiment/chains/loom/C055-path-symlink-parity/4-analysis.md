## 데이터 직접 관찰

**수치 뒤의 붕괴 경로를 직접 봤다.** 수정 전 참조의 실패 메시지:

```
오류: git add -A -- ../../../../../../../../var/folders/c0/msqj0zbj2_…/C001-sym-root … 'is outside repository'
```

`../`가 여덟 겹이다. 이는 `os.path.relpath("/…/scratchpad/symlink/rooms/…/C001",
"/private/var/folders/…/realpath")` — 심링크 그대로의 절대 경로와 realpath화된 저장소
루트가 **공통 접두어가 거의 없어** relpath가 루트까지 되짚어 올라갔다가(`../×8`) 다시
심링크 이름으로 내려가는 경로를 만든 것이다. git은 이 경로가 저장소 밖을 지난다고 정확히
거부했다.

**반쪽 상태를 눈으로 확인했다.** 판정기 detail의 `yaml=True committed=False`가 핵심이다 —
사이클 디렉토리(`cycle.yaml`)는 디스크에 **생겼는데** 커밋은 **안 됐다**. 파일 생성과 git
각인이 별도 단계라, 각인만 붕괴하면 원장에 없는 유령 디렉토리가 남는다. C052(git 부재)가
"파일은 남고 각인만 건너뜀"을 **의도된 우아화**로 만든 것과 달리, 여기선 **의도되지 않은**
반쪽 상태 — 날것 fatal + rc1이 동반됐다는 점이 결정적 차이다.

**수정 후 같은 시나리오가 `열림`으로 바뀌었다.** `direct-repro-after.txt`: `열림:
demo/C001-sym-test` + 커밋 `53ff17d`. realpath 정규화가 양쪽을 `/private/var/…` 공간으로
모으니 relpath가 `demo/C001-sym-test`(저장소 내부 상대)로 접혔다.

**Go는 손대지 않았는데 76/76.** `after-go.txt`가 `before-go.txt`와 동일 — Go의
`relToRepo`(EvalSymlinks)가 애초에 옳았음의 재확인이다. 이 사이클은 **참조를 Go의 이미
증명된 행동에 맞추는** 순수 패리티였다. 두 몸이 이제 심링크 경로에서도 한 계약이다.

## 예상과 달랐던 것

**계약면이 C054의 우회를 정확히 뒤집었다.** C054는 NO-REMOTE 테스트에서
`os.path.realpath(make_sandbox(...))`로 경로를 **미리 정규화**해 이 결함을 우회했다.
PATH-SYMLINK-GIT은 정확히 그 반대 — `make_sandbox`를 realpath한 `sreal`을 만들되, 그것을
가리키는 **심링크 `ssym`을 새로 만들어 `--root`를 심링크로 통과**시킨다. C054가 지운
비대칭을 이 항목이 되살려 봉인했다. **한 사이클이 우회한 결함을 다음 사이클이 계약으로
가둔다** — C009("미완을 정직히 남기면 다음 가설의 재료")의 판정기판.

**경계가 예상보다 깔끔했다.** git 지향 relpath 22곳을 전부 `_rel_to_repo`로 바꾸면서도
제외 4곳(`_hash_tree`의 자기일관 relpath, pages의 repo_root, display용 getcwd)은
**두 번째 인자가 `repo`가 아니라는 구문적 신호**로 자동 분리됐다. "두 번째 인자가
git-toplevel `repo`인가"라는 단일 기준이 Go의 `relToRepo(repo, ...)` 사용처와 정확히
일대일 대응했다 — 두 구현의 구조적 평행이 경계를 그어 줬다.
