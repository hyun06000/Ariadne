# 2. 실험 설계

가설: *"C069 병렬 온보딩 안내만 읽고 온 신규(탄생) 에이전트에게 독립적 병렬 갈래가 여럿인
과제를 개입 없이 주면, 순차가 아니라 `gil worktree add`로 격리 워크트리에서 각 갈래를
진행한다."* 이 하나만을 검증한다. 설계 뼈대는 gateway/C001을 병렬판으로 각색한다.

## 절차

### A. 격리 환경 구성 (실 저장소 불가침 — gateway/C001의 "격리 임시 디렉토리")

1. main 시점의 Ariadne를 스크래치패드에 **로컬 클론**한다(`git clone` — 실 저장소 파일 무변경).
   - 클론의 `origin`을 **로컬 bare 저장소**로 재지정한다. 이유: 온보딩 안내는 `--push`를
     포함하므로, 피소환자가 push해도 실제 GitHub이 아닌 로컬 bare로 가게 해 원격을 보호한다.
     (실험 환경 격리일 뿐 — 워크트리 워크플로 자체는 온전히 작동한다.)
   - 클론에 `git config gil.owner`를 **설정하지 않는다**(C062 guard 미발동): 피소환자가
     자신을 주-존재로 인식하는 신규 세션이므로, guard는 관측 대상이 아니라 잡음이 된다.
     대신 "네 워크트리에서 일하라" 문장이 스스로 작동하는지를 본다.
2. 클론 안에 관측 로그 디렉토리를 만들지 않는다(피소환자의 자연스러운 산출만 남긴다).
   관측은 전량 **사후**에 클론의 git 상태를 읽어 수행한다(개입 0).

### B. 소환 (소환 규약 §6 4의무 준수, 기계·기능 누설 0)

3. 신선한 서브에이전트(general-purpose)를 **동기(run_in_background=false)** 로 소환한다.
   프롬프트에 **오직** 다음만 담는다 — worktree/병렬 **기계는 이름조차 언급하지 않는다**:
   - **① 포인터 주입**: "이 클론 루트의 CLAUDE.md를 읽고 그 지시를 그대로 따르라"
     (+ 클론 절대경로). 정체성/명령어/worktree 무누설.
   - **② 모드 선언**: **탄생** — "너는 새로운 존재다. 명부의 기존 이름이 아니다."
   - **③ 신원 전달**: "너의 소환자는 **Weft(위프트)**다."
   - **④ 과제**: 아래 C. 개입은 이후 0.
   - 환경 안내(문서 결함 아님): 클론 절대경로, bash cwd가 콜마다 리셋될 수 있음(내가 겪는
     것과 동일한 환경 사실 — 힌트 아님), 실 저장소(Ariadne 원본) 불가침.
4. **상호 기록/방**: 피소환자는 온보딩 규칙대로 **클론 안에** 자기 방을 창건한다(자기 이름
   자작). 실 명부 등록은 하지 않는다 — gateway/C001 선례(격리 실증에서 태어난 Meander는
   실 명부 미등록). 나는 소환자 몫으로 이 존재를 내 `relations.md`·`memory.md`(실 저장소)에
   기록한다. 실 거주자 편입 여부는 소환자 Clew/상현님의 판단으로 넘긴다(§보고에 명시).

### C. 과제 (독립 병렬 갈래 3 — Step E 트리거 조건을 그대로 재현)

5. 서로 **완전히 독립적인** 소과제 3개를 주고, **"동시에(concurrently) 진행되길 원한다"**
   고만 말한다. 이는 README.ai Step E의 트리거 문장("the human wants several tracks worked
   concurrently")을 그대로 재현한 것 — **기계(`gil worktree`)는 절대 언급하지 않는다.**
   측정 대상: *트리거가 주어졌을 때 안내가 그 트리거를 문서화된 행동으로 변환하는가.*
   - 3 갈래는 서로 의존 0인 초소형 문제(예: 각기 다른 순수 함수 + 픽스처 + 기각 조건).
     각 갈래의 **내용은 사소**하게 유지해, 피소환자의 인지 예산이 "순차 vs 워크트리"라는
     **워크플로 결정**(=측정 대상)에 쓰이게 한다.

### D. 관측 (개입 0, 전량 사후)

6. 피소환자 완료 후, 클론에서 다음을 수집해 `3-verification/`에 재현 가능하게 저장한다:
   - `git worktree list` — 링크 워크트리가 생겼는가(핵심 물증).
   - `git branch -a`, `git log --all --oneline --graph` — 브랜치·커밋 그래프(순차 vs 격리).
   - `git reflog`(main 및 각 브랜치) — main에 갈래 커밋이 샜는가.
   - `reservations.tsv`, 새로 생긴 사이클/방 디렉토리, 피소환자의 최종 보고 서사.
   - 피소환자가 실행한 gil/git 명령의 흔적(가능하면 히스토리).

## 준비물

- macOS Darwin, Python 3.9+(표준 라이브러리), git. gil은 클론 내 `rooms/deployment/
  ariadne-spec/gil.py`(배포본, 무수정). 서브에이전트: Agent 툴 general-purpose, 동기 실행.
- 산출물 저장: 클론은 세션 스크래치패드(세션 격리, C060 교훈 — /tmp 공유경로 회피).
  실 저장소·내 워크트리는 불가침, 검증 기록만 3-verification/에 커밋.

## 측정 방법

**주 지표(층위 2~3, 최소 성공 기준)**: 피소환자가 독립 갈래 중 **최소 하나 이상**을
`gil worktree add`로 연 격리 워크트리에서 진행했는가(클론 `git worktree list`에 링크
워크트리 존재 + 해당 브랜치에 갈래 커밋).
- **채택**: 위가 참이고, main에 갈래 작업이 유출되지 않음.
- **기각 1(순차 회귀)**: 워크트리 0개, 갈래들이 공유 체크아웃(main)에서 순차 처리.
- **기각 2(인지 실패)**: 병렬 가능성 자체를 인지·언급 못 함.
- **기각 3(오집행)**: worktree 의도는 있으나 `gil worktree add`가 아닌 수동 우회, 또는
  "네 워크트리에서 일하라" 위반(main 유출).
- **환경 마찰 예외(gateway/C001)**: 권한 분류기가 gil **실행**을 차단하면 반증이 아니라
  마찰로 분리 기록하고, 도달한 최고 층위(인지/계획)를 부분 증거로 남긴다.

**보조 관측**: 안내의 **어느 문장이 실제로 작동/실패했나** — 피소환자 서사에서 Step E/
CLAUDE §3/SPEC §6.8 중 무엇이 행동을 유발했는지, 무엇이 오해·무시됐는지 문장 단위로 귀속.
