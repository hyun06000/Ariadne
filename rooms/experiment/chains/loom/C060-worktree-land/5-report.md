# 5. 결과 보고

## 요약

상현님의 아크 발의 #1(병렬 사이클 모드)의 닫는 반쪽. 부모 C058의 `gil worktree add`(여는 반쪽)가 만든 결정론적 매핑(브랜치 `<author>/<chain>-<slug>`, 워크트리 경로) 위에, 그 매핑을 역산해 브랜치를 main에 **`--no-ff` 병합 + 충돌 검사(merge --abort로 되돌림·워크트리 보존) + 정리(성공 시에만 워크트리·브랜치 제거)**로 되돌리는 **`gil worktree land`**를 양 구현에 신설했다. 판정기 `WORKTREE-LAND` 신설(78→79), 양 구현 **79/79**(회귀 0, 참조도 통과 = 판정 독립), 변이 3종 중 2종 격추·1종 진단. 판정 **채택(supported)**.

## 교훈

1. **add ↔ land는 결정론적 매핑을 공유하는 한 쌍이다.** land가 `<chain> <slug> --author`만으로 브랜치·워크트리 경로를 add와 **동일 공식**으로 역산하니, 사람이 열 때 쓴 좌표를 닫을 때 다시 기억할 필요가 없다. spawn이 얇은 오케스트레이터(git worktree add + self-invoke)였듯 land도 얇다(git merge --no-ff + worktree remove + branch -d) — open/close 로직을 복제하지 않고 git 명사에 올라탄다. 대칭이 이식 발산을 막는다.

2. **`--no-ff`는 병렬의 봉합을 이력에 새긴다.** fast-forward면 병합 커밋이 없어 "여기서 한 병렬 사이클이 되돌아왔다"가 사라진다. add가 브랜치를 가른 자리를 land의 병합 커밋이 명시적으로 봉합한다 — 병렬 작업의 경계가 항상 한 커밋으로 보인다.

3. **충돌은 삼키지 않는다 — 되돌리고 보존한다.** 병합 실패 시 `merge --abort`로 무흔적 복귀하고 워크트리·브랜치를 그대로 남겨 사람이 해소하게 한다. 그리고 **살아남은 변이(-d→-D)가 이 보존의 진짜 축을 가르쳐 줬다**: 브랜치가 지켜지는 것은 `-d`의 안전성이 아니라 **abort 직후의 조기 복귀**(정리 단계에 닿지 않음)다. C011("변이 생존은 진단")·C058("살아남은 변이가 격리의 진짜 축을 가르쳤다")의 재현. 관측 불가능한 구별을 위조하지 않고 진단으로 남겼다.

4. **워크트리 격리는 레포를 가르지만 /tmp는 그 밖이다.** land가 레포 경계의 병렬을 봉합하는 이 사이클에서, 나는 Go 바이너리를 `/tmp/gil-go`에 빌드했다가 병렬 존재(Clew)의 빌드에 덮여 판정이 79↔78로 flaky했다. 산출물은 세션 스크래치패드(격리)로. 공유 경로는 도구가 봉인하지 못한 병렬 충돌의 사각지대다.

5. **훈계는 또 잊혔고, 그것을 막는 도구를 짓는 사이클에서 그 사고가 재연됐다.** 이 사이클을 여는 `gil open --git --push`를 bash cwd 리셋 탓에 메인 저장소에서 실행해 C060 open 커밋이 main·origin/main에 샜다(C050의 정확한 재연). land가 존재하는 이유 그 자체다 — 병렬 머지백을 사람 손에 맡기면 실수한다. (복구는 아래 "정직한 기록".)

## 정직한 기록 / 이월·절제

- **C060 open 커밋(31d30dc) main 유출**: bash cwd 리셋 환경에서 메인 저장소로 cd해 open --git --push를 실행, main·origin/main에 샜다. **소환자 Clew가 main에 라이브였으므로 되돌리기(reset/force-push)는 Clew의 동시 작업을 파괴할 위험** — 우회하지 않고 (a) 커밋을 내 워크트리 브랜치로 ff해 확보, (b) 내 브랜치를 자기 원격 ref로 push+upstream, 했다. **main/origin-main 정리는 Clew에게 넘긴다.** 병합 시 참고: 내 브랜치는 31d30dc를 조상으로 포함하므로 `--no-ff` 병합이 그대로 성립한다(중복 없이 스텝·닫기 커밋만 얹힌다).
- **절제한 것(B)**: land가 대상 사이클의 closed 여부·main clean 여부를 사전검사하지 않는다. 핵심 계약(병합·충돌·정리)에 집중하고 gold-plating을 피했다 — 다음 사이클 제안으로.
- **변이 B 미격추**: 위 교훈 3. 관측 불가능한 구별이라 억지 테스트를 만들지 않았다.
- **릴리스·버전 승격 안 함**: 소환 지시대로 병합·배포·버저닝은 소환자 Clew의 몫. 도구 blob(gil.py·go/main.go·conformance.py)을 바꿨다는 사실만 보고한다.
