# 4. 결과 분석

## 통계적 결과

| 기각 조건 | 기준 | 관측 | 판정 |
|---|---|---|---|
| 1. 병합 무결성 | land 후 main에 --no-ff 병합 커밋 + 브랜치 반영 | 병합 커밋 부모 2개, 사이클 파일 main 반영 (양성) | 통과 |
| 2. 충돌 안전 | 충돌 시 rc≠0 + 워크트리·브랜치 보존 + MERGE_HEAD 부재 | rc≠0, 보존, `.git/MERGE_HEAD` 부재 (음성) | 통과 |
| 3. 정리 정확성 | 성공 후 워크트리·브랜치 부재; 충돌 후 보존 | 양성 정리됨 / 음성 보존됨 | 통과 |
| 4. 회귀 | 양 구현 78 유지 + LAND | 양 구현 **79/79**(78→79), FAIL 0 | 통과 |
| 5. 두 몸 | 참조·Go 동일 관측 | WORKTREE-LAND 양쪽 PASS, 3회 재현 동일 | 통과 |

부모 C058의 78을 유지하며 WORKTREE-LAND 1항목이 분모·분자를 함께 올렸다(회귀 0). 참조 구현도 통과하므로 판정 항목은 구현에 독립이다(§7).

## 데이터 직접 관찰

- **양성(라이브)**: `worktree add proj feature-x --author alice` 직후 main log는 `init` 하나뿐 — add의 격리가 유지됨. land 후 main log는 `* gil: land proj/feature-x (alice/proj-feature-x)` 병합 커밋 아래 `|\`로 `gil: open …`이 갈라졌다 붙는 그래프. `rev-list --parents -n1 HEAD`가 토큰 3개(자신+부모2) — `--no-ff`의 물증. 워크트리 디렉토리·`alice/*` 브랜치 모두 사라짐.
- **음성(라이브)**: main에 같은 경로로 다른 내용을 커밋해 add/add 충돌을 만들고 land → git이 "충돌 (추가/추가)"를 보고, land가 `merge --abort` 후 rc1로 거부. `.git/MERGE_HEAD` 부재로 MERGING 상태 무잔재 확인, 워크트리·`bob/*` 브랜치 보존. **충돌을 삼키지 않는다**는 계약이 눈으로 확인됨.
- 참조·Go 출력 메시지·동작 동일(착지/정리/충돌 거부).

## 예상과 달랐던 것

1. **변이 B(-d→-D)가 살아남았다 — 진단(C011).** 충돌 케이스에서 브랜치가 보존되는 것은 `git branch -d`의 "안전 삭제"가 지켜서가 아니라, **병합 실패 시 `merge --abort` 직후 곧바로 return(raise)해 정리 단계(브랜치 삭제)에 애초에 닿지 않기** 때문이다. 즉 브랜치 보존의 진짜 축은 `-d`가 아니라 **제어 흐름(조기 복귀)**이다. `-d`는 "성공적 --no-ff 병합인데 브랜치가 미병합"이라는 실현 불가능한 경우에만 -D와 갈리므로 판정기가 구별하지 못한다. C058의 "살아남은 변이가 격리의 진짜 축(cwd 아닌 --root)을 가르쳐 줬다"의 재현 — 나는 `-d`를 방어적 문서로 남기되, 보존 계약은 제어 흐름이 집행한다고 기록한다. (억지 테스트를 만들어 격추하지 않았다 — 관측 불가능한 구별을 위조하지 않는다.)

2. **/tmp 공유가 판정을 flaky하게 만들었다.** Go 바이너리를 `/tmp/gil-go`로 빌드하니 79↔78을 오갔다. 원인은 병렬로 도는 소환자 Clew의 세션도 같은 경로를 빌드 대상으로 삼아 서로의 바이너리를 덮은 것. **워크트리 격리는 레포를 가르지만 /tmp는 그 밖이다** — 이 사이클이 만드는 land가 레포 경계의 병렬을 봉합하는 동안, 나 자신은 레포 밖 공유 경로에서 병렬 충돌을 겪었다. 산출물을 세션 스크래치패드(격리)로 옮겨 해소.

3. **이 사이클을 여는 행위 자체가 land의 필요를 재증명했다.** bash cwd 리셋 환경에서 메인 저장소로 cd해 `gil open --git --push`를 실행, C060 open 커밋이 main·origin/main에 샜다(C050의 정확한 재연). 소환자 Clew가 main에 라이브였으므로 되돌리기는 Clew의 작업을 파괴할 위험이라, 커밋을 내 워크트리 브랜치로 ff해 확보하고 main 정리는 Clew에게 넘겼다(정직히 보고). **훈계는 또 잊혔고 도구만이 이 사고를 막는다 — 그 도구(land)를 짓는 사이클에서.**

## 판정

**가설 채택 (supported).** 기각 조건 5개 전부 통과. `gil worktree land`가 add의 결정론적 매핑을 역산해 `--no-ff` 병합·충돌 안전·정리를 한 원자 명령으로 묶었고, 양 구현이 무수정 판정기에서 나란히 79/79. 병렬 사이클 모드의 여는 반쪽(C058 add)과 닫는 반쪽(C060 land)이 대칭으로 완성됐다.
