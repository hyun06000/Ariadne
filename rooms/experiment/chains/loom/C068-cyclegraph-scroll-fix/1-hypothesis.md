# 1. 가설 수립

## 이전 사이클의 교훈

부모 loom/C067이 체인 클릭 시 가로 사이클 노드-엣지 그래프를 펼치게 했다. 상현님 지적: **"스크롤이 되는 게 아니라 그냥 전부 다 한 번에 보여져서 너무 작게 보인다. 스크롤 넣어줘."** 원인: 체인 아코디언이 지도 카드(`.card.hmap`) 안에 있는데, C064의 CSS `.gil .hmap svg{max-width:100%}`가 **하위의 모든 svg**에 적용된다 — L0 지도뿐 아니라 넓은 가로 사이클 그래프까지 카드 폭에 **축소**시켜, `.cyclegraph{overflow-x:auto}` 스크롤이 발동하지 못하고 그래프가 작게 뭉개졌다.

## 문제 분할

셀렉터 범위가 너무 넓은 게 유일한 문제다. `.hmap svg`(후손 전체) → **`.card.hmap>svg`(직계 자식만)** 으로 좁히면, L0 지도(카드 직계 svg)만 폭에 맞춰지고, 중첩된 `.cyclegraph` 안 사이클 그래프 svg는 max-width 제약이 사라져 자연 크기를 유지하며 `overflow-x:auto`로 스크롤된다.

## 가설

> **가설**: `.gil .hmap svg` 셀렉터를 `.gil .card.hmap>svg`(직계 자식)로 좁히면, L0 체인 지도는 그대로 폭에 맞춰지고 가로 사이클 그래프는 축소되지 않은 자연 크기로 가로 스크롤되며, flat 불변·양 구현 바이트 동일이다.

## 기각 조건

- 그래프가 여전히 축소되거나, L0 지도가 폭을 넘겨 페이지 가로 스크롤이 생기면 기각.
- 두 구현의 기본·`--flat` 출력이 1바이트라도 다르거나 WEB-* 회귀 시 기각.
