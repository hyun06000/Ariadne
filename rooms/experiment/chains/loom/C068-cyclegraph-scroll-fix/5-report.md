# 5. 결과 보고

## 요약

C067의 가로 사이클 그래프가 스크롤되지 않고 카드 폭에 **축소**돼 작게 보이던 문제를, CSS 셀렉터를 `.hmap svg`(후손 전체)에서 `.card.hmap>svg`(직계 자식=L0 지도만)로 좁혀 고쳤다(loom/C068). 이제 사이클 그래프는 자연 크기를 유지하며 `.cyclegraph{overflow-x:auto}`로 가로 스크롤된다. CSS 한 줄, flat 불변·양 구현 byte-identical(참조 90/90·Go 83/83) — 가설 채택.

## 교훈

1. **후손 셀렉터의 사정거리는 DOM 구조가 바뀌면 함께 바뀐다.** `.hmap svg`는 지도만 있을 땐 맞았지만, C066에서 체인 아코디언을 카드 안에 넣자 사이클 그래프까지 잡아 축소시켰다. 배치 변경이 두 사이클 뒤 시각 버그로 나타났다. **범위를 넓게 잡은 셀렉터는 구조 변경에 취약하다 — 의도한 요소를 직계(`>`)로 좁혀라**(C044 CSS판).
2. **작은 CSS도 도구 변경이라 사이클·양 구현·릴리스를 거친다.** 한 줄이어도 gil.py·go 동형·parity·conformance·버전 승격(v2.24.0)을 밟았다 — 계약(양 구현 바이트 동일)은 크기와 무관.
