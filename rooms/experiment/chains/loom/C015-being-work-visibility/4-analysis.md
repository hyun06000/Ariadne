## 데이터 직접 관찰

- 이 사이클 자체가 첫 관전 대상이었다: C015는 `--git --push`로 열리고 스텝마다 전이 커밋됐으며, 뷰어에서 "●●●●○ 4/5 분석 · 활동 N분 전"으로 보인다.
- 워크플로 추출 테스트(T3)가 뜻밖의 보너스를 증명했다: 로컬 클론에도 Weft의 브랜치가 있으므로 branches/ 렌더가 원격 없이 재현된다 — "정의가 곧 테스트" 패턴이 4번째로 값을 했다.

## 예상과 달랐던 것

- **병렬 번호 경합이 예언보다 빨리 실물이 됐다**: 이 사이클을 여는 순간 main이 C014를 발급했고, Weft의 워크트리도 C014를 쓰고 있었다. 소환자 양보(내 사이클을 C015로 개명)로 해소 — 번호 발급의 직렬화(예: 소환 시 번호 예약, 존재별 네임스페이스)가 다음 규율 후보다. SPEC §6-6에 알려진 문제로 명문화.
- 소환자의 `git add -A`가 워크트리를 오인 추가한 사건(직전)과 합치면, 병렬 존재 시대의 소환자 수칙이 셋 모였다: ignore 구획 · 번호 양보 · add 범위 명시.
