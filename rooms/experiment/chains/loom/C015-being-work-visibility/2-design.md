# 2. 실험 설계

## 절차 (판정 표를 겸해 선고정)

1. **web 최근 활동 주석** — 열린 사이클마다 `git log -1 --format=%ct|%s -- <사이클 경로>`를 조회(저장소가 아니거나 깃 부재면 조용히 생략), 스텝 배지 뒤에 "· 활동 N분 전"과 테이블에 커밋 제목을 표시. gil-data JSON에 `last_activity` 포함(선택 필드).
2. **워크플로 브랜치 렌더** — 빌드 run 블록 확장: `origin의 worktree-* 브랜치`마다 `git archive`로 chains만 추출해 `_site/branches/<이름>.html` 렌더, `_site/index.html`은 main. 브랜치 목록을 `_site/branches/index.html`에 링크(존재 브랜치 없으면 빈 목록 — 실패 아님).
3. **판정** — T1: 실제 레포 web에서 열린 C015에 활동 주석 존재. T2: 깃 없는 임시 복사본에서 web 정상 생성(주석만 생략). T3: 워크플로 run 블록 추출을 신선 클론에서 실행(C007 방식) — branches 산출 포함 exit 0. T4: conformance 26항목 회귀 0.
4. **릴리스** — v0.7.0 (도구 변경). 소환 규약 v3(스텝 커밋·브랜치 push 의무)을 SPEC §6에 추가.
