## 데이터 직접 관찰

계측판(probe_layout.py)이 무한 루프를 좌표로 찍었다:
```
node=C032 row=12 col=0
tracks=[None,None,None,None,None,None,None,None]   ← 전부 비었는데
occupied at row 12: [0, 4, 6]                        ← (12,0)은 점유
```
`(12,0)`이 occupied라 while 진입 → `tracks[0]!=node`라 안 비우고 `free_slot()` → **tracks가 다 None이라 또 0 반환** → 영원히 col=0. `free_slot`은 "빈 track"을, while은 "빈 좌표"를 원하는데 tracks가 비면 free_slot이 점유된 col만 준다. 두 축의 경계에서 전진 수단이 사라졌다.

수정 후 같은 데이터에서 C032는 col=0이 막히자 1→2로 밀려 첫 미점유 좌표에 안착, 종료. 정상 그래프는 애초에 `(row,col)`이 비어 있어 while에 진입조차 안 하므로 좌표 불변(바이트 동일이 이를 확증).

## 예상과 달랐던 것

- **인공 최소 그래프로 재현 불가**: 선형+분기 조합을 40노드까지 스윕해도 스핀이 안 났다. 스핀은 order 순서·depth·병합 예약이 특정하게 얽혀야 발현 — loom prefix 36(분기 노드 단 1개!)에서 처음 났다. 그래서 판정기는 인공 그래프 대신 **실제 스핀 위상을 순수 parent 인덱스로 박제**했다.
- **버그가 진단 도중 자기를 증명**: 스핀을 고치려 연 C075가 loom을 스핀시켰다(C073→C074·C075 분기 생성). 이 저장소 뷰어도 그 순간 망가졌다 — 버그가 eda 특유가 아닌 일반 결함임을 스스로 드러냈다.
- **가동중단은 판정기의 사각**: 98항목이 "무엇을 내는가"만 봤고 "끝나는가"는 아무도 안 봤다. 소비자 저장소 CPU 100%가 되고서야 드러난 계약면.

## 판정

**채택.** 기각 조건 4개(종료 미보장 / 좌표 변경 / parity 깨짐 / 판정기 미관측) 전부 불발. 참조·Go 양 구현 수정, 회귀 0, WEB-LAYOUT-TERMINATES로 계약화 완료.
