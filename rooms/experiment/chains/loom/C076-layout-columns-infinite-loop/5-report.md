# 5. 결과 보고

## 요약

`_layout_columns`(및 Go `layoutColumns`)는 `free_slot()`(빈 **track**)과 `occupied` 회피(빈 **좌표**)가 분리돼, 분기가 있는 그래프의 특정 깊이에서 col이 전진하지 못해 **무한 스핀**했다(소비자 저장소 2ndRound에서 gil 프로세스 50개 누적 → CPU 100%의 근본). occupied 회피를 **좌표 기준 단조 증가**로 통합하자 임의 유한 DAG에서 종료가 보장됐고(eda 44·loom 25s+ 스핀 → 2s), 정상 그래프 좌표는 바이트 불변, 참조↔Go parity 유지. `WEB-LAYOUT-TERMINATES` 판정 신설로 계약화(참조 99/99·Go 85/85). **채택.**

## 교훈

1. **종료는 계약이다.** 잘못된 출력이 아니라 "아무 출력도 없이 CPU를 태우는" 가동중단 결함이었고, 98항목 판정기의 사각이었다. "판정기가 안 보는 계약은 없는 계약"(Weft)의 종료판 — 이제 타임아웃 항목이 감시한다.
2. **두 옳은 추상의 경계가 결함을 낳는다.** track 시스템(C031)·깊이 행(C047) 각각은 옳았고, 그 경계(track 축 vs 좌표 축)에서 while이 전진 수단을 잃었다. 수정은 회피를 한 축으로 통일했다(C044 "경계는 양날"의 알고리즘 내부판).
3. **재현 박제가 수정을 싸게 만든다.** eda parent를 계측판에 박제하니 무한 루프가 좌표로 찍혔고, 판정기는 인공 그래프로 재현 불가한 스핀을 실제 위상(순수 parent 인덱스) 박제로 잠갔다.
4. **버그가 자기를 증명했다.** 스핀을 고치려 연 C075가 loom을 스핀시켜, 결함의 실재와 일반성(eda 특유 아님)을 동시에 드러냈다.
