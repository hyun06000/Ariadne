# 2. 실험 설계

가설(1-hypothesis): steps.yaml을 원형 노드(kind 색) + 실선 parent + 파선 backtrack의
자기완결 SVG로 렌더하면 (a) 위상 보존 (b) 사이클 DAG와 같은 시각 언어 (c) 상현님 방향 승인.

## 확정할 설계 (검증 대상)

### 시각 언어 — v2 사이클 DAG에서 가져온다

v2 `gil web`의 사이클 DAG 관찰(캡쳐):
- **노드 = 원(circle)** + 안에 라벨, 아래에 텍스트.
- **엣지 = 실선**, 부드러운 곡선/직선.
- 색이 상태를 나타냄.

스텝 트리도 이 언어로:
- **노드 = 원** (반경 ~22px), 원 안에 id(s1·s2…), 원 아래에 kind 텍스트.
- **kind 색 구분** (원의 채움/테두리):
  - `define` = 파랑, `hypothesis` = 보라, `verify` = 청록, `analyze` = 회색.
- **잎 상태**는 원 스타일로:
  - 산 잎(analyze/success) = 초록 채움 + 굵은 테두리.
  - 죽은 잎(analyze/backtrack) = 회색 파선 테두리 + 흐림.
- **parent 엣지 = 실선**(회색), **backtrack 엣지 = 파선**(주황) + 화살촉(잎→조상).

### 레이아웃

C004 좌표계 재사용(depth=세로, 형제=가로 col 분배) — 검증된 것. 노드만 박스→원으로,
텍스트를 원 밖(아래)로. 정수 좌표 유지(이식 시 바이트 동일 가능, C004 교훈).

### 목업 스크립트

C004 `steptree.py`를 복사(닫힌 사이클 불변)해 **노드 렌더만 원형으로 교체**한 목업
`circletree.py`. 나머지(파서·트리·엣지·좌표·범례)는 최대한 유지 — 변화를 노드 표현에
국소화해 "원형이 유일 변수"가 되게.

## 절차

1. **circletree.py 작성** (3-verification/): C004 steptree 복사 → 노드를 `<circle>` +
   원 아래 텍스트로, kind 색·잎 상태 스타일, parent 실선/backtrack 파선.
2. **C002 데이터로 렌더** → out.html (자기완결).
3. **헤드리스 캡쳐** → PNG. `_viewer-design/captures/`에 R1로 박는다.
4. **위상 검증** (measure): 원형 트리도 노드 10·parent 엣지 9·backtrack 2·죽은 잎 2·
   산 잎 1을 담는가 (C004 회귀 0 = 정보 손실 0).
5. **상현님께 캡쳐 제시** → 방향 판정 (K3: 사람 눈이 판정자). "맞다"면 supported,
   "아니다"면 rejected하고 피드백을 다음 사이클 가설로.

## 준비물

- Python 3 stdlib. C004 steptree.py(복사 기반, 닫힌 사이클), C002 case steps.yaml.
- 헤드리스 Chrome (캡쳐). 산출물 3-verification/ + 캡쳐 _viewer-design/captures/.

## 측정 방법

- **M1 (위상 보존, K1)**: 원형 렌더가 노드 10·parent 9·backtrack 2·죽은 잎 2·산 잎 1을
  전부 담는다(박스 대비 정보 손실 0). **기준: 5수치 일치면 PASS.**
- **M2 (시각 언어, K2)**: 노드=원·parent=실선·backtrack=파선이 캡쳐에 보이고, kind 4색이
  구별된다. **기준: 캡쳐 육안 + SVG에 circle 10·실선/파선 클래스 존재면 PASS.**
- **M3 (방향 판정, K3)**: 상현님이 캡쳐를 보고 방향 승인. **기준: "맞다"=PASS(supported),
  "아니다"=FAIL(rejected, 피드백→다음 가설).**
