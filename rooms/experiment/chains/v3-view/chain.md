# Chain: v3-view

## 이 체인이 정복하려는 문제 / 이뤄야 하는 목표

**v3의 세 층(체인·사이클·스텝)을 한 시각 언어로 드릴다운하는 뷰어를 만든다.**

v3-build 체인이 v3의 데이터·명령(스텝 트리·open/step/close·봉인)을 짓는다면, v3-view는
그 세 층을 **사람 눈에 드러내는** 뷰어를 짓는다. Sheen의 loomlight 계보(뷰어 사이클)를
v3로 잇는다.

### 상현님이 확정한 형태 (2026-07-21) — 3층 드릴다운

1. **체인** 클릭 → 카드가 펼쳐지며 **사이클 DAG**를 본다 (v2 뷰어의 그 그래프).
2. 사이클 DAG에서 **사이클 노드** 클릭 → 아래 카드가 열리며 **스텝 트리**를 본다.
3. 스텝 트리는 사이클 DAG처럼 **원형 노드 + 실선 그래프**로, kind를 **색으로 구분**.
   본문 펼침(C007)·상태 보존은 유지.

### 통일된 시각 언어 (핵심 원칙)

세 층이 **같은 시각 언어**를 쓴다 — 원형 노드, 실선 엣지, 색 구분. 층을 드릴다운해도
"같은 종류의 그래프"라는 연속감을 준다. 지금 스텝 트리(v3-build C004/C007)는 박스형
노드라 사이클 DAG와 시각이 다르다 — 이걸 통일하는 게 이 체인의 첫 정복 대상.

- **backtrack 엣지는 파선으로 구별 유지** (상현님 결정): parent 가지=실선,
  backtrack=파선. 실선 통일하면 "되돌아가 난 형제"와 "자란 자식"이 안 갈리므로,
  v3의 핵심(백트래킹 명시화)을 살리려면 구별이 남아야 한다.

### 유지되는 것 (v3-build에서)

- 자기완결(외부 리소스 0, file://로 열림) — v2 뷰어 계약.
- 스텝 트리 데이터는 v3-build C002의 `steps.yaml` 표현 그대로.
- 본문 인라인 임베드 + 클릭 펼침 + 다중 열림 상태 보존 (C007, Sheen).

### 작업 방식 — 캡쳐 → 피드백 → 사이클

상현님과 **실제 렌더 캡쳐를 보고 피드백을 주고받으며** 사이클을 돈다
(`_viewer-design/`에 라이브 기록). 목업 먼저(C067), 방향 확인 후 구현.
