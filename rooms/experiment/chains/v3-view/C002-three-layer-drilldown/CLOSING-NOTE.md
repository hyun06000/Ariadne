# C002 봉인 노트 — 왜 지금 닫는가

**판정: supported (채택), 단 이 구조의 마지막 사이클로 봉인.**

## 무엇이 검증됐나

3층 드릴다운(체인→사이클 DAG→스텝 트리)이 **한 시각 언어**로 연속됨을 목업 캡쳐로 검증했다
(5측정 PASS, `captures/01-collapsed.png`·`02-drilled-C004.png`). 드릴다운 상호작용·자기완결·
상태 색 어휘 분리 전부 확인. **이 시각 언어 결론은 구조가 바뀌어도 유효하다** — 뷰어가 무엇을
그릴지의 답이지, 데이터를 어디에 담을지의 답이 아니기 때문.

## 왜 여기서 닫나 (구조 전환)

상현님이 이 사이클 도중 두 근본 문제를 짚었다:

1. **`C00x/` 폴더 난립** — v3 open이 디렉토리를 사이클 정체성으로 써, 사이클마다 폴더가 쌓인다.
   그러나 v3 정신은 "깃=append-only 커밋 그래프"(C001)다. 사이클은 폴더가 아니라 **커밋으로**
   담겨야 한다.
2. **본문이 아직 md 파일** — 계보 trailer는 커밋 메타로 옮겼지만(C041), 스텝 본문(`steps/sN.md`)은
   여전히 디스크 파일이다. "문서를 md로 남기지 말고 깃 로깅을 쓰자"는 방향의 **절반만** 실현됨.

## 상현님이 확정한 새 구조 (다음 프로젝트에서 재설계)

- **rooms 확장**: `existence`(정체성)·`deployment`(배포)·**`project`**(신설). project 아래에
  `source`·`figure`·`md`·`field` 등 사용자가 자유롭게 만든다.
- **모든 버저닝·로깅은 git**. gil은 **템플릿화된 로깅**(위계 있는 노드 생성)만 담당.
- **본문 → 커밋 로그**: 짧은 커밋 제목(노드 위계·이름·id) + 긴 커밋 본문(스텝 디테일). 뷰어는
  `git log`에서 본문을 읽는다. `steps/sN.md` 폐지.
