# 2. 실험 설계

가설 하나만 검증: 넓은 체인 가로 그래프 위 **미니맵**이 JS 0줄·기본 레이아웃 불변·양 구현
바이트 동일로 전체 조망 + 임의 사이클 도달을 준다.

## UI 목업 — 전/후 (C067 철칙: 코드보다 목업이 먼저)

이 병렬 세션에서는 상현님 실시간 컨펌이 어렵다. 그래서 **가장 보수적이고 기존 상과 정렬된 선택**을
목업으로 명시하고 근거를 남긴 뒤 진행한다(land 시 Clew/상현님 검토). 함께 `design-mockup-before-after.html`
(다크테마 SVG)도 둔다.

### 후(後) — 제안 · 미니맵을 그래프 위에 얹음

```
▼ loom                                                        [체인 아코디언 카드]
┌──────────────────────────────────────────────────────────────────────────────┐
│  ┌── 미니맵 — 전체 개요 · 노드 클릭 → 그 사이클 문서 ─────────────────────────┐  │
│  │ ·················•···•···•···•···•···•···•···•···•···•···•···•···•···•·· │  │  ← max-width:100%
│  │ ·   (전체 70노드가 한 줄로 압축, 폭 맞춰 축소 — 형제 갈래는 아래 레인으로)     │  │     로 폭 맞춤
│  │ ·········································•······································ │  │     (한 화면에 전부)
│  └──────────────────────────────────────────────────────────────────────────┘  │
│  .cyclegraph  (전과 동일 — 자연 크기 가로 스크롤, 불변) ────────────────────────▶ │
│   0──o──o──o──o──o──o──o──o──o──o──o──o──o──o──o──o──o──o──o──o──o─ … (스크롤) │
│  C001 002 003 004 005 006 007 008 009 010 011 012 013 014 …                    │
└──────────────────────────────────────────────────────────────────────────────┘
   미니맵 노드 클릭 → 본 그래프와 동일한 #cycdoc-* 로 점프 → 그 문서가 :target으로 열림
```

- 미니맵은 전체 사이클을 **작은 정수 좌표**(COLW≈9·R≈3)로 압축한 SVG. 본 그래프와 같은 `_layout_columns`
  (깊이,레인)을 재사용하되 상수만 작게. **CSS `max-width:100%`로 카드 폭에 맞춰 축소** — 좌표 바이트는
  그대로(정수), 축소는 표시(presentational)뿐이라 parity에 영향 0. C068의 "자연크기 vs 폭맞춤" 통찰을
  미니맵에는 반대로(폭맞춤) 적용.
- 각 미니맵 노드 = `<a href="#cycdoc-name-cid">` — 본 그래프 노드와 **같은 타겟**. 클릭하면 그 사이클
  문서가 :target으로 열린다. 라벨은 없음(툴팁 `<title>`=cid만) — 개요니까.
- **좁은 체인(깊이 < 임계 `_MINI_MIN_DEPTH=12`)은 미니맵을 안 그린다** → genesis·loomlight 등은 출력이
  개선 전과 **바이트 동일**. 오직 loom(넓은 것)만 미니맵을 얻는다 — 필요한 곳에만.

### 왜 이 선택이 가장 보수적인가 (근거)

1. **JS 0줄 계약 유지**: 미니맵은 순수 SVG + 앵커. "현재위치 표시"는 스크롤 추적 = JS 필요라 기각.
2. **기본 레이아웃 불변**: 본 그래프(`_render_cycle_graph_h`)는 한 줄도 안 건드린다. "깊이 접기"는
   본 그래프를 변형해 회귀 위험 → 기각. 미니맵은 순수 **추가**(위에 얹기).
3. **parity 무위험**: 좌표 전부 정수(삼각함수·나눗셈 없음, C064). 폭맞춤은 CSS라 출력 바이트 불변.
4. **중복 회피**: lineage 호버-전체는 이미 노드 `<title>`이 제공(C067) → 다시 안 만든다.
5. **회귀 최소화**: 임계 미만 좁은 체인은 미니맵 미출력 → 그 체인들 바이트 동일(conformance 충격 최소).

## 절차

1. **목업 작도**(본 문서 + `design-mockup-before-after.html` → 헤드리스 Chrome PNG). ← 코드보다 먼저.
2. **gil.py `_render_cycle_minimap(name, chain)` 신설**(≈ `_render_cycle_graph_h` 인접):
   - `_layout_columns`로 (깊이,레인). `max_depth < _MINI_MIN_DEPTH`면 `return ""`.
   - 상수 `_MINI_COLW, _MINI_ROWH, _MINI_R, _MINI_X0, _MINI_Y0 = 9, 9, 3, 6, 8`, `_MINI_MIN_DEPTH = 12`(전부 정수).
   - parent 엣지 = 직선 `<line>`(정수). 노드 = 작은 원, `<a href="#cycdoc-*"><title>cid</title><circle/></a>`.
     closed=채움(rejected면 빨강)·open=surface+node 테두리 — 본 그래프와 같은 색 규칙.
3. **`_render_hierarchy_body` 배선**(gil.py:1673): `.cyclegraph` 앞에 `{_render_cycle_minimap(name, chain)}` 삽입.
4. **CSS `_WEB_HIER_CSS`에 `.minimap`·`.minimap-cap`·`.minimap svg{max-width:100%;height:auto}`** 추가(기존 var만 사용).
5. **go/main.go 동형 이식**: `renderCycleMinimap`·hbody 배선·`webHierCSS`. 정수 좌표라 이식=바이트 동일.
   CSS 이스케이프는 원문자열(백틱)로 보존(C003 교훈).
6. **검증**: Go 빌드 → `diff -q` 참조↔Go(기본·`--flat`) → conformance 양 구현 → 헤드리스 Chrome:
   loom 카드에서 (a) 미니맵이 폭 맞춰 전체 표시, (b) 미니맵 노드 클릭 → 그 문서 :target,
   (c) 좁은 체인엔 미니맵 없음. + 개선 전 기본 출력과 좁은 체인 부분 `cmp` 동일 확인.

## 측정 방법

- **parity**: 참조↔Go `gil web`(기본)·`--flat` `diff -q` 바이트 동일.
- **좁은 체인 회귀 0**: 개선 전 기본 출력에서 genesis·loomlight 등 좁은 체인의 hbody 구간이 개선 후와 동일.
- **conformance**: 참조·Go 각각 회귀 0(WEB-* 포함). gil-data JSON 계약 불변이므로 통과 기대.
- **상호작용**(스크린샷): 미니맵 폭맞춤 표시 · 미니맵 노드 클릭 → 문서 :target · 좁은 체인 미니맵 부재.
- **자기완결**: 외부 리소스 0(WEB-SELFCONTAINED) 유지, JS 0줄.
