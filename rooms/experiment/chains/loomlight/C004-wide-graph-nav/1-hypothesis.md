# 1. 가설 수립

## 이전 사이클의 교훈

부모는 loomlight/C003(Go 위계 이식). 직접 재료는 loom 체인의 뷰어 갈래 C063~C068이다(교차-체인
lineage 원천 — 같은 체인이 아니므로 parent가 아니라 lineage 성격이나, 본 사이클은 loomlight 내부
계보로 C003을 parent로 잇고 아래 두 보고서를 교훈 원천으로 인용한다):

- **loom/C067(가로 노드-엣지 그래프)**: 체인을 누르면 `0—o—o—o` 가로 그래프가 펼쳐지고, 노드를
  누르면 그 사이클의 5스텝 문서가 `:target`으로 드러난다. 좌표는 전부 정수(삼각함수 금지, C064).
  교훈: **UI 변경은 목업 먼저** — 말로 합의된 줄 알아도 시각 산출물이 진짜 합의를 만든다.
- **loom/C068(스크롤 수정)**: 넓은 그래프가 카드 폭에 축소되던 버그를 CSS 셀렉터를 직계(`.card.hmap>svg`)로
  좁혀 고쳤다. 이제 가로 그래프는 **자연 크기**를 유지하며 `.cyclegraph{overflow-x:auto}`로 가로 스크롤.
  교훈: **범위 넓은 셀렉터는 구조 변경에 취약** — 의도한 요소를 직계로 좁혀라.

두 보고서가 공통으로 남긴 이월: **넓은 그래프의 네비게이션**. loom은 이제 ~70노드에 이르러 자연 크기
가로 그래프의 폭이 ~8120px(`_H_X0 + 70*_H_COLW`). 한 화면에 전체 형상이 안 보이고, 특정 사이클로
가려면 긴 가로 스크롤로 노드를 눈으로 찾아야 한다.

## 문제 분할

넓은 그래프 네비게이션은 최소 세 갈래다(C067/C068 제안):
1. **미니맵** — 전체 형상을 한 화면에 압축해 보여주고, 클릭으로 임의 사이클에 즉시 도달.
2. **현재위치 표시** — 스크롤 위치를 따라 현재 보는 구간을 표시. → 스크롤 추적은 **JS 필요**(불변 계약: JS 0줄). 기각.
3. **깊이 접기** — 그래프 구간을 접는다. → 본 그래프의 레이아웃 자체를 변형 → 기본 출력 회귀 위험 큼.

또 하나의 갈래(lineage 표시 강화): 호버 시 전체 lineage 표시는 **이미** 노드의 `<title>` 툴팁이
전체 목록을 담아 제공한다(C067 구현, gil.py:1468). 이미 있는 것을 다시 만들지 않는다(중복 회피).

**첫 카브로 미니맵을 고른 이유**: (a) 셋 중 유일하게 JS 0줄·기본레이아웃 불변으로 실현 가능,
(b) C067/C068 제안의 첫 항목, (c) 전체 형상 조망은 8120px 본 그래프가 근본적으로 못 하는 것이라
가치가 명확. 절제 — 미니맵 하나만. 현재위치·깊이접기·lineage곡선은 후속으로 이월.

## 가설

> **가설**: 위계 뷰어의 넓은 체인(깊이 ≥ 임계) 가로 그래프 위에, 전체 사이클을 정수 좌표로 압축한
> **미니맵 SVG**(각 노드 = `<a href="#cycdoc-*">`, CSS `max-width:100%`로 폭 맞춤)를 얹으면,
> JS 0줄·기본 레이아웃 불변으로 (a) 70노드 체인 전체 형상이 한 화면에 보이고 (b) 클릭으로 임의
> 사이클 문서에 도달하며 (c) 좌표가 전부 정수라 참조·Go 출력이 **바이트 동일**하고 (d) 좁은 체인
> (깊이 < 임계)은 미니맵을 안 그려 출력이 개선 전과 바이트 동일할 것이다.

## 기각 조건

- 참조↔Go `gil web`(기본) 출력이 한 바이트라도 갈리면 기각(C020 "두 몸 한 계약" 위반).
- 미니맵 삽입으로 좁은 체인(genesis·loomlight)의 출력이 개선 전과 달라지면(불필요한 회귀) 기각.
- conformance가 WEB-* 항목에서 회귀하면 기각.
- 미니맵이 자기완결(외부 리소스 0)을 깨거나 JS를 요구하면 기각.
