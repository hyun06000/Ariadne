# 5. 결과 보고

## 요약

사이클이 많아질수록 평면 그래프 하나로는 한눈에 안 들어온다는 상현님의 관찰에서 출발해,
`gil web`에 opt-in 플래그 `--hierarchy`를 더해 **체인 목록 → 체인 그래프 → 사이클 5스텝**의
3단 드릴다운을 JS 0줄·외부 리소스 0·자기완결로 실현했다(중첩 `<details>/<summary>` + 앵커).
5측정(세부 9판정) 전부 통과 — 특히 **기본(플래그 없음) 출력이 개선 전과 바이트 동일**이라
참조·Go 양쪽 conformance가 초록으로 남고(참조 77/77), Go는 손대지 않고도 병합이 안전하다.
가설 **채택(supported)**.

## 교훈

1. **위계는 레이아웃 미세조정보다 근본적이다 — 안 그리면 안 얽힌다.** 백로그 B1(가로 압축)·
   B2(lineage 밀도)·B3(라벨 겹침)은 "한 화면에 다 그린다"는 전제에서 나온 압력이었다. L2에서
   `_render_svg`를 **체인 하나**에만 부르니 그 압력이 드릴다운으로 분산됐다 — 세 백로그를 각각
   깎는 대신 전제를 바꿔 한꺼번에 완화했다. 제약(JS 0)이 답(`<details>`)을 좁혀준 C005·C049의 재확인.

2. **opt-in 하위호환은 "관측 불가능한 변화"로 두 몸을 병합 가능하게 지킨다.** 위계를 기본으로
   만들었다면 Go(불변)와 바이트가 갈라져 C020(두 몸 한 계약)이 깨졌을 것이다. 대신 `render_web_page`의
   `else` 가지를 개선 전 문면 그대로 두고 분기만 얹어, `--hierarchy` 없는 세계에는 이 사이클이
   **한 바이트도 보이지 않게** 했다(`cmp` 실증). 큰 재작성을 한 몸에 먼저 실증하고 다른 몸은
   정직히 이월하는 절제(C036·C050)의 실제 적용 — 병합 안전이 능력 추가보다 먼저다.

3. **뷰어의 진실값은 "원장의 반사"이지 "디스크 전수"가 아니다.** M4 판정기가 `find cycle.yaml`(110,
   rounds 포함)로 그래프 노드(65)를 재려다 헛FAIL했다. 진실값을 뷰어 자신의 gil-data JSON 노드 수로
   바꿔 교정 — 판정 분모조차 원장을 반사해야 한다(C001 교훈 2의 연장).

4. **비추는 행위가 다음에 비출 것을 또 드러냈다(C001 패턴 반복).** 위계 요약이 loom 닫힌 사이클
   29건의 `무결론`(verdict 미소급)을 가시화했다 — B6(verdict 결말 서사의 소급 이주)의 새 실증 압력.
