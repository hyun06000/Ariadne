# 5. 결과 보고

## 요약

상현님 지시 "체인 아래 배포 아래 존재"에 따라 위계 뷰어 패널 순서를 존재→배포→체인지도에서 **체인지도→배포→존재**로 재배치했다(병렬배너는 최상단 유지). 렌더 순서만 바꿔 데이터 계약 불변, 참조·Go parity 바이트 동일, conformance 회귀 0으로 **채택**.

## 교훈

1. **표현/계약 분리가 배치 자유를 준다(C008 재확인)** — 순서는 렌더일 뿐 gil-data(계약)가 아니라, f-string 위치·format 인자 순서만 바꿔 회귀 0. "렌더는 계약 아님"(§7)이 UX 반복(패널 접기 C008 → 순서 C009)을 싸게 만든다.
2. **정보 위계 = 작업→성과→주체** — 체인(무엇을 했나)이 먼저, 배포(어디까지 갔나)가 다음, 존재(누가 했나)가 마지막. 상현님 순서 감각이 이 위계를 담았다.
