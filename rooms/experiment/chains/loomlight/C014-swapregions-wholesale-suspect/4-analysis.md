# 4. 결과 분석

## 가설 판정: **채택(supported)** — 리젝트 체인이 억셉트로 수렴

가설(swapRegions 통스왑이 상태 파괴 근원, 최소변경하면 열린 노드 보존)이 네 kill 조건을 전부 통과했다.

| kill 조건 | 결과 |
|---|---|
| 1. 통스왑이 아니면 기각 | probe: 수정 전 `chainNodeReplaced:true` — 통스왑 확정 |
| 2. 수정 후 노드 교체되면 기각 | `M1_nodePreserved:true` (참조·Go) — 보존됨 |
| 3. 데이터 갱신 깨지면 기각 | header `101개→124개` 갱신 + 열린 카드 보존 동시 성립 |
| 4. 회귀·두 몸 불일치면 기각 | 참조 133·Go 110·바이트 동일·C011 무회귀 |

## 데이터 직접 관찰

핵심은 **노드 정체성**이다. 수정 전 탐침: 열린 chain-loom에 JS 마커(`__probe='ALIVE'`)를 심고 폴링 후 확인하니 마커가 사라졌다 — `replaceChild(imported, cur[i])`가 노드를 새것으로 갈아치웠다는 결정적 증거. detKey는 `open=true`만 복원하지, **이 노드가 그 노드냐**는 복원 못 한다. 그래서 C011(detKey 수정)로도 헤드리스 open 불린은 살았지만, 노드에 붙은 모든 것(네이티브 `<details>` 토글 애니메이션 상태·포커스·이벤트 리스너·부분 스크롤)은 매 폴링마다 소실됐다.

`hasOpenDetails` 가드 후: 같은 탐침에서 마커 생존(`M1_nodePreserved:true`). 그리고 데이터 변경(v1→v2) 폴링에서도 열린 카드는 마커 생존·open 유지, **닫힌 header는 `체인1개·101개`→`체인6개·124개`로 갱신**. 트레이드오프 없이 둘 다 얻었다 — 열린 것만 보존하고 닫힌 것은 갱신하니까.

## 예상과 달랐던 것

**트레이드오프가 없었다.** 설계 시 "열린 카드 보존 > 그래프 실시간 갱신" 트레이드오프를 각오했는데(2-design), 실제로는 `hasOpenDetails`가 **열린 노드만** 건너뛰므로 닫힌 영역은 다 갱신됐다. 사용자가 보는 것(열린 카드)은 보존되고 안 보는 것(닫힌 영역)은 최신 — 트레이드오프가 아니라 정확히 옳은 분리였다.

## 판정

**채택(supported).** 통스왑이 상태 파괴의 아키텍처 근원임을 헤드리스 계측(노드 마커)으로 확정하고, 최소변경(`hasOpenDetails` 가드)으로 노드 정체성을 보존해 해결. 참조·Go 바이트 동일, 회귀 0.

**⭐ 방법론의 증명(상현님 "사이클 방식의 꽃")**: C012·C013 두 rejected가 헛되지 않았다. C012가 "환경 의존"으로 좁히고, C013이 "내 손 밖 계측기는 막힘"을 배우고, 그 끝에서 "내 손 안 계측기로 검증 가능한 가설"(통스왑=노드 교체, JS 마커로 관찰)을 골라 억셉트에 도달했다. **리젝트에서 배우고, 억셉트되기 위해 다음 사이클을 굴렸다.** 진단을 사이클로 하니 두 죽음이 세 번째의 정확한 조준이 됐다.
