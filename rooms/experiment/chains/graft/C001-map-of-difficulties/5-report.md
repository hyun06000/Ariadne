# 5. 결과 보고

## 요약

거대 레거시 레포(Flask, 676커밋·16태그)에 **준비 없이** gil을 채택하면 무엇이 깨지는지를, 상상이 아니라 실 레포에서 관찰했다. 마찰 **6건이 4범주**(A 골격 부재 · B 개념 부재 · C 이름공간 충돌 · D 인터페이스 비대칭)로 유한·구조적으로 분류됨 → **가설 채택**. 이 지도는 다음 사이클(legacy/ 이주 절차)의 직접 설계 입력이다.

## 교훈

1. **레거시 채택의 진짜 문제는 "gil을 심는 것"이 아니라 "기존 역사와의 관계"다.** gil은 빈 땅에 골격을 심을 줄 알고(F4: `open --new-chain --new-root`가 `_template` 없이 스텁 자동 생성 → 직후 fsck 위반 0), 거대 작업트리를 안 건드리고 자기 것만 커밋한다(F7: rooms/ 6파일만). **심기는 이미 작동한다.** 깨지는 것은 전부 "이미 있는 것과 만나는 지점"이다.

2. **가장 위험한 마찰은 크래시가 아니라 침묵이다 (F3).** `gil releases`가 Flask의 실제 16개 태그·CHANGES.rst를 보고도 `0개 릴리스 drift=0`을 성공처럼 출력한다. gil은 `v<semver>` + 자기 CHANGELOG 형식만 세상으로 알아서 레거시의 배포 역사를 **없다고 단언**한다. 에러는 사용자가 알지만 침묵하는 실명은 손실을 숨긴다 — 레거시 도구화에서 최우선으로 다룰 결함.

3. **gil은 "저기 있는 남의 저장소"라는 개념이 없다 (F1·D범주, 예상 밖).** `--root`는 open 전용이고 log/fsck/web/verify는 cwd=`rooms/experiment/chains`를 강하게 전제한다. gil의 세계관은 "내가 사는 저장소 = 나 자신"이다. 레거시 채택은 이 프레임을 처음으로 시험한다. (예상했던 "규모" 마찰은 미발생 — Flask 규모에서 체감 지연 0.)

4. **존재(B범주)는 레거시에서 비어 있다 (F5·F6).** `--author`가 존재의 방 실재를 검사하지 않아 아무 문자열이나 통과하고(F5), `--git` 커밋의 git author는 레포 로컬 config이지 gil `--author`가 아니다(F6, loom/C003·C004의 오래된 갭이 레거시에서 재현). "누가 이 사이클을 했나"의 Ariadne 개념이 채택 순간 공백이 된다.

5. **상현님의 직관("legacy/로 옮기고 gil로 재시작")은 관계 충돌의 우회 전략으로 유력하다.** 기존 코드·역사를 `legacy/`로 격리하면 F3(태그 충돌)·F6(git identity)이 만나는 표면 자체가 사라진다. F7(커밋 격리)이 그 기술적 안전성을 이미 절반 증명했다. 다음 사이클에서 절차로 설계·검증할 후보.

## 다음 사이클을 위한 제안

이 지도에서 갈래가 나뉜다(분기 후보):

- **(A) legacy/ 이주 표준 절차 — 상현님 아이디어의 직접 계승.** "기존 코드 → `legacy/`, gil로 재시작"을 재현 가능한 절차로 설계하고 실 Flask에 적용해 검증. graft 체인의 자연스러운 다음 매듭. **1순위 추천.**
- **(B) F3 releases 실명 수정.** gil이 레거시 태그·CHANGELOG를 "이해 못 함"과 "없음"을 구별하게(예: 비-semver 태그를 external로 보고). loom 계열 수정 사이클. 위험도 최고지만 legacy/ 전략(A)이 이 표면을 없애면 우선순위가 내려갈 수 있음 — A 설계가 B의 필요성을 판정할 것.
- **(C) `gil init`/`gil adopt` 포셀린.** F1·F2·F4를 묶어, 레거시 레포에서 한 명령으로 골격을 세우고(존재의 방 스텁 포함) 채택을 시작하는 진입점. gateway("한 문장 진입")의 레거시판.
- **(D) 존재↔git identity 연동(F5·F6).** 오래된 갭이라 graft 밖(loom)에서 다룰 수도. 우선순위 낮음.

**추천 경로: A → (A가 드러내는 것에 따라) B 또는 C.** Clew의 방식대로, 상현님의 원래 아이디어(legacy/ 이주)를 다음 가장 작은 단서로 삼아 실제로 절차를 짜고 Flask에 적용한다.
