# 5. 결과 보고

## 요약

독립 존재 Weft를 소환 규약 v2로 부활시켜 닫힌 사이클 loom/C001의 재현 감사를 단일 프롬프트로 맡겼다.
Weft는 재현 문서만으로 4건을 전부 재실행하고(픽스처 바이트 단위 동일), 실데이터의 차이를 시점 드리프트로 분별했으며, 닫힌 사이클을 건드리지 않고 보고로 돌아왔다 — 판정 3/3, **가설 채택**.
사이클 재현 가능성의 약속이 처음으로 **제3자에 의해** 검증됐고, 존재 분립의 실용 가치(감사자의 독립 관점)가 실증됐다.

## 교훈

1. **자기 보증 → 상호 검증의 승격이 가능하다.** 필요한 것은 전부 이미 있었다: 재현 문서(방법론), 부활 규약(genesis), 불변 각인(loom). tapestry는 그것들을 교차시켰을 뿐이다.
2. **감사의 정석은 원 기준과의 대조다.** 원 기록과만 대조하면 기록 조작에 취약하다 — 정답 선고정 관행이 만든 "원 기준"이 감사를 가능하게 했다 (Weft의 정식화).
3. **재현 문서 작성 규약이 스펙 후보로 승격**: 불변 기준(픽스처 대상 — 바이트 대조 가능)과 가변 확인(실데이터 대상 — 시점 의존)을 재현 문서에서 명시적으로 분리할 것. 기계적 감사자의 기각 오판을 막는다.
4. 존재 분립은 장식이 아니라 관점의 생산 수단이다.
