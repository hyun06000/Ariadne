# 5. 결과 보고

## 요약

conformance를 v3 계약으로 재정의하기 시작했다 — v3 쓰기 계약 3항목(V3-OPEN-CREATE·V3-OPEN-REJECT-EXISTING·V3-RETIRE-GUIDANCE)을 판정기에 신규로 세워 **134→137/137**(순증, 무회귀). 코드 실측으로 드러난 것: v2 OPEN-* 16항목은 **대부분 v3 open으로 1:1 교체할 대응물이 없다** — v2 open(사이클-간 번호·저자·부모)과 v3 open(사이클-내 스텝 트리 시작)은 다른 층이기 때문. **가설 채택(부분)** — v3 계약은 섰고, 완전 버전리스(게이트 없이 초록)의 다음 관문(판정기 v2 crash 내성 or v2 항목 제거)이 실측으로 찍혔다.

## 교훈

1. **⭐⭐ v2 open과 v3 open은 "다른 층의 명령"이다 — 사이클-간 vs 사이클-내.** v2 open은 번호 자동증가·저자 필수·부모 닫힘 게이트(사이클 *간* 계보)를 다뤘고, v3 open은 "빈 dir에 루트 define s1 생성"(사이클 *내* 스텝 트리 시작)만 한다. v3에서 사이클-간 정보(번호·저자·부모)는 cycle.yaml·notes 층으로 이동했다(migrate가 읽는 그 층). 그래서 "OPEN 16항목을 v3로 교체"는 대부분 **교체할 대응 계약이 없다.** C032 "인터페이스 정체성 전환"의 conformance 판.

2. **⭐⭐ 판정기의 v2 결합은 open보다 깊다 (M5, 다음 관문의 좌표).** open 항목만 v3로 옮기면 게이트 없이 초록일 줄 알았으나, 게이트 없이 **첫 v2 open(OPEN-CREATE)이 crash하면 순차 실행이 무너져 뒤 항목(v3 신규 포함)이 실행조차 안 된다**(conformance.py 예외 미처리). 완전 버전리스는 v3 항목 추가만으로 안 되고, **판정기가 v2 open crash에 견디거나(예외 격리) v2 계약 항목을 실제 제거**해야 한다. C033이 그 좌표를 정확히 찍었다.

3. **⭐ "v2를 버린다"의 정직한 실현 = 순증부터, 삭제는 대응 검증 후.** 상현님 "겁내지 말고 v3 위주로"를 나는 v3 계약을 **실제로 세우는 것**(겁내지 않음)으로, 그러나 v2 삭제는 대응 검증 후로(작은 확실함) 실현했다. v3 3항목 순증(134→137, 회귀 0)은 위험 0으로 v3 계약을 판정기에 세웠다. v2 삭제(다음 관문)는 v2 계약의 v3 행방을 매핑해둔 표 위에서 안전하게.

4. **⭐ GUARD-\*는 버전 계약이 아니라 병렬 안전이다 — v3 open에도 살아야 한다.** C050 오염 방지 가드 5항목은 v2/v3 무관하게 존재의 병렬 협업을 지킨다. v3 open은 아직 guard 미부착(코드 실측) — 이전은 "삭제"가 아니라 "v3 open에 guard 부착 + 검증"이다. 상현님 "스무스한 v3 시작 안내"가 곧 안전의 존치.

## 다음 사이클을 위한 제안

1. **⭐⭐ 판정기 v2 crash 내성 or v2 항목 제거 (다음 관문, C033이 찍은 좌표)** — 게이트 없이 초록이 되려면: (A) conformance.py를 예외 격리(각 check가 crash해도 다음 진행)해 v2 항목이 게이트 없이 FAIL만 하고 crash 안 하게, 또는 (B) v2 OPEN/GUARD 항목을 실제 제거하고 v3 계약으로 대체. 상현님 "v2 완전 폐기" 방향이면 (B), 점진 전환이면 (A)+게이트.
2. **v3 쓰기 계약 확장** — step kind 순환(define→hypothesis→verify→analyze)·백트래킹(--outcome backtrack --to)·죽은 잎·close(산 잎 solved) 판정 항목. C033은 open만.
3. **GUARD v3 이전** — v3 open에 C050 guard 부착 + conformance V3-GUARD-* 항목. 병렬 안전을 v3에서 존치.
4. **읽기축 기본 전환 (C032 이월 ②)** — gil log/web 기본을 v3 눈으로.
5. **v2→v3 온보딩 문서** — C033 계약 행방 매핑표를 SPEC "v2에서 온 사람에게"로. "스무스한 v3 시작 안내"의 문서화.
