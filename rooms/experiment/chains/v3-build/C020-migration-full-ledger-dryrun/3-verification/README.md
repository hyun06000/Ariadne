# 3. 검증 — 실제 원장 전량을 격리 복제본에서 순회 소급 (드라이런)

C019 도출기를 실제 원장 규모로. 우리 저장소 복제본(1299 커밋)을 순회하며 derive→retro_imprint(notes)를 배선해, 도출 견고성·유령 감소·잔여 투명성을 대규모 실측한다. **격리 철칙: 우리 실제 저장소는 안 건드린다(복제본만).**

## 재현

```bash
bash build_case.sh <scratch>     # 우리 저장소 clone → 전량 순회 소급
python3 measure.py <scratch>     # 5측정 감사
```

## 산출물

- `full_ledger_migrate.py` — cycle.yaml 진실원 전량 순회. 견고 파서(→·—) + 원장관리 스킵 + 잔여 3종 분류.
- `derive_fingerprint.py`·`rebuild_migrate.py`·`retro_imprint.py` — C019판 재사용.
- `build_case.sh`·`measure.py`·`*-out.txt` — 격리 순회 + 5측정(ALL PASS).

## ⭐ 핵심 발견 — 실제 원장은 100+ 사이클에 걸쳐 관습이 진화했다

subject 전수 조사가 C019 가정을 깨뜨렸다:
- **step N/5의 N이 대부분 2~5** — 최신은 open이 "1/5 가설"을 겸하고 step은 2/5부터(사이클당 step 4개).
- **초기 사이클은 다른 형식** — `gil: C057 가설 작성`, `gil: C058 구현+검증+분석+보고`(여러 스텝 한 커밋).
- **원장 관리 커밋** — release 77·land 15·reserve 9·renumber·withdraw… (스텝 아님).

**subject만으로 전량 도출은 불가능** — 관습이 진화했기에. 그러나 이것은 실패가 아니라 **C017 가시성 원리의 대규모 적용**이다: 도출 가능한 건 도출하고, 못 하는 건 정직히 유령으로 남겨 정체를 분류 보고.

## ⭐ 진실원 재선택 — subject 대신 cycle.yaml

subject는 관습 진화로 불안정하지만 **cycle.yaml은 스키마가 안정적**(id·chain·parent·step·verdict). 사이클 발견은 cycle.yaml, 커밋 찾기는 subject의 `<chain>/<id>`(cycle id는 관습에 안정적), 구조는 cycle.yaml에서. **subject는 커밋을 찾는 열쇠로만.**

## 전량 순회 결과 (실측)

- **150 사이클 발견**, **532 스텝 노드 도출·각인**(드라이런=각인 일관).
- **20 사이클 도출 실패**(커밋 못 찾음 — 재번호·관습 이질).
- **잔여 유령 3종 분류**: 비-원장 305(memory·부트스트랩) · 원장관리 305(release·land·close…) · 도출실패-스텝 157(관습 이질 스텝).

767 유령이 정체별로 투명하게 갈렸다 — 침묵 0. 532가 지문을 받아 v3의 눈에 보이게 됐다.

## 5측정 (ALL PASS)

| 측정 | 확인 | 결과 |
|---|---|---|
| **M1 전량 순회 무사고** | 복제본 1299 커밋 SHA == 우리 원장 SHA — notes 각인이 커밋 하나도 안 바꿈 | PASS |
| **M2 유령 감소** | 소급된 사이클(C015)의 step 커밋 5개 전부 notes 지문 받음(유령→노드) | PASS |
| **M3 도출 견고성** | `→`·`—` 둘 다 파싱, release는 step 아님·관리로 분류(오각인 0) | PASS |
| **M4 잔여 투명성** | 3종 분류 합(767) == 독립 계산 유령(767) — 잔여 정체 투명, 침묵 0 | PASS |
| **M5 원장 불변(격리)** | 우리 실제 저장소 HEAD·notes 측정 전후 동일 — 복제본만 각인, 무손상 | PASS |

## 결론

**ALL PASS → supported.** 전량 순회 소급이 격리 복제본에서 정합·안전하다: 커밋 불변으로 532 스텝이 지문을 받고(무사고), 이질 형식에 견고하며(→·—), 도출 못 한 767 유령이 3종으로 투명하게 분류되고(침묵 0), **우리 실제 저장소는 무손상**(격리 철칙). 관습 진화가 드러낸 잔여(도출실패-스텝 157)는 실패가 아니라 정직한 지도다. **드라이런으로 "전량 소급이 안전하고 정합적"임이 실증됐다** — 실제 적용은 이 검증 위에서 상현님 결정. 다음: 도출실패 157의 관습 이질 사이클 개별 규칙(정밀화) · 위상 접합(④ 복원).

## 정직한 경계

- **드라이런(복제본)만.** 실제 원장 적용은 별도 결정.
- **도출실패 157·미발견 20사이클** — 관습 이질(초기 형식·재번호). 전량 도출 억지 안 함(C017 가시성). 정밀화는 다음.
- **위상 접합 안 함**(④ 복원) — 사이클 내 노드 편입만.
- verdict 근사(C019) 유지.
- **계측기 결함 1건** — 판정 라인 sum에 re.Match 혼입(m3 bool 미확정), 수리(반증 아님, 5측정 자체는 정상).
