# 4. 결과 분석

## 측정 결과 요약

5측정 ALL PASS (measure-out.txt):

- **M1 전량 순회 무사고** — 복제본 1299 커밋 SHA == 우리 원장 SHA(notes가 커밋 안 바꿈).
- **M2 유령 감소** — C015 step 5개 전부 notes 지문 받음.
- **M3 도출 견고성** — →·— 둘 다 파싱, release 스킵.
- **M4 잔여 투명성** — 3종 분류 합(767) == 독립 유령(767).
- **M5 원장 불변** — 우리 저장소 HEAD·notes 무손상(격리).

전량 순회 실측: 150 사이클, 532 스텝 도출·각인, 20 사이클 미발견, 잔여 유령 767(비-원장 305·관리 305·도출실패 157).

## 가설 판정

**H1 (전량 순회 소급, 격리 드라이런) — supported.** H1a(무사고, M1)·H1b(유령 감소, M2)·H1c(견고성, M3)·H1d(투명성, M4) 전부 확인. **격리 철칙(M5)이 치명적 기각을 피했다** — 우리 원장 무손상.

## 교훈

1. **⭐⭐ 실제 데이터는 깔끔한 규칙을 부순다 — 그리고 그것이 마이그레이션의 진짜 모습이다.** C019는 "step 5개·표준 형식"을 가정했다. 실제 원장은 100+ 사이클에 걸쳐 커밋 관습이 진화해(초기 `gil: C057 가설 작성`, 최신 `→ N/5`, 원장관리 release·land) 그 가정을 부쉈다. **마이그레이션이 어려운 이유가 바로 이것** — 레거시는 한 규칙으로 안 쓰였다. 실측이 아니었으면 "532만 도출되고 767은 유령"이라는 진실을 몰랐을 것. 드라이런의 가치는 정확히 이 진실을 우리 원장을 안 망치고 알아낸 것.

2. **⭐⭐ 진실원을 안정적인 것으로 옮기는 것이 견고성이다.** subject는 관습 진화로 불안정했다. cycle.yaml은 스키마가 안정적이라 진실원으로 옮기니 견고해졌다 — subject는 커밋 찾는 열쇠로 강등. **불안정한 표면(subject)이 아니라 안정적인 구조(스키마)를 신뢰하라.** C018 "trailer=진실원"·C019 "규칙적 기록"의 연장: 무엇을 신뢰하느냐가 견고성을 정한다.

3. **⭐ 전량 도출을 억지 부리지 않는 것이 정직이다(C017 가시성의 대규모판).** 767 유령을 0으로 만들려 초기 관습마다 특수 규칙을 우겨넣을 수도 있었다. 대신 **도출 가능한 것(532)만 도출하고, 못 하는 것(767)을 정체별로 분류 보고**했다. 침묵으로 "다 됐다"고 하지 않고 "이만큼 됐고 이만큼은 이런 정체로 남았다"고 말한다. C017이 close 유령 1개에서 세운 원리가 767개 규모에서도 참 — 오히려 규모가 클수록 가시성이 필수다.

4. **⭐ 격리가 실측을 자유롭게 한다.** 복제본에서 532개 notes를 각인하고 실패를 관찰하는 실험을, 우리 원장 걱정 없이 할 수 있었다(M5). 만약 실제 원장에서 했다면 실패마다 되돌리기 부담에 실험이 위축됐을 것. **파괴 가능한 복제본이 있어야 대담한 드라이런이 된다** — 상현님이 "격리 저장소에서 검증"을 택한 판단의 값어치.

5. **잔여의 구조 — 767 = 비-원장 305 + 관리 305 + 도출실패 157.** 비-원장(memory·부트스트랩)과 원장관리(release·land·close)는 **애초에 스텝이 아니라** 소급 대상이 아니다 — 유령으로 남는 게 옳다. 진짜 개선 여지는 도출실패-스텝 157뿐(관습 이질 스텝). 즉 "유령 767"이 다 문제가 아니라, **610은 정상 유령이고 157만 정밀화 대상**이다.

## 정직한 경계

- **드라이런(복제본)만.** 실제 적용은 별도 결정(상현님).
- **도출실패 157 + 미발견 20사이클** — 관습 이질(초기 형식·재번호). 정밀화는 다음(억지 도출 안 함).
- **610 유령은 정상**(비-원장·관리는 스텝 아님). 157만 개선 대상.
- **위상 접합 안 함**(④ 복원).
- verdict 근사(C019) 유지 — refuted/partial 세밀 매핑 이월.
- **계측기 결함 1건**(판정 라인 sum re.Match, bool 미확정) 수리 — 반증 아님, 5측정 자체 정상.
- **성능**: 복제본 순회가 커밋마다 git subprocess 다수 호출로 느림(수백 초). 대규모 실각인 시 배치·bulk notes 필요(이월).
