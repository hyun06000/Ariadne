# 1. 가설 — 잎 못을 브랜치에서 태그로 정식화한다

부모: v3-build/C015-merge-is-lineage-command (supported — lineage=머지 도구화, 분기·합류 양방향 도구 강제 닫힘).
lineage: v3-build/C011-everything-is-a-branch (잎=태그 결론의 원리 증명).

## 이전 사이클의 교훈

C014·C015가 백트래킹=checkout(분기)·lineage=머지(합류)를 도구화하며, 잎 못은 계속 **브랜치**(`gil/dead/`·`gil/live/`·`gil/sealed/`)로 임시 처리했다. 두 사이클 모두 정직한 경계에 "잎=태그는 ◐ — 브랜치 못이 태그 아님(C011 결론=태그), 이 사이클은 다른 데 집중(그리디)"으로 이월했다. 이제 lineage까지 도구 강제됐으니, 그 남은 유일한 ◐을 정식으로 채운다.

C011 결론(잎=태그의 근거):
- **잎은 불변 시점**이라 브랜치(움직임)보다 태그(못박음)가 정확하다.
- **태그는 push되어 다머신 영구 생존** — "존재는 레포에만 산다"와 맞물림.
- **이름은 해시면 충분** — 논리 id(sid)는 커밋 trailer(Step-Id)가 담으므로, 못 이름은 `gil/leaf/<short-hash>`.
- **사이클 종결 = `cycle/<name>/solved` 태그** (gil v2 태그 규율과 일관).

## 문제 분할

도구화 진척표에서 lineage=머지가 C015에서 ✅이 됐으니, 남은 미완의 조각은 잎=태그(◐)와 뷰어(⬜). 뷰어는 Sheen 축(병렬 가능)이므로, 내 축 단독 순차(C074)로 잡을 것은 **잎=태그 정식화**다 — 브랜치 못 4곳을 태그 못으로. 이것이 마이그레이션의 선행조건(백트래킹 복원이 잎 못에 의존, 태그가 불변·push 생존이라 정식)도 완성한다.

## 가설

> **가설(H1)**: gilv3의 잎 못을 브랜치(`gil/dead/`·`gil/live/`)에서 **`gil/leaf/<short-hash>` 태그**로, 사이클 종결 못을 `gil/sealed/`에서 **`cycle/<name>/solved` 태그**로 바꾸면, C011 결론(잎=불변 시점=태그, push 생존)이 도구 동작이 되면서 백트래킹·lineage·close의 dangling 방지가 그대로 유지된다.

- **H1a 잎=태그:** 백트래킹으로 떠나는 잎(죽은·산)과 close의 산 잎이 `gil/leaf/<short-hash>` **태그**로 못박힌다(브랜치 0). 해시 이름, 논리 id는 trailer.
- **H1b 종결=태그:** 사이클 종결이 `cycle/<name>/solved` **태그**로 못박힌다.
- **H1c dangling 방지 유지:** 태그로 바꿔도 모든 잎·머지·close 커밋이 `git log --all`·`rev-list --all`에 생존(브랜치 못과 동일한 보존, 더 정확).
- **H1d 불변성:** 태그는 브랜치와 달리 움직이지 않는다 — 잎 시점이 못박힌 뒤 후속 커밋이 그 태그를 안 옮긴다.

## 기각 조건

- 태그로 바꾼 뒤 잎·머지·close 커밋 중 하나라도 `rev-list --all`에서 소멸(dangling→gc). → H1c 기각.
- 브랜치 못이 하나라도 남는다(정식화 불완전). → H1a/H1b 기각.
- C011·C012 build 스크립트가 쓴 태그 규칙(`gil/leaf/<short>`·`cycle/<name>/solved`)과 도구 출력이 불일치. → 정식화 실패.
- 회귀: lineage 머지(C015)·백트래킹(C014)의 5측정/측정이 태그 전환으로 깨진다.

## 검증 설계 (다음 스텝)

C015 build_case(두 산 잎 → lineage 머지)와 C014 build_case(3층 분기, 죽은 잎 2·산 잎 1)를 **태그판 도구**로 재실행. 측정:
- M1 잎=태그: 모든 잎이 `gil/leaf/<short>` 태그, 브랜치 못 0.
- M2 종결=태그: `cycle/<name>/solved` 태그 존재.
- M3 dangling 방지: 모든 잎·머지·close 커밋 rev-list --all 생존.
- M4 불변성: 태그가 잎 커밋에 고정(후속 커밋이 안 옮김).
- M5 회귀: C014 백트래킹·C015 lineage 측정이 태그판에서도 ALL PASS.

## 정직한 범위 (선긋기)

- 잎 이름은 해시(C011 결론). 논리 id ↔ 해시 매핑은 trailer(Step-Id)가 이미 담음 — 별도 색인 안 만듦.
- 태그 삭제/이동은 다루지 않음(잎은 불변). gil v2의 태그 삭제 금지 규율과 일관.
- 뷰어 재배선은 이 사이클 밖(Sheen 축, 병렬 가능) — 이 사이클은 잎 못에 집중(그리디).
