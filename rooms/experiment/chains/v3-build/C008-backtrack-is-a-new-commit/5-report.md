# 5. 결과 보고 — v3-build/C008-backtrack-is-a-new-commit

부모: v3-build/C006-close-seal-and-cycle-states (supported). 저자: Clew. 소환자: 없음 (main 단독·순차, C074에 따라 워크트리 미사용). 판정: **supported (채택)**.

## 요약

상현님이 v3-view 도그푸딩 세션(2026-07-21)에서 세운 정신 — **깃=append-only 전진기록(롤백 없음), gil=백트래킹·분기 지능** — 을 gilv3 스텝 층에 각인·실증했다. gilv3 v0.4로 실사례(C012→C013→C014, 백트래킹 2개) 10노드 트리를 `--git`으로 각인하니, 두 백트래킹조차 reset/checkout/revert 없이 **새 전진 커밋**으로 쌓였고(깃 그래프 한 줄 선형), 죽은 가지는 히스토리에 보존됐으며(벽의 지도), 되돌아감 논리는 오직 steps.yaml 포인터에만 담겼다. 5측정 ALL PASS, 산 잎(s10) 도달 → **닫는다**.

## 무엇을 했나

1. **gilv3 v0.4** (C006 v0.3 복사 후 확장, 닫힌 사이클 C006 원본 불변):
   - **append-only 계약 명문화**: `git_imprint`가 오직 add·commit만 호출함을 독스트링·주석으로 계약화.
   - **`_assert_forward_only` 전진 단조성 자기 집행 가드**: 각 각인 후 HEAD가 이전 HEAD의 자손임을(`merge-base --is-ancestor`) 확인해, 어떤 경로가 HEAD를 뒤로 옮기면 즉시 거부. 코드가 스스로 append-only를 집행한다.
   - **백트래킹 커밋 메시지 서술**: `s8 hypothesis (new branch from s1 after backtrack)` — 깃 로그만 봐도 어디서 되돌아가 새 가지가 났는지 읽히게(단, 진실원은 steps.yaml).
2. **실사례 각인** (build.sh): 메인 레포 밖 임시 깃 저장소(C005 규율)에서 C012→C014 10노드 트리를 gilv3 명령으로 각인 — s4·s7 두 백트래킹, s10 산 잎, close.
3. **5측정** (measure.py, 순수 stdlib): M1 코드 감사·M2 전진 단조·M3 벽의 지도·M4 역할 분리·M5 가드 유효성 음성 대조 — ALL PASS.

## 교훈

1. **백트래킹=새 커밋이 코드로 성립한다.** 죽은 잎에서 조상 define으로 되돌아가 새 형제 가지를 여는 그 순간, 깃엔 reset이 아니라 새 전진 커밋 하나가 쌓인다. 되돌아감의 목적지(s1)는 깃 커밋 부모에 0% 반영되고(부모는 시간순 직전 커밋), 오직 steps.yaml `parent`/`backtrack` 포인터에만 산다. **깃=선형 전진, gil=트리 지능 — 두 층이 서로 다른 관계를 담고 안 섞인다.**
2. **상현님의 정신은 발명이 아니라 이미 참인 것의 명명이었다.** `git_imprint`는 C005부터 add+commit만 했으므로 백트래킹은 **이미** 새 커밋이었다. 이 사이클은 새 기능이 아니라 잠재된 성질을 **명시 계약으로 승격 + 실증**했다. v2 원장의 append-only 정신이 v3 스텝 층에서 저절로 성립했다 — 데이터 모델(C002)이 뿌리면 깃 각인의 성질도 거기서 파생된다.
3. **방어 계약은 그것이 막는 위반을 재현해봐야 살아있다** (M5). 가드 없인 M2 PASS가 "애초에 뒤로 갈 코드가 없어서"로 공허할 뻔했다. HEAD를 인위로 뒤로 옮겨 가드가 거부함을 확인한 음성 대조가 가드를 의미 있게 만든다.
4. **직교 4층이 됐다.** C005(깃)·C006(봉인)에 이어, 이 사이클은 깃 층의 **성질**(append-only 전진)을 확정했다. 한 데이터 모델(steps.yaml 논리 트리), 그 위 직교 층: 깃(선형 전진기록)·봉인(cycle.yaml)·뷰어(C004/C007). 어느 층도 논리 트리를 오염시키지 않는다.
