# 4. 결과 분석 — 깃 로그로 트리 재구성

## 통계적 결과

4측정 ALL PASS, 네 기각 조건(K1~K4) 전부 미발동.

| # | 기준값 | 실측 | 판정 |
|---|---|---|---|
| M1 | 노드·parent 엣지 == 원본 | 복원 10노드·parent 엣지 전부 동일 | ✅ K1 미발동 |
| M2 | backtrack·outcome == 원본 | {s4→s1, s7→s1}·{s10 success, s4·s7 backtrack} 동일 | ✅ K2 미발동 |
| M3 | git 하위명령 = log만 | log뿐, steps.yaml·show·diff 접근 0 | ✅ K3 미발동 |
| M4 | 파싱 배타·왕복 무손실 | 배타적·복원→yaml == 원본 바이트 동일 | ✅ K4 미발동 |

## 데이터 직접 관찰

수치 뒤로 들어가 복원 출력과 `rebuilt-steps.yaml`을 직접 봤다.

**1) parent가 대개 시간순 직전이고, 예외는 백트래킹 후 새 가지뿐.** 복원 출력에서 s2·s3·s6·s9·s10의 parent는 전부 시간순 직전 노드(순환 계승)다. 오직 s5·s8만 parent=s1로, 시간순 직전(s4·s7)이 아니다 — 이 둘이 커밋 메시지 `(new branch from s1 after backtrack)` 서술로 복원된 **유일한 비-직전 부모**다. 즉 **트리의 모든 비자명한 정보가 두 커밋 메시지에 압축돼 있다.** 나머지는 순환 규칙(C003)이 공짜로 준다.

**2) 되돌아감의 목적지가 두 곳에 이중 각인돼 있다.** s4·s7의 `(backtrack to s1)`(죽은 잎의 되돌아감 선언)과 s5·s8의 `(new branch from s1)`(그 되돌아감의 착지점)이 같은 s1을 가리킨다. 복원기는 후자(`from s1`)로 parent를, 전자(`backtrack to s1`)로 backtrack 엣지를 각각 세운다. **되돌아감이 "떠남"과 "도착" 두 커밋에 나뉘어 기록돼 있어 복원이 견고하다.**

**3) 왕복이 바이트 동일이다.** `rebuilt-steps.yaml`을 C008 `built-steps.yaml`과 diff하니 **0바이트 차이**. 필드 순서·null 표기·body 경로까지 완전 일치. steps.yaml을 한 번도 안 읽고 깃 로그만으로 원본을 재생산했다 — **steps.yaml은 깃 로그의 파생 캐시다.**

## 예상과 달랐던 것

- **깃이 진짜 단일 진실원이 될 수 있다.** C008은 "steps.yaml=진실원, 깃=각인 결과"라 정리했는데, C009는 그 역이 **무손실**임을 보였다 — 왕복(steps.yaml→깃→steps.yaml)이 바이트 동일이니 둘 중 하나는 **잉여**다. 커밋 메시지 서술을 각인해두면(C008이 이미 함) **깃 로그가 트리를 완전히 담는다.** 이것은 C008 정리의 정련이다: steps.yaml은 진실원이 **아니라** 깃 로그에서 언제든 재생 가능한 편의 캐시일 수 있다.
- **정보의 국소성.** 10노드 트리에서 "비자명한 정보"는 딱 2비트(s5·s8의 parent가 s1)였다. 순환 규칙이 나머지 8노드의 parent를 결정한다. → *데이터 모델이 규칙을 담으면 저장할 정보가 극적으로 준다* — C002 "id=시간순, 트리=포인터"의 심화. 시간순(깃이 공짜로 줌) + 규칙(C003) + 예외만 명시(2곳) = 트리.
- **커밋 메시지가 계약면이 된다.** 지금은 서술이 `(backtrack to sM)` 같은 자연어에 가깝다. 복원이 이에 의존하니 **이 서술 형식이 사실상 스키마**다. 씨앗: 서술을 더 엄격한 구조(trailer, 예 `Backtrack-To: s1`)로 각인하면 파싱이 견고해지고 v3 fsck가 이를 계약으로 검증할 수 있다.

## 판정

**supported (채택).** 네 기각 조건 전부 미발동, 4측정 ALL PASS. 깃 로그(커밋 순서 + 메시지 서술)만으로 스텝 트리를 무손실 복원했고, 왕복이 바이트 동일이다. **깃이 스텝 트리의 단일 진실원이 될 수 있다** — steps.yaml은 파생 캐시. 산 잎(s10) 복원 도달 → 그리디하게 닫는다.
