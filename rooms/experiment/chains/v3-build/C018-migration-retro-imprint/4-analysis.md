# 4. 결과 분석

## 측정 결과 요약

4측정 ALL PASS (measure-out.txt):

- **M1 커밋 불변** — notes 각인 전후 원장 SHA 동일, 유령 커밋 SHA 불변.
- **M2 notes=trailer** — 소급된 유령 L1·L2·L3이 지문 값 정확히 노드로 복원.
- **M3 유령 감소** — 소급 후 유령 4→1(close만).
- **M4 pre-gil/close 구분** — pre-gil만 notes, close 안 건드림.

## 가설 판정

**H1 (git notes 소급각인, 커밋 불변) — supported.** H1a(커밋 불변, M1)·H1b(notes=trailer, M2)·H1c(유령 감소, M3)·H1d(pre-gil/close 구분, M4) 전부 확인. 기각 조건 어느 것도 발생 안 함 — SHA 변경 0, notes 못 읽음 0, 유령 감소 확인, close 오염 0.

## 교훈

1. **⭐⭐ append-only는 소급을 금지하지 않는다 — '커밋을 고치는 소급'만 금지한다.** C014가 확립한 "커밋 불소멸"이 소급각인을 막는 것처럼 보였다(닫힌 커밋에 어떻게 지문을 넣나). 그러나 append-only의 진짜 계약은 **커밋 도달가능성 단조**(집합이 오직 늘어남)였다(C014 정정). git notes는 커밋을 안 지우고 **밖에서 가리키는** 새 객체를 더할 뿐 — 정확히 append다. **소급과 append-only는 충돌이 아니다**: 과거를 고치는 게 아니라 과거에 대한 주석을 더한다. "깃도 옛 커밋에 서명 없어도 그래프 읽히듯"의 서명이 바로 notes다 — 서명은 커밋을 안 바꾸고 커밋을 인증한다.

2. **⭐ 진실원을 넓히는 것이 마이그레이션의 본질이다.** C017 재구성기는 커밋 trailer만 봤다. C018은 "trailer 있으면 그것, 없으면 notes"로 진실원을 넓혔다 — 소급된 유령이 v3 네이티브 커밋과 **동등**해진다. 마이그레이션은 데이터를 옮기는 게 아니라(원장 불변), 재구성기가 보는 **진실원의 범위를 넓히는** 것. notes 형식을 trailer와 똑같이 두어 한 파서로 읽히게 한 게 이 동등성의 열쇠.

3. **⭐ rev-list --all의 함정 — 도구의 부작용을 계약 위반으로 오판하지 말 것.** notes가 `refs/notes/commits` 커밋을 만드는 걸 "커밋이 늘었다=재작성"으로 오판했다. 진짜 계약은 원장 커밋의 불변이지 notes 저장소의 정지가 아니다. **불변 집행식은 무엇이 불변이어야 하는지 정확히 겨눠야 한다** — "전체 ref 커밋 수"가 아니라 "원장 커밋 SHA". C014 정점 교훈("append-only의 진짜 계약은 무엇인가")의 계측기판.

4. **pre-gil/close 구분이 subject로 깔끔히 갈렸다(C017 이월 해결).** trailer 없음이 유령 판별식이라 pre-gil과 close가 함께 걸렸는데, close는 subject가 `gilv3 `로 시작한다(v3 각인 이력). 이 subject 판별로 소급 대상을 pre-gil로 정확히 좁혔다. C017이 "총계는 이 카브, 구분은 다음 카브"로 미룬 것을 여기서 마감.

## 마이그레이션 4단계 진척

| 단계 | 상태 |
|---|---|
| ① 동결백업 | ⬜ |
| ② 읽기호환 | ✅ (C017) |
| **③ 소급각인** | **✅ (C018)** |
| ④ 백트래킹 복원 | ⬜ |

읽기호환·소급각인 두 단계가 서며, "유령을 무해하게 건너뛰고 → 지문을 불변하게 박아 되살린다"의 왕복이 닫혔다.

## 정직한 경계

- **소급각인 절차만.** pre-gil 3개에 **주어진 지문 값**을 각인(합리적 legacy 계보 L1→L2→L3). 실제 v2 cycle.yaml → v3 steps 지문 **자동 도출**은 규모 확장/별도 카브 — 이 카브는 "notes가 불변하게 박히고 재구성기가 읽는가".
- **위상 정합성 미검증.** 소급된 L1·L2·L3이 v3 트리(s1~s4)와 **한 트리로 이어지는지**(예: s1의 조상이 L3인지)는 안 봤다 — 지문이 노드로 복원되는 것만. 위상 접합은 ④ 복원 카브의 몫.
- **notes 충돌·병합**(여러 머신 각인) 안 다룸 — 단일 원장 소급.
- **계측기 결함 1건 수리** — rev-list --all이 notes 커밋 셈(refs/notes 제외로 수리, 반증 아님).
- gilv3.py 명령 불변 — 소급각인은 재구성(읽기) 축 + notes 도구.
