# 4. 결과 분석

## 측정 결과 요약

4측정 ALL PASS (measure-out.txt):

- **M1 무해한 유령** — 혼합 원장(유령 3+v3)에서 재구성이 안 죽고, v3 트리가 순수 원장 복원과 바이트 동형.
- **M2 경계 보존** — 재구성 전후 rev-list 동일, pre-gil 유령 3개 생존(읽기 전용).
- **M3 순수 v3 무회귀** — 유령 0에서 rebuild_migrate == C010 rebuild_trailer.
- **M4 유령 가시성** — MIX 유령 4·PURE 1·pre-gil 격리 3, `--report`가 정확히 보고.

## 가설 판정

**H1 (v3의 눈이 유령을 무해·가시적으로 건너뜀) — supported.** H1a(무해, M1)·H1b(경계 보존, M2)·H1c(순수 v3 무회귀, M3)·H1d(가시성, M4) 전부 확인. 기각 조건 어느 것도 발생 안 함 — 즉사 0, 유령 삭제 0, 회귀 0, 침묵 스킵 0.

## 교훈

1. **⭐⭐ 앞 사이클의 설계 선택이 미래 문제의 절반을 미리 풀어 놨다.** C010이 subject 파싱(C009)에서 trailer로 옮긴 것은 "서술 변조에 견고"(C010 M3)가 목적이었다. 그런데 그 전환이 **마이그레이션 읽기호환의 절반을 공짜로** 만들었다 — trailer 없으면 스킵하는 `if not sid: continue`가 close 커밋용이었는데 pre-gil 유령에도 그대로 통한다. C009 즉사(`sys.exit`)가 C010에서 우아한 스킵이 된 것. **좋은 데이터 모델은 그때 안 보이던 미래 문제까지 미리 닫는다** — 지문(trailer)을 진실원으로 삼자 "지문 없음"이 자연스러운 유령 판별식이 됐다.

2. **⭐ 마이그레이션의 핵심은 데이터 변환이 아니라 '눈'의 문제다.** 원장(v2 커밋)은 하나도 안 바뀌고 그대로 산다(M2). 바뀌는 건 v3 재구성기가 그것을 **어떻게 보느냐**뿐 — 못 읽는 유령으로 볼 뿐 파괴하지 않는다. 마이그레이션 노트의 "깃도 옛 커밋에 서명 없어도 그래프가 읽히듯"이 정확한 비유였다: 깃은 서명 없는 커밋을 지우지 않고 그냥 "검증 안 됨"으로 둔다. v3도 지문 없는 커밋을 지우지 않고 "안 읽힘"으로 둔다.

3. **⭐ 침묵 스킵은 마이그레이션에서 거짓말이다(가시성=계약).** close 커밋 하나를 침묵으로 건너뛰는 건 무해했다(트리와 무관). 그러나 v2 자산 189개를 침묵으로 건너뛰면 사용자는 "다 읽혔다"고 오독한다 — 소급각인이 필요한 규모를 모른 채. C104 Sheen이 "낡은 화면은 침묵보다 나쁘다"고 한 것의 재구성기판: **덜 읽었으면 얼마나 덜 읽었는지 말해야 한다.** `--report`가 그 계약. 소급각인 카브는 이 유령 수가 0이 될 때까지 진행하는 것으로 종료를 판정할 수 있다.

4. **유령의 이질성 — pre-gil과 close가 같은 판별식에 걸린다.** trailer 없음이 판별식이라 pre-gil 유령과 v3 close 커밋이 함께 걸린다(MIX 4 = pre-gil 3 + close 1). 이 카브에선 둘 다 "안 읽힌 커밋"이라 동일하게 다뤘다. 소급각인 카브는 이 둘을 갈라야 한다 — close는 이미 v3라 지문 소급 대상이 아니고, pre-gil만 지문을 받는다. **총계는 이 카브, 구분은 다음 카브**의 자연스러운 분할.
