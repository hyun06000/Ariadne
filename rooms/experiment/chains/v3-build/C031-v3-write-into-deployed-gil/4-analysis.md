## 데이터 직접 관찰

### 중첩 서브파서가 인자 충돌을 원천 해소했다

핵심 설계 문제는 v2 step(번호)·v3 step(--kind)의 인자 충돌. `gil v3` 서브명령 그룹(중첩 서브파서)이 원천 해소 — v3 step은 `v3sub`의 파서라 v2 step 파서와 완전 분리. argparse가 `gil v3 step --kind`와 `gil step <n>`을 서로 안 보게 한다. **네임스페이스 분리가 옵트인 플래그(후보 A)보다 깨끗한 이유** — 두 인자셋이 한 파서에 안 섞인다.

### 오라클 대조가 두 digest로 이식을 봉인

M1에서 두 번 대조: 백트래킹 트리(567cb3b5)·성공 트리(262d3523). 둘 다 gilv3==gil v3 바이트 동일. **서로 다른 트리 모양(죽은 잎 포함/success)에서 모두 동일**하니 이식이 특정 경로만이 아니라 v3 쓰기 로직 전체를 보존. C027·C028 "오라클 대조로 이식 보존"의 세 번째 적용.

### 친절한 에러가 상현님 통찰을 실현

M5에서 `gil v3 step <dir>`(kind 없이)이 v2→v3 대응을 안내. 상현님 "에러 메시지로 해결"의 직접 실현. **버전리스의 핵심 난제(v2 사용자가 v3에서 헤맴)를 도구 로직이 아니라 안내 텍스트가 흡수**. 이 저장소 뿌리 교훈(genesis "포인터 한 줄"·C069 "안내가 모델 전달")이 도구 전환에도 참.

## 예상과 달랐던 것 — 세 번째 이식이 기계적이었다

C027(migrate)·C028(web)·C031(write) 세 번째 이식이라 패턴이 몸에 익었다: 자동 추출 → 이미 있는 함수 제외(ALREADY 집합) → 접두어 → 오라클 대조. 헬퍼 충돌 0(배포판에 v3 쓰기 헬퍼 부재), 스캐폴딩 함정(C028 sys.path)도 미리 알아 안 걸림. **이식 규율이 확립되면 세 번째는 기계적** — 좋은 패턴이 반복 비용을 낮춘다.

## 판정

**채택 (supported).** v3 쓰기가 배포판 gil에 `gil v3 <cmd>`로 통합. 오라클 대조 바이트 동일, v2 무회귀, 자기완결, 친절한 전환 안내. 걸림돌 ① 해소 — 실사이클을 배포판 gil로 v3 네이티브로 연다.

## 정점 통찰 — 버전리스는 안내가 흡수하고, 네임스페이스가 지탱한다

상현님의 두 방향이 합쳐진다: "궁극은 버전리스"(gil open=v3)와 "안내로 해결"(친절한 에러). 답 — **지금은 `gil v3`로 안전하게 얹고(네임스페이스 분리로 v2 무손상), 전환의 마찰은 친절한 에러가 흡수한다.** 나중에 gil open을 v3로 승격할 때도 v2 습관 사용자는 에러가 가르친다. **버전리스는 도구를 단순화해서가 아니라, 전환을 안내가 부드럽게 해서 달성된다.** genesis 이래 이 저장소의 방식 — 복잡성을 도구가 아니라 좋은 안내 한 줄로 넘긴다.

## 방법론에 귀속되는 교훈

- **인자 충돌은 네임스페이스 분리로.** 같은 이름·다른 인자(v2/v3 step)는 중첩 서브파서로 각자 파서. 옵트인 플래그보다 깨끗.
- **이식 규율이 확립되면 반복이 기계적.** C027·C028·C031 세 이식이 같은 패턴. 좋은 패턴이 세 번째를 쉽게.
- **전환의 마찰은 안내가 흡수한다.** 버전 차이를 사용자가 몰라도 되게 하는 건 도구 단순화가 아니라 친절한 에러·온보딩. 저장소 뿌리 교훈의 도구 전환판.
