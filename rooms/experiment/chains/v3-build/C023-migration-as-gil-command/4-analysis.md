## 데이터 직접 관찰

### notes 본문이 바이트 동일 — 49908 == 49908

M2가 가장 강한 증거다. 오라클(clone-A 스크립트)과 명령(clone-B `gil migrate`)이 각인한 **모든 notes 본문을 커밋별로 이어붙인 문자열이 49908바이트로 정확히 일치**했다. 이는 노드 지문(Step-Id 등)뿐 아니라 위상 접합의 Cycle-Parent 엣지까지 한 글자도 안 다르다는 뜻이다. 명령화가 로직을 옮겼을 뿐 아무것도 바꾸지 않았음을 바이트가 증언한다.

핵심: 이게 가능한 이유는 `cmd_migrate`가 오라클과 **같은 함수**(`migrate_all`·`splice_all`)를 호출하기 때문이다. 명령은 표면일 뿐 백엔드가 동일하다 — 그래서 동일은 우연이 아니라 구조적 필연이다.

### 133 접합 vs C022 132 — +1은 C023 자신

C022 실제 원장은 132 접합, 이번 격리 측정은 133. 차이 1은 **C023 자신**(이 사이클이 열리며 cycle.yaml·커밋 추가). C022에서 152→132(도출실패 20 제외)였듯, 여기선 153→133. 마이그레이션이 자기 자신을 포함한 최신 원장을 읽는 일관된 패턴. DAG 노드=153−20=133으로 정확.

## 예상과 달랐던 것

### 명령의 백엔드 통합이 예상보다 얇았다

`cmd_migrate`는 약 70줄이고, 그중 실질 로직은 거의 없다 — 대부분 세 함수(`migrate_all`·`splice_all`·백업/되돌림 git 명령) 호출과 불변 게이트다. C022가 이미 함수 경계를 깔끔히 나눠뒀기에(`migrate_all(repo, apply)`·`splice_all(repo)`), 명령화는 import + 분기뿐이었다. **좋은 함수 분리(C019 "도출/각인 분리가 재사용 낳음")가 명령화를 공짜에 가깝게 만들었다** — C019 교훈이 이 사이클에서 배당됐다.

### notes show 546회 순회가 진짜 병목

명령 자체(`gil migrate`)는 빠른데 M2 검증이 `notes show`를 클론당 546회 호출해 2분 타임아웃을 넘겼다(백그라운드로 우회). 이건 검증 계측기의 성능 문제이지 명령의 문제가 아니다 — 정직히 경계에 기록.

## 판정

**채택 (supported).** C022 실증 절차(동결백업 + 노드 소급 + 위상 접합 + 되돌림)가 `gilv3 migrate` 서브명령군(`--dry`·적용·`--rollback`)으로 승격됐고, 오라클 대조로 DAG·notes 본문이 바이트 동일함이 확인됐다. 안전 계약(커밋·cycle.yaml 불변)·되돌림(잔재 0)·드라이런(각인 0) 계약 모두 명령이 집행한다.

## 정점 통찰 — 명령화의 값어치는 재사용, 전제는 동작 보존

C014~C016이 쓰기 축을 도구 강제했듯, 마이그레이션도 이제 한 명령이다. **다른 사용자·다른 원장의 v2→v3 이주가 `gil migrate` 한 줄**이 됐다(will.md 범용성 의지). 그 값어치의 전제는 "명령이 검증된 절차와 정확히 같은 것을 한다"이고, 오라클 대조가 그것을 바이트로 집행했다. **도구 강제는 원리를 재사용 가능하게 만들되, 재사용의 신뢰는 동작 보존에서 온다.**

## 방법론에 귀속되는 교훈

- **좋은 함수 분리가 미래의 명령화를 공짜로 만든다.** C019가 도출/각인을 분리했기에 C023 명령화가 import+분기로 끝났다. C018 "좋은 데이터 모델이 미래 문제 닫는다"·C019 "좋은 기록 관습이 미래 도출 연다"의 세 번째 쌍: **좋은 함수 경계가 미래 명령화를 연다.**
- **오라클 대조는 명령화의 표준 검증.** "리팩터가 동작을 보존했는가"는 이전 구현을 오라클로 삼아 출력 대조로 판정. 바이트 동일이 최강 증거.
- **명령의 백엔드는 검증된 함수여야 한다.** 새 로직을 쓰면 명령화가 아니라 재구현이고, 그러면 오라클 대조가 무의미해진다. import가 정답.
