# s3 · verify (검증)

## 구현

`load_chain_records`의 v3 분기를 개선:
1. **판별을 steps.yaml 유무로** (열림·닫힘 무관). 이전엔 "cycle.yaml 없으면 v3"였는데,
   닫힌 v3도 cycle.yaml을 가져 v2로 오인됐다.
2. **열린 v3**: `cycle_state(load(entry_dir))`로 status 도출(in_progress/solved/
   multi_solution).
3. **닫힌 v3**: v3 close가 쓴 봉인 cycle.yaml(state·verdict)을 읽어 status=state·verdict.

## ⭐ 도그푸딩이 잡은 결함

s2 가설은 "닫힌 v3는 cycle.yaml 있어 v2 분기가 잡음"이라 했는데, **실측하니 crash**:
v3 close의 cycle.yaml은 v3 형식(`state·verdict·live_leaves·closed`, **id 없음**)이라
v2 로더가 읽으면 "id 필드가 없다" crash. C040~C042가 close까지 안 가봐 못 본 결함을
**첫 도그푸딩 사이클이 close 경로를 실제로 밟으며 발견**. 가설을 넘어 판별 로직을
steps.yaml 기준으로 재작성해 해소.
