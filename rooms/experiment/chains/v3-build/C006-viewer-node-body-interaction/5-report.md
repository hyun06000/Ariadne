## 요약

C004가 정직히 이월한 "상호작용(노드 클릭→본문)"을 되찾았다. C004 스텝 트리 생성기를
**복사 후 확장**(닫힌 사이클 불변)해, 각 노드의 본문 steps/<id>.md를 HTML에 **인라인 임베드**
하고 노드 클릭에 본문 패널 토글을 배선했다. 정적 실측(M1~M5·M4b)으로 C004 회귀 0 + 본문
진실원 일치 10/10 + 자기완결을, 실 Chrome CDP 실측(M6a~c)으로 클릭→펼침 + **다중 열림
상태보존**을 증명했다. 산 잎(가설 채택) 도달 → 닫는다.

## 무엇을 했나

1. **생성기 확장** (`steptree.py`, C004 복사 후): 본문 인라인 임베드(`<article class="stepbody"
   hidden>` + escape된 `<pre>`), 노드를 클릭 가능하게(`clickable`·`role=button`·`tabindex`·
   `data-body`), 인라인 JS(클릭/Enter/Space → 해당 패널 `hidden` 토글, 각 패널 독립), 열린
   노드 SVG 강조(`.node.open`), 패널 닫기 버튼.
2. **CLI** (`render.py`): steps.yaml + steps/*.md → `out.html`(17696 B, 재현물). `base_dir`로
   본문 파일을 읽어 임베드.
3. **정적 실측** (`measure.py`, stdlib html.parser): M1~M4(C004 회귀 0) + M5(본문 임베드·진실원
   10/10) + M4b(상호작용 자기완결). 진실원 독립 재파싱 대조.
4. **상호작용 실측** (`interact.py`, stdlib raw-WebSocket CDP, C010 계보): 실 Chrome headless로
   M6a 클릭→펼침, M6b 상태보존, M6c 닫기 국소성.

## 핵심 발견

- **정적 트리에 상호작용을 얹되 자기완결을 지키는 열쇠 = fetch가 아니라 임베드.** v2 폴링
  (C010)은 fetch로 새 데이터를 당겼지만, 여기 본문은 render 시점에 다 아는 정적 자산이라
  임베드가 옳았다. fetch였다면 file://에서 죽어 계약을 깼다. 문제의 시간성이 다르면 같은
  정직함("지금을 비춘다")이 다른 기제(폴링 vs 임베드)로 실현된다.
- **상태보존을 사후 복원이 아니라 구조로 보장.** C010은 통스왑을 피하고 열린 마운트를 복원
  했다. 여기선 각 패널을 독립시키고 `hidden` 하나만 뒤집어, 상호작용끼리 서로 상태를 건드릴
  **경로 자체를 없앴다**. C010의 교훈이 "복원 로직"에서 "파괴 경로를 안 만드는 구조"로 갔다.
- **한 사이클 안 두 실측 층위.** 임베드·배선 정합은 stdlib 파서(M5)로, "정말 클릭에 펼쳐지고
  다른 클릭에도 유지되는가"는 실 Chrome CDP(M6)로. 도구를 문제에 맞춘다(C004 재확인).
- **표식에서 서사로.** C004가 s4·s7을 "죽은 잎" 배지로 표식만 했다면, C006은 그 잎이 왜
  죽었는지(본문 "벽의 지도")를 같은 화면에서 펼쳐 보인다. 비추는 행위가 다음 깊이를 열었다.

  가능. 정적 measure.py는 브라우저 무의존으로 완전 이식적.
