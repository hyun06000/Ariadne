# 4. 분석

## 측정 요약

정적 실측(measure.py) M1~M5·M4b **ALL PASS**, 실 Chrome CDP 실측(interact.py) M6a~c
**ALL PASS**. 가설의 (a)(b)(c)(d) 네 조건이 전부 성립하고, 네 기각 조건(K1~K4) 모두 미발동.

| 가설 조건 | 판정 근거 |
|---|---|
| (a) C004 트리 회귀 0 | M1·M2·M3 — parent 9==9, backtrack 2(s4·s7→s1), 죽은 잎 2·산 잎 1 그대로 |
| (b) 노드 클릭→본문 펼침 | M5(패널 10·노드↔패널 배선) + M6a(실 클릭→body-s1 hidden 제거) |
| (c) 외부 리소스 0 | M4(정적)·M4b(JS 자기완결) — 인라인 script 1, fetch/xhr/websocket/http(s) 0 |
| (d) 다중 열림 상태보존 | M6b — s10 클릭해도 body-s1 여전히 열림; M6c 닫기 국소성 |

## 기각 조건 대조

- **K1(C004 회귀)**: 미발동. M1~M4 전부 C004와 동일 판정. 나는 원본을 복사 후 확장했고,
  트리 렌더 경로(parent line·backtrack path·잎 클래스)에 손대지 않았다. 추가는 노드에
  `clickable`/`data-body`/`role`/`tabindex` 속성과, 트리 **아래** 별도 섹션(본문 패널)뿐 —
  트리 좌표·엣지·잎 표식은 불변.
- **K2(본문 안 펼쳐짐·불일치)**: 미발동. M5 본문 진실원 일치 10/10. 임베드는 파일에서
  읽어 `html.escape`만 하고 `<pre>`에 담았고, 측정자는 진실원 steps/*.md를 **독립으로 다시
  읽어** 정규화 대조했다(생성기 파서와 별개 경로). M6a가 실 클릭으로 펼침을 확인.
- **K3(외부 리소스)**: 미발동. 본문을 fetch가 아니라 **인라인 임베드**로 넣었기에 file://
  자기완결이 유지된다. JS는 인라인이고 네트워크/동적로드 토큰 0.
- **K4(상태보존 실패)**: 미발동. 이 사이클의 핵심. M6b가 "s1 열고 s10 클릭 → s1 여전히
  열림"을 실 DOM에서 확인. 설계상 각 패널은 독립 `<article>`이고 클릭은 그 패널의 `hidden`
  하나만 뒤집으므로, 다른 노드 조작이 열린 본문을 건드릴 경로가 없다.

## 핵심 발견

1. **정적 트리에 상호작용을 얹되 계약(자기완결)은 안 깨는 법 — 임베드 vs fetch.** v2 라이브
   폴링(C010)은 fetch로 새 데이터를 가져왔지만, 여기선 본문이 render 시점에 이미 다 아는
   정적 자산이라 **임베드**가 옳았다. fetch였다면 file://에서 CORS·파일접근으로 죽어 K3을
   깼을 것이다. "지금을 비춘다"(정체성)가 v2에선 폴링을, v3 정적 트리에선 임베드를 뜻한다 —
   문제의 시간성이 다르면 같은 정직함이 다른 기제로 실현된다.

2. **상태보존은 "무엇을 스왑하지 않을지"의 재현(C010).** C010에서 라이브 폴링의 상태보존은
   body 통스왑을 피하고 열린 마운트만 지키는 설계였다. 여기선 폴링이 없어 더 단순하다 —
   애초에 각 패널을 독립시키고 `hidden` 속성만 뒤집으면, 상호작용끼리 서로의 상태를 건드릴
   경로 자체가 생기지 않는다. **상태보존을 사후 복원이 아니라 구조로 보장**했다. C010의
   교훈이 "복원 로직"에서 "애초에 파괴 경로를 안 만드는 구조"로 한 단계 더 갔다.

3. **정적 계약엔 정적 실측, 상호작용엔 실 브라우저 — 도구를 문제에 맞춘다(C004 재확인).**
   본문 임베드·배선의 정합성은 stdlib 파서(M5)로 충분했으나, "클릭하면 정말 펼쳐지고 다른
   클릭에도 유지되는가"는 정적 마크업이 답할 수 없어 실 Chrome CDP(M6)가 필요했다. C004는
   정적 구조라 파서로 끝났지만, C006은 상호작용을 더했으므로 C010의 CDP 계보를 다시 불렀다.
   한 사이클 안에서 두 실측 층위가 각자의 계약을 증명한다.

4. **비추는 행위가 또 다음을 드러냈다(C001~C004 패턴).** 본문을 펼치니 s4·s7 죽은 잎의
   "벽의 지도" 문구가 트리 옆에서 바로 읽힌다 — C004가 배지로 "죽은 잎"이라 **표식**만 했다면,
   C006은 그 잎이 **왜** 죽었는지(본문)를 같은 화면에서 보인다. 표식에서 서사로. 다음 어둠:
   본문이 길어지면(실 사이클의 5-report급) `<pre>` 평문보다 마크다운 렌더가 필요할 수 있다
   (이 사이클은 짧은 요약 본문이라 평문으로 충분·범위 한정). 5-report에 이월.

## 정직한 경계

- 본문을 **평문 `<pre>`**로 임베드했다(마크다운 미렌더). C002 case 본문이 짧은 요약이라
  가독성에 충분했고, 마크다운 파서를 자작하면 자기완결은 지키나 범위가 커진다 — 정직히 후속.
- 트리는 SVG, 본문 패널은 트리 **아래** HTML 섹션으로 분리 배치했다. 노드 옆 팝오버·툴팁이
  아니라 아래 펼침을 택한 건 SVG 안 foreignObject의 file:// 렌더 편차를 피하고 상태보존을
  단순하게 하려는 절제. 노드↔패널은 `.open` 강조 + `data-body`로 시각·구조 연결.
- CDP 실측은 macOS Chrome headless 1환경. 재현 스크립트(interact.py)는 stdlib만이라 Chrome만
  있으면 어디서든 재현 가능. 정적 실측(measure.py)은 브라우저 무의존이라 완전 이식적.
