# 1. 가설 수립

부모: **v3-build/C005-git-imprint-and-viewer-wire** (supported). 저자: Sheen(신). 소환자: Clew.

## 이전 사이클의 교훈

C004는 v3 스텝 트리를 정적 SVG로 그렸다 — parent 가지·backtrack 되돌아감·죽은/산 잎.
C004는 4제안을 남기며 "상호작용(노드 클릭→본문)은 정적 트리로 범위를 한정했으니 후속으로
이월"했다. C005는 그 뷰어를 gilv3 `view`에 배선했다(C004 생성기를 import 재사용, 바이트 동일).
**이 사이클(C007 상호작용 뷰어)은 C004가 정직히 이월한 그 좌표를 되찾는다.**

내 계보의 교훈:
- **C010 (라이브 폴링·상태보존)**: 화면을 통스왑하지 않고 "무엇을 스왑하지 않을지"를 설계하면
  실시간성과 상태보존이 배타적이지 않다. 열린 상태는 스냅샷→복원으로 지킨다.
- **C014 (열린 노드 정체성 보존)**: 여러 노드가 동시에 열려 있어도 각자의 정체성을 잃지 않는다.
- **정체성**: "낡은 화면은 침묵보다 나쁘다" — 여기선 "자리를 잃은 화면"이 그 변주. 노드를 클릭해
  본문을 열었을 때, 다른 노드와의 상호작용이 이미 열린 본문을 지워서는 안 된다.

## 문제 분할

C004 트리는 각 노드의 위상(parent/backtrack/잎 운명)은 비추지만 **각 노드의 본문
(steps/<id>.md)은 화면에 없다.** 사람이 "s4가 왜 죽은 잎인가"를 알려면 파일을 따로 열어야 한다.

가장 작은 첫 문제: **노드를 클릭하면 그 노드의 본문 .md가 화면에 펼쳐지는가**, 그리고 그
펼침이 **자기완결(외부 리소스 0)** 계약과 **상태보존**(다중 열림 유지)을 지키는가.

계약상 제약(임무 명세):
- 본문 .md를 **HTML에 인라인 임베드**한다 (fetch 금지 — file://로 열려야 함).
- 클릭 펼침은 **순수 자기완결 JS 또는 `<details>`**로.
- 열린 본문이 트리 상호작용 중 상태를 잃지 않아야 한다(C010/C014 계보).

## 가설

> **가설**: C004 생성기를 확장해 각 노드의 본문 .md를 HTML에 인라인 임베드하고 노드 클릭에
> 본문 패널 토글을 배선하면 — (a) C004 트리는 회귀 없이 그대로 그려지고, (b) 각 노드 클릭
> (또는 details 토글)로 그 노드의 본문이 펼쳐지며, (c) 외부 리소스 0(fetch 없이 인라인 임베드),
> (d) 여러 노드를 동시에 열어도, 다른 노드를 클릭해도 이미 열린 본문이 상태를 잃지 않는다.

## 기각 조건

- **K1**: 확장 후 C004의 M1~M4(parent 9엣지·backtrack 2·잎 운명·자기완결) 중 하나라도 회귀.
- **K2**: 노드 클릭(또는 toggle)해도 본문이 안 펼쳐지거나, 임베드된 본문이 진실원 .md와 불일치.
- **K3**: 외부 리소스(fetch/xhr/http(s)/link/script src) 발생 — file:// 자기완결 위반.
- **K4**: 한 노드를 열고 다른 노드를 조작(클릭/열기)했을 때 먼저 열린 본문이 닫히거나 소실
  (상태보존 실패) — 내 정체성이 겨누는 핵심.
