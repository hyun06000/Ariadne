# 4. 결과 분석

## 측정 결과 요약

4측정 ALL PASS (measure-out.txt):

- **M1 결정성** — 같은 커밋 2회 도출 → 동일 지문 5노드(순수 함수).
- **M2 파싱 정확** — 실제 C015 원장 open 1·step 5·close 1 분류, N/5 1..5 추출.
- **M3 C018 통합** — 도출 지문 → retro_imprint → rebuild가 s1~s5 노드 편입.
- **M4 근사 명시** — 매핑 테이블 문서화, 근사 3개(2·4·5) 표기.

## 가설 판정

**H1 (v2 메타 → v3 지문 자동 도출) — supported.** H1a(결정성, M1)·H1b(파싱 견고, M2)·H1c(C018 통합, M3)·H1d(근사 정직, M4) 전부 확인. 실제 v2 원장(C015)에서 손 없이 지문이 도출됐다.

## 교훈

1. **⭐⭐ v2가 이미 규칙적으로 기록했기에 자동 도출이 가능했다.** v2 커밋 subject(`gil: step .../<cycle> → N/5 <스텝명>`)가 처음부터 규칙적이었던 것이 — 그때는 사람이 읽기 위한 것이었는데 — 지금 기계가 v3 지문을 뽑는 파싱 대상이 됐다. **좋은 기록 규율은 미래의 자동 변환을 공짜로 만든다.** C018이 "좋은 데이터 모델(trailer)이 미래 문제를 미리 닫는다"였다면, C019는 그 쌍대: **좋은 기록 관습(규칙적 subject)이 미래 도출을 미리 연다.** gil v2가 100+ 사이클 동안 지켜온 커밋 규율이 v3 마이그레이션의 재료가 됐다.

2. **⭐ 위상은 보존하고 kind만 근사한다 — 무엇을 잃는지 정확히 알 때 근사가 정직하다.** v2 5스텝을 v3 4kind로 매핑하며 억지로 무손실을 주장하지 않았다. 위상(선형 5노드)은 완전 보존하고, kind 3개(설계·분석·보고)만 근사로 표기. **그리고 그 손실조차 실은 손실이 아니다** — v2 스텝명 원문이 커밋 subject에 그대로 남는다(notes는 지문만, subject 불변). 지문은 근사, 원문은 보존. "무엇을 잃는가"를 정확히 아는 근사는 거짓말이 아니라 정직한 사영이다.

3. **⭐ 도출과 각인의 분리가 재사용을 낳았다.** 도출기(순수 함수)를 각인(C018 retro_imprint)과 분리하니, 도출된 지문을 C018 왕복에 그대로 먹일 수 있었다(M3). 만약 도출기가 notes를 직접 박았다면 C018과 중복됐을 것 — **순수 함수는 부작용 있는 도구와 조합될 때 재사용이 극대화된다.** derive(읽기·계산) → retro_imprint(부작용) → rebuild(읽기)의 파이프라인.

4. **v2→v3는 스텝 단위 변환이지 사이클 단위가 아니다.** v2 사이클(고정 5스텝)이 v3 사이클(스텝 트리)로 통째 바뀌는 게 아니라, **v2 스텝(커밋) 하나하나가 v3 노드(커밋)로** 매핑된다 — 커밋은 그대로 두고 notes만 첨부(C018). 마이그레이션의 최소 단위가 커밋이라, 189 사이클이 아니라 ~1300 커밋의 순회가 실제 규모(다음 카브).

## 정직한 경계

- **한 사이클 도출 규칙만.** 189 전량 순회·전체 notes 배선은 규모 확장(다음). 이 카브는 "도출 규칙이 맞는가".
