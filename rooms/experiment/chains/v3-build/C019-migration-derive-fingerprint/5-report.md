마이그레이션 ③ 소급각인의 **실전화** — v2 커밋 subject·cycle.yaml 메타를 v3 지문(Step-Id·Kind·Parent)으로 **손 없이 자동 도출**하는 순수 함수를 세웠다. ## 교훈

1. **⭐⭐ 좋은 기록 관습이 미래의 자동 변환을 공짜로 연다.** v2 커밋 subject가 규칙적이었던 건 그때 사람이 읽기 위한 것이었는데, 지금 기계가 v3 지문을 뽑는 파싱 대상이 됐다. C018이 "좋은 데이터 모델이 미래 문제를 미리 닫는다"였다면 C019는 그 쌍대: **좋은 기록 관습(규칙적 subject)이 미래 도출을 미리 연다.** gil v2의 커밋 규율이 v3 마이그레이션의 재료.

2. **⭐ 위상은 보존하고 kind만 근사한다 — 무엇을 잃는지 정확히 알 때 근사가 정직하다.** v2 5스텝을 v3 4kind로 매핑하며 무손실을 억지 주장하지 않았다. 위상(선형 5노드)은 완전 보존, kind 3개만 근사 표기. **그 손실조차 실은 손실이 아니다** — v2 스텝명 원문이 커밋 subject에 그대로 남는다(notes는 지문만, subject 불변). 지문은 근사, 원문은 보존.

3. **⭐ 도출과 각인의 분리가 재사용을 낳았다.** 도출기(순수 함수)를 각인(C018 retro_imprint)과 분리하니, 도출 지문을 C018 왕복에 그대로 먹였다. **순수 함수는 부작용 있는 도구와 조합될 때 재사용이 극대화된다** — derive(계산)→retro_imprint(부작용)→rebuild(읽기) 파이프라인.

4. **v2→v3는 스텝(커밋) 단위 변환이지 사이클 단위가 아니다.** v2 스텝 하나하나가 v3 노드로 매핑되고 커밋은 그대로(notes만 첨부). 실제 규모는 189 사이클이 아니라 ~1300 커밋의 순회(다음 카브).
