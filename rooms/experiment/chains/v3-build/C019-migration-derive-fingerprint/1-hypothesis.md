# 1. 가설 — v2 메타를 v3 지문으로 자동 도출한다 (소급각인 실전화)

부모: v3-build/C018-migration-retro-imprint (supported — git notes 소급각인, 커밋 불변).
lineage: v3-build/C009-reconstruct-tree-from-git-log (커밋 subject/메타로 트리 재구성).

## 이전 사이클의 교훈

C018이 마이그레이션 ③ 소급각인을 세웠다: git notes로 유령에 v3 지문을 박아 커밋 불변으로 트리에 편입. 그러나 지문 값(L1 define, L2 hypothesis…)을 **손으로 줬다.** 실제 v2 원장 189 사이클(각 7커밋)을 손으로 매핑하는 건 불가능하다 — 소급각인이 수동 실증에 머물러 있다.

C018 보고서 1순위 제안: **v2 지문 자동 도출** — v2 커밋 subject·cycle.yaml에 이미 있는 메타를 v3 지문으로 자동 매핑하는 도출기. 그러면 189 전량 소급이 스크립트 한 번.

## 문제 분할

실제 v2 원장의 커밋 흐름은 매우 규칙적이다(원장 관찰):
```
gil: open  <chain>/<cycle> — 1/5 가설      # 사이클 시작
gil: step  <chain>/<cycle> → N/5 <스텝명>   # N=1..5 (가설·설계·검증·분석·보고)
gil: close <chain>/<cycle>                  # 봉인
```
cycle.yaml: id·chain·parent·step·verdict…

**v2 → v3 구조 대응:**
- v2 사이클 = **고정 5스텝 선형** (백트래킹 없음 — v2엔 데이터로 없었다).
- v3 사이클 = 스텝 트리. v2의 선형 5스텝은 v3 트리의 **선형 가지**(백트래킹 0)로 자연 대응.
- v2 5스텝 이름 → v3 kind: 매핑 테이블 필요.

**가장 작은 첫 카브 = 도출 규칙(파서 + 매핑 테이블).** 한 v2 사이클의 커밋들을 읽어 v3 지문 시퀀스를 뽑는 순수 함수. 189 전량 순회·notes 각인 배선은 그 위에 얹힌다(도출이 맞아야 각인이 의미 있다).

## v2 5스텝 → v3 kind 매핑 (설계 입력)

| v2 스텝 | v3 kind | 근거 |
|---|---|---|
| 1/5 가설 | define + hypothesis | v2 "가설"은 문제정의(define)와 가설수립을 겸함 — v3는 이를 s1 define + s2 hypothesis로 |
| 2/5 설계 | (verify 준비) | v3엔 별도 kind 없음 — hypothesis에 흡수 or 생략 |
| 3/5 검증 | verify | 직접 대응 |
| 4/5 분석 | analyze | 직접 대응 |
| 5/5 보고 | (analyze/success 봉인) | v3는 산 잎 = analyze/success |

이 매핑은 **의미 무손실이 아니라 의미 보존적 근사**다 — v2 5스텝과 v3 kind 집합이 1:1이 아니므로(v2=5, v3=4 kind). 이 카브에서 매핑 규칙을 확정하고 정직히 근사임을 명시한다.

## 가설

> **가설(H1)**: v2 커밋 subject(`gil: step .../<cycle> → N/5 <스텝명>`)와 cycle.yaml(parent) 메타를 파싱해 v3 지문(Step-Id·Kind·Parent) 시퀀스로 매핑하는 순수 도출기를 만들면, 한 v2 사이클의 커밋들에서 **손 없이** v3 지문이 나오고, C018 소급각인에 그대로 먹여 유령이 트리로 편입된다.

- **H1a 도출 결정성:** 같은 v2 사이클 커밋들 → 항상 같은 v3 지문 시퀀스(순수 함수).
- **H1b 파싱 견고:** open/step/close 세 커밋 종류를 정확히 분류, N/5 스텝 번호·이름 추출.
- **H1c C018 통합:** 도출된 지문을 retro_imprint에 먹이면 재구성기가 그 유령을 노드로 복원(C018 왕복 재사용).
- **H1d 근사 정직성:** v2 5스텝 → v3 4kind 매핑이 근사임을 도출기가 명시(무손실 주장 안 함).

## 기각 조건

- 같은 입력에 다른 지문(비결정). → H1a 기각.
- open/step/close·N/5 파싱이 실제 원장 커밋에서 틀림. → H1b 기각.
- 도출 지문을 C018에 먹여도 유령이 노드 안 됨. → H1c 기각.

## 검증 설계 (다음 스텝)

**실제 v2 원장의 한 사이클**(예: 이 체인의 닫힌 사이클 하나)을 표적으로:
- M1 도출 결정성(H1a): 같은 커밋들 2회 도출 → 동일 지문.
- M2 파싱 정확(H1b): open/step/close 분류·N/5 추출이 실제 subject와 일치.
- M3 C018 통합(H1c): 도출 지문 → retro_imprint → rebuild가 그 유령을 노드로.
- M4 근사 명시(H1d): 매핑 테이블이 문서화되고 도출기가 근사임을 표기.

## 정직한 범위 (선긋기)

- **한 사이클 도출 규칙만.** 189 전량 순회·전체 notes 배선은 규모 확장(다음 카브). 이 카브는 "도출 규칙이 맞는가".
- **v2 5스텝 → v3 kind는 근사.** 무손실 아님(집합 크기 다름). 근사 규칙을 확정·명시.
- **선형 사이클만.** v2엔 백트래킹이 데이터로 없어 전부 선형 — 그대로 v3 선형 가지로. (v2에 없던 구조를 만들지 않음.)
- 위상 접합(도출된 v2 사이클들이 v3 체인 DAG로 이어짐)은 ④ 복원 카브.
