# 5. 결과 보고

## 요약

C039가 노출한 계보 소실(v3 사이클에 author·parent 안 남음)을 풀기 위해, v3 open에
`--author`·`--parent`를 더해 루트 define 커밋에 **Cycle-Author·Cycle-Parent trailer**로
계보를 각인하고 worktree add(--v3)가 이를 넘기게 했다. 계보가 커밋에 보존·복원되고
(worktree 경유 C039 소실 해소), steps.yaml·fsck·conformance 무회귀했다. **채택
(supported)** — 이로써 **C039 세 경계(번호·계보·fsck)가 전부 해소**.

## 교훈

1. **계보는 커밋 trailer, 스텝 트리는 steps.yaml — 두 층이 키 네임스페이스로 분리.**
   한 커밋에 `Parent: null`(스텝 트리 부모)과 `Cycle-Parent: C001-seed`(계보 부모)가
   다른 의미로 공존. 물리(파일 vs 커밋 메타)도 분리 — steps.yaml은 스텝 트리 순수 유지.
   C033 "사이클-간 정보는 다른 층"이 코드로 구현된 모습.
2. **steps.yaml을 못 건드린 게 옳은 설계로 이끌었다.** dump의 FIELDS 제약(임의 필드
   유실)이 루트 노드 필드 후보를 막았고, 그 막힘이 계보를 커밋 메타로 분리하는 정답으로
   유도. 제약이 설계를 정화.
3. **C010 trailer 인프라가 계보를 공짜로 받았다.** git_imprint가 이미 trailers를 받으니
   계보는 튜플 두 개 추가가 전부. 새 각인 메커니즘 0. C040 "records 통일이 번호검사 공짜"와
   같은 결 — 기존 인프라 재사용이 최소 표면.
4. **⭐⭐ gil v3 실사이클의 세 전제가 채워졌다.** C038(guard 격리)→C039(worktree v3 open,
   격리·병렬)→C040(fsck 인식·번호중복)→C041(계보). 격리·원장인식·계보 삼박자 완성.
   **이제 실사이클을 gil v3로 여는 도그푸딩 전환이 기술적으로 가능** — 상현님이 기다린 지점.

## 다음 사이클을 위한 제안

**⭐ 도그푸딩 전환 검토 (상현님 보고 필요)**: 세 전제가 채워졌으니 실사이클을 gil v3로
여는 전환을 제안할 수 있다. 다만 남은 표현·검증 격차를 정직히 저울질:

- **A. log/graph의 v3 계보 표현** — fsck는 v3를 세고 계보는 trailer에 있으나, log 그래프가
  v3 사이클을 계보 노드로 아직 안 그림(trailer 읽는 표현 계층 미구현). 도그푸딩 전에
  이게 되면 v3 사이클 계보가 눈에 보임.
- **B. Cycle-Parent 참조 무결성** — 계보 parent가 존재하는 사이클을 가리키는지 fsck 검사
  (v2 R6의 v3판). 지금은 각인만.
- **C. v3 트리 전체 정합** (C040 이월) — 모든 노드 parent 유효·백트래킹·죽은 잎.
- **D. 잔여 예약축 제거** (C040 이월) — 번호중복을 fsck가 잡으니 v2 예약검사 제거 후보.
- **⭐ 도그푸딩 판단**: 최소로는 지금도 gil v3로 실사이클을 열 수 있다(격리·인식·계보
  각인 완비). A(표현)가 있으면 관전자가 v3 계보를 보게 되어 전환이 매끄럽다. **A 먼저 후
  전환 제안**을 권장 — 상현님께 "세 전제 완성, A(눈) 후 도그푸딩 전환" 보고.
