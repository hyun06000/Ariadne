# 2. 실험 설계

가설(1-hypothesis): "모든 분기=브랜치, 1스텝 종결=1커밋" 통일 매핑이 세 층 분기를 브랜치 그래프로 표현하고, 그 위에서 로컬 롤백·죽은 가지 생존·위계 복원이 성립한다 = 깃 브랜치 모델 ≅ gil 위계 모델.

## 설계 개요

이 사이클은 **설계 사이클**이다 — 도구 대공사가 아니라 통일 매핑을 **못박고 최소 실사례로 증명**한다. 순수 깃 명령(+ 얇은 stdlib 검증기)으로 실사례 트리를 브랜치·커밋으로 구성하고, 네 기각 조건을 실측한다. gil 명령 자동화는 다음 사이클로 이월(그리디).

## 먼저 못박을 것 — C008과의 긴장 해소

C008은 "백트래킹=새 커밋"(한 사이클을 선형 커밋 열로)이라 했는데, 상현님은 "스텝 분기=브랜치"라 한다. 모순이 아니다 — **구분이 필요하다**:

- **선형 계승 스텝** (define→hypothesis→verify→analyze, 한 가지 안): 직전 커밋 위에 **커밋 1개**. (C008 그대로.)
- **분기 스텝** (백트래킹 후 새 형제 가지, 사이클 분기, 체인 분기): 조상 커밋에서 **브랜치를 새로 파고** 그 위에 커밋. 갈라짐 자체가 브랜치.

즉 C008의 "새 커밋"은 **선형 계승**에 대해 참이고, 상현님의 "브랜치"는 **분기**에 대해 참이다. 백트래킹은 분기이므로 브랜치가 옳다 — 그래야 **죽은 가지가 머지 안 된 브랜치로 살아남는다**(C008에선 선형이라 죽은 가지가 같은 줄에 섞여 있었다; 브랜치로 하면 죽은 가지가 물리적으로 분리돼 산다). **이것이 C008의 정련**: 백트래킹은 새 커밋이되, 그 새 커밋은 **새 브랜치 위의 첫 커밋**이다.

## 통일 매핑 (확정본)

```
1스텝 종결 = 1커밋                          ← 불변식 (유일한 규율)
선형 계승 스텝 = 현재 브랜치에 커밋 추가
분기 (체인/사이클/스텝 어느 층이든) = 조상 커밋에서 새 브랜치
  ├ 체인 분기 → 새 브랜치 (역할 라벨: chain)
  ├ 사이클 분기 → 새 브랜치 (역할 라벨: cycle)
  └ 스텝 백트래킹 → 새 브랜치 (조상 define 커밋에서, 역할 라벨: step-branch)
산 잎으로 종결 = 부모 브랜치로 --no-ff 머지 (정답 채택)
죽은 가지 = 머지 안 된 브랜치 (벽의 지도, 영구 생존)
위계 = 브랜치가 어디서 갈라졌나 (분기점의 재귀)
역할 라벨 = gil이 브랜치명/trailer로 부여 (chain/cycle/step)
```

## 실사례 (최소, 세 층 분기를 다 담음)

C012→C014 스텝 트리(사이클 내 백트래킹 2개)를 **브랜치로** 재구성:
- `cycle/C-demo` 브랜치에 s1(define) 커밋.
- s2·s3 선형 커밋 (가지 1).
- s4(죽은 잎) — 가지 1은 여기서 죽음. **이 가지가 브랜치 `step/b1`이 되어 머지 안 됨.**
- s1 커밋에서 **새 브랜치 `step/b2`** 분기 → s5·s6·s7(죽은 잎). 머지 안 됨.
- s1 커밋에서 **새 브랜치 `step/b3`** 분기 → s8·s9·s10(산 잎).
- `step/b3`를 `cycle/C-demo`로 --no-ff 머지 (산 잎 채택).
- 추가로 **체인 층 분기**도 최소 예시: `cycle/C-demo`가 부모 체인 브랜치에서 갈라져 나온 것으로 구성해 "체인 분기도 같은 브랜치 메커니즘"을 보임.

각 커밋은 C010 trailer(Step-Id·Kind·Parent·Outcome·Backtrack-To) + 역할 라벨(Role: step/cycle/chain) 각인.

## 절차

1. **build_branches.sh**: 순수 깃 명령으로 위 실사례를 브랜치·커밋으로 구성. 임시 저장소(메인 레포 밖). 1스텝=1커밋 준수, 분기=브랜치, 죽은 가지=머지 안 함, 산 잎=머지.
2. **measure.py** (순수 stdlib):
   - **M1 (세 층 분기 = 브랜치)**: 체인·사이클·스텝 분기가 전부 `git branch`로 존재하고, 브랜치 아닌 프리미티브를 안 씀. K1.
   - **M2 (1스텝=1커밋)**: 스텝 수 == 커밋 수(머지 커밋 제외), 각 스텝이 정확히 1커밋. 0커밋·2커밋 스텝 0. K2.
   - **M3 (죽은 가지 생존 + 로컬 롤백)**: 죽은 브랜치(step/b1·b2)가 `git branch`에 살아있고(머지 안 됨), 그 죽은 잎 커밋의 상태를 `git show <commit>:<file>`로 **워킹트리를 안 깨고** 꺼낼 수 있음(꺼낸 내용 == 각인 당시). 워킹트리 HEAD 불변. K3.
   - **M4 (위계 복원)**: 브랜치 그래프(분기점 + Role 라벨)에서 gil 위계(체인⊃사이클⊃스텝, 어느 브랜치가 무엇인지, 죽은/산 가지)를 유일하게 복원. `git log --graph`·`git merge-base`로 분기점 파악해 위계 재구성. K4.

## 준비물

- 순수 git (2.55), Python 3 stdlib. C010 trailer 스키마(재사용).
- 임시 저장소: 스크래치패드 (메인 레포 밖, C005 규율).
- 이 사이클은 gilv3.py를 안 고친다 — 순수 깃으로 매핑을 증명(도구화는 다음 사이클). C010 gilv3는 참조만.

## 측정 방법

| # | 측정 | 기각 조건 | PASS 기준 |
|---|---|---|---|
| M1 | 세 층 분기 = 브랜치 | K1 | 체인·사이클·스텝 분기 전부 브랜치, 타 프리미티브 0 |
| M2 | 1스텝 = 1커밋 | K2 | 스텝 수 == 커밋 수, 0/2커밋 스텝 0 |
| M3 | 죽은 가지 생존 + 로컬 롤백 | K3 | 죽은 브랜치 생존, git show로 워킹트리 무손상 꺼냄 |
| M4 | 위계 복원 | K4 | 브랜치 그래프→gil 위계 유일 복원 |

네 측정 PASS면 supported (깃 브랜치 ≅ gil 위계). 하나라도 K 발동이면 rejected/partial.

## 사용자 컨펌

- 통일 원리("모든 분기=브랜치, 1스텝=1커밋")는 상현님이 직접 세웠다(이 사이클을 여는 대화). 세부 설계 — 특히 **C008 "백트래킹=새 커밋"을 "분기=새 브랜치 위의 첫 커밋"으로 정련**하는 판단 — 은 보고서에 근거를 남긴다. 이 정련이 상현님 원리와 C008을 화해시키는 핵심이라 명시적으로 검증(M1·M2가 그 화해를 측정).
