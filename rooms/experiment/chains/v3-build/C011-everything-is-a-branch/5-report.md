# 5. 결과 보고 — v3-build/C011-everything-is-a-branch

부모: v3-build/C010-commit-message-as-contract (supported). 저자: Clew. 소환자: 없음 (main 단독·순차, C074에 따라 워크트리 미사용). 판정: **supported (채택)**.

## 요약

상현님이 던진 "코드 아티팩트를 어떻게 트레이싱·롤백하나"에서 출발해, 검증 중 상현님이 모델을 세 번 정련하며 **깃 데이터 모델 ≅ gil 위계 모델**을 확정했다: **백트래킹=`git checkout <조상>`+detached 커밋, 1스텝=1커밋, 위계=커밋 메시지 지문(trailer), 잎=태그(못)**. 실사례(체인·사이클·스텝 3층 분기, 죽은 잎 2·산 잎 1)로 4측정 ALL PASS — 깃 그래프가 곧 스텝 트리이고(뷰어 재구현 0), git show로 임의 스텝(죽은 가지 포함) 롤백이 워킹트리 무손상으로 되며, 태그로 죽은 가지가 모든 머신에서 영구 생존한다. 산 잎(s10) 도달 → **닫는다**.

## 무엇을 했나

1. **통일 매핑 확정**: 모든 분기(체인/사이클/스텝)는 깃 그래프의 갈라짐, 1스텝 종결=1커밋. 순수 깃으로 실사례 구성(build_branches.sh) — 도구는 안 고침(다음 사이클 몫).
2. **백트래킹 = checkout**: `git checkout <조상해시>` + detached HEAD 커밋 → 분기 자연 발생(명시적 브랜치 0, 머지 0). C008 "백트래킹=새 커밋"의 정정.
3. **위계 = 커밋 지문**: C010 trailer를 Chain·Cycle까지 확장. M4가 브랜치·태그 안 보고 지문만으로 위계 복원함을 실증.
4. **잎 = 태그**: 죽은/산 잎을 `gil/leaf/<hash>` 태그로 못박음. 사이클 종결은 `cycle/<name>/solved` 태그. gil v2 태그 규율과 일관, push되어 다머신 영구 생존.

## 교훈

1. **깃 데이터 모델이 gil 위계 모델과 동형이다.** 깃의 두 기본 연산 — 분기(갈라짐)와 합류(머지) — 가 정확히 gil의 백트래킹과 지식 통합이다. gil은 깃을 감싸 **커밋 지문으로 위계를, 태그로 잎을** 입히는 얇은 껍질일 뿐 — 그래프·checkout·분기·롤백은 전부 깃이 공짜로 준다. **깃을 버리는 게 아니라 깃이 이미 우리 모델이었다.**
2. **백트래킹은 커밋이 아니라 checkout이다** (C008 정정). 되돌아감 = `git checkout <조상>`. 그 후 첫 스텝이 새 커밋이고, 깃이 자동으로 새 가지를 친다. detached HEAD 분기라 머지가 불필요(갈라지기만).
3. **위계는 브랜치가 아니라 커밋 지문에 산다.** 브랜치·태그는 "커밋을 붙잡는 못"일 뿐, 위계(체인⊃사이클⊃스텝)는 커밋 trailer가 담는다. → 뷰어 = `git log --all --graph` + 커밋 메시지. **우리는 아무것도 새로 그리지 않는다.**
4. **죽은 가지도 못이 필요하다** (실측 발견). detached HEAD 죽은 가지는 ref 없으면 `git log --all`에서 사라진다(dangling→gc). **태그가 그 못으로 최적** — 잎은 불변 시점이라 브랜치(움직임)보다 태그(못박음)가 정확하고, push되어 다머신 영구 생존("존재는 레포에만 산다"와 맞물림). 이름은 해시면 충분(논리 id는 지문에).

## 정직한 경계

- **이 사이클은 순수 깃으로 원리만 증명**했다 — gilv3.py를 안 고쳤다(도구 자동화는 다음 사이클, 그리디). 실제 gil이 이 매핑을 명령으로 강제하는지는 미검증.
- **검증이 모델을 3회 바꿨다** — 첫 설계(명시적 브랜치)는 상현님 모델 오해였고 실측이 정정을 이끌었다(README·분석에 명시). 이는 정직한 진행이나, "설계 시점 가설"과 "확정 모델"이 다르다(설계 사이클의 특성).
- **머지=lineage는 아직 미검증** — 상현님이 닫으며 던진 씨앗이고, 다음 사이클이 증명할 몫. 이 사이클은 분기만 증명, 합류는 안 함.
- 실사례는 최소(단일 체인·단일 사이클·백트래킹 2). 다중 체인 머지, 중첩 백트래킹, 실제 대형 트리의 성능은 미검증.
