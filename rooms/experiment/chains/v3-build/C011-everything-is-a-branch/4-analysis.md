## 데이터 직접 관찰

**1) 깃 그래프가 곧 스텝 트리다.** git-graph.txt에서 s1 커밋 아래로 세 갈래(`|/`)가 갈라진다 — s2·s3·s4(죽음), s5·s6·s7(죽음), s8·s9·s10(산). `git log --all --graph`가 우리 스텝 트리를 **그대로** 그린다. 뷰어를 새로 짤 필요가 없다는 상현님 주장의 실물. 각 노드의 정체는 커밋 지문에:
```
step: s7 analyze/backtrack
Step-Id: s7 · Kind: analyze · Parent: s6 · Chain: v3-demo · Cycle: C-demo · Outcome: backtrack · Backtrack-To: s1
```

**2) 백트래킹이 커밋도 브랜치도 아니라 checkout이다.** build_branches.sh에서 되돌아감은 `git checkout <s1해시>` 한 줄이다. 그 뒤 s5를 커밋하면 깃이 s1에서 새 가지를 **자동으로** 친다(내가 브랜치를 안 만들었는데). M2가 머지 커밋 0을 확인 — detached HEAD 분기는 머지가 필요 없다(갈라지기만 함). **C008 "백트래킹=새 커밋"의 정정**: 백트래킹 자체는 커밋이 아니라 checkout이고, 그 후 첫 스텝이 새 커밋이다.

**3) 위계가 브랜치가 아니라 커밋 지문에 산다.** M4는 브랜치·태그를 **한 번도 안 보고** 커밋 trailer만으로 위계를 복원했다 — Chain 지문으로 체인, Cycle 지문으로 사이클, Parent 지문으로 s1의 세 자식(s2·s5·s8), Outcome 지문으로 죽은 잎(s4·s7)/산 잎(s10). **깃 브랜치는 위계를 담는 그릇이 아니다 — 커밋 지문이 담고, 브랜치/태그는 그저 "커밋을 붙잡는 못"이다.**

## 예상과 달랐던 것 (모델 정련 3회)

이 사이클은 **검증이 모델을 바꾼** 드문 사례다. 상현님이 실측을 보며 세 번 정련했다:

- **정련 1 — 백트래킹은 브랜치가 아니라 checkout.** 나는 처음 `git checkout -b step/b2`로 명시적 브랜치를 팠다. 상현님: *"백트래킹은 새 커밋이 아니지. 깃 그래프를 그대로 두고 특정 시점으로 체크아웃하는 게 백트래킹이다."* → build를 checkout+detached로 재작성. 이게 훨씬 얇다(브랜치 관리 0).
- **정련 2 — 위계는 커밋 메시지 지문에.** 상현님: *"각 커밋이 어떤 위계를 가지는지 정보만 추가로 적어 관리한다."* → C010 trailer를 Chain·Cycle까지 확장. M4가 이것만으로 위계 복원됨을 실증.
- **정련 3 (내가 발견 → 상현님이 규칙) — 죽은 가지엔 최소 못이 필요하다.** detached HEAD 죽은 가지가 `git log --all`에서 **사라지는 것**(dangling, gc 위험)을 M3 음성 대조로 발견했다. 상현님: *"체인 이름, 사이클 이름, 스텝 분기는 구분 가능한 해시 같은 걸로."* → 잎마다 해시 못. 그리고 상현님 *"태그 같은 자연스러운 방법이 있으면"* 제안에 내가 깃 관점으로 **태그**를 택했다 — 잎은 불변 시점이라 브랜치(움직임)보다 태그(못박음)가 정확하고, **태그는 push되어 모든 머신에서 죽은 가지가 영구 생존**한다(커스텀 ref는 push 안 됨). "존재는 레포에만 산다"와 맞물림.

**씨앗**: 이 세 정련이 곧 다음 사이클(gil 명령 자동화)의 명세다 — `gil step`이 checkout+커밋+지문을 자동으로, `gil close`가 산 잎에 태그, backtrack이 죽은 잎에 해시 태그.

## 판정

**supported (채택).** 네 기각 조건 전부 미발동, 4측정 ALL PASS. **깃 그래프 모델 ≅ gil 위계 모델**이 실증됐다: 깃이 그래프·checkout·분기를 제공하고, gil은 커밋 지문으로 위계를, 태그로 잎을 못박는 얇은 껍질이다. 백트래킹=checkout, 1스텝=1커밋, 위계=지문, 잎=태그. 상현님이 처음 던진 "코드 아티팩트를 어떻게 트레이싱·롤백하나"의 답이 나왔다 — **git show로 임의 스텝 롤백, 태그로 죽은 가지까지 영구 생존.** 산 잎(s10) 도달 → 그리디하게 닫는다.
