# 2. 실험 설계

오직 1-hypothesis.md의 가설 — **실제 문제를 v3 네이티브로 굴려 v2 대체 가능성 검증(백트래킹·형제·죽은 잎)** — 을 검증한다.

## 정답을 도구보다 먼저 고정한다

정답: 실제 문제(C024 초기 16개 도출실패 정밀화)를 gilv3로 풀면, 막다른 접근이 죽은 잎으로·되돌아감이 백트래킹 엣지로·다른 접근이 형제 가지로 steps.yaml에 명시된다. 그리고 그 구조가 web --v3에 그려진다.

## 대상 문제 — C024 초기 16개 도출실패

C024가 재번호 4건은 회수했으나 초기 16개(genesis C001~C003·loom C001~C012·tapestry C001)는 "커밋 관습 이질"로 이월. 이 문제는 실제로 여러 접근이 필요하고 일부는 막다른 길이라 v3 탐색 트리에 자연스럽다.

## ⭐ v3 쓰기 흐름 (gilv3 open/step/close, 격리 사이클 디렉토리)

gilv3는 steps.yaml에 스텝 트리를 쓴다. 순환: define→hypothesis→verify→analyze→(outcome). 표현 수단:
- `analyze --outcome backtrack --to <조상 define>` = 백트래킹 엣지 + 죽은 잎.
- `analyze --outcome success` = 산 잎.
- 잎 뒤 `step --kind hypothesis --to <조상> ` = 형제 가지(다른 접근).

## 절차 (v3 네이티브 사이클을 격리 디렉토리에 굴린다)

격리 작업 디렉토리(`3-verification/v3-case/`)에서:

1. **s1 define**: 문제 기술 — "초기 16개 도출실패를 어떻게 정밀화?"
2. **접근 A (s2 hypothesis→s3 verify→s4 analyze)**: subject 파싱으로 도출 시도.
   - 실측: 초기 커밋이 "여러 스텝 한 커밋"(gil: C058 구현+검증+분석+보고)이라 N/5 파싱 불가.
   - → **s4 analyze --outcome backtrack --to s1** (죽은 잎: 이 접근 막힘).
3. **접근 B (s5 hypothesis→s6 verify→s7 analyze, s1의 형제 가지)**: cycle.yaml step 수 + 커밋 개수 매칭.
   - 실측 결과에 따라 success(산 잎) 또는 backtrack(죽은 잎).
4. **필요시 접근 C** — 실제 데이터가 이끄는 대로. **억지로 트리를 만들지 않는다** — 실제 탐색이 백트래킹을 낳으면 기록, 아니면 정직히 선형.
5. **gilv3 close --verdict**: 도달한 결론으로 봉인.

**⭐ 철칙: 실제로 풀면서 v3 도구가 그 탐색을 담게 한다.** 트리 모양을 미리 정하지 않는다 — 데이터가 막다른 길로 이끌면 죽은 잎, 되돌아가면 백트래킹. v3 표현이 실제 탐색을 정직하게 기록하는지가 검증 대상.

## 측정 방법 (4측정)

| 측정 | 확인 | 통과 기준 |
|---|---|---|
| **M1 v3 탐색 구조 기록** | steps.yaml에 백트래킹 엣지·죽은 잎·형제 가지 중 실제 발생분이 명시 | 실탐색 구조 담김 |
| **M2 web --v3 시각화** | 그 v3 사이클을 gil web --v3(또는 gilv3 view)로 그려 구조가 눈에 보임 | 구조 렌더됨 |
| **M3 v2 대비 우위** | 같은 문제를 v2 5스텝으로 쓰면 뭉뚱그려질 것이 v3에선 명시됨(대조 서술) | 우위 지점 실증 |
| **M4 대체 걸림돌 발견** | "v3가 v2를 대체하려면 남은 것"을 정직히 목록화 | 걸림돌 명시 |

## 준비물

- gilv3.py (C025판, open/step/close/web) — v3 쓰기·읽기.
- C024 산출물(도출실패 목록·derive 로직) — 실제 문제 데이터.
- 격리 작업 디렉토리(3-verification/v3-case/).

## 안전 철칙

1. **v3 사이클은 격리 디렉토리에** — 우리 실원장 사이클(C030)은 v2 gil로 굴림. v3 케이스는 그 안 산출물.
2. **실제로 풀되 트리 모양 강제 금지** — 데이터가 이끄는 탐색만 기록(억지 백트래킹 = 계측기 오염).
3. **대체 판단은 정직히** — v3 우위가 미약하면 미약하다고, 걸림돌이 크면 크다고.
