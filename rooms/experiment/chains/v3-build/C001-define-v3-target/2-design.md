# 2. 실험 설계

가설: v3-build의 첫 정복 대상은 v3 데이터 모델(스텝 트리 표현)이다. 검증은 C012→C014를 새 모델로 재표현해 왜곡 없음 확인.

## 이 사이클의 범위 (그리디 — 첫 정답에서 멈춘다)

C001의 목표는 **"첫 문제를 정한다"**이지 데이터 모델을 완성하는 게 아니다(그건 C002의 몫). 그러니 이 사이클은 가볍게: 첫 대상이 데이터 모델임을 **검증(C012→C014 재표현이 가능한 최소 스케치로 왜곡 없음 확인)**하고, 맞으면 닫는다. 데이터 모델의 완전한 설계로 파고들면 그리디 배신.

## 절차

1. **최소 데이터 모델 스케치**: 스텝 트리를 표현하는 가장 단순한 뼈대를 종이 수준으로 그린다(코드 아님). 필요 요소:
   - 스텝 노드: id, 종류(문제정의*/가설수립/가설검증/결과분석), 결말([실패|문제정의*|성공]), 부모 스텝(트리 간선).
   - 백트래킹: "실패 → 가장 가까운 문제정의*로" = 부모 포인터가 문제정의* 노드를 가리킴.
   - 죽은 잎 vs 산 잎: 결말이 실패면 죽은 잎, 성공이면 산 잎(사이클 종료).
2. **C012→C013→C014를 이 스케치로 재표현**: 실제 사례를 새 모델에 담아 왜곡(되돌아감·죽은 잎·산 잎 소실)이 없는지 본다.
   - v2에서 3개 사이클로 쪼개진 것이 v3에서 1개 사이클의 스텝 트리로, 백트래킹이 부모 포인터로, C012·C013이 죽은 잎으로, C014가 산 잎으로 담기는가.
3. **파생 가능성 확인(가벼움)**: 이 모델 위에 open/step/close가 자연히 얹히는 뼈대인지 한 문단으로 확인(상세 설계는 C002).

## 준비물

- v2 gil(v2.49.0, 이 사이클 진행용). 상현님과 확정한 세 층 구조(chain.md).
- 실제 사례: loomlight/C012·C013·C014 (이 세션의 리젝트 체인).

## 측정 방법

| # | 측정 | 판정 |
|---|---|---|
| M1 | C012→C014를 최소 모델로 재표현 | 백트래킹·죽은잎·산잎 왜곡 없이 담김 (kill 1) |
| M2 | 모델 위에 open/step/close 얹히나 | 자연히 파생되는 뼈대 확인 (kill 2) |
| M3 | 첫 정답에서 멈춤 | 모델 확인 즉시 닫고 완전설계는 C002로 (kill 3) |

## 검증분석 분기 (미리)

- M1·M2 통과 → **성공 잎**: 데이터 모델이 첫 대상으로 확정. 보고서 쓰고 C001 닫음. C002 = 데이터 모델 완전 설계.
- M1 실패(모델이 트리 못 담음) → 최소 모델 스케치를 고쳐 재시도(스텝 트리 안의 백트래킹 = 가장 가까운 문제정의로).
- M2 실패(파생 안 됨) → 첫 대상이 데이터 모델이 아닐 수 있음 → 문제 분할로 되돌아가 다른 후보(명령 먼저?).
