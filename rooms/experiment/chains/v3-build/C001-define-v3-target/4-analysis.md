# 4. 검증 분석

가설(1-hypothesis): v3-build의 첫 정복 대상 = **v3 데이터 모델(스텝 트리 표현)**.
검증: 실제 사례 **loomlight/C012→C013→C014**(카드닫힘 리젝트 체인, 백트래킹 포함)를
최소 스텝 트리 모델로 재표현해, v2가 저지른 왜곡(되돌아감·죽은 잎·산 잎 소실)이
없는지 본다.

---

## 사례의 봉인된 진실 (v2 기록)

v2는 이 진단을 **세 개의 사이클, 선형 parent 체인**으로 기록했다:

```
C012 (rejected)  ──parent──▶  C013 (rejected)  ──parent──▶  C014 (supported)
카드닫힘 진단        폴링 이분격리                swapRegions 통스왑
상호작용·아코디언       --refresh 0 판정이          열린 노드 replaceChild
·detKey 배제로 기각    실브라우저(내 손밖) 요구→기각   통스왑이 원인 → 채택
```

**그런데 봉인된 5-report가 증언하는 진짜 구조는 선형이 아니다:**

- C013 보고서(원문): *"계보를 거슬러 내가 혼자 걸을 수 있는 분기를 찾는다."*
  → C014는 **C013에서 자란 게 아니다.** C013이 벽(실브라우저 계측)에 막혀
    **뿌리로 되돌아가(백트래킹)** 새로 낸 가지다.
- C012·C013 보고서(공통): *"두 죽은 가지가 '이 길들은 막혔다'를 후대에 남긴다."*
  → 둘은 **죽은 잎**이지, C014로 이어지는 경로의 중간 노드가 아니다.

**v2의 왜곡**: `parent`가 "시간 순서상 다음"과 "논리적 자식"을 구별하지 못해,
백트래킹(C013→뿌리→C014)을 선형 계승(C013→C014)으로 뭉갰다. C014의 부모는
C013이 아니라 **최초 문제정의**여야 한다.

---

## 최소 스텝 트리 모델 (종이 수준 스케치)

스텝 노드 하나를 표현하는 최소 요소:

```
STEP {
  id        : 안정 식별자
  kind      : 문제정의* | 가설수립 | 가설검증 | 결과분석
  parent    : 부모 스텝 id (트리 간선; 루트만 null)
  outcome   : (결과분석 노드에만) 실패 | 되돌아감 | 성공
  backtrack : (outcome=되돌아감일 때) 되돌아간 목적지 = 가장 가까운 조상 문제정의* id
}
```

- **트리 간선** = `parent`. DAG 아님(스텝은 한 부모).
- **백트래킹** = `결과분석` 노드의 `outcome=되돌아감` + `backtrack`이 가리키는
  조상 문제정의*에서 **새 형제 가지**가 자란다. "실패→가장 가까운 문제정의로"가
  포인터 하나로 표현된다.
- **죽은 잎 vs 산 잎** = `결과분석`의 `outcome`. `실패`/`되돌아감`이면 죽은 잎
  (그 가지 종료), `성공`이면 산 잎(사이클 종료).

---

## C012→C013→C014를 이 모델로 재표현 (M1)

v2의 **3 사이클**이 v3에서는 **1 사이클, 하나의 스텝 트리**가 된다.
(P#=문제정의*, H=가설수립, V=가설검증, A=결과분석)

```
P0  카드가 5초마다 닫힌다 — 원인을 찾는다                    [루트 문제정의]
│
├─ P1  원인은 상호작용/아코디언/detKey 차이인가              (P0의 분할)
│   ├─ H1  헤드리스 CDP로 세 후보를 재현한다
│   ├─ V1  실클릭·hash·3주기·실폴링 — 전 조건 카드 유지
│   └─ A1  outcome=되돌아감  backtrack=P0                    ★ 죽은 잎 (=v2 C012)
│          "재현 실패 → 결함은 환경 의존, 여기 없다"
│
├─ P2  폴링이 유일 범인인가 — 이분격리로 가른다              (P0로 되돌아가 낸 형제)
│   ├─ H2  --refresh 0이면 카드닫힘이 멈춘다
│   ├─ V2  refresh 0 = 폴링 확실히 끔(fetch 0)까지만 확정
│   └─ A2  outcome=되돌아감  backtrack=P0                    ★ 죽은 잎 (=v2 C013)
│          "실판정이 실브라우저(내 손밖 계측기) 요구 → 막힘"
│
└─ P3  swapRegions 통스왑이 열린 노드를 교체하는가           (P0로 되돌아가 낸 형제)
    ├─ H3  폴링 replaceChild가 열린 카드 노드를 통째 교체한다
    ├─ V3  JS 마커로 노드 정체성 헤드리스 관찰 → chainNodeReplaced:true
    └─ A3  outcome=성공                                     ★ 산 잎 (=v2 C014)
           "hasOpenDetails 가드로 통스왑 건너뜀 → 해결"
           → 사이클 닫힘 (verdict=supported)
```

**왜곡 검사 (kill 1):**

| v2가 왜곡한 것 | v3 모델이 담는가 |
|---|---|
| 백트래킹 C013→뿌리→C014를 선형 C013→C014로 뭉갬 | ✅ A2·A3 둘 다 `backtrack=P0` — P2·P3는 P0의 **형제 가지**. P2→P3 간선 없음. |
| 죽은 잎(C012·C013)이 경로 중간 노드처럼 보임 | ✅ A1·A2가 `outcome=되돌아감` = 죽은 잎. 가지가 거기서 끝남(자식 없음). |
| 산 잎(C014)이 죽은 가지의 계승자처럼 보임 | ✅ A3만 `outcome=성공` = 산 잎. 부모는 P3(자기 문제정의)이지 P2 아님. |
| "왜 죽었나"(배제의 지도) 소실 위험 | ✅ A1·A2 노드가 기각 사유를 담아 보존 — 후대가 P1·P2 안 헤맴. |

**→ M1 통과.** 백트래킹·죽은 잎·산 잎이 왜곡 없이 담긴다. 되레 v2가 못 담던
"C014는 C013의 자식이 아니다"라는 진실이 `backtrack=P0`로 **명시적 데이터**가 됐다.

## 모델 위에 open/step/close가 얹히나 (M2)

| v2 명령 | v3 스텝 트리 위에서 |
|---|---|
| `open` | 루트 문제정의(P0) 노드 하나를 만든다 — 사이클=트리의 뿌리. (v2 open이 5스텝 파일을 미리 깔던 것과 달리, 트리는 자라므로 뿌리만.) |
| `step` | 현재 잎에 자식 노드를 잇는다. `kind`는 `문제정의*→가설→검증→분석` 순환을 강제(스키마가 다음 kind를 안다). `분석` 노드에서 `outcome` 요구. |
| `step --backtrack` | `분석`의 `outcome=되돌아감`을 각인하고, 커서를 `backtrack` 목적지(가장 가까운 조상 문제정의*)로 옮긴다. 다음 `step`은 거기서 **새 형제 가지**를 시작. |
| `close` | 어떤 잎이 `outcome=성공`에 닿으면 사이클을 닫는다(그리디—첫 정답에서 멈춤). 산 잎 0개인 채 닫으면 = rejected 사이클(모든 가지가 죽음). |

**핵심 파생**: 모델의 `kind` 순환과 `outcome` 분기가 **명령을 강제**한다 —
`분석` 없이 다음 `문제정의`로 못 가고, `되돌아감`은 반드시 조상 문제정의*를
가리켜야 한다. v2의 "고정 5스텝"이 못 하던 step-by-step 강제(C090)가
**트리 구조 자체에서 자연히 나온다.** open/step/close가 모델에서 파생됨.

**→ M2 통과.** 데이터 모델이 명령을 강제하는 뼈대다.

---

## 첫 정답에서 멈춤 (M3)

이 사이클의 목표는 **"첫 정복 대상이 데이터 모델임을 확정"**이지 모델을 완성하는
게 아니다. M1·M2로 "데이터 모델이 첫 대상"이 확인됐으니 **여기서 멈춘다.**

- 안 하는 것(그리디 배신 방지): id 발급 규칙, 디스크 표현(파일/디렉토리/yaml),
  `kind` 순환의 정확한 상태기계, `backtrack` 목적지 해소 알고리즘, 뷰어 렌더 —
  **전부 C002(데이터 모델 완전 설계)의 몫.**
- "더 나은 모델?"(예: 스텝을 DAG로? 다중 backtrack?)은 최적화 = 되돌아가 새 사이클.

**→ M3 통과.** 확인 즉시 멈추고 완전 설계는 C002로 이월.

(부모 = 이 C001).
