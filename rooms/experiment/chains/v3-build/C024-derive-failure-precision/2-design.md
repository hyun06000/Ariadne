# 2. 실험 설계

오직 1-hypothesis.md의 가설 — **재번호 provenance 추적으로 후기 5개 도출실패를 회수** — 을 검증한다.

## 정답을 도구보다 먼저 고정한다

후기 5개의 "정답"은 옛 번호 커밋이다. 조사로 이미 확인:

| 새 id | 옛 id | 매핑 진실원 |
|---|---|---|
| v3-build/C004 | C003 | 5-report "번호 재발급 주석" (`C003→C004`) |
| v3-build/C007 | C006 | 5-report (`C006→C007`) |
| loom/C083 | C082 | git 커밋 `gil: 재번호 C082→C083` |
| loom/C106 | C105 | git `gil: renumber ...C105... → C106` + 5-report |
| loom/C104 | C103 | git `재번호 C103→C104` (※ C104는 도출실패 목록에 없음 — 이미 도출됨? 검증에서 확인) |

## ⭐ 핵심 설계 — 재번호 매핑의 진실원은 두 곳, git이 더 안정적

재번호 매핑이 두 곳에 산다:
1. **git 커밋** `gil: 재번호 C<old>→C<new>` 또는 `gil: renumber ... → C<new>` — 원장에 각인, 파싱 안정적.
2. **5-report.md** "번호 재발급 주석" — Clew가 병합 시 재번호한 경우(v3-build C004·C007).

**C020 교훈("불안정한 표면 아니라 안정적 구조를 신뢰")의 연장**: git 재번호 커밋을 1순위 진실원으로, 5-report를 보완으로. 둘을 합쳐 `{new_id: old_id}` 매핑을 만든다.

## 절차

1. **재번호 매핑 수집** (`renumber_map.py`, 순수 함수):
   - git log에서 `재번호 C<old>→C<new>` / `renumber ...C<old>... → C<new>` 파싱.
   - 5-report.md에서 `C<old>→C<new>` 재발급 주석 파싱(도출실패 사이클 디렉토리만, 오염 방지).
   - 합쳐 `{chain/new_id: old_id}` 반환.
2. **도출기 정밀화** (`derive_precise.py`, 기존 `cycle_step_commits` 위):
   - 새 id로 커밋 검색 실패 시 → 매핑에서 옛 id 조회 → 옛 id로 재시도.
   - 회수한 커밋으로 v3 지문 도출(C019 규칙 그대로).
3. **회수 측정**: 후기 5개(및 목록의 재번호 사이클)에 대해 옛 id 커밋 회수 성공 여부, 도출된 지문이 실제 스텝과 정합.
4. **오염 방지 검증**: 초기 15개(비-재번호)는 매핑에 안 잡혀 종전대로 도출실패 유지(정밀화가 엉뚱한 걸 안 건드림).

## 준비물

- `renumber_map.py`·`derive_precise.py` — 신규(순수 함수 + 도출 래퍼).
- C022판 `full_ledger_migrate.py`·`derive_fingerprint.py` — 기준 도출 로직 재사용.
- 격리 복제본(우리 원장 안 건드림 — 조회·측정만).

## 측정 방법 (4측정)

| 측정 | 확인 | 통과 기준 |
|---|---|---|
| **M1 매핑 수집** | git+5-report에서 재번호 매핑 정확 수집(C004→C003 등) | 5개 매핑 정확 |
| **M2 커밋 회수** | 옛 id로 후기 5개 스텝 커밋 회수 | 커밋 찾음 |
| **M3 도출 감소** | 도출실패 20 → 15 (회수분 5 감소) | 5 감소, 잔여 15 |
| **M4 오염 없음** | 초기 15개는 매핑 미포함 → 종전대로 실패 유지, 오도출 0 | 초기 15 불변 |

## 안전 철칙

1. **격리 복제본만** — 조회·측정. 실제 원장 재적용은 별도(원한다면 `gil migrate` 재실행).
2. **provenance 파싱은 도출실패 사이클만** — 초기 15개나 정상 사이클을 안 건드려 오염 0.
3. **억지 도출 금지** — 초기 15개(커밋 관습 이질)는 이 카브 밖. 명확한 재번호 부류만.
