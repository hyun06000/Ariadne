# 1. 가설 수립

## 이전 사이클의 교훈

부모 **C023**(마이그레이션 gil 명령화)까지 마이그레이션이 도구 명령이 됐다. 그러나 C020 이래 일관되게 이월된 잔여가 있다: **20 도출실패 사이클** — `full_ledger_migrate`가 커밋을 못 찾아 v3 지문을 소급 못 하는 섬들. C022/C023이 모두 "적용의 안전성이지 도출 완전성이 아니다"라며 정직히 남겼다.

C017 교훈: **침묵 스킵은 마이그레이션에서 거짓말** — 덜 읽었으면 얼마나 덜인지 말해야. C020이 767 유령을 3종 분류(비-원장·관리·도출실패)했고, 도출실패-스텝만이 진짜 개선 여지였다.

## 문제 분할 — 20 도출실패는 한 부류가 아니다

20 사이클을 원인별로 조사하니 **두 부류**로 갈린다:

1. **초기 15개** (genesis 3 + loom C001~C012 + tapestry C001): cycle.yaml에 `step` 필드 부재(`step=?`) 또는 초기 커밋 관습(여러 스텝 한 커밋, `gil: C057 가설 작성` 형식). 커밋 관습 확립 전.
2. **후기 5개** (loom/C083·C106, v3-build/C004·C007): `step=5`로 정상인데 커밋을 못 찾음. 조사 결과 **재번호된 사이클** — cycle.yaml id는 새 번호(C004)인데 git 커밋 subject는 옛 번호(C003)를 담는다. 도출기가 새 id로 검색하니 못 찾는다.

**첫 번째로 정복할 것: 후기 5개(재번호 부류).** 이유:
- 초기 15개는 커밋 관습 자체가 달라 근본적 이질(억지 도출의 경계 — C020 교훈). 개별 특수 규칙이 필요해 위험.
- **후기 5개는 명확한 단일 원인(재번호)** 이고, 매핑이 이미 데이터로 존재한다 — 5-report.md의 "번호 재발급 주석"에 `C003→C004`·`C006→C007` 형식으로. **provenance를 따라가면 결정적으로 회수 가능.** 억지가 아니라 정직한 추적.

## 가설

> **가설**: 재번호된 사이클의 5-report.md provenance 주석(`C<old>→C<new>` 재번호 매핑)을 파싱해, 도출기가 커밋 검색 시 새 id로 실패하면 옛 id로 재시도하면, 후기 5개 도출실패 사이클(loom/C083·C106, v3-build/C004·C007)의 스텝 커밋을 회수해 v3 지문을 소급할 수 있다 — 즉 도출실패 20이 15로 줄고, 그 감소분이 전부 재번호 부류로 정확히 설명된다.

## 기각 조건

- provenance 매핑을 따라가도 옛 id 커밋을 못 찾으면 → **기각**(재번호가 원인이 아니거나 매핑이 불완전).
- 회수한 커밋의 도출 지문이 실제 스텝과 안 맞으면 → **기각**(옛 커밋이 다른 사이클).
- 5개 중 일부만 회수되고 나머지가 다른 원인이면 → **부분 성공**(회수분만 보고, 잔여는 정직히 분류).
- provenance 파싱이 초기 15개까지 잘못 건드려 오도출하면 → **기각**(정밀화가 아니라 오염).
