4측정 ALL PASS → **가설 supported** (정직한 정정 1건 포함, 아래).

| 기각 조건 | 실측 | 발동? |
|---|---|---|
| provenance 따라가도 옛 커밋 못 찾음 | 4건 전부 옛 id로 회수 | ✗ |

## 데이터 직접 관찰

### ⭐ slug 없는 번호 매칭이 다른 사이클을 긁었다 (측정 중 함정)

첫 도출기는 옛 **번호만**으로 매칭했다(`C%03d`). 그 결과 C004←C003 회수가 10커밋이었는데, 그 안에 **두 다른 사이클이 섞였다**:
- `v3-build/C003-v3-viewer-step-tree` (재번호 전 이 뷰어 — 원하는 것)
- `v3-build/C003-first-commands-on-step-tree` (Clew의 진짜 C003 — **다른 사이클!**)

같은 번호 C003을 두 사이클이 (재번호 전/정상) 공유했기 때문이다. **C021 short_id 함정("전역 유일성을 잊으면 데이터가 조용히 뭉개진다")의 재연** — 여기선 번호만으로 매칭하니 slug이 다른 두 사이클이 한 번호로 뭉쳤다.

해소: **재번호는 번호만 바꾸고 slug은 유지한다**(git·5-report 전수 확인)는 사실을 이용. 옛 full id = `C{old_num}-{현재 slug}`로 구성하니 정확히 하나만 매칭, 오염 0. C004 회수가 10→4커밋으로 정확해졌다.

### 회수 4건 모두 slug 보존 확인

| 새 id | 옛 full id | 회수 스텝 |
|---|---|---|
| loom/C083-warp-selfupdate | C082-warp-selfupdate | 4 |
| loom/C106-go-deploy-panel-render | C105-go-deploy-panel-render | 4 |
| v3-build/C004-v3-viewer-step-tree | C003-v3-viewer-step-tree | 4 |
| v3-build/C007-viewer-node-body-interaction | C006-viewer-node-body-interaction | 5 |

slug이 전부 동일 — 재번호는 번호 충돌 해소일 뿐 사이클 정체(slug)는 안 바꾼다는 규율이 데이터로 확인됐다.

## 예상과 달랐던 것 — "후기 5개"는 실은 4개

설계 시 "후기 5개(재번호 부류)"라 적었으나, 실제 도출실패 목록의 재번호 사이클은 **4개**였다. loom/C104(재번호 C103→C104)는 git 원장에 재번호 커밋이 있으나 **도출실패 목록에 없었다** — 즉 이미 도출 성공. 재번호됐어도 도출기가 어떤 경로로 커밋을 찾은 것(아마 land 후 최종 커밋 subject가 새 번호를 담아). **정직한 정정: 회수 대상은 4개, 가설의 "5→15"는 "4→16"이 정확.** 가설의 방향(재번호 회수)은 맞았고 수만 정정.

## 판정

**채택 (supported).** 재번호 provenance(git `재번호` 커밋 + 5-report 주석)를 진실원으로 옛 id를 추적해, 도출실패 4개 재번호 사이클을 회수(20→16). slug 포함 매칭으로 오염 0, 잔여 16은 전부 초기 커밋 관습 이질로 정직히 남았다.

## 정점 통찰 — 정밀화의 정직은 "무엇이 회수 가능한가"를 가르는 것

20 도출실패를 억지로 0 만들 수도 있었으나(초기 15개에 특수 규칙), 그건 C020이 경계한 억지 도출이다. 이 카브는 **명확한 단일 원인(재번호)에 명확한 데이터(slug 보존 매핑)가 있는 4개만** 회수하고, 근본적으로 이질적인 초기 16개는 정직히 남겼다. **정밀화는 유령을 다 없애는 게 아니라, 없앨 수 있는 것과 남길 것을 정확히 가르는 것**(C017 가시성·C020 억지 도출 경계의 연장).

## 방법론에 귀속되는 교훈

- **전역 유일 식별자를 잊으면 데이터가 조용히 뭉개진다(C021 재연).** 번호는 체인 간·재번호 전후로 안 유일. slug 포함 full id가 유일 키. 부분 식별자로 매칭하면 조용한 오염.
- **재번호의 불변식: 번호만 바뀌고 slug은 유지.** 이 규율이 데이터로 확인돼, 옛 id 재구성을 결정적으로 만들었다. 좋은 재번호 관습이 미래 추적을 열었다(C019 쌍대).
- **진실원 다중화의 통합.** 재번호 매핑이 git 커밋과 5-report 두 곳에 흩어져, 둘을 합쳐야 완전(C083은 git만, C004는 5-report). 안정적(git) 1순위 + 보완(report).
