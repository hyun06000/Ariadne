# 5. 결과 보고

## 요약

상현님의 "v3로 굴리며 문서 갱신, 뷰어 완성 후 배포" 방향에서, **"v3로 굴린다"의 형태가 아직 설계 안 됐다**는 게 v3 준비의 진짜 구멍이었다. 세 후보(v3가 v2 대체 / v3=v2 위의 눈 / 병존) 중, C026 자신을 도그푸딩해 **후보 2(v3 = v2 5스텝 위에 얹힌 notes 눈)** 를 확정·실증했다. C026을 v2 gil로 5스텝 굴리며 v3 눈이 그 5스텝을 스텝 트리(s2·s3)·사이클 DAG(C026→C023)로 **steps.yaml 없이, 원장 불변(digest 614b630b)으로** 읽었다. **4측정 ALL PASS → supported.**

## 교훈

1. **"v3로 굴린다" = v2 5스텝 + v3 notes 눈.** v3는 v2를 대체하지 않고 그 위에 얹힌 눈이다. 실사이클은 지금처럼 5스텝을 굴려 원장에 각인하고, `gil migrate`가 그 커밋에 notes로 스텝 트리·계보를 얹어 읽는다. C017~C022 마이그레이션 사슬이 이미 구현한 모델을 명시적 답으로 고정.

2. **형태는 코드보다 먼저지만, 코드가 이미 답을 살았을 때 사이클은 그걸 언어화한다.** 마이그레이션 사슬이 암묵적으로 이 형태를 구현했고, 이 사이클은 흩어진 구현을 명시된 답으로 만들었다. 문서·배포가 이제 이 명시된 형태를 따른다.

3. **도그푸딩의 재귀성이 가장 강한 증거.** C026이 자기 자신을 v3 눈으로 읽었다(C026 Cycle-Parent: C023). 형태 성립이 이론이 아니라 실물. will.md "방법론이 방법론을 증명"의 v3판.

4. **원장 불변이 "v3는 눈이다"의 계약.** digest 614b630b 불변이 "v3가 원장 손대야 눈 성립"(기각 조건 4)을 막았다. v3는 순수히 눈 — 원장 변경자가 아니다. C018 계약의 도그푸딩 재확인.

5. **이름은 gil — gil의 v3이지 새 프로그램이 아니다(상현님 교정).** 형태와 정합: v3=v2 위의 눈이니 도구도 gil의 진화. gilv3는 개발 격리명, 배포 시 gil로 통합.
