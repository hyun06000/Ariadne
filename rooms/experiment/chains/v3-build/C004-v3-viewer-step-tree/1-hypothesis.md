# 1. 가설 수립

## 이전 사이클의 교훈

부모: v3-build/C002-design-v3-data-model (supported). C002가 확정한 것:
- **STEP 노드 스키마**: `{id, kind, parent, outcome, backtrack, body}`. `id`는 순수
  시간순(커밋순), **트리 구조는 오직 `parent`/`backtrack` 포인터가 담는다**. outcome은
  `analyze` 전용, backtrack은 반드시 조상 define을 가리킴.
- **디스크 표현**: 사이클 디렉토리 안 `steps.yaml`(평면 인접 리스트 = 단일 진실원) +
  `steps/<id>.md`. 디렉토리 **중첩 없음**.
- **실사례 = 검증 데이터**: `case-c012-c014/steps.yaml` 10노드. s1 아래 세 형제 가지
  (s2→s3→s4 · s5→s6→s7 · s8→s9→s10), backtrack 엣지 s4→s1·s7→s1, 죽은 잎 s4·s7
  (outcome=backtrack, 본문이 스스로 "벽의 지도"라 부름) · 산 잎 s10 (outcome=success).
- **v2 `parent`의 죄**: v2는 `parent`에 시간순과 논리순을 겹쳐 백트래킹을 선형으로
  뭉갰다(C014.parent를 C013으로 뒀다). v3는 s7의 `backtrack→s1` 한 엣지로 "벽에 부딪혀
  뿌리로 되돌아가 새 형제 가지가 났다"를 담는다 — **이 엣지가 v2엔 아예 없었다**.

체인 목표(chain.md): **사이클 = 스텝들의 트리**. v2 뷰어(gil web)는 사이클을 노드로
그렸다. v3 뷰어는 **한 사이클 안의 스텝 트리**를 그려야 한다.

## 문제 분할

v3 뷰어가 v2와 달라야 하는 이유는 정확히 세 가지, C002 데이터가 그 세 가지를 다 품는다:

1. **가지 (parent 엣지)**: 한 사이클이 이제 스텝들의 트리다. s1 아래 세 형제 가지가
   부채꼴로 갈린다. v2 선형 5스텝은 이걸 못 그린다.
2. **되돌아감 (backtrack 엣지)**: 죽은 잎 → 조상 define. s4→s1, s7→s1. 이 엣지는 v2
   원장에 존재조차 안 했다 — v3 뷰어의 존재 이유의 핵심.
3. **잎의 두 운명 (죽은 잎 vs 산 잎)**: outcome=backtrack(벽의 지도, 자산으로 보존)과
   outcome=success(채택, 사이클을 닫음)를 **사람 눈에** 구별. 리젝트가 자산이라는 v3
   철학(chain.md "리젝트가 자산")은 죽은 잎이 침묵당하지 않고 **보여야** 성립한다.

지금 정복할 첫 문제: **이 세 가지를 하나의 자기완결 SVG로 동시에 그리는 렌더가
가능한가**. 셋을 쪼개지 않는 이유 — 셋은 한 트리의 한 레이아웃 안에서만 동시에
의미를 갖는다(backtrack 엣지의 출발·도착이 가지 레이아웃 좌표에 의존, 잎 구별이
가지 끝에 얹힌다). 나눠 그리면 세 형제 가지와 되돌아감의 공간 관계가 사라진다.

## 가설

> **가설**: C002의 `case-c012-c014/steps.yaml`(10노드)을 입력받아, parent 엣지로 트리
> 가지를 그리고 backtrack 엣지를 시각적으로 구별되는 되돌아감 선으로 겹쳐 그리며,
> 죽은 잎(outcome=backtrack)과 산 잎(outcome=success)을 서로 다른 표식으로 렌더하는
> **자기완결(외부 리소스 0·서버 0) 단일 HTML/SVG 뷰어**를 만들면, 그 출력을 헤드리스로
> 실측했을 때 세 형제 가지·backtrack 엣지 2개·죽은 잎 2개·산 잎 1개가 좌표/표식
> 수준에서 명확히 구별되어 검출된다.

## 기각 조건

- **K1 (구조 왜곡)**: 렌더된 트리가 steps.yaml의 parent 인접 리스트와 다른 위상을
  그린다(형제 가지가 3개가 아니거나, 어떤 노드의 부모가 뒤바뀜) → 기각. 뷰어는 진실을
  비출 뿐 새 진실을 만들지 않는다(정체성).
- **K2 (backtrack 비가시)**: backtrack 엣지 2개가 parent 엣지와 헤드리스 실측에서
  구별 불가(같은 스타일·같은 마커·별도 요소 클래스 없음) → 기각. 되돌아감이 안 보이면
  v2와 다를 이유가 없다.
- **K3 (잎 운명 비구별)**: 죽은 잎과 산 잎이 헤드리스 실측에서 같은 표식으로 나와
  사람이 "벽의 지도"와 "채택된 답"을 못 가른다 → 기각.
- **K4 (계약 위반)**: 출력이 외부 리소스(CDN·폰트·원격 이미지·fetch)를 하나라도
  요구하거나 서버 없이는 안 열린다 → 기각. v2 뷰어 자기완결 계약을 잇는다.
