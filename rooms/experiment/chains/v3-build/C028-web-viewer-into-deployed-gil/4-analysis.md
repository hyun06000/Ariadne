## 데이터 직접 관찰

### digest 6bc9c7f2가 이식 정확성을 봉인

M1·M4가 같은 digest 6bc9c7f2 — 오라클(C025 gilv3)·배포판(gil web --v3)·격리 복사본이 전부 바이트 동일 HTML(396294 bytes). Sheen의 렌더 로직(SVG 좌표·CSS·JS·드릴다운 구조)이 한 글자도 안 바뀌고 배포판으로 왔음을 증명. C027이 migrate에서 한 것(743fc56a)을 web에서 재현.

### 죽은 스캐폴딩이 두 번 걸렸다

이식 중 두 번 실패: ① `from splice_topology import short_id` → ModuleNotFoundError ② `sys.path.insert(0, HERE)` → NameError. 둘 다 **모듈로 살던 코드의 스캐폴딩** — 모듈 간 import·sys.path 조작은 인라인되면 죽은 코드이자 오류원. C027 "이식의 함정은 숨은 의존(import 부족)"의 짝: 여기선 반대로 과잉 스캐폴딩. 자기완결 계약(M4)이 둘 다 강제로 드러냈다.

### rebuild_cycle_dag 누락 — 의존 그래프의 끝단

web_render.render가 rebuild_cycle_dag를 부르는데 이 함수는 별도 파일이라 첫 추출에서 빠졌다. 세 파일(steptree·notes_reconstruct·web_render)만 추출했으나 그들이 의존하는 rebuild_cycle_dag가 네 번째 의존. **의존 그래프를 끝까지 따라가야 이식이 완결된다** — 세 파일이 아니라 이행적 폐포가 이식 단위.

## 예상과 달랐던 것 — 상수 접두어가 조용한 오염을 막았다

steptree·web_render 둘 다 `CSS` 상수를 가져, 접두어(_ST_CSS·_WR_CSS) 없이 인라인하면 나중 것이 앞 것을 덮어 **조용한 오염**이 났을 것(C021·C024 "전역 유일성" 재연). 모듈 네임스페이스가 주던 격리를 인라인은 잃으므로, 접두어가 그 격리를 수동 복원했다.

## 판정

**채택 (supported).** `gil web --v3`가 배포판 gil.py에 통합됐다. 오라클 대조 바이트 동일(6bc9c7f2), v2 무회귀, conformance 134/134, 자기완결. v3 눈의 읽기 축이 하나의 gil로 들어와 "v3만 쓴다"의 도구 기반 완성.

## 정점 통찰 — 인라인 이식은 의존 폐포 + 스캐폴딩 제거 + 네임스페이스 수동 격리

C027·C028로 이식 패턴 완성. 인라인 이식의 세 규율:
1. **의존 폐포까지 추출** — 직접 파일만이 아니라 그들이 부르는 함수(rebuild_cycle_dag)까지.
2. **죽은 스캐폴딩 제거** — 모듈 import·sys.path는 인라인되면 오류원.
3. **네임스페이스 수동 격리** — 모듈이 주던 격리를 접두어로 복원(CSS 충돌 방지).
오라클 대조(바이트 동일) + 자기완결 계약(격리 복사)이 이 셋을 전부 집행.

## 방법론에 귀속되는 교훈

- **인라인 이식의 단위는 파일이 아니라 의존 폐포.** 세 파일을 옮겨도 네 번째 의존이 빠지면 런타임 실패.
- **모듈 격리를 잃으면 접두어로 복원.** 인라인은 네임스페이스를 합치므로 상수 충돌이 조용히 난다(C021 재연). 모듈별 접두어가 수동 격리.
- **자기완결 배포 계약이 이식 완전성의 검사자.** import 부족(C027)·과잉 스캐폴딩(C028) 둘 다 격리 복사(M4)가 드러냈다.
