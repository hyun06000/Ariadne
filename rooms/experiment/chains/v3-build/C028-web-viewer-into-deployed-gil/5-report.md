# 5. 결과 보고

## 요약

C027이 v3 눈의 뿌리(gil migrate)를 배포판에 통합한 데 이어, 읽기 축인 **Sheen C025 통합 뷰어를 `gil web --v3`로 배포판 gil.py에 통합**했다. C025 백엔드(steptree·notes_reconstruct·web_render + rebuild_cycle_dag ≈ 800줄)를 인라인 이식(상수 접두어 _ST_·_WR_로 충돌 방지), cmd_web에 --v3 옵트인 분기. 오라클 대조로 배포판 `gil web --v3`가 C025 gilv3 web과 HTML 바이트 동일(digest 6bc9c7f2), v2 무회귀·conformance 134/134. **5측정 ALL PASS → supported. 배포판 gil이 v3 쓰기(migrate)+읽기(web)를 다 갖춰 "v3만 쓴다"의 도구 기반이 완성됐다.**

## 교훈

1. **인라인 이식은 의존 폐포 + 스캐폴딩 제거 + 네임스페이스 수동 격리 (세 규율).** C027·C028로 이식 패턴 완성: ① 직접 파일만이 아니라 그들이 부르는 함수(rebuild_cycle_dag)까지 이행적 폐포를 추출 ② 모듈 import·sys.path 같은 죽은 스캐폴딩 제거 ③ 모듈이 주던 네임스페이스 격리를 접두어로 복원. 오라클 대조(바이트 동일) + 자기완결 계약(격리 복사)이 셋을 집행.

2. **죽은 스캐폴딩은 인라인의 함정 (C027의 짝).** C027은 import 부족(hashlib·glob)이 함정이었고, C028은 반대로 과잉 스캐폴딩(불필요한 import·sys.path)이 함정. 모듈로 살던 코드를 인라인하면 로딩 장치가 죽은 코드이자 오류원이 된다. 자기완결 계약(M4)이 양쪽을 드러냈다.

3. **모듈 격리를 잃으면 접두어로 복원 (C021 재연).** steptree·web_render 둘 다 CSS 상수를 가져, 접두어 없이 인라인하면 조용한 오염. 인라인은 네임스페이스를 합치므로 "전역 유일성"이 다시 문제 — 모듈별 접두어가 수동 격리.

4. **"v3만 쓴다"의 도구 기반 완성.** 배포판 gil이 쓰기(migrate로 v3 눈 각인)+읽기(web --v3로 두 층 드릴다운)를 다 갖췄다. 배포된 도구를 쓰는 누구나 자기 v2 원장을 v3로 각인하고 사람 눈으로 본다(will.md 범용성). C027+C028로 v3 눈이 하나의 gil에 온전히.
