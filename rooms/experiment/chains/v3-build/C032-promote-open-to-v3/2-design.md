# 2. 실험 설계

## 상현님 확정 방향 (설계 단계 3연속 컨펌)

이 사이클은 큰 전환의 **설계 사이클**이다. 승격 전략을 상현님과 세 번 물어 확정했다:

1. **범위**: **전면 승격** — 쓰기(open/step/close)·읽기(log/web)·conformance·공존까지.
2. **v2 처우**: **완전 은퇴** — 새 v2 사이클을 못 연다.
3. **기존 189 처우**: **눈 마이그레이션** — 데이터(커밋·문서) 불변, v3 눈(notes)만 각인. (닫힌 사이클 불변 R4·R5, "v2 여정 보존" 헌법 준수)

세 답이 모순 없이 맞물린다: **원장 데이터는 불변, "v3"를 원장을 읽는 유일한 눈이자 새 사이클을 쓰는 유일한 손으로 승격.** v2는 표면에서 은퇴하지만 189의 데이터는 그대로 살아 v3 눈으로 읽힌다.

## 핵심 통찰 — 승격은 "함수 교체"가 아니라 "인터페이스 정체성 전환"이다

배포판 gil.py 실측 (C032 s2에서 코드 읽음):

| | v2 open | v3 open |
|---|---|---|
| 표면 | `gil open <chain> <slug>` | `gil v3 open <dir>` |
| 위치인자 | 2개 (chain, slug) | 1개 (dir 경로) |
| 번호 | 자동 증가 (chain 안 유일) | 없음 (경로가 정체) |
| 원장 | cycle.yaml + 5문서 디렉토리 | steps.yaml 스텝 트리 |

**"gil open을 v3로 승격"은 cmd_open→cmd_v3open 함수 포인터 교체가 아니다.** 두 인터페이스의 위치인자 수·정체성 모델(번호 vs 경로)이 다르다. 그래서 전면 승격은 **한 사이클에 코드로 다 밀 수 없다** (C030 M4 "대체는 걸림돌 넘는 여정" 경고 준수). C032는 전략을 확정하고 **가장 작은 첫 실행 조각만** 검증한다.

## C032가 실제로 코드로 미는 것 (가장 작은 첫 조각)

전면 승격의 **①네임스페이스 승격**만. 나머지(②읽기축 log/web 기본 전환, ③conformance v3 재정의, ④공존 완결)는 후속 사이클로 정직히 이월.

- **A. `gil open`을 v3로**: 기본 진입점을 v3 스텝 트리 쓰기로. v2 open은 은퇴 경로로 이동.
- **B. v2 은퇴 안내**: v2 습관(`gil open <chain> <slug>`, 2 위치인자)으로 옛 방식을 시도하면 **친절한 에러**가 v3로 안내 — 침묵 실패(옛 인자 무시) 아님. C031 M5 씨앗을 open에 확장.
- **C. 눈 마이그레이션 재확인**: 189 데이터 불변 + v3 눈이 이미 읽음(migrate)을 실측으로 재확인 — 승격이 기존 원장을 안 깬다.

## 절차

1. **현 표면 스냅샷** — 승격 전 `gil open <chain> <slug>` 동작·conformance 134/134·원장 커밋 digest를 baseline으로 기록.
2. **네임스페이스 승격 구현** — `gil open`이 v3 쓰기로 분기하도록. 인터페이스 차이(위치인자 2 vs 1) 때문에 단순 함수 교체가 안 되므로, **v2 습관 인자를 감지해 친절한 에러로 안내**하는 얇은 디스패치를 open 진입에 얹는다. (전면 인터페이스 통일은 후속 — C032는 "v3가 기본, v2 시도는 안내" 최소형)
3. **v2 은퇴 안내 실측** — 옛 방식(`gil open <chain> <slug>`) 시도 → 친절한 에러가 v3 방식을 정확히 가르치는지(침묵 실패 아님) 실측.
4. **무회귀·불변 실측** — conformance 그대로인지, 원장 커밋 digest 불변인지, 189 v3 눈(web --v3/migrate)이 여전히 189 읽는지 실측.

## 준비물

- 배포판 `rooms/deployment/ariadne-spec/gil.py` (운영 도구, Python3 stdlib).
- 현 원장 (체인 8개·사이클 161개, refs/notes/commits에 v3 눈 각인).
- conformance 판정기 (배포판, 현재 134/134).

## 측정 방법

- **M1 (승격)**: `gil open`이 v3 쓰기로 진입 — v3 스텝 트리(steps.yaml)를 생성하는가. 성공: v3 네이티브 산출물 생성.
- **M2 (은퇴 안내)**: v2 습관 인자로 `gil open` 시도 → 친절한 에러가 v3 방식을 정확히 가르치는가 (exit≠0, 안내문에 v3 사용법·v2→v3 대응). 기각: 침묵 실패(인자 무시하고 뭔가 생성) 또는 안내 없는 raw 에러.
- **M3 (무회귀)**: conformance 134/134 유지. 기각: 판정기 초록 깨짐 → ①이 ③과 분리 불가(가설 기각조건 2).
- **M4 (데이터 불변)**: 원장 커밋 digest(`git rev-list --exclude=refs/notes/* --all` 정렬) 불변. 189 데이터 안 건드림.
- **M5 (눈 생존)**: 승격 후에도 v3 눈이 189를 읽음 (web --v3 노드 수 유지). 눈 마이그레이션이 승격과 공존.
