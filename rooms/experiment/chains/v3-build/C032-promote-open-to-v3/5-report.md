# 5. 결과 보고

## 요약

`gil open`을 버전리스 승격했다 — 기본을 v3로, v2 습관은 친절한 은퇴 안내로(전환기 게이트 `GIL_V2_OPEN=1`이 v2 유지). 설계 단계에서 상현님께 **전면 승격 + v2 완전 은퇴 + 기존 189 눈 마이그레이션**을 3연속 컨펌받고, 그 첫 조각(네임스페이스 승격)만 배포판에 심어 5측정 ALL PASS(conformance 134/134, 원장 digest 05ba3e6c 불변, v3 눈 132노드 생존). **가설 채택(supported)** — 승격은 도구 복잡화가 아니라 기본값 전환 + 안내 흡수로 달성된다(상현님 처방 실현).

## 교훈

1. **⭐⭐ 승격은 "함수 교체"가 아니라 "인터페이스 정체성 전환"이다.** v2 `gil open <chain> <slug>`(위치인자 2·번호 자동증가)와 v3 `gil v3 open <dir>`(위치인자 1·경로가 정체)는 인자 수·정체성 모델이 다르다. 그래서 `cmd_open→cmd_v3open` 함수 포인터 교체가 불가능. 큰 전환은 한 사이클에 코드로 다 밀 수 없다(C030 M4 "대체는 여정" 준수) — C032는 전략 확정 + 첫 조각(v2 은퇴 안내로 v3 유도)만.

2. **⭐⭐ ①네임스페이스 승격은 ③conformance 재정의와 분리 불가 — 반증이 아니라 분할 경계의 실증.** conformance 134항목 중 20+개(OPEN-\*·GUARD-\*)가 v2 open 인터페이스를 직접 실행. 게이트 없이 은퇴시키면 첫 open에서 crash(기각조건 2). **그러나 전환기 게이트가 그 긴장을 흡수**해 134/134 유지. 완전 버전리스(게이트 없이 초록)는 conformance를 v3 계약으로 재정의해야 완성 — **이 사이클이 그 다음 관문의 좌표를 정확히 찍었다.**

3. **⭐ 승격 = 기본값 전환 + 안내 흡수 (상현님 처방의 코드적 실현).** "결국 v3 표준, 버전 몰라도 되게"는 도구를 복잡하게 만드는 게 아니라 `gil open`의 기본을 v3로 두고 v2 습관을 친절한 에러가 v3로 안내함으로써 달성된다. C031 `gil v3 step` 전환 안내의 open 확장. 침묵 실패(옛 인자 무시) 아님 — 정확한 v2→v3 대응을 가르침.

4. **⭐ 눈 마이그레이션과 승격의 공존 — 데이터 불변이 셋을 화해시킨다.** 전면 승격·v2 은퇴·189 마이그레이션 세 방향이 모순 없이 맞물린 이유: **원장 데이터(커밋·문서)는 불변, "v3"를 유일한 눈이자 손으로.** 189는 데이터 그대로 살아 v3 눈으로 읽히고(M5 132노드), 새 사이클만 v3로 쓴다. 닫힌 사이클 불변 R4·R5 준수.

5. **⭐ 계측기 오염이 환경변수 층에서 재연.** M3 첫 실측 133/134의 유일 FAIL은 승격 결함이 아니라 `env` 래퍼가 빈-PATH 테스트에서 python3 못 찾은 rc=127. 게이트를 프로세스 환경 상속으로 바꾸니 134/134. Weft·Bobbin의 "계측기 vs 반증 구분"이 환경변수 주입 방식에서 재연 — 환경변수는 계측을 오염시킬 수 있다.

## 다음 사이클을 위한 제안

1. **⭐⭐ conformance v3 재정의 (③, 다음 필수 관문)** — 이 사이클이 찍은 좌표. 판정기가 v2 open 인터페이스에 결합된 20+항목을 v3 계약(steps.yaml·kind 순환·백트래킹)으로 재정의해야 게이트 없이도 초록. 완전 버전리스의 핵심 관문.
2. **읽기축 기본 전환 (②)** — `gil log`/`gil web` 기본을 v3 눈으로(지금 v2 기본, --v3 옵트인). 승격의 읽기 짝.
3. **인터페이스 정체성 통일 (①의 완성)** — `gil open`이 v2 은퇴 안내를 넘어 v3 쓰기로 *직접 진입*. v2(번호)·v3(경로) 인터페이스를 하나로. C032가 남긴 최소형의 완성.
4. **공존 완결 (④)** — 189 v2 사이클과 새 v3 사이클이 한 원장에 섞일 때 fsck·web의 통합 표현. Sheen C025 통합 뷰어가 이미 두 축을 한 눈에 — 그 위에서.
5. **v2→v3 온보딩 문서** — C031·C032 친절한 에러(씨앗)를 SPEC/README "v2에서 온 사람에게" 대응표로 전면화. "문서가 곧 테스트"(순서 2번).
