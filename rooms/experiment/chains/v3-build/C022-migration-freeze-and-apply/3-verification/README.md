# 3. 검증 — 마이그레이션 동결백업 + 실제 원장 적용 (①동결백업, 실전화)

C017~C021이 v3 마이그레이션 왕복(②읽기호환·③소급각인·④위상접합)을 **격리 복제본**에서 안전 실증했다. 이 카브는 그 마이그레이션을 **우리 실제 원장에 되돌림 가능하게 적용**하고, 4단계 중 유일한 미완 ①동결백업을 세운다.

## ⭐ 무엇이 "동결백업"인가 — refs/notes 뿐

마이그레이션 노트의 4단계는 원래 v2 데이터 변환을 상정했다. 그러나 **C018이 밝힌 대로 원장 커밋은 애초에 안 바뀐다**(notes는 refs/notes에만 append). 따라서 실제 "되돌림 대상"은 커밋이 아니라 **refs/notes/\* 뿐**이다.

→ **동결백업 = 적용 직전 refs/notes 상태를 백업 ref(`refs/notes-backup/pre-c022`)로 못박는 것.** 적용 전 refs/notes가 부재였으므로(실측) 백업 = "부재 기록", 되돌림 = refs/notes 삭제로 부재 복원. 커밋·cycle.yaml·작업 트리는 애초에 백업 대상이 아니다.

## 재현

```bash
python3 apply_migration.py <repo>       # 백업(A) + 적용(B): 노드 소급 + 위상 접합
python3 rollback_migration.py <repo>    # 되돌림(C): 백업 ref로 리셋 / notes 삭제
python3 measure.py <repo> <scratch>     # 5측정 (격리 복제본 독립 재현)
```

## 산출물

- `apply_migration.py` — 국면 A(동결백업) + B(적용) + 불변 게이트 자기 집행.
- `rollback_migration.py` — 국면 C(되돌림): 백업 ref로 리셋, notes 완전 제거 실측.
- `snapshot.py` — 커밋 SHA·트리·cycle.yaml 해시 캡처(불변 검증 계측기, refs/notes 제외).
- `measure.py`·`measure-out.txt` — 5측정 감사(격리 복제본에서 왕복 독립 재현, ALL PASS).
- `apply-real-log.txt` — 실제 원장 적용 로그(백업→적용→검증→되돌림 실증→재적용).
- C020/C021 재사용: `full_ledger_migrate.py`·`splice_topology.py`·`derive_fingerprint.py`·`retro_imprint.py`·`rebuild_cycle_dag.py`.

우리 진짜 저장소에 왕복 전 국면을 실행했다(`apply-real-log.txt`):

|---|---|

| **B 적용** | 152 사이클, **542 노드 각인** + **132 사이클 접합**(루트5·선형123·머지4). 커밋 digest `c9cb8090` **불변** |

## ⭐ 정점 교훈 — 되돌릴 수 있음을 코드로 보인 뒤에만 적용을 남긴다

실제 원장을 건드리는 유일한 문턱에서, 순서가 안전을 만들었다: **백업 → 적용 → 검증 → 되돌림 실증 → 되돌림 확인 → 재적용.** 국면 C(되돌림)를 실제 원장에서 코드로 실증(notes 삭제→DAG 0노드→커밋 불변)한 **뒤에야** 국면 D(재적용)로 최종 상태를 남겼다. "되돌릴 수 있다"는 주장이 아니라 실측이 됐다. C018 "append-only는 커밋을 고치는 소급만 금지한다"가 **실제 우리 원장에서 참**임이 확인됐다.

## ⭐ 교훈 — 마이그레이션의 부작용면은 refs/notes 하나로 봉인됐다

불변 게이트(snapshot 자기 집행)가 매 국면 후 커밋 SHA·트리·cycle.yaml을 원장 전(c9cb8090)과 대조해 하나라도 바뀌면 중단하도록 설계됐다. 실측에서 커밋 digest가 적용·되돌림·재적용을 거쳐 전부 `c9cb8090`으로 고정 — **마이그레이션이 refs/notes 밖을 하나도 안 건드림이 코드로 봉인됐다.** "격리에서 안전"(C020/C021)이 "실제 원장에서 안전"으로 승격됐다.

## 마이그레이션 4단계 진척

- ①**동결백업 ✅(C022)** / ②읽기호환 ✅(C017) / ③소급각인 ✅(C018/C019/C020) / ④위상접합 ✅(C021)
- **네 단계 전부 섰고, 격리를 넘어 실제 원장에 적용됐다.**

## 결론

**ALL PASS → supported.** 동결백업(①)을 세우고 C020/C021 파이프라인을 실제 우리 원장 refs/notes에 되돌림 가능하게 적용했다. 커밋·트리·cycle.yaml 전부 불변(C018 계약 실증), 실제 원장 DAG 132노드·131엣지 복원(머지 완전일치), 되돌림 완전(잔재 0)·되돌림 후 무손상. **v3 마이그레이션이 드라이런을 넘어 실전이 됐다** — 우리 자신이 v3의 첫 마이그레이션 대상이다.

## 정직한 경계

- **원격 push는 이 사이클 밖**(설계 안전철칙 4) — 로컬 refs/notes 각인·검증까지. refs/notes 원격 push는 외부로 나가는 행동이라 별도 결정(상현님).
- **20 도출실패 사이클은 여전히 섬**(C020/C021 이월) — 초기 커밋 관습 이질. 이 사이클은 적용의 안전성이지 도출 완전성이 아니다. DAG 132(=152−20)가 이를 정직히 반영.
- **notes ref 해시는 재적용마다 다름**(notes 커밋 timestamp) — 내용은 동일. 백업/되돌림은 ref 값 기준이라 무해.
- **뷰어 렌더는 Sheen 축** — 재구성기는 검증 계측기이지 사람 눈 뷰어가 아니다.
- 계측기 결함 1건(measure.py `sh()`가 repo=None일 때 git 미부착) 수리 — 반증 아님.
