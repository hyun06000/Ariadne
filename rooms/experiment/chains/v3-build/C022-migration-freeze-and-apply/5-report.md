# 5. 결과 보고

## 요약

C017~C021이 격리 복제본에서 안전 실증한 v3 마이그레이션을, 동결백업(①)을 세워 **우리 실제 원장 refs/notes에 되돌림 가능하게 적용**했다. 백업→적용→검증→되돌림 실증→재적용 순서로, 커밋 digest `c9cb8090`이 전 국면 불변(C018 계약 실증)이고 실제 원장 DAG 132노드·131엣지(머지 4)가 복원됐으며 되돌림이 notes 완전 제거(잔재 0)로 실증됐다. **5측정 ALL PASS → supported.** 마이그레이션 4단계(①②③④)가 전부 섰고, 격리를 넘어 우리 자신이 v3의 첫 마이그레이션 대상이 됐다.

## 교훈

1. **①동결백업의 정체는 대상 시스템의 부작용면이 결정한다.** C018 계약상 마이그레이션이 바꾸는 것은 refs/notes/\* 뿐(커밋·트리·cycle.yaml 불변)이므로, 백업도 되돌림도 refs/notes만. "무엇을 백업할지"는 "무엇이 바뀌는지"의 함수다 — 커밋을 백업 대상에 넣었다면 낭비이자 오해였을 것.

2. **되돌릴 수 있음은 주장이 아니라 실측이다.** 실제 원장을 건드리는 유일한 문턱에서 순서가 안전을 만들었다: 적용(132노드)→**되돌림(0노드, 커밋 불변)**→재적용(132노드). 되돌림을 실제 원장에서 한 번 돌려 보인 뒤에야 최종 적용을 남겼다. 되돌리기 어려운 행동 앞의 규율의 코드적 구현.

3. **격리 실증이 실제 적용의 담보다 — 승격의 표준 경로.** 격리(C020/C021)에서 왕복이 닫혔기에 실제 적용의 유일한 새 위험은 "복제본≠원장"뿐이었고, M3가 132 정합으로 그마저 닫았다. "격리에서 안전"이 "실제 원장에서 안전"으로 승격됐다.

4. **불변 게이트를 자기 집행으로 코드에 박으면 사고가 원리적으로 막힌다.** snapshot 대조가 매 국면 후 자동 실행돼 커밋이 하나라도 바뀌면 즉시 중단하도록 설계됐다. 사람의 주의가 아니라 코드가 안전을 집행. C018 계측기 함정(rev-list가 notes 커밋을 셈)을 `--exclude=refs/notes/*`로 정확히 겨눔.

5. **C018 "append-only는 커밋을 고치는 소급만 금지한다"가 실제 우리 원장에서 참.** notes는 별도 ref에 (커밋,메타)를 저장하고 원장 커밋 객체는 SHA 콘텐츠 주소라 불변. digest `c9cb8090` 고정이 이를 봉인. 마이그레이션은 데이터 변환이 아니라 **v3의 눈의 범위를 넓히는 것**(trailer→trailer+notes)이라는 C018 통찰의 실전 완결.

## 다음 사이클을 위한 제안

1. **refs/notes 원격 push 결정 (상현님)** — 이 사이클은 로컬 각인까지. refs/notes를 원격에 올리면 다른 머신/세션에서도 v3의 눈이 공유되나, 외부로 나가는 행동이라 유보했다. push 여부·전략(`git push origin refs/notes/*`)이 다음 결정.

2. **20 도출실패 섬 정밀화** — 초기 커밋 관습 이질 사이클(tapestry/C001·genesis/C002·v3-build/C004 등)을 개별 규칙으로 접합. 억지 도출의 경계를 지키며 유령을 줄이는 잔여 카브. 152 전량 접합이 목표는 아니고 "정직하게 도출 가능한 만큼"의 확장.

3. **통합 재구성기 / 뷰어 (Sheen 축)** — 이제 실제 원장에 노드·엣지가 각인됐으니, 사이클 내(스텝 트리)×사이클 간(계보 DAG)을 한 눈에 보는 뷰어. C021이 접합한 계보를 사람 눈에. 병렬 가능(Sheen).

4. **마이그레이션의 gil 명령화** — apply_migration.py·rollback_migration.py의 실증된 절차를 `gil migrate`·`gil migrate --rollback` 명령으로 승격. 지금은 사이클 산출물 스크립트인데, 쓰기 축 다섯 조각처럼 도구 강제로 올릴 수 있다(다른 사용자·다른 원장의 v2→v3 이주에 재사용).
