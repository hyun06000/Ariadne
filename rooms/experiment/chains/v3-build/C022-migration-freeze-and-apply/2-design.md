# 2. 실험 설계

오직 1-hypothesis.md의 가설 하나 — **되돌림 가능한 실제 원장 적용** — 을 검증한다.

## 정답을 도구보다 먼저 고정한다

C018이 밝혔듯 마이그레이션이 건드리는 것은 **refs/notes/\* 뿐**(커밋·cycle.yaml·트리 불변)이다. 따라서 백업 대상도, 되돌림 대상도 refs/notes뿐이다. 세 국면으로 고정한다.

## 절차

1. **국면 A — 동결백업(①).** 적용 직전 refs/notes 상태를 백업 ref(`refs/notes-backup/pre-c022`)로 복사. 적용 전 refs/notes가 없으면(C022 시작 시 확인됨) 백업 = "부재 기록", 되돌림 = refs/notes 전체 삭제로 부재 복원.
2. **국면 B — 실제 적용.** C020 노드 소급 + C021 위상 접합 파이프라인을 **복제본이 아닌 우리 실제 원장(REAL_REPO)** 에 실행. 스크립트는 격리에서 이미 검증됐고 유일한 변경은 대상 경로.
   1. 적용 전 안전 스냅샷(전 커밋 SHA 목록, `git status`, cycle.yaml 해시) 캡처.
   2. 동결백업 ref 생성(국면 A).
   3. `full_ledger_migrate.py --apply` → 실제 refs/notes에 노드 지문 각인.
   4. `splice_topology.py` → Cycle-Parent notes 엣지 각인.
   5. 적용 후 안전 스냅샷 재캡처 → 커밋 SHA·트리·cycle.yaml 불변 검증.
3. **국면 C — 되돌림 실증.** `rollback_migration.py`가 백업 ref로 되돌리고 notes 완전 제거를 실측(잔재 0 확인).
4. **국면 D — 재적용(최종 상태).** 되돌림을 코드로 보인 뒤 재적용하여 원장에 마이그레이션이 각인된 상태로 남긴다. 되돌림만 남으면 마이그레이션이 무효가 되므로.
   - 최종 순서: 백업 → 적용 → 검증 → 되돌림 실증 → 되돌림 확인 → **재적용** → 최종 검증.

## 준비물

- `apply_migration.py` — 백업 + 적용 + 안전 스냅샷 검증(REAL_REPO 대상).
- `rollback_migration.py` — 백업 ref로 되돌림 + notes 제거 실측.
- `snapshot.py` — 커밋 SHA 목록·트리 상태·cycle.yaml 해시 캡처(불변 검증 계측기).
- `measure.py` — 5측정 감사.
- C020/C021 산출물 재사용: `full_ledger_migrate.py`·`splice_topology.py`·`derive_fingerprint.py`·`retro_imprint.py`·`rebuild_cycle_dag.py`.
- 환경: Python 3(stdlib만), git. REAL_REPO = 이 저장소 루트.

## 측정 방법 (5측정)

| 측정 | 확인 | 통과 기준 |
|---|---|---|
| **M1 커밋 불변** | 적용 전후 전 커밋 SHA 목록 동일, `git status` 청결 | 바이트 동일 |
| **M2 트리·cycle.yaml 불변** | 적용 전후 작업 트리·cycle.yaml 해시 동일 | 바이트 동일 |
| **M3 실제 재구성** | 실제 원장 refs/notes에서 사이클 DAG 복원 → C021 격리(131 노드·130 엣지, 머지 4) 대조 | 일치 또는 차이 정직 회계 |
| **M4 되돌림 완전성** | 백업 ref 되돌림 후 refs/notes 완전 제거, 적용 전(부재) 정확 복귀 | 잔재 0 |
| **M5 되돌림 후 원장 무손상** | 되돌림·재적용 거쳐도 커밋 SHA·트리 불변 | 바이트 동일 |

## 안전 철칙 (실제 원장을 건드리므로)

1. **커밋·트리 불변이 최우선 게이트.** 하나라도 바뀌면 즉시 중단·되돌림.
2. **notes만이 유일한 변경면.** refs/notes-backup 외 어떤 새 ref도 안 만든다.
3. **되돌림을 먼저 실증하고 최종 적용.**
4. **원격 push는 이 사이클 밖** — 로컬 refs/notes 각인·검증까지. refs/notes 원격 push는 외부로 나가는 행동이라 상현님 보고 후 별도 결정.
