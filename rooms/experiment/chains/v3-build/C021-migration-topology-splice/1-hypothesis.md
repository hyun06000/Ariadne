# 1. 가설 — cycle.yaml parent를 사이클 간 v3 엣지로 접합

부모: v3-build/C020-migration-full-ledger-dryrun (supported — 전량 순회 소급 안전 실증).
lineage: v3-build/C015-merge-is-lineage-command (lineage=머지, 다중부모 계보).

## 이전 사이클의 교훈

C020이 전량 순회로 532 스텝 노드를 각 사이클 **안에서** notes 지문으로 편입했다. 그러나 그것들은 아직 **150개의 분리된 섬** — 사이클 내부 트리는 이어졌지만 사이클 *사이*가 안 이어졌다. cycle.yaml의 `parent`가 이미 사이클 간 계보를 담고 있으니(단일 162·머지 6·null 27), 그걸 v3 지문 엣지로 소급하면 섬들이 한 그래프가 된다.

C015에서 도구화한 **lineage=머지**가 정확히 이 재료다: cycle.yaml `parent: [A, B]`는 다중부모 계보(C012 명세), 단일 parent는 선형 계보, null은 체인 루트.

## 문제 분할

마이그레이션 4단계 중 ①②③이 섰고 **④ 백트래킹 복원**만 남았다. "복원"의 핵심은 v2 원장의 **전체 위상**을 v3의 눈에 되살리는 것 — 사이클 내부(C020 완료)와 **사이클 간 계보**(이 카브).

**가장 작은 단위 = 사이클 간 엣지 접합.** chain.md 정의대로 "체인 = 사이클들의 DAG"이므로, 사이클 간 관계는 각 사이클의 지문에 **부모 사이클 포인터**를 담는 것. 노드 위상(s1~s5)은 C020이 했고, 이 카브는 사이클 위상(cycle→parent cycle).

## 접합 설계 (개념)

각 사이클의 루트 지문(s1)에 **Cycle-Parent** 필드를 추가:
- 단일 parent(`parent: C014`) → `Cycle-Parent: C014` (선형 계보)
- 머지 parent(`parent: [C020, C016]`) → `Cycle-Parent: C020, C016` (다중부모, C015 lineage=머지)
- null → `Cycle-Parent: null` (체인 루트)

이러면 재구성기가 사이클 내 노드(Parent)와 사이클 간 엣지(Cycle-Parent)를 둘 다 읽어, 150 섬이 한 DAG로 이어진다. **notes로 소급**(C018 커밋 불변) — cycle.yaml은 안 바꾸고 notes에 Cycle-Parent 첨부.

## 가설

> **가설(H1)**: 각 사이클의 루트 지문(notes)에 cycle.yaml parent를 `Cycle-Parent` 엣지로 소급하고 재구성기가 그것을 읽으면, C020의 150개 분리된 섬(사이클 내 트리)이 **사이클 간 계보로 이어진 한 v3 체인 DAG**가 된다 — 단일·머지·루트 세 형태 모두.

- **H1a 엣지 접합:** 각 사이클 루트에 Cycle-Parent가 notes로 붙고, 재구성기가 사이클 간 엣지를 복원.
- **H1b 세 형태:** 단일 parent·머지 parent(다중)·null(루트)이 각각 정확히 엣지로(머지는 C015 lineage와 동형).
- **H1c 커밋 불변:** Cycle-Parent 소급이 notes라 커밋 SHA 안 바꿈(C018 계약, cycle.yaml도 불변).
- **H1d DAG 정합:** 접합된 그래프가 실제 체인 DAG와 일치(사이클 수·엣지 수·루트 수가 cycle.yaml과 대조).

## 기각 조건

- Cycle-Parent가 안 붙거나 재구성기가 사이클 간 엣지를 못 읽음. → H1a 기각.
- 머지 parent(리스트)를 단일로 뭉개거나 놓침. → H1b 기각.
- 소급이 커밋·cycle.yaml을 바꿈. → H1c 기각.
- 접합 DAG가 실제 계보와 불일치(엣지 수·루트 수 틀림). → H1d 기각.

## 검증 설계 (다음 스텝)

C020 격리 복제본 방식 재사용(우리 원장 안 건드림). 대표 사이클 부분집합으로 접합 실증.
- M1 엣지 접합(H1a): 사이클 루트 notes에 Cycle-Parent, 재구성기가 사이클 간 엣지 복원.
- M2 세 형태(H1b): 단일·머지·루트 각각 정확히(머지는 다중 Cycle-Parent).
- M3 커밋 불변(H1c): 소급 전후 커밋 SHA·cycle.yaml 불변.
- M4 DAG 정합(H1d): 접합 그래프의 사이클/엣지/루트 수 == cycle.yaml 집계.

## 정직한 범위 (선긋기)

- **사이클 간 엣지 접합만.** 노드 위상(C020)·도출실패 정밀화(C020 이월)는 별개. 이 카브는 "섬을 잇는가".
- **격리 복제본(드라이런).** 실제 원장 적용은 별도 결정(상현님).
- Cycle-Parent는 루트(s1) 지문에 — 사이클을 대표. 스텝별 사이클 엣지는 불필요(사이클이 단위).
- 대표 부분집합으로 세 형태 실증 + 전량 집계 대조. 전량 그래프 렌더는 뷰어 축(Sheen).
