## 교훈

1. **⭐⭐ 살아있는 계보와 죽은 계보는 같은 개념, 다른 각인 수단이다.** C015가 살아있는 v3 사이클의 lineage를 `git merge --no-ff` 다중부모 커밋으로 만들었다면, C021은 죽은 v2 사이클의 lineage를 `git notes` Cycle-Parent로 담았다 — **같은 lineage 개념(다중부모 계보), 다른 각인 수단.** 살아있으면 머지 커밋(전진), 죽었으면 notes 소급(불변에 주석). C018 "append-only는 과거에 주석을 더한다"가 여기서 계보 층에도 적용됐다: 살아있는 것은 커밋으로 잇고, 죽은 것은 notes로 잇는다. **한 개념이 상태(생/사)에 따라 두 수단으로 실현된다.**

2. **⭐ 전역 유일성을 잊으면 데이터가 조용히 뭉개진다(short_id 함정).** DAG 키를 short_id(C001)로만 하니 체인 간 충돌(loom/C001·v3-build/C001)로 131이 104로 조용히 줄었다 — 에러 없이 dict가 덮었다. cycle id는 **체인 안에서만** 유일하다는 걸 잊은 것. 마이그레이션은 여러 체인을 한 그래프로 모으므로 전역 유일 키(chain/id)가 필수. **부분에서 유일한 식별자를 전체에서 유일하다 가정하면 조용한 손실이 난다** — C106에서 Weft가 "판정기가 못 본 걸 두 구현 대조가 짚었다"던 그 종류의 조용한 결함.

3. **⭐ 머지의 완전 일치가 검증의 앵커였다.** 총계는 gap 20이 있었지만 머지가 4==4로 완전 일치한 게 접합이 옳다는 강한 신호였다 — 머지 사이클(계보의 가장 복잡한 형태)이 정확히 잡히면 단순 형태(단일·루트)는 더 쉽다. 그리고 gap 20이 정확히 C020 도출실패 수와 같아, 새 손실이 아니라 알던 잔여임이 확인됐다. **가장 복잡한 케이스의 완전 일치 + 잔여의 정확한 회계 = 정합의 이중 증거.**

4. **위상 접합으로 마이그레이션 왕복이 닫혔다.** C017(읽기호환)→C018(소급각인)→C019(자동도출)→C020(전량)→C021(위상). 이제 "유령을 무해히 건너뛰고 → 지문 박아 노드로 → 자동 도출로 규모화 → 전량 안전 실증 → 사이클 간 계보로 접합"의 전체 사슬이 섰다. v2 원장의 위상(사이클 내 트리 + 사이클 간 DAG)이 v3의 눈에 복원 가능하다.
