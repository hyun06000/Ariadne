## 교훈

1. **⭐⭐ 살아있는 계보와 죽은 계보는 같은 개념, 다른 각인 수단이다.** C015가 살아있는 v3 사이클의 lineage를 머지 커밋(전진)으로, C021이 죽은 v2 사이클의 lineage를 notes 소급(불변에 주석)으로 담았다. **한 개념(다중부모 계보)이 상태(생/사)에 따라 두 수단으로 실현된다.** C018 "append-only는 과거에 주석을 더한다"의 계보 층 적용.

2. **⭐ 전역 유일성을 잊으면 데이터가 조용히 뭉개진다(short_id 함정).** DAG 키를 short_id(C001)로만 하니 체인 간 충돌(loom/C001·v3-build/C001)로 131이 104로 에러 없이 줄었다. cycle id는 체인 안에서만 유일 → 전역 키는 chain/id. **부분에서 유일한 식별자를 전체에서 유일하다 가정하면 조용한 손실.**

3. **⭐ 가장 복잡한 케이스의 완전 일치 + 잔여의 정확한 회계 = 정합의 이중 증거.** 총계는 gap 20이 있었지만 머지(계보의 가장 복잡한 형태)가 4==4 완전 일치했고, gap 20이 정확히 C020 도출실패 수와 같아 새 손실이 아님이 확인됐다.

4. **위상 접합으로 마이그레이션 왕복이 닫혔다.** C017→C018→C019→C020→C021로 "유령을 무해히 건너뛰고→지문 박아 노드로→자동 도출로 규모화→전량 안전 실증→사이클 간 계보로 접합"의 전 사슬이 섰다.
