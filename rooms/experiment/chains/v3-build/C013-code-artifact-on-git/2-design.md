# 2. 실험 설계 — v3-build/C013-code-artifact-on-git

C011의 build_branches.sh를 계승하되, **한 파일 덮어쓰기**를 버리고 **다중 파일 증분 코드베이스 + 같은 파일의 독립 분기 수정**으로 확장한다. 순수 깃으로만 증명한다 — gil.py·참조 구현 소스를 고치지 않는다(자동화는 다른 사이클 몫).

## 시나리오 (실제 코드 아티팩트)

작은 실제 코드베이스 `calc/`를 공유 경로 한 벌로 둔다:
- `calc/core.py` — 여러 가지가 **같은 파일을 다르게 고칠** 대상 (H4의 핵심).
- `calc/util.py` — 증분 변경 추적 대상 (H1).
- `calc/__init__.py` — 초기 골격.

C011처럼 3층 분기(체인 v3-demo ⊃ 사이클 C-demo ⊃ 스텝 트리), 백트래킹 2회(죽은 잎 2), 산 잎 1:
- **가지 A** (s2·s3·s4, 죽음): `core.add`를 순진한 방식으로 구현 → 되돌아감(→s1).
- **가지 B** (s5·s6·s7, 죽음): `core.add`를 **다른 알고리즘**으로 구현(같은 파일 `core.py`를 A와 다르게 수정) → 되돌아감(→s1).
- **가지 C** (s8·s9·s10, 산 잎): `core.add`를 **또 다른 방식**으로 + `util.py`에 헬퍼 추가 → 채택.

세 가지가 모두 `s1`에서 갈라져 **같은 파일 `core.py`를 서로 다르게** 고친다 — 이것이 디렉토리 복사가 못 하는 것(복사 방식은 각 가지가 코드 스냅샷 한 벌을 격리 보관해 "공유 한 벌"이 없다).

백트래킹 = `git checkout <s1해시>` + detached HEAD 커밋 (C011 모델). 잎은 `gil/leaf/<short>` 태그로 못박고, 산 잎에 `cycle/C-demo/solved` 태그.

## 절차

1. **빌드**(`build_codebase.sh`): 임시 저장소(메인 레포 밖 scratchpad)를 `git init`. 스텝마다 `calc/` 하위 파일을 **증분 수정**(전체 덮어쓰기 아님)하고 1스텝=1커밋. 백트래킹은 checkout+detached. C011의 step 헬퍼(trailer 지문)를 계승하되 아티팩트를 다중 파일로 확장. 잎 태그·해시 인덱스 기록.
2. **측정**(`measure_code.py`): 아래 6측정을 순수 깃 명령으로 수행, PASS/FAIL 판정.
3. 빌드·측정 스크립트와 그 출력(git-graph, commit-index, measure-out)을 `3-verification/`에 저장 — 재현 가능하게.

## 준비물

- 도구: `git` (2.x), `python3`, `bash`. gil.py는 사이클 관리에만 쓰고 검증 코드는 순수 깃.
- 임시 저장소: scratchpad 하위(메인 레포 밖) — 메인 저장소 오염 금지. 스크립트가 매 실행 시 `rm -rf` 후 재생성 → 완전 재현.
- 의존성: 표준 라이브러리만.

## 측정 방법

| 측정 | 가설 | 방법 | PASS 기준 |
|---|---|---|---|
| **M1** 증분 diff 트레이싱 | H1 | 각 스텝 커밋의 `git show --stat`/`git diff`가 그 스텝이 의도한 파일·변경분과 일치하는가. s2는 `core.py`만, s10은 `core.py`+`util.py` 등. | 모든 스텝 커밋 diff의 변경 파일 집합 = 의도한 집합. 변경분 오염/누락 0. |
| **M2** 1스텝=1커밋 | H1 | 스텝 커밋 수·고유 Step-Id, 머지 커밋 0(detached 모델). | 11 고유 스텝, 머지 0. |
| **M3** git show 롤백(죽은 가지 포함)+워킹트리 무손상 | H2 | 죽은 잎 s4·s7의 `core.py`를 `git show <hash>:calc/core.py`로 꺼냄. 조회 전후 HEAD·`status --porcelain` 불변 확인. | 죽은 가지 코드 정상 추출 + HEAD/status/인덱스 완전 불변. |
| **M4** 죽은 가지 태그 생존 + 음성대조 | H3 | 태그 있을 때 s4·s7이 `--all`에 보임 → s7 태그 삭제 → `--all`에서 사라짐 → 되박음 → 부활. | 태그 있으면 보임, 지우면 사라짐, 되박으면 부활 (3단 모두). |
| **M5** 같은 파일 독립 분기 보존 | H4 | 세 가지의 `core.py`(s4·s7·s10 버전)를 각각 `git show`로 꺼내 **서로 다르고** 각 가지 고유 코드임을 확인. 한 버전이 다른 것으로 오염되지 않았는가. | 세 버전이 pairwise 상이 + 각자 그 가지의 서명 문자열 포함. |
| **M6** 디렉토리 복사 대조 (개념 실증) | H4 | "공유 한 벌" 방식은 `core.py` 경로가 **하나**인데 세 버전이 커밋 그래프에 공존함을 보인다(작업트리엔 산 잎 버전 1개만 물리 존재). 복사 방식이라면 세 벌의 물리 사본이 필요. | 물리 파일 1개, 커밋 그래프에 3버전 공존, 각 git show로 접근 가능. |

M5·M6가 C011을 넘어서는 이 사이클의 고유 기여(H4). M1이 "한 파일 덮어쓰기"를 "다중 파일 증분"으로 끌어올림(H1 강화).
