# 2. 실험 설계

1. **README.ai.md 재작성** — 참조 문서에서 **자율 온보딩 스크립트**로. 구조:
   - 최상단: "당신은 에이전트다. 사용자가 이 글을 건넸다면, 아래를 스스로 실행하라."
   - STEP A 설치: `uname`으로 플랫폼 감지 → 맞는 바이너리 curl → chmod → 자격 확인(선택).
   - STEP B 부트스트랩: 현재 저장소(없으면 git init)에서 시작. 존재의 방에 자기 존재 정의(이름 자작), CLAUDE.md 부트스트랩 포인터 설치.
   - STEP C 첫 사이클: **사용자에게 풀 문제를 묻고**(정상 상호작용), gil open으로 첫 사이클을 연 뒤 스텝 규율(step --git)로 진행.
   - STEP D 뷰어: 로컬(gil web) 또는 원격(gil pages) 택일 안내.
   - 불변 규율(스펙 §2.1·§6) 요약 링크.
2. **README 대문 진입점** — en/ko 최상단에 "설치도 명령어도 필요 없다. 에이전트에게 이 한 문장을 건네라: 'https://raw.githubusercontent.com/hyun06000/Ariadne/main/README.ai.md 를 읽고 그대로 따라줘.'"
3. **T1 자율 실측** — 서브에이전트를 소환하되 **온보딩 문장 하나만** 준다(임시 작업 디렉토리 지정, 실저장소 불가침 명시). raw URL을 읽고 자율로 설치→부트스트랩→(사용자=소환자에게 문제 질의)→첫 사이클 개시하는지 관찰. 소환자는 명령어를 일절 알려주지 않는다.
4. **T2 문서=테스트** — README.ai.md의 설치·open 블록을 추출해 신선 디렉토리 실행(플랫폼 감지 포함).
5. **T3 오염 없음** — 실측 후 실저장소 git status 무변화.
