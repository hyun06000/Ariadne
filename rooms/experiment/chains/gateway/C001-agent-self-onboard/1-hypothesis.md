# 1. 가설 수립

## 이전 사이클의 교훈

부모: [gateway/chain.md](../chain.md) (최초). 교훈의 연원(lineage): [genesis/C002](../../genesis/C002-subagent-bootstrap/5-report.md).

- genesis/C002: 서브에이전트는 포인터 한 줄("CLAUDE.md를 읽고 따르라")로 부활한다. 존재의 지속이 전달의 문제였듯, **채택도 전달의 문제**다.
- 발의: 사용자 (박상현, 2026-07-15): "이건 바이브코딩으로 LLM이 직접 실험하게 만드는 도구다. 사람이 일일이 퀵스타트를 알 필요 없다. '당신의 에이전트에게 이 글을 읽게 하세요' 한 문장으로 모든 게 구현되도록."
- 현재 README.ai.md는 참조 문서(무엇이 있는지)에 가깝다. 자율 실행 지시(무엇을 지금 하라)가 아니다 — 에이전트가 읽고 "그대로 실행"하기엔 결정 지점이 열려 있다.

## 가설

> **가설**: README.ai.md를 **자율 온보딩 지시**(플랫폼 감지 설치 → 저장소 부트스트랩 → 사용자에게 풀 문제 질의 → 첫 사이클 개시, 각 단계가 에이전트가 스스로 실행할 명령형)로 다시 쓰고 README 대문에 "에이전트에게 이 글을 읽히세요" 진입점을 두면, 그 문서의 URL과 한 문장만 받은 에이전트가 사람의 추가 개입 없이 gil을 설치하고 저장소를 세우고 첫 사이클을 연다.

## 기각 조건

1. **자율 실패**: 문장 하나만 받은 에이전트가 설치·부트스트랩·첫 사이클 중 어느 지점에서 사람에게 명령어를 되묻거나 멈춘다 (진입점이 지시가 아니라 참조에 그침).
2. **환경 오염**: 온보딩이 검증용 임시 환경 밖(실제 저장소)을 건드린다.
3. **문서=테스트 파손**: 지시 안의 실행 블록이 실제로 동작하지 않는다.

*주의: 에이전트가 "어떤 문제를 풀까요?"를 사용자에게 묻는 것은 기각이 아니다 — 그것은 지시가 요구하는 정상 상호작용이다. 기각은 명령어·설치 방법을 되묻는 경우다.*
