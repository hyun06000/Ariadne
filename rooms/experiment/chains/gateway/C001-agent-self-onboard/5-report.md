# 5. 결과 보고

## 요약

README.ai.md를 자율 온보딩 스크립트로 재작성하고 README 대문에 "에이전트에게 이 한 문장을 건네라" 진입점을 두었다. 실측: 한 문장만 받은 서브에이전트가 명령어를 되묻지 않고 설치·부트스트랩·자기 존재 정의(스스로 Meander로 명명)·첫 문제 해결(회문+유니코드, 놀라운 발견 포함)을 자율 완수 — **조건부 채택**. 유일한 미완은 gil 바이너리 실행이 auto-mode 보안에 차단된 것(문서 결함 아님)이며, 승인 마찰을 문서에 명시했다.

## 교훈

1. **진입점은 지시여야 하지 참조가 아니다** — "무엇이 있다"가 아니라 "지금 무엇을 하라"의 명령형으로 쓰자 에이전트가 자율 실행했다. genesis의 포인터 주입이 raw URL 한 문장으로 저장소 밖까지 일반화됐다.
2. **바이브코딩 딸깍의 현실 경계**: 웹-유래 바이너리 실행은 잠긴 에이전트에게 한 번의 사람 승인을 요구할 수 있다. 이를 숨기지 않고 "사람 몫의 유일한 단계"로 명시하는 것이 정직하다.
3. 에이전트가 우회 대신 정직한 승인 요청을 택하고, 지시 충돌을 스스로 판단한 것 — 신뢰할 수 있는 자율성의 모습.

## 다음 사이클을 위한 제안

- **(A) 승인 모드 end-to-end** (사용자 실전): 상현님이 자기 에이전트를 승인/interactive 모드로 두고 한 문장을 건네 gil 기록까지 완주 — 이 사이클이 남긴 미검증 구간의 완결.
- (B) 설치 신뢰 강화: SHA256 검증을 온보딩 문서에 선택 단계로, 혹은 설치 원리를 사람이 신뢰하도록.
- (C) Go 단독 완결(open --git·release) — 이월.
