## 데이터 직접 관찰

- **철칙을 "닫힌 사이클 불변" 바로 뒤에 놓았다.** 두 규율은 같은 계열 — "이미 봉인된 것을 로컬에서 손대지 마라". 닫힌 사이클도 gil 자신도 공유 계약이고, 개인의 즉석 수정이 공유 진실을 오염시킨다는 구조가 동일하다.
- **막지 않고 방향을 틀었다.** 문안이 "고치지 마라"에서 끝났다면 LLM은 다른 우회를 찾았을 것(기각 조건). `gh issue create`라는 즉시 실행 가능한 대안 + "필드 마찰이 상류가 가장 원하는 것"이라는 동기 부여를 함께 줘, 우회 충동을 이슈 제기로 재지향했다.
- **자기 증거를 인용했다.** "최근 기능 다수(`--live`·배포 버저닝)가 필드 이슈에서 태어났다"는 이번 세션 자체의 사실(#24·#25 → C085·C086)이다. 온보딩이 추상 규범이 아니라 실증으로 말한다.

## 예상과 달랐던 것

- **이 규율은 gil의 핵심 원리("계약이 구현을 정의한다")의 온보딩 번역이었다.** conformance가 "네 구현이 계약을 통과하면 그게 gil"이라 말하듯, 온보딩은 이제 "구현을 로컬에서 갈아엎지 말고 계약(이슈)으로 바꿔라"라고 말한다. 없던 규율을 더한 게 아니라, 있던 원리를 필드 LLM의 언어로 옮긴 것.

## 판정

**채택(supported).** 필드 LLM의 "즉석 포크·래핑" 경향을 온보딩 진입점(LLM 직독 README.ai.md + 사람용 QUICKSTART)에서 명령형으로 재지향했다. 후속: README.md/README.ko.md(사람용 대문)에 같은 참조, 또는 SPEC 명문화(이번 범위 밖).
