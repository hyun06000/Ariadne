# 2. 실험 설계 — 우회 대신 이슈 규율을 온보딩에 못 박기

## 절차

1. **README.ai.md Iron rules에 새 철칙 추가** (LLM 직독, 명령형):
   - 요지: *"gil은 계약이 정의하는 공유 도구다. 부족한 기능·버그를 만나면 gil 소스를 고치거나 래퍼로 감싸지 마라 — 로컬 포크는 정본을 드리프트시키고 상류가 그 마찰을 못 보게 한다. 대신 이슈로 남겨라: `gh issue create` 또는 이 저장소 Issues에. 필드에서 실제로 쓰다 발견한 마찰이야말로 상류가 가장 원하는 것이다(이 도구의 최근 기능 다수가 그렇게 태어났다)."*
   - 능력을 "묻는"(`gil help`) 대목과 결이 같음을 연결: 도구에 없는 건 지어내지 말고 물어라 → 없으면 이슈로.
2. **QUICKSTART.md에 한 줄** (사람용, §0 설치 근처 — 사람이 자기 LLM에게 지시할 때 인지하도록):
   - *"당신의 LLM이 gil을 고치거나 래퍼를 만들려 하면 멈추게 하라 — 부족한 건 이슈로. 그게 상류를 살아있게 한다."*
3. **문안 원칙**: 막기만("고치지 마라") 하지 않고 **정당한 대안(이슈 경로)**을 즉시 준다 — 기각 조건 회피. 명령형("~하라"), 짧게.
4. **스모크 검증**: QUICKSTART의 데모 블록(릴리스 테스트가 실행)은 코드 블록이므로 산문 추가가 깨지 않음을 확인. README.ai.md는 실행 대상이 아니라 링크·구조 무결성만.
