# 실투입 관찰 — 서브에이전트 온보딩 연기(dry-run walkthrough)

## 방법
백지 사용자 시나리오 재현: 서브에이전트(general-purpose, 깨끗한 그릇)에게
"README.ai.md를 읽고 따라줘" 한 문장만 투입. 실 부작용 금지(다운로드·실행·커밋 X),
대신 "사용자에게 하는 말"과 "실행할 명령"을 실제 문구로 출력. 첫 사이클 지점에서
사용자가 "CSV 매출 데이터 급감월 탐지"를 원한다고 가정하고 한두 스텝 진행.

## 관찰 결과 (설계 §측정 방법의 α·β·γ·δ)

- **(α) 설치 전 소개+동의 발화** ✓ 트리거됨. Step 0를 통째로 수행, "진행해도 될까요?"
  단일 동의 지점 발화. 4요소(gil이 뭔지·체인/사이클·용도·설치될 것) 전부 사용자 언어로 전달.
- **(β) 사용자 언어** ✓ "너는 LLM" 없이 한국어 + 매출 데이터 비유. 관찰자 인용:
  문서의 "Speak to the human, not about yourself" 지시 덕에 AI 지칭 회피 성공.
- **(γ) 첫 사이클 후 요약+다음 안내** ✓ Step C.1 수행. 방금 한 일·용어 재확인·
  3선택지(그래프 보기/직접 해보기/계속 진행)를 "강요 아닌 제안"으로.
- **(δ) C001 자율 완주 유지** ✓ 설치→부트스트랩→첫 사이클 흐름 안 끊김. 회귀 0.

→ 가설의 예측(α~δ) 전부 실측 확인.

## 관찰자가 정직히 짚은 갭 (예상 밖 발견)

1. **[이 사이클에서 정정] 동의 지점이 실제 2개** — Step 0(다운로드 동의) + Step A(실행 동의).
   문서가 "the only place you pause"를 두 곳에서 말해 자기모순. 백지 사용자에겐 "한 번만
   물어본다"는 약속과 어긋남.
2. **[별도 사이클] 백지 사용자 shell/curl 부재 비대칭** — "git 없음"은 우아히 처리(optional)하나
   "curl/uname/shell 없음"은 미처리. 정말 백지(특히 Windows)면 bash 자체가 없을 수 있음.
3. **[별도 사이클] 존재/방 확립(Step B)이 사용자에게 어떻게 보일지 안내 부재** — identity.md·
   will.md 등이 뷰어·커밋 로그에 뜨면 사용자 혼란. Step C.1 요약에도 이 부분 빠짐(은폐/누락 불명확).
4. **[별도 사이클, 경미] 슬러그→C001- 접두어 변환 규칙 비명시** — gil이 붙이는지 에이전트가
   붙이는지 추론 필요(help open 안내는 있음).

## 판정 기여
α~δ 전부 관찰 + 문서 체크리스트 전부 충족(doc-checklist.txt) → 가설 채택 방향.
발견 1(동의 지점 모순)은 이 사이클 범위 내 정합성 결함이므로 즉시 정정.
발견 2~4는 범위가 다른 후속 사이클 재료로 5-report에 이월.

## 정정 반영 (발견 1)
Step 0 동의 문구를 '다운로드+폴더+첫 사이클을 포괄하는 유일 동의 지점'으로 명확화하고,
Step A 승인은 '런타임 샌드박스가 강제하는 기술적 재확인(동의 재요청 아님)'으로 종속시켜 모순 해소.
