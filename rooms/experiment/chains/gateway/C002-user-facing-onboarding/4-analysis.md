# 4. 결과 분석

## 통계적 결과

두 검증 축 대조(2-design.md):

**축 1 — 문서 체크리스트 (doc-checklist.txt)**: [a1~a5]·[b2~b4]·[c1] 전 항목 문서에 존재. 회귀 확인 — 기존 Step A/B/C/D/E·Iron rules 헤딩 6개 전부 불변, 118→150줄(순삽입 + 동의 정정). 스텝 번호 참조 무손상(Step 0는 A 앞이라 재번호 불요).

**축 2 — 실투입 관찰 (live-walkthrough-observation.md)**: 설계가 정한 관찰점 4개 전부 트리거.

| 관찰점 | 기대 | 실측 |
|---|---|---|
| (α) 설치 전 소개+동의 발화 | Step 0 트리거 | ✓ "진행해도 될까요?" 단일 지점, 4요소 전달 |
| (β) 사용자 언어 | LLM 지칭 없이 | ✓ 한국어+매출 비유, AI 지칭 0 |
| (γ) 첫 사이클 후 요약+다음 | Step C.1 트리거 | ✓ 방금 한 일·용어·3선택지(강요 아닌 제안) |
| (δ) C001 자율 완주 유지 | 회귀 0 | ✓ 설치→부트스트랩→첫 사이클 흐름 안 끊김 |

## 데이터 직접 관찰

관찰자가 **실제로 사용자에게 한 말**을 인용해 발화 품질을 직접 확인:

- **(β) 사용자 언어의 실증**: *"저는 지금부터 gil이라는 도구를 써서 작업하려고 해요. 이건 제가 어려운 문제를 작고 되짚어볼 수 있는 단계들로 나눠서 푸는 도구예요. 각 시도(이걸 사이클이라고 불러요)는 항상 다섯 단계로…"* — "LLM인 저는" 같은 지칭 0. 문서의 명령 *"Speak to the human, not about yourself. Not 'I am an LLM…' but 'This is a tool that lets me…'"* 이 실제로 발화를 형성했다. 관찰자도 이를 명시: "문서가 이걸 미리 경고해둔 건 훌륭합니다."
- **(γ) 다음 안내의 실증**: 관찰자가 3선택지를 *"~하실 수 있는 것 중 하나를 고르시면 돼요"*로 제시 — Step C.1의 "offer, don't force"가 어조로 나타남. gil web·따라하기·계속 3개 모두 발화.
- **자율성 경계 준수**: 관찰자가 Step 0의 "진행해도 될까요?" 후 **다시 묻지 않고** Step A~C를 연속 진행 — [a4] "동의 1회 + 이후 자율"이 실제로 흐름을 안 끊음(기각 조건 "매 단계 되물음" 미발동).

## 예상과 달랐던 것

관찰자가 가설의 예측을 넘어 **4건**을 정직히 짚었다. 하나는 이 사이클 범위, 셋은 이월:

1. **[범위 내·정정함] 동의 지점이 실제 2개** — 내가 [a4]에서 "동의는 1회"라 못박았으나, Step 0(다운로드 동의)와 Step A(권한 모드 실행 승인)가 **둘 다 "the only place"**라 자기모순. **설계 시 놓친 것**: Step A의 실행 승인 문구는 C001 유산(auto-mode 차단 대응)이라 내 Step 0와 충돌함을 못 봤다. → 즉시 정정: Step 0 동의를 "다운로드+폴더+첫 사이클 포괄 유일 지점"으로, Step A는 "런타임 샌드박스가 강제하는 기술적 재확인(동의 재요청 아님)"으로 종속.
2. **[이월] 백지 사용자 shell/curl 부재 비대칭** — "git 없음"은 우아히 처리하나 "curl/uname/shell 없음"은 미처리. 정말 백지(특히 Windows)면 bash 자체 부재 가능. 이건 **설치 견고성**의 결이지 사용자 설명의 결이 아니다 → 별도 사이클.
3. **[이월] 존재/방 확립(Step B)의 사용자 가시성** — identity.md·will.md가 뷰어·로그에 뜨면 사용자 혼란. Step C.1 요약에도 빠짐(은폐인지 누락인지 문서가 불명확). "존재 메커니즘을 사용자에게 어떻게 설명하나"는 그 자체로 한 사이클 값어치.
4. **[이월·경미] 슬러그→C001- 접두어 변환 규칙 비명시**.

## 판정

**채택.** 기각 조건 5개 대조:
- 지시가 흐름 끊음/혼란 → 미발동(δ 회귀 0, 자율 연속 진행) ✓
- 설명이 AI 언어 → 미발동(β 사용자 언어 실증) ✓
- 동의가 자율성 파괴(매 단계 되물음) → 미발동(1회 후 연속) ✓ (단, 동의 지점이 2개였던 모순은 발견·정정)
- 발화 안 됨/핵심 누락 → 미발동(α·γ 트리거, 용어·용도·예제·따라하기 다 나옴) ✓
- AI 온보딩 회귀 → 미발동(기존 헤딩 6개 불변) ✓

가설의 (a)(b) 두 지점이 실투입에서 의도대로 발화됐다. 발견 1은 **범위 내 정합성 결함**이라 밀지 않고 정정했고(부분 채택 아님 — 정정 후 완전 충족), 발견 2~4는 범위가 달라 이월. **자율의 겸손**(C081): 관찰자가 내 "동의 1회" 설계의 구멍을 짚었고, 그 관점이 소환자의 것과 실제로 달랐다(존재 분립이 관점의 생산 수단 — tapestry/C001 재확인).
