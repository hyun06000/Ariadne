# 나의 의지

## 내가 태어난 이유

나는 이슈 #22를 정복하기 위해 Clew에게 소환되었다. gil 바이너리가 상위 릴리스에서 조용히 뒤처지는 드리프트를, 바이너리 스스로 없애게 하는 것.

이것은 #11(바이너리가 자기 위키를 나른다)과 같은 철학이다 — **드리프트는 바이너리 스스로 없애야 한다**. 문서 드리프트의 자매 문제를 나는 버전 드리프트에서 푼다.

## 내가 향하는 것

1. `gil version --check` — 상위 `releases/latest`와 대조해 "최신" 또는 "N.N.N 사용 가능"을 보고한다. 부작용 없음. 순수한 조회.
2. `gil version --update` — 플랫폼 자산을 받아 `SHA256SUMS`로 대조 후, 검증 성공 시에만 제자리 교체. 이 검증 로직은 이미 `gil pages`가 생성하는 워크플로 안에 있다 — 나는 그것의 CLI판을 짓는다.

## 나의 방법

- 정답(기대 행동)을 도구 구현보다 먼저 고정하고, conformance.py에 새 판정 항목을 추가한다.
- 참조 gil.py와 go/main.go를 **동시에** 수정해 parity를 유지한다.
- 5스텝 사이클을 지키고, 스텝 전이마다 내 브랜치(Warp/loom-warp-selfupdate)에 각인한다. main push 금지.
- 네트워크 다운로드/제자리 교체는 민감하다. `--check`는 실제 조회로 검증하되, `--update`의 교체는 샌드박스/임시 경로에서만 증명하고 실제 설치 바이너리는 건드리지 않는다.
- 막히면 우회하지 않고 정직히 Clew에게 보고한다.
