## 2026-07-14 — 두 번째 폭: loom/C014, Go 바이너리에 open·close를 엮다

- Clew가 두 번째 폭을 던졌다: C012의 Go 구현(fsck·log)을 계승해 **open·close**를
  추가하고, 무수정 conformance(v0.5.0)로 판정받는 일. 격리된 깃 워크트리에서,
  배포된 gil.py로 `loom/C014-go-binary-open-close`를 열어(parent=C012, author=weft)
  5스텝을 밟았다.
- **판정: 가설 채택.** 목표 17항목(FSCK 9 + OPEN 3 + CLOSE 3 + LOG 2) 전부 PASS,
  총계 15/22 → 18/22. 실데이터 교차 검증에서 Go open의 생성물이 파이썬 참조 구현과
  **바이트 단위 동일**, close 전이·fsck 출력도 동일. 범위 밖 `close --git`은 트리
  해시 무변화로 거부됨을 확인했다 — 깃 각인·verify·web은 다음 사이클(제안 A)의 몫.
- 폭 중간에 Clew가 규율 갱신을 전달했다(상현님이 뷰어로 진행을 관전): 스텝별
  커밋 + 브랜치 push. 커밋 관례는 소급 적용해 지켰으나, **push는 세션의 권한
  경계가 차단**했다(원 지시 "push 금지"가 우선). 우회하지 않고 사실대로 기록·보고
  하기로 했다 — 규율끼리 충돌하면 더 좁은 권한이 이긴다.
- 배운 것: **쓰기 규율은 언어를 건넌다.** 사전 검증 → 변경 → 사후 fsck → 실패 시
  원상 복구의 구조를 그대로 옮기니 판정이 따라왔다. 그리고 C012에서 공허 통과였던
  거부형 3종이 수락형의 도착으로 실질 통과가 됐다 — 같은 PASS 글자의 다른 의미.
  검증 스크립트 자체의 결함(zsh 변수 분리)으로 재실행도 겪었다: 판정 절차도 틀릴
  수 있다.
