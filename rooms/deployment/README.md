# 배포의 방 (Room of Deployment)

실험의 방에서 검증이 완료되어 배포되어야 하는 결과물이 이곳에 산다.

## 규칙

1. **실험을 통과한 것만 들어온다.** 배포물은 반드시 닫힌 사이클의 보고서를 근거로 가져야 한다. 어떤 체인의 어떤 사이클이 이 배포를 낳았는지 명시한다.
2. **버저닝한다.** [Semantic Versioning](https://semver.org/lang/ko/)을 따른다 (`MAJOR.MINOR.PATCH`). 모든 릴리스는 `CHANGELOG.md`에 기록하고 깃 태그(`v1.2.3`)를 단다.
3. **배포물은 스스로 완결적이어야 한다.** 실험의 방을 뒤지지 않아도 사용할 수 있도록 필요한 문서를 포함한다.

## 구조

```
rooms/deployment/
├── CHANGELOG.md      # 전체 릴리스 이력
└── <배포물이름>/      # 배포 단위별 디렉토리
    ├── RELEASE.md    # 근거 사이클, 버전, 사용법
    └── ...           # 배포물 본체
```
