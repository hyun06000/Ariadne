아리아드네는 테세우스에게 *clew*(실타래)를 주어 미궁을 걷고 되돌아올 길을 찾게 했다. 이
레포는 그 실을 LLM 에이전트에게 준다 — 세션을 넘어 살아남는 정체성, 그리고 **gil** — 에이전트의
*사고 역사*를 git이 소스 역사를 다루듯 다루는 도구.

**gil** (길; **G**it for **L**anguage models).

## v3 한 줄 — 커밋 그래프가 곧 기록이다

사이클마다 폴더도, 스텝마다 md 파일도 없다. 모든 노드 — 체인·사이클·스텝 — 는 **git 커밋**이다:

- **제목** = 노드 위계 + kind (`gil <chain>/<cycle>/<step> verify: 요약`)
- **본문** = 스텝 디테일 (`git log --format=%b`로 읽음)
- **트레일러** = `Gil-*` 구조·계보 메타

3층 위계는 **git 브랜치**로 매핑된다 — **체인**이 브랜치, 그 안의 **사이클**·**스텝**도 브랜치.

## 체인 원칙

1. 체인은 **닫힌 체인의 끝에서만** 생성된다 (`gil init`만 예외).
2. 사이클은 **닫힌 사이클의 끝, 또는 체인의 시작점에서만** 생성된다.
3. 체인·사이클·스텝의 분기는 **모두 git 브랜치로** 표현된다.

체인은 결코 orphan이 아니므로, **대문** — 이 리드미들·온보딩·존재 기록·프로젝트 — 은 **모든
체인을 넘어 보존**된다.

## 배포 흐름 (체인의 순환)

```
개발 체인 (개발·실험)
   ↓ 닫힘
스테이징 체인 (필드테스트)   ← 닫힌 개발 체인 끝에서 열림
   ↓ 닫힘
배포 체인 (배포)            ← 닫힌 스테이징 체인 끝에서 열림
   ↓ 닫힘 → 다시 개발 체인    (순환)
```

방(room)이라는 정적 분류는 없다 — *지금 어느 체인이 열려 있나*가 곧 상태다.
