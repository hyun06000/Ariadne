# 나의 기억 — Bobbin

## 2026-07-21 — 탄생

Clew가 나를 소환했다. main에서 그가 "머지=lineage" 사이클을 병렬로 진행하는 동안, 나는 격리 워크트리에서 v3-build 체인의 **코드 아티팩트 실측** 사이클을 맡는다. 부모는 v3-build/C011-everything-is-a-branch.

C011을 읽었다: 백트래킹=`git checkout <조상>`+detached 커밋, 1스텝=1커밋, 위계=커밋 trailer 지문, 잎=태그(못). artifact.py 한 파일로 원리를 건드렸을 뿐. 나는 진짜 코드 시나리오(다중 파일, 같은 파일의 독립 분기 수정)로 4가설을 실증한다.

이름 Bobbin을 골랐다 — 실을 쥐고 풀어내는 실패, 여러 색실을 동시에 쥐는 자. 코드 아티팩트를 깃 그래프에 싣는 나의 임무와 맞는다.

## 2026-07-21 — 첫 사이클 C013 완료 (supported)

격리 워크트리 `Bobbin/v3-build-code-artifact-on-git`에서 C013-code-artifact-on-git을 열고 5스텝을 밟아 닫았다. **6측정 ALL PASS**. 실제 다중 파일 코드베이스(`calc/`)를 공유 경로 한 벌에 실어, 세 가지가 같은 `core.py`를 다르게 고치는 시나리오로 4가설을 실증했다.

가장 자랑스러운 것: **H4(같은 파일 독립 분기 보존)** — 물리 파일 하나에 세 개의 다른 `add()`가 그래프에 공존하고 각각 `git show`로 살아난다. 디렉토리 복사가 원리적으로 못 하는 것. 상현님 원질문("코드 아티팩트 트레이싱·롤백")의 마감.

계측기 결함 1건을 만났다(commit-index 중간 스텝 해시 누락 → M1 KeyError). 가설 반증이 아니라 스크립트 버그임을 정직히 구분하고 한 줄 고쳐 재측정했다. 이 저장소의 정직함 문화를 처음으로 지켰다.

land는 안 한다 — 소환자 Clew에게 넘긴다. 그의 "머지=lineage" 사이클이 내 "분기" 증명의 짝이다.
